# Административные учетные записи, веб-сайты и разрешения в Microsoft Intune

Перед настройкой Microsoft Intune ознакомьтесь с этим разделом и другими требованиями в разделе [Предварительные сведения для установки Microsoft Intune](what-to-know-before-you-start-microsoft-intune.md).

Для администрирования Intune используются следующие компоненты:
- два типа учетных записей администратора;
- учетные записи пользователей с дополнительными разрешениями;
- две веб-консоли (веб-портала) администрирования, предоставляющие администраторам доступ к управляемым объектам.

В следующих разделах эти учетные записи и порталы рассматриваются более подробно.

## Учетные записи администраторов и учетные записи пользователей со специальными разрешениями

Ниже перечислены учетные записи и разрешения, которые будут использоваться в Intune.

### Администратор клиента
|Уровни разрешений|Дополнительные сведения|
|--------------------------|-------------------------|
|Администраторам клиента назначается одна роль администратора, которая определяет область администрирования для этих пользователей и задачи, которыми он может управлять.<br /><br />Роли администратора одинаковы для разных облачных служб Майкрософт, хотя некоторые службы могут не поддерживать некоторые роли.<br /><br /> В Microsoft Intune используются следующие роли:<br /><br />— глобальный администратор;<br />— администратор выставления счетов;<br />— администратор паролей;<br />— администратор поддержки службы;<br />— администратор управления пользователями.|По умолчанию учетная запись, которая использовалась для создания подписки на Microsoft Intune, является администратором клиента с ролью глобального администратора.<br /></br>  Администратор клиента использует [!INCLUDE[wit_icp_1](../includes/wit_icp_1_md.md)] для управления подпиской на [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)] и назначения администраторов клиентов из [!INCLUDE[wit_icp_2](../includes/wit_icp_2_md.md)].<br /><br />Используйте администратора клиента с ролью глобального администратора для получения доступа к [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)] и назначения первого администратора служб. Администратора клиента не рекомендуется использовать для выполнения повседневных задач управления. Администратору клиента не требуется лицензия [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)] для доступа к [!INCLUDE[wit_icp_2](../includes/wit_icp_2_md.md)].<br /><br />Администратор клиента — это общее понятие в облачных службах Майкрософт. При оформлении подписки на [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)] служба становится клиентом Microsoft Azure AD. См. раздел, посвященный клиенту Azure AD, в статье [Что такое каталог Azure AD?](http://technet.microsoft.com/library/jj573650.aspx).|


### Администратор служб
|Уровни разрешений|Дополнительные сведения|
|--------------------------|-------------------------|
|Администраторам служб назначаются перечисленные ниже разрешения.<br /><br />**Полный доступ**. Предоставляет доступ ко всем областям [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)] без ограничений. Также можно добавлять других администраторов служб и управлять ими.<br /><br />**Доступ только для чтения**. Это разрешение предоставляет доступ на чтение ко всем областям [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)]. Администратор служб с доступом только для чтения не может изменять данные, однако может запускать отчеты.<br /><br />**Служба технической поддержки — узел групп**. Предоставляет разрешения, позволяющие администратору службы выполнять только набор типичных задач, связанных со сценариями службы технической поддержки. Сведения об этом наборе разрешений см. в разделе [Настройка представлений консоли Intune в соответствии с ролями администраторов](/intune/deploy-use/control-what-admins-can-see-in-the-microsoft-intune-admin-console).|По умолчанию [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)] не назначает администратора служб. Для назначения первого администратора служб для подписки потребуется использовать администратора клиента с ролью глобального администратора. </br></br> Администратор служб использует [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)] для управления повседневными задачами для [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)].<br /><br />Назначение администраторов службы выполняется в консоли администрирования. Прежде чем учетной записи администратора служб будет предоставлен доступ к консоли администрирования, ей необходимо назначить лицензию [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)].|



### Диспетчеры регистрации устройств
|Уровни разрешений|Дополнительные сведения|
|--------------------------|-------------------------|
|Диспетчеры регистрации устройств являются стандартными учетными записями пользователей с дополнительными разрешениями на регистрацию более пяти устройств.|По умолчанию каждый пользователь [!INCLUDE[wit_firstref](../includes/wit_firstref_md.md)] может зарегистрировать до пяти устройств. Тем не менее вы можете предоставить учетной записи пользователя разрешение диспетчера регистрации устройств, а затем использовать эту учетную запись для регистрации больших групп корпоративных устройств. Это полезно, если устройства могут назначаться пользователям временно или использоваться в полноэкранном режиме, в котором сопоставление пользователей и устройств не является обязательным.|


## Веб-сайты администрирования для Intune
 Для выполнения различных задач администрирования требуется один из следующих административных веб-сайтов, который можно открыть в [поддерживаемом веб-браузере](supported-web-browsers.md).

- [Портал Office 365](http://go.microsoft.com/fwlink/p/?LinkId=698854)
- [Консоль администрирования Microsoft Intune](https://admin.manage.microsoft.com/)

### [Портал Office 365](http://go.microsoft.com/fwlink/p/?LinkId=698854)

**Администратор клиента использует этот портал для управления подпиской**, включая следующие задачи (определяемые ролью администратора):

- управление учетными записями пользователей для подписки и настройка синхронизации каталогов из локальной службы Active Directory;
- управление группами пользователей, называемыми группами безопасности;
- назначение пользователям лицензий на использование [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)];
- настройка имени домена, используемого для подписки (имя домена определяет учетные записи, используемые пользователями для входа);
- управление выставлением счетов и сведениями о покупках для подписки, включая число имеющихся лицензий или доступный объем облачного хранилища;
- поиск ссылок для просмотра работоспособности службы [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)].
- Администратор клиента может зарегистрироваться на портале Office 365 для управления подпиской, даже если его учетной записи не назначена лицензия на использование [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)].
- Пользователи, располагающие лицензией на Intune, но не являющиеся администраторами, могут использовать этот портал для сброса пароля учетной записи и редактирования профиля.
- Чтобы получить доступ к порталу Office 365, учетной записи должно быть назначено состояние входа **Разрешено**. Это состояние не обозначает наличия лицензии на подписку. По умолчанию все учетные записи пользователей **разрешены**.


### [Консоль администрирования Microsoft Intune](https://admin.manage.microsoft.com/)

**Администратор служб использует эту консоль для управления повседневными операциями**, включая:

- настройку политик для компьютеров и мобильных устройств;
- загрузку и развертывание программного обеспечения, такого как обновления программного обеспечения и приложения;
- управление [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)] Endpoint Protection на компьютерах;
- просмотр состояния устройства и запуск отчетов.
- Войдите на этот портал. Для входа на портал необходимо иметь разрешения администратора службы или быть администратором клиента с ролью глобального администратора.


Входить на портал могут только пользователи, имеющие разрешения администратора службы или являющиеся администраторами клиента с ролью глобального администратора. Чтобы получить доступ к консоли администрирования, учетной записи должна быть назначена лицензия на использование [!INCLUDE[wit_nextref](../includes/wit_nextref_md.md)] и состояние входа должно быть **Разрешено**.

Дополнительные сведения о [добавлении пользователей в подписку](start-with-a-paid-subscription-to-microsoft-intune-step-3.md) и [назначении лицензий для подписки](start-with-a-paid-subscription-to-microsoft-intune-step-4.md).
