# <a name="sync-active-directory-and-add-users-to-intune"></a>Синхронизация Active Directory и добавление пользователей в Intune
Можно настроить синхронизацию каталогов для импорта учетных записей пользователей из локальной службы Active Directory в Microsoft Azure Active Directory (Azure AD). Соединение локальной службы Active Directory со всеми вашими службами, использующими Azure Active Directory, существенно упрощает управление удостоверениями пользователей. Можно также настроить единый вход, чтобы взаимодействие с пользователями при проверке подлинности происходило просто и привычно.

При использовании нескольких служб с одним и тем же [клиентом Azure AD](https://azure.microsoft.com/documentation/articles/active-directory-aadconnect/) ранее синхронизированные учетные записи пользователей доступны всем облачным службам с общей подпиской клиента Azure AD, что еще больше упрощает работу.

## <a name="synchronize-on-premises-users-with-azure-ad"></a>Синхронизация локальных пользователей с помощью Azure AD
Для синхронизации учетных записей пользователей с Azure AD нужно всего одно средство — [Мастер Azure AD Connect](https://www.microsoft.com/download/details.aspx?id=47594). Мастер Azure AD Connect — это удобное средство для подключения локальной инфраструктуры удостоверений к облаку.  Выберите топологию и потребности (один или несколько каталогов, синхронизация паролей или федерация), и мастер сможет развернуть и настроить все компоненты, необходимые для работы подключения. Сюда входят службы синхронизации, службы федерации Active Directory (AD FS) и модуль Azure AD PowerShell.

> [!TIP]
> Azure AD Connect включает в себя функции, которые ранее выпускались как Dirsync и Azure AD Sync. См. дополнительные сведения об [интеграции каталогов](http://technet.microsoft.com/library/jj573653.aspx). Дополнительные сведения о преимуществах синхронизации учетных записей пользователей из локального каталога c Azure AD см. в статье [Сходство Active Directory и Azure AD](http://technet.microsoft.com/library/dn518177.aspx).

## <a name="grant-administrator-permissions"></a>Назначение прав администратора

Для администрирования Intune используются следующие компоненты:
- два типа учетных записей администратора;
- учетные записи пользователей с дополнительными разрешениями;
- две веб-консоли (веб-портала) администрирования, предоставляющие администраторам доступ к управляемым объектам.

После добавления пользователей в подписку рекомендуется задать некоторым из них административные учетные данные. Консоль, которая будет использоваться для назначения административных учетных данных, зависит от типа администратора.

-   **Администратор клиента.** Используйте **портал учетных записей Microsoft Intune**, чтобы назначить эту роль администратора для управления подпиской, включая выставление счетов, облачное хранилище и управление пользователями, которым разрешено использовать Intune.

-   **Администратор служб.** Используйте **консоль администратора Microsoft Intune**, чтобы назначить эту роль администратора для выполнения повседневных задач, включая управление мобильными устройствами и компьютерами, развертывание политик и ПО и выполнение отчетов.

В следующих разделах эти учетные записи и порталы рассматриваются более подробно.

## <a name="administrator-accounts-and-user-accounts-with-special-permissions"></a>Учетные записи администраторов и учетные записи пользователей со специальными разрешениями

Ниже перечислены учетные записи и разрешения, которые будут использоваться в Intune.

### <a name="tenant-administrator"></a>Администратор клиента
|Уровни разрешений|Дополнительные сведения|
|--------------------------|-------------------------|
|Администраторам клиента назначается одна роль администратора, которая определяет область администрирования для этих пользователей и задачи, которыми он может управлять.<br /><br />Роли администратора одинаковы для разных облачных служб Майкрософт, хотя некоторые службы могут не поддерживать некоторые роли.<br /><br /> В Microsoft Intune используются следующие роли:<br /><br />— глобальный администратор;<br />— администратор выставления счетов;<br />— администратор паролей;<br />— администратор поддержки службы;<br />— администратор управления пользователями.|По умолчанию учетная запись, которая использовалась для создания подписки на Microsoft Intune, является администратором клиента с ролью глобального администратора.<br /></br>  Администратор клиента использует [!INCLUDE[wit_icp_1](../includes/wit_icp_1_md.md)] для управления подпиской на Intune и назначения администраторов клиентов из [!INCLUDE[wit_icp_2](../includes/wit_icp_2_md.md)].<br /><br />Используйте администратора клиента с ролью глобального администратора для получения доступа к [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)] и назначения первого администратора служб. Администратора клиента не рекомендуется использовать для выполнения повседневных задач управления. Администратору клиента не требуется лицензия на Intune для доступа к [!INCLUDE[wit_icp_2](../includes/wit_icp_2_md.md)].<br /><br />Администратор клиента — это общее понятие в облачных службах Майкрософт. При оформлении подписки на Intune служба становится клиентом Microsoft Azure AD. См. раздел, посвященный клиенту Azure AD, в статье [Что такое каталог Azure AD?](http://technet.microsoft.com/library/jj573650.aspx).|


### <a name="service-administrator"></a>Администратор служб
|Уровни разрешений|Дополнительные сведения|
|--------------------------|-------------------------|
|Администраторам служб назначаются перечисленные ниже разрешения.<br /><br />**Полный доступ**. Предоставляет доступ ко всем областям [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)] без ограничений. Также можно добавлять других администраторов служб и управлять ими.<br /><br />**Доступ только для чтения**. Это разрешение предоставляет доступ на чтение ко всем областям [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)]. Администратор служб с доступом только для чтения не может изменять данные, однако может запускать отчеты.<br /><br />**Служба технической поддержки — узел групп**. Предоставляет разрешения, позволяющие администратору службы выполнять только набор типичных задач, связанных со сценариями службы технической поддержки. Сведения об этом наборе разрешений см. в разделе [Настройка представлений консоли Intune в соответствии с ролями администраторов](/intune/deploy-use/control-what-admins-can-see-in-the-microsoft-intune-admin-console).|По умолчанию Intune не назначает администратора служб. Для назначения первого администратора служб для подписки потребуется использовать администратора клиента с ролью глобального администратора. </br></br> Администратор служб использует [!INCLUDE[wit_adminconsole](../includes/wit_adminconsole_md.md)] для управления повседневными задачами для Intune.<br /><br />Назначение администраторов службы выполняется в консоли администрирования. Прежде чем учетной записи администратора служб будет предоставлен доступ к консоли администрирования, ей необходимо назначить лицензию на использование Intune.|



### <a name="device-enrollment-managers"></a>Диспетчеры регистрации устройств
|Уровни разрешений|Дополнительные сведения|
|--------------------------|-------------------------|
|Диспетчеры регистрации устройств являются стандартными учетными записями пользователей с дополнительными разрешениями на регистрацию более пяти устройств.|По умолчанию каждый пользователь [!INCLUDE[wit_firstref](../includes/wit_firstref_md.md)] может зарегистрировать до пяти устройств. Тем не менее вы можете предоставить учетной записи пользователя разрешение диспетчера регистрации устройств, а затем использовать эту учетную запись для регистрации больших групп корпоративных устройств. Это полезно, если устройства могут назначаться пользователям временно или использоваться в полноэкранном режиме, в котором сопоставление пользователей и устройств не является обязательным.|
