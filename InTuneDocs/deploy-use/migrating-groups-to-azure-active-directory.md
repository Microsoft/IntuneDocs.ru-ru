## <a name="the-new-admin-experience-for-groups"></a>Новый интерфейс администратора для групп
    
На основе ваших отзывов о том, что необходимо единообразие среды выбора целей и работы с группами в компонентах Enterprise Mobility и Enterprise Security, мы преобразовали группы Intune в группы безопасности на основе Azure Active Directory. Это позволит унифицировать управление группами в Intune и Azure Active Directory (Azure AD). Новая среда работы позволяет отказаться от дублирования групп между службами и обеспечивает расширяемость с помощью PowerShell и Graph. 

### <a name="how-and-when-will-i-migrate-to-the-new-groups-experience"></a>Как и когда будет выполняться переход на новые группы?
Миграция текущих клиентов будет выполняться постепенно и начнется не раньше декабря 2016 г. Перед миграцией своих групп вы получите уведомление. Если при миграции у вас возникнут проблемы, обратитесь к рабочей группе по миграции по адресу [intunegrps@microsoft.com](mailto:intunegrps@microsoft.com).

### <a name="what-new-features-will-be-available-to-me"></a>Какие новые возможности будут доступны?
Будут доступны новые функции, перечисленные ниже. 
 
-    Группы безопасности Azure AD будут поддерживаться в Intune для всех типов развертывания. 
-    Группы безопасности Azure AD будут поддерживать не только группы пользователей, но и устройств.
-    Группы безопасности Azure AD будут поддерживать динамические группы с атрибутами устройств Intune. Например, вы сможете динамически группировать устройства на базе платформы, например iOS. Таким образом, при регистрации нового устройства iOS в вашей организации оно будет автоматически добавляться в динамическую группу устройств iOS.
-    Возможности совместного администрирования для управления группами Azure AD и Intune.
- *Роль администратора службы Intune* будет добавлена в Azure AD, чтобы разрешить администраторам служб в Intune выполнять задачи управления группами в Azure AD.

 
### <a name="what-intune-functionality-wont-be-available"></a>Какие функции Intune не будут доступны?
Работа с группами станет удобнее, но после миграции некоторые функции Intune будут недоступны.

#### <a name="group-management-functionality"></a>Функции управления группами

-   Вы не сможете исключать членов и группы при создании новой группы. Тем не менее динамические группы Azure AD позволяют использовать атрибуты для создания расширенных правил, с помощью которых можно исключать членов на основе условий. Например, можно создать расширенное правило, включающее в себя всех сотрудников отдела продаж в группе безопасности, кроме тех, у кого в названии должности есть слово "Помощник". Для этого используется следующее расширенное правило: `(user.department -eq "Sales") -and -not (user.jobTitle -contains "Assistant")`. Дополнительные сведения см. в разделе [Использование атрибутов для создания дополнительных правил](https://azure.microsoft.com/en-us/documentation/articles/active-directory-accessmanagement-groups-with-advanced-rules/).
-   Будет прекращена поддержка групп **Несгруппированные пользователи** и **Несгруппированные устройства**. Эти группы переноситься не будут.

#### <a name="group-dependent-functionality"></a>Функции, зависящие от групп

-   Роль администратора служб утратит разрешения на **управление группами**.
-   Вы не сможете группировать устройства Exchange ActiveSync.  Группа **Все управляемые устройства EAS** будет преобразована из представления группы в представление отчета.
-  Сведение с группами в отчетах будет недоступно.
-  Настраиваемое нацеливание правил уведомлений для групп будет недоступно.

### <a name="what-should-i-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
 Ниже приведены рекомендации, которые помогут вам упростить переход.
 
- Удалите все нежелательные или ненужные группы Intune перед миграцией.
- Оцените использование режима исключения в группах и рассмотрите возможность преобразования групп таким образом, чтобы отказаться от исключений (для этого также можно применить расширенные правила).
-  Если в организации есть администраторы, у которых нет разрешений на создание групп в Azure AD, попросите администратора Azure AD добавить их в роль Azure AD **Администратор служб Intune**.

Дополнительные сведения о группах безопасности Azure AD:
-  Общие сведения см. в разделе [Управление доступом к ресурсам с помощью групп Azure Active Directory](https://azure.microsoft.com/en-us/documentation/articles/active-directory-manage-groups/).
-  Сведения о способах создания групп Azure AD и управления ими см. в разделе [Управление группами в Azure Active Directory](https://azure.microsoft.com/en-us/documentation/articles/active-directory-accessmanagement-manage-groups/).
-  Дополнительные сведения о расширенных правилах для групп безопасности см. в разделе [Использование атрибутов для создания дополнительных правил](https://azure.microsoft.com/en-us/documentation/articles/active-directory-accessmanagement-groups-with-advanced-rules/).

> [!NOTE]
Вы могли заметить, что в документации по группам безопасности Azure AD не рассматривается создание групп для устройств. Эта возможность включается в Azure AD перед началом переноса групп Intune.

## <a name="migration-details"></a>Сведения о переносе
Здесь приведены сведения о том, как производится перенос групп Intune в группы безопасности Azure AD.

### <a name="migration-of-existing-groups"></a>Перенос существующих групп

| Группа Intune становится...|...группой безопасности Azure AD|
|-----------------------------------------------------------------------|-------------------------------------------------------------|
|Статические группы пользователей, к которым применяется политика Intune|Статические группы безопасности Azure AD, содержащие тех же пользователей|
|Динамические группы пользователей, к которым применяется политика Intune|Статические группы безопасности Azure AD с иерархией групп безопасности Azure AD|
|Статические группы устройств, к которым применяется политика Intune|Статические группы безопасности Azure AD с устройствами|
|Динамические группы устройств с атрибутами устройств, к которым применяется политика Intune|Динамические группы безопасности Azure AD с атрибутами устройств|
|Группа с условием включения|Отдельная статическая группа безопасности Azure AD, которая будет содержать группу, а также любые статические и динамические члены, разрешенные условием включения в Intune|
|Группа с условием исключения|...не переносится. Условия исключения при создании статических групп в Azure AD не поддерживаются. Условие исключения можно использовать при создании динамической группы в Azure AD.|
|Группы по умолчанию **Все пользователи**, **Несгруппированные пользователи**, **Все устройства**, **Несгруппированные устройства**, **Все компьютеры**, **Все мобильные устройства**, **All MDM managed devices** (Все устройства, управляемые MDM) и **All EAS managed devices** (Все устройства, управляемые EAS), используемые в политике Intune  |Группы безопасности Azure AD. Группы по умолчанию, которые не используются, должны по мере необходимости создаваться клиентом с помощью динамических групп.|

### <a name="changes-in-hierarchical-views"></a>Изменения в иерархических представлениях
Иерархическим представлением для групп в рамках отношения "родитель-потомок" Intune было отношение "надмножество-подмножество", но в Azure AD это не так. Дочерний объект может иметь члены, которых нет у родительского объекта. Кроме того, группы в Azure AD могут иметь циклический характер — родительская группа может быть дочерним объектом дочерней группы.

### <a name="attribute-conversion-during-migration"></a>Преобразование атрибутов во время миграции
Атрибуты являются свойствами устройства, которые можно использовать при определении групп. В этой таблице описано, как такие критерии будут перенесены в группы безопасности Azure AD.

| Атрибут Intune|Атрибут Azure AD|
|-----------------------------------------------------------------------|-------------------------------------------------------------|
|Атрибут подразделения для групп устройств|Атрибут подразделения для динамических групп. Значения атрибутов, относящиеся к каждому клиенту, которые предоставляются администратору посредством добавления в качестве одного из компонентов клиента для просмотра.|
|Атрибут доменного имени для групп устройств|Атрибут доменного имени для динамических групп. Значения атрибутов, относящиеся к каждому клиенту, которые предоставляются администратору посредством добавления в качестве одного из компонентов клиента для просмотра.|
|Группа безопасности в качестве атрибута для групп пользователей|Группы не могут быть атрибутами в динамических запросах Azure AD. Динамические группы могут содержать только атрибуты для определенного пользователя или устройства.|
|Атрибут Manager для групп пользователей|Расширенное правило для атрибута *manager* в динамических группах|
|Все пользователи из родительской группы пользователей|Статическая группа с этой группой в качестве члена|
|Все мобильные устройства из родительской группы устройств|Статическая группа с этой группой в качестве члена|
|Все мобильные устройства, к которым применяется прямое управление Microsoft Intune|Атрибут типа управления с "MDM" в качестве значения для динамической группы|
|Все мобильные устройства, управляемые EAS|Группировать устройства EAS в Azure AD нельзя. Группа **Все управляемые устройства EAS** будет преобразована из представления группы в представление отчета.|
|Вложенные группы в статических группах |Вложенные группы в статических группах|
|Вложенные группы в динамических группах|Динамическая группа с одним уровнем вложенности|


## <a name="migration-of-policies"></a>Перенос политик
Во время переноса групп вы продолжаете управлять политиками в консоли Intune. В консоли Intune появится ссылка на консоль управления Azure, где можно управлять группами. Политики продолжат развертываться в перенесенных группах безопасности Azure AD, используемыми параллельно со старыми группами Intune.

После переноса всего функционала Intune на портал управления Azure (ориентировочно в первом квартале 2017 г.) управление политиками и группами будет осуществляться там.
