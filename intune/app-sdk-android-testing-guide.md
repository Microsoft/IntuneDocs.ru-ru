# <a name="microsoft-intune-app-sdk-for-android-developers-testing-guide"></a>Руководство по тестированию для разработчиков по пакету SDK для приложений Intune для Android

Microsoft Intune App SDK for Android руководство по тестированию призвана помочь вам протестировать приложения Android под управлением Intune.  

## <a name="prerequisite-test-accounts"></a>Учетные записи тестирования готовности к установке
Можно создать новые учетные записи, с и без предварительно созданных данных. Создание новой учетной записи
1. Перейдите к [демонстрации Microsoft](https://demos.microsoft.com/environments/create/tenant) сайта. 
2. [Настройка Intune](https://docs.microsoft.com/intune/setup-steps) для включения управления мобильными устройствами (MDM).
3. [Создание пользователей](https://docs.microsoft.com/intune/users-add).
4. [Создание групп](https://docs.microsoft.com/intune/groups-add).
5. [Назначение лицензий](https://docs.microsoft.com/intune/licenses-assign) соответствующим образом для тестирования.


## <a name="azure-portal-policy-configuration"></a>Настройка политики Azure портала
[Создание и назначение политик защиты приложений](https://docs.microsoft.com/intune/app-protection-policies) в [на портале Azure колонку Intune](https://portal.azure.com/?feature.customportal=false#blade/Microsoft_Intune_Apps/MainMenu/14/selectedMenuItem/Overview). Ваш [политика конфигурации приложений](https://docs.microsoft.com/intune/app-configuration-policies-overview) можно также создать и назначить в колонке Intune.

> [!NOTE]
> Если приложение отсутствует на портале Azure, вы можете целевую с политикой, выбрав **дополнительные приложения** параметр и указав имя пакета в текстовом поле.

> [!IMPORTANT]
> Для политики конфигурации приложений для применения, регистрацию пользователя должна распространяться [политику защиты приложений Intune](https://docs.microsoft.com/intune/app-protection-policy).

## <a name="test-cases"></a>Тестовые случаи

Конфигурации и подтверждение действия описаны в следующих тестовых случаев. Используйте это руководство для устранения проблем приложений Android, управляемых Intune.

### <a name="required-pin-and-corporate-credentials"></a>Требовать PIN-код и корпоративных учетных данных

Вы можете потребовать ПИН-код для доступа к корпоративным ресурсам. Кроме того можно принудительно задать корпоративной проверки подлинности, чтобы пользователи могли использовать управляемые приложения. Следуйте инструкциям ниже, чтобы задать эти требования:

1. Настройка **требование ПИН-кода для доступа к** и **запрашивать корпоративные учетные данные для доступа к** для **Да**. Дополнительные сведения см. в статье [Параметры политики защиты приложений Android в Microsoft Intune](app-protection-policy-settings-android.md#access-requirements).
2. Убедитесь в следующем:
    - Запуск приложений должны представлять запрос для ввода ПИН и/или рабочей среде пользователя, который использовался во время регистрации с помощью корпоративного портала.
    - Сбой, чтобы представлять допустимый входа в систему может быть из-за неправильно настроенной манифеста android, специально значения для интеграции ADAL (SkipBroker ClientID и центра).
    - Сбой для представления запроса может быть из-за неправильно интегрированной `MAMActivity` значение. Дополнительные сведения о `MAMActivity`, см. в разделе [Microsoft Intune App SDK for руководство разработчика Android по](app-sdk-android.md).

> [!NOTE] 
> Если выше тест не работает, приведенных ниже тестах скорее всего также завершится неудачно. Просмотрите [SDK](app-sdk-android.md##sdk-integration) и [ADAL](app-sdk-android.md#configure-azure-active-directory-authentication-library-adal) интеграции.

### <a name="restrict-transferring-and-receiving-data-with-other-apps"></a>Ограничение передачи и приема данных с другими приложениями
Можно управлять передачей данных между корпоративным управляемые приложения следующим образом:

1. Задайте **разрешить приложению передавать данные другим приложениям** для **, управляемыми политикой**.
2. Для поля **Разрешить приложению принимать данные от других приложений** установите значение **Все приложения**. Использование намерения и поставщики содержимого будет оказывать воздействие этих политик.
3. Убедитесь в следующем:
    - Открывается из неуправляемого приложения в свои приложения функции правильно.
    - Допускается совместном использовании содержимого между управляемыми приложениями.
    - Передавать данные из управляемых приложений в неуправляемых приложениях (например, Chrome) блокируется.

### <a name="restrict-cut-copy-and-paste"></a>Блокировать вырезание, копирование и вставку
Можно ограничить в буфер обмена для приложений с управляемым кодом следующим образом:

1. Задайте **ограничение вырезания, копирования и вставки между приложениями** для **управляемые политикой, с добавлением из буфера**.
2. Убедитесь в следующем:
    - Копировать текст из приложения в управляемый блокируется неуправляемого приложения (например, сообщения).

### <a name="prevent-save-as"></a>Запретить **Сохранить как**
Вы можете управлять **Сохранить как** функциональные возможности, как показано ниже:

1. Если вашему приложению требуется [интегрированной элементы управления «Сохранить как»](app-sdk-android.md#example-determine-if-saving-to-device-or-cloud-storage-is-permitted), задайте **запретить «Сохранить как»** для **Да**.
2. Убедитесь в следующем:
    - Сохранить ограничивается только соответствующие управляемые расположения.

### <a name="file-encryption"></a>Шифрование файлов
Вы можете зашифровать данные на устройстве следующим образом:

1. Задайте **шифровать данные приложения** для **Да**.
2. Убедитесь в следующем:
    - Не влияет на поведение нормальную работу приложения.

### <a name="prevent-android-backups"></a>Запретить резервное копирование на Android
Резервное копирование приложения можно управлять следующим образом:

1. Если вы задали [интегрированных ограничений резервного копирования](app-sdk-android.md#protecting-backup-data), Настройка **резервное копирование на Android предотвратить** для **Да**.
2. Убедитесь в следующем:
    - Резервные копии запрещены.

### <a name="unenrollment"></a>Отмена регистрации
Вы можете выполнить удаленную очистку управляемых приложений из содержащего корпоративной электронной почты и документов и персональных данных расшифровывается в том случае, если он больше не управляются следующим образом:

1. На портале Azure [выполнять очистку](https://docs.microsoft.com/intune/apps-selective-wipe).
2. Если приложение не регистрирует для обработчиков стирания подтверждение следующих условий:
    - Полную очистку приложения происходит.
3. Если ваше приложение зарегистрировано для `WIPE_USER_DATA` или `WIPE_USER_AUXILARY_DATA`, подтвердите следующих условий:
    - Управляемые содержимое удаляется из приложения. Дополнительные сведения см. в разделе [пакета SDK для приложений Intune для Android developer guide — Выборочная очистка](app-sdk-android.md#selective-wipe).

### <a name="multi-identity"></a>Множественная идентификация
Интеграция [поддержка множественной идентификации](app-sdk-android.md#multi-identity-optional) отличается высоким уровнем риска, необходимо тщательно проверить. Наиболее распространенных проблем, которые будут из-за неправильной настройки удостоверения (контекст и уровень угрозы) и также файлы отслеживания (`MAMFileProtectionManager`).

Как минимум следует повторно проверить следующие сценарии для множественных удостоверений:

- **Сохранить как** для управляемых удостоверений политика работает правильно.
- Копирование и вставка, принудительно реализованы ограничения правильно из управляемого в личный.
- Шифруются только те данные, относящиеся к управляемым удостоверением и личные файлы не изменяются.
- Выборочная Очистка во время отмены регистрации только удаляет данные управляемого удостоверения.
- Конечному пользователю предлагается ввести условного запуска, если переход с неуправляемых на управляемую учетную запись (только в первый раз).

### <a name="app-configuration-optional"></a>Настройка приложения (необязательно)
Поведение управляемых приложений можно настроить следующим образом:

1. Если приложение использует все параметры конфигурации приложения, следует проверить, что приложение правильно обрабатывает все значения, которые вы (администратор) можно задать. [Политики конфигурации приложений](https://docs.microsoft.com/intune/app-configuration-policies-overview) можно создать и назначить с помощью Intune.
