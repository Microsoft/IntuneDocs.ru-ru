# <a name="endpoint-protection-settings-for-windows-10-and-later-in-intune"></a>Параметры защиты конечных точек для Windows 10 и более поздних версий в Intune

[!INCLUDE [azure_portal](./includes/azure_portal.md)]

Профиль защиты конечных точек позволяет управлять функциями безопасности, такими как BitLocker и Защитник Windows, на устройствах с Windows 10.

Ознакомившись с этой статьей, вы научитесь создавать профили защиты конечных точек.

> [!NOTE]
> Эти параметры не поддерживаются в выпусках Windows 10 Домашняя и Windows 10 Pro.

## <a name="windows-defender-application-guard"></a>Служба Application Guard в Защитнике Windows

Application Guard в Защитнике Windows изолирует среду в Microsoft Edge от всех сайтов, которые не определены в вашей организации как доверенные. Если сайт не относится к изолированной корпоративной сети, он открывается в виртуальном сеансе просмотра Hyper-V. Надежные узлы определяются границами сети, которые можно настроить в конфигурации устройства. 

Служба Application Guard доступна только для устройств с Windows 10 (64-разрядная версия). При использовании этого профиля устанавливается компонент Win32 для активации Application Guard.

- **Application Guard** — открытие неутвержденных сайтов в виртуализированном контейнере Hyper-V для просмотра.
- **Поведение буфера обмена** — выберите действия копирования и вставки, которые разрешено выполнять между локальным компьютером и виртуальным браузером Application Guard.
- **Внешнее содержимое на сайтах предприятия** — заблокируйте загрузку содержимого с недоверенных сайтов.
- **Печать из виртуального браузера** — разрешите печать содержимого из виртуального браузера в PDF, XPS, на локальных или сетевых принтерах.
- **Собрать журналы** — сбор журналов событий, происходящих в сеансе просмотра Application Guard.
- **Сохранять пользовательские данные браузера** — сохранение пользовательских данных (таких как пароли, избранное и файлы cookie), создаваемых в сеансе работы с виртуальным браузером Application Guard.
- **Ускорение графики** — быстрая загрузка веб-сайтов с большим количеством графики при работе в виртуальном сеансе Application Guard. Ускорение загрузки достигается за счет доступа к виртуальному графическому процессору.
- **Скачивание файлов в основную файловую систему** — разрешите пользователям скачивать файлы из виртуального браузера в основную операционную систему.

## <a name="windows-defender-firewall"></a>Брандмауэр Защитника Windows

### <a name="global-settings"></a>Глобальные параметры

Эти параметры применимы к сетям всех типов.

- **Протокол FTP** — блокирование протокола FTP с отслеживанием состояния.
- **Время простоя для сопоставлений безопасности перед удалением** — сопоставления безопасности удаляются, если сетевой трафик не регистрируется в течение *n* секунд.
- **Кодирование предварительных ключей** — общие ключи кодируются с использованием стандарта UTF-8.
- **Исключения IPsec** — настройте трафик, к которому не должна применяться защита IPsec. Доступные значения: **Коды типов ICMP IPv6 для обнаружения соседей**, **ICMP**, **Коды типов ICMP IPv6 для обнаружения маршрутизаторов** и **Сетевой трафик DHCP по протоколам IPv4 и IPv6**.
- **Проверка для списка отзыва сертификатов** — выбранное здесь значение определяет, как применяется список отзыва сертификатов. Доступные значения: **Отключить проверку списка отзыва сертификатов**, **Сбой проверки для списка отзыва сертификатов только при отозванном сертификате** и **Сбой проверки для списка отзыва сертификатов при любой ошибке**.
- **Рационально сопоставлять набор проверки подлинности по модулю ключей** — укажите, должны ли модули ключей игнорировать весь набор аутентификации, если они поддерживают не все пакеты аутентификации в этом наборе.
- **Очередь пакетов** — укажите, как обеспечивается масштабирование для ПО на принимающей стороне в сценарии приема зашифрованных данных и пересылки в виде обычного текста при использовании шлюза туннеля IPsec. Этот параметр позволяет сохранить очередность пакетов.

### <a name="network-settings"></a>Параметры сети

Эти параметры применяются к сетям определенных типов, включая следующие: **Доменная (рабочая) сеть**, **Частная сеть (с обнаружением)** и **Общедоступная сеть (без обнаружения)**.

#### <a name="general-settings"></a>Общие параметры

- **Брандмауэр Защитника Windows** — включите этот параметр, чтобы блокировать сетевой трафик.
- **Скрытый режим** — запрет работы брандмауэра в скрытом режиме. Блокировка скрытого режима позволяет также блокировать **исключение пакетов, защищенных IPsec**.
- **Экранированный** — включение этого параметра и параметра брандмауэра позволяет блокировать весь входящий трафик.
- **Одноадресные отклики на многоадресный широковещательный трафик** — блокирование одноадресных ответов на многоадресную рассылку. Обычно нет необходимости получать одноадресные ответы на сообщения многоадресной или широковещательной рассылки. Такие ответы могут означать, что злоумышленники выполняют атаку "отказ в обслуживании" (DOS) или пытаются проверить, какие компьютеры работают в сети.
- **Входящие уведомления** — не показывать уведомления пользователям, если приложению запрещено ожидать передачу данных через порт.
- **Действие по умолчанию для входящих подключений** — блокирование действия, которое выполняется по умолчанию в брандмауэре для входящих подключений.

#### <a name="rule-merging"></a>Объединение правил

- **Авторизованные правила брандмауэра Защитника Windows для приложения из локального хранилища** — распознавание и применение авторизованных правил брандмауэра в локальном хранилище.
- **Глобальные правила для портов брандмауэра Защитника Windows из локального хранилища** — распознавание и применение правил брандмауэра в отношении глобальных портов в локальном хранилище.
- **Правила брандмауэра Защитника Windows из локального хранилища** — распознавание и применение глобальных правил брандмауэра в локальном хранилище.
- **Правила IPsec из локального хранилища** — применение правил защиты подключений из локального хранилища независимо от схемы или версии правила.

## <a name="windows-defender-smartscreen-settings"></a>Параметры SmartScreen Защитника Windows

- **SmartScreen для приложений и файлов** — применение Windows SmartScreen для выполнения файлов и приложений.
- **Выполнение непроверенных файлов** — запрет запуска пользователями файлов, которые не проверены Windows SmartScreen.

## <a name="windows-encryption"></a>Шифрование Windows

### <a name="windows-settings"></a>Параметры Windows

Следующие два параметра применяются ко всем версиям Windows 10:

- **Шифрование устройств** — когда этот параметр включен, пользователям предлагается включить шифрование устройств. Кроме того, им предлагается убедиться, что не включено шифрование от другого поставщика. Если включить шифрование Windows когда активен другой метод шифрования, устройство может работать нестабильно.
- **Шифрование карты памяти** — включите этот параметр, чтобы шифровать все съемные карты памяти на устройстве.


### <a name="bitlocker-base-settings"></a>Базовые параметры BitLocker

Базовые параметры — это универсальные параметры BitLocker для всех типов дисков с данными. Параметры групповой политики BitLocker определяют то, какие задачи шифрования дисков или параметры конфигурации пользователь может изменять для всех типов дисков с данными.

- **Предупреждение при шифровании других дисков** — отключите вывод предупреждения об использовании другого типа шифрования дисков на компьютерах пользователей.
- **Настройка методов шифрования** — включите этот параметр, чтобы настроить алгоритмы шифрования для операционной системы, данных и съемных дисков.
  - **Шифрование дисков операционных систем**— выберите метод шифрования для дисков операционной системы. Рекомендуется использовать алгоритм XTS-AES.
  - **Шифрование несъемных дисков с данными** — выберите метод шифрования для фиксированных (встроенных) дисков с данными. Рекомендуется использовать алгоритм XTS-AES.
  - **Шифрование съемных дисков с данными** — выберите метод шифрования для съемных дисков с данными. Если съемный диск используется с устройствами, работающими не под управлением Windows 10, рекомендуется использовать алгоритм AES-CBC.

### <a name="bitlocker-os-drive-settings"></a>Параметры дисков ОС в BitLocker

Эти параметры применяются к дискам с данными операционной системы.

- **Дополнительная проверка подлинности при запуске** — настройте требования к аутентификации при запуске компьютера, включая использование доверенного платформенного модуля (TPM).
  - **BitLocker с несовместимой микросхемой доверенного платформенного модуля**
  - **Запуск совместимого доверенного платформенного модуля** — здесь вы можете разрешить, запретить или потребовать использование доверенного платформенного модуля.
  - **ПИН-код запуска совместимого доверенного платформенного модуля** — здесь вы можете разрешить, запретить или потребовать использовать ПИН-кода запуска для доверенного платформенного модуля.
  - **Ключ запуска совместимого доверенного платформенного модуля** — здесь вы можете разрешить, запретить или потребовать использование ключа запуска для доверенного платформенного модуля.
  - **Ключ и ПИН-код запуска совместимого доверенного платформенного модуля** — здесь вы можете разрешить, запретить или потребовать использование ключа и ПИН-кода запуска для доверенного платформенного модуля.
- **Минимальная длина ПИН-кода** — включите этот параметр, чтобы настроить минимальную длину ПИН-кода запуска для доверенного платформенного модуля (TPM).
  - **Минимальное число символов** — введите обязательное число символов для ПИН-кода запуска из диапазона **4**-**20**.
- **Восстановление дисков ОС** — включите этот параметр определяет, как будут восстанавливаться диски операционной системы, защищенных с помощью BitLocker, при отсутствии требуемых сведений о запуске.
  - **Агент восстановления данных на основе сертификатов** — включите этот параметр, чтобы агенты восстановления данных можно было использовать вместе с дисками операционной системы, защищенными BitLocker.
  - **Создание пользователем пароля восстановления** — укажите, является ли создание 48-значного пароля восстановления разрешенной, запрещенной или обязательной операцией для пользователей.
  - **Создание пользователем ключа восстановления** — укажите, является ли создание 256-значного ключа восстановления разрешенной, запрещенной или обязательной операцией для пользователей.
  - **Варианты восстановления в мастере установки BitLocker** — включите этот параметр, чтобы запретить пользователям просматривать или изменять параметры восстановления при включении BitLocker.
  - **Сохранять сведения о восстановлении BitLocker в AD DS** — включите хранение сведений о восстановлении BitLocker в Active Directory.
  - **Сведения о восстановлении BitLocker, хранящиеся в AD DS** — укажите, какие элементы сведений о восстановлении BitLocker хранятся в Active Directory. Выберите один из следующих типов.
    - **Архивировать пароли и пакеты ключей восстановления**
    - **Архивировать только пароли восстановления**
  - **Сохранить сведения о восстановлении в AD DS перед включением BitLocker** — включите этот параметр, чтобы разрешить пользователям включать BitLocker, только если устройство присоединено к домену, а сведения о восстановлении BitLocker успешно сохранены в Active Directory.
- **Предзагрузочное сообщение о восстановлении и URL-адрес** — включите этот параметр, чтобы настроить сообщение и URL-адрес, отображаемые на предзагрузочном экране восстановления ключей.
  - **Предзагрузочное сообщение о восстановлении** — укажите, как будет отображаться для пользователей предзагрузочное сообщение о восстановлении. Выберите один из следующих типов.
    - **Использовать сообщение о восстановлении и URL-адрес по умолчанию**
    - **Использовать пустое сообщение о восстановлении и URL-адрес**
    - **Использовать пользовательское сообщение о восстановлении**
    - **Использовать пользовательский URL-адрес восстановления**

### <a name="bitlocker-fixed-data-drive-settings"></a>Параметры жестких дисков с данными в BitLocker

- **Доступ на запись к несъемным дискам с данными, не защищенным BitLocker** — если этот параметр включен, запись данных будет разрешена только на те фиксированные или встроенные диски, для которых включена защита BitLocker.
- **Восстановление несъемных дисков** — этот параметр определяет, как выполняется восстановление жестких дисков, защищенных с помощью BitLocker, при отсутствии требуемых сведений о запуске.
  - **Агент восстановления данных** — включите этот параметр, чтобы агенты восстановления данных можно было использовать вместе с жесткими дисками, защищенными BitLocker.
  - **Создание пользователем пароля восстановления** — выберите, является ли создание 48-значного пароля восстановления разрешенной, запрещенной или обязательной операцией для пользователей.  
  - **Создание пользователем ключа восстановления** — выберите, является ли создание 256-разрядного ключа восстановления разрешенной, запрещенной или обязательной операцией для пользователей.
  - **Варианты восстановления в мастере установки BitLocker** — включите этот параметр, чтобы запретить пользователям просматривать или изменять параметры восстановления при включении BitLocker.
  - **Сохранять сведения о восстановлении BitLocker в AD DS** — включите хранение сведений о восстановлении BitLocker в Active Directory.
  - **Сведения о восстановлении BitLocker в AD DS** — укажите, какие элементы сведений о восстановлении BitLocker хранятся в Active Directory. Выберите один из следующих типов.
    - **Архивировать пароли и пакеты ключей восстановления**
    - **Архивировать только пароли восстановления**
  - **Сохранить сведения о восстановлении в AD DS перед включением BitLocker** — включите этот параметр, чтобы разрешить пользователям включать BitLocker, только если устройство присоединено к домену, а сведения о восстановлении BitLocker успешно сохранены в Active Directory.

### <a name="bitlocker-removable-data-drive-settings"></a>Параметры удаляемых дисков с данными в BitLocker

- **Доступ на запись к съемным дискам с данными, не защищенным BitLocker** — укажите, требуется ли шифрование BitLocker для съемных запоминающих устройств.
  - **Доступ на запись к устройствам, настроенным в другой организации** — укажите, можно ли записывать данные на съемные диски, принадлежащие другой организации.

## <a name="windows-defender-exploit-guard"></a>Exploit Guard в Защитнике Windows

С помощью [Exploit Guard в Защитнике Windows](https://docs.microsoft.com/windows/threat-protection/windows-defender-exploit-guard/windows-defender-exploit-guard) можно управлять приложениями, используемыми сотрудниками, и снижать их уязвимость.

### <a name="attack-surface-reduction"></a>Уменьшение уязвимой зоны

- **Помечать кражу учетных записей из локальной системы безопасности Windows**

Позволяет [блокировать действия и приложения](https://docs.microsoft.com/windows/threat-protection/windows-defender-exploit-guard/attack-surface-reduction-exploit-guard), которые обычно используются вредоносным ПО для поиска уязвимостей и заражения компьютеров.

#### <a name="rules-to-prevent-office-macro-threats"></a>Правила предотвращения угроз, связанных с макросами Office

Вы можете блокировать выполнение следующих действий приложениями Office:

- **внедрение приложений Office в другие процессы (без исключений);**
- **создание исполняемого содержимого приложениями или макросами Office;**
- **запуск дочерних процессов приложениями Office;**
- **импорт Win32 из кода макросов Office.**

#### <a name="rules-to-prevent-script-threats"></a>Правила предотвращения угроз, связанных со скриптами

Для защиты от угроз, связанных со скриптами, можно блокировать следующие компоненты и действия:

- **код JS, VBS, PS или макросов с маскированием;**
- **выполнение полезных данных, скачанных из Интернета, кодом JS или VBS (без исключений).**
- **создание процессов с помощью команд PSExec и WMI**;
- **ненадежные и неподписанные процессы, запускаемые с USB-накопителей**;
- **исполняемые файлы, которые не соответствуют условиям распространенности, возраста или включения в список доверенных**.

#### <a name="rules-to-prevent-email-threats"></a>Правила предотвращения угроз, связанных с электронной почтой

Для защиты от угроз, связанных с электронной почтой, можно блокировать следующее действие:

- **выполнение исполняемого содержимого (EXE, DLL, PS, JS, VBS и т. д.), полученного по электронной почте через веб-клиент или почтовую программу (без исключений).**

 2. ## <a name="windows-defender-application-control"></a>Управление приложениями в Защитнике Windows

При использовании режима **Только аудит** приложения не блокируются. В режиме **Только аудит** в журнал записываются все события из журналов событий локальных клиентов.

После включения управление приложениями можно отключить, только изменив режим **Принудительно** на **Только аудит**. Смена режима **Принудительно** на **Не настроено** не приводит к отключению управления приложениями на назначенных устройствах.

## <a name="windows-defender-credential-guard"></a>Credential Guard в Защитнике Windows
Credential Guard в Защитнике Windows защищает от атак, направленных на кражу учетных данных. Он изолирует секретные данные и разрешает доступ к ним только системному программному обеспечению с соответствующими привилегиями.

Настройки **Credential Guard** содержат следующие параметры.

- **Отключено** — удаленное отключение Credential Guard при включении с параметром **Включено без блокировки UEFI**.
- **Включено с блокировкой UEFI** — блокировка удаленного отключения Credential Guard через раздел реестра или групповую политику.

    > [!NOTE]
    > Если вы выберете этот параметр, для отключения Credential Guard потребуется **отключить** групповую политику. После отключения физически удалите сведения о конфигурации UEFI с каждого компьютера. Компонент Credential Guard остается включенным, пока сохраняется конфигурация UEFI.

- **Включено без блокировки UEFI** — удаленное отключение компонента Credential Guard с помощью групповой политики. Этот параметр можно использовать только на устройствах под управлением Windows 10 версии 1511 и выше.

При включении Credential Guard автоматически включаются следующие обязательные компоненты:

- **Включить средство обеспечения безопасности на основе виртуализации** — активируется при следующей перезагрузке. Средство обеспечения безопасности на основе виртуализации использует гипервизор Windows для поддержки служб безопасности.
- **Безопасная загрузка с прямым доступом к памяти** — включает средство обеспечения безопасности на основе виртуализации с безопасной загрузкой и прямым доступом к памяти (DMA). Средства защиты DMA требуют аппаратной поддержки и применяются только на правильно настроенных устройствах.

- **Require CTRL+ALT+DEL to log on** (Требовать нажатия CTRL+ALT+DEL для входа в систему) — если настроен этот параметр, пользователь должен нажать сочетание клавиш CTRL+ALT+DEL, чтобы войти.

- **Logon message text** (Текст сообщения при входе в систему) — здесь можно настроить текст сообщения для пользователя, пытающегося войти в систему.

### <a name="network-access-and-security"></a>Доступ к сети и безопасность

- **Anonymous access to Named Pipes and Shares** (Анонимный доступ к именованным каналам и общим файловым ресурсам) — запрет анонимного доступа к параметрам общих файловых ресурсов и именованных каналов. Эта настройка применяется только к параметрам, допускающим анонимный доступ.
- **Anonymous enumeration of SAM accounts** (Анонимное перечисление учетных записей SAM) — разрешает анонимным пользователям перечислять учетные записи SAM. Windows разрешит анонимным пользователям получать списки имен доменных учетных записей и сетевых папок.
- **Anonymous enumeration of SAM accounts and shares** (Анонимное перечисление учетных записей SAM и общих папок) — позволяет запретить анонимным пользователям получать списки учетных записей SAM и сетевых папок. Windows разрешит анонимным пользователям получать списки имен доменных учетных записей и сетевых папок.
- **LAN Manager hash value stored on password change** (Сохранение хэш-значения LAN Manager при изменении пароля) — выберите, будет ли сохраняться хэш-значение LAN Manager для нового пароля при следующей смене пароля. По умолчанию оно не сохраняется.
- **Запросы проверки подлинности PKU2U** — блокировка использования сетевых удостоверений в запросах аутентификации PKU2U к этому устройству.
- **Restrict remote RPC connections to SAM** (Ограничить удаленные подключения RPC к SAM) — измените используемую по умолчанию строку SDDL (языка определения дескрипторов безопасности), чтобы разрешить или запретить удаленные вызовы SAM для конкретных пользователей и (или) групп.
- **Дескриптор безопасности**

### <a name="recovery-console-and-shutdown"></a>Консоль восстановления и завершение работы

- **Clear virtual memory pagefile when shutting down** (Очистка файла подкачки виртуальной памяти при выключении) — файл подкачки виртуальной памяти будет очищаться, когда устройство завершает работу.
- **Завершение работы без входа в систему** — блокирует возможность завершить работу компьютера прямо на экране входа в систему Windows. Если выбран этот параметр, пользователь должен выполнить вход на компьютер, прежде чем завершать работу системы.

### <a name="user-account-control"></a>Контроль учетных записей пользователей

- **Уровень целостности UIA при запуске из небезопасных расположений** — позволяет выполнять приложения из незащищенного расположения в файловой системе на уровне целостности UIAccess.
- **Virtualize file and registry write failures to per-user locations** (Виртуализация в размещение пользователя при сбоях записи в файл или реестр) — перенаправляет неудачные операции записи, выполняемые приложениями, в определенные расположения в реестре и файловой системе. Если этот параметр не установлен, происходит сбой приложения.
- **Only elevate executable files that are signed and validated** (Повышение привилегий только для подписанных и проверенных файлов) — принудительное подтверждение пути сертификации PKI перед разрешением выполнения для исполняемого файла.

#### <a name="uia-elevation-prompt-behavior-settings"></a>Параметры для приглашения на повышение привилегий в модели автоматизации пользовательского интерфейса Microsoft Windows

- **Elevation prompt for admins** (Запрос на повышение прав для администраторов) — определяет поведение запроса на повышение прав для администраторов в режиме одобрения администратором:
  - **Повышение без запроса**
  - **Запрос учетных данных на безопасном рабочем столе**
  - **Запрос согласия на безопасном рабочем столе**
  - **Запрос учетных данных**
  - **Запрос согласия**
  - **Не настроено** (запрос согласия для двоичных файлов, не входящих в состав Windows)
- **Elevation prompt for standard users** (Запрос на повышение прав для обычных пользователей) — определяет поведение запроса на повышение прав для администраторов в режиме одобрения администратором:
  - **Автоматически отклонять запросы на повышение прав**
  - **Запрос учетных данных на безопасном рабочем столе**
  - **Не настроено** (запрос учетных данных)
- **Route elevation prompts to user’s interactive desktop** (Направлять запросы на повышение прав в интерактивный рабочий стол пользователя) — все запросы на повышение прав будут передаваться на интерактивный рабочий стол, а не безопасный рабочий стол. Используются параметры политики запросов, установленные для администраторов и обычных пользователей.
- **Elevated prompt for app installations** (Запрос на повышение прав для установки приложений) — если установка приложения требует повышенных привилегий, система предложит ввести учетные данные администратора.
- **Запрос на повышение привилегий UIA без использования безопасного рабочего стола** — приложения UIAccess смогут запрашивать повышение прав, не используя безопасный рабочий стол.

#### <a name="admin-approval-mode-settings"></a>Параметры режима одобрения администратором

- **Режим одобрения администратором для встроенной учетной записи администратора** — определяет, будет ли встроенная учетная запись администратора использовать режим одобрения администратором или выполнять все приложения с полными правами администратора.
- **Запуск всех администраторов в режиме одобрения администратором** — определяет, будет ли использоваться режим одобрения администратором и все параметры политики контроля учетных записей.

### <a name="microsoft-network-client"></a>Клиент сети Microsoft

- **Связь с цифровой подписью (с согласия сервера)** — определяет, будет ли клиент SMB согласовывать подписывание SMB-пакетов. Если этот параметр включен (режим по умолчанию), клиент сети Microsoft запрашивает у сервера подписывание SMB-пакетов перед началом сессии. Если на сервере включено подписывание пакетов, это будет согласовано для сессии. Если этот параметр политики отключен, клиент SMB никогда не обращается за подписыванием SMB-пакетов.
- **Send unencrypted password to third-party SMB servers** (Отправлять незашифрованный пароль серверам SMB сторонних разработчиков) — если этот параметр включен, перенаправитель Server Message Block (SMB) может отправлять пароли в виде простого текста тем серверам SMB сторонних разработчиков, которые не поддерживают шифрование паролей для аутентификации.

### <a name="microsoft-network-server"></a>Сервер для сетей Майкрософт

- **Связь с цифровой подписью (с согласия клиента)** — определяет, будет ли сервер SMB согласовывать подписывание SMB-пакетов с теми клиентами, которые его запрашивают. Если этот параметр включен, сервер для сетей Майкрософт согласовывает подписывание SMB-пакетов по запросу клиента. Это означает, что подписывание согласовывается с теми клиентами, для которых включена связь с цифровой подписью. Если этот параметр политики отключен (по умолчанию), клиент SMB никогда не обращается за подписыванием SMB-пакетов.
- **Связь с цифровой подписью (всегда)** — определяет, будет ли сервер SMB требовать подписывание пакетов. Если включен этот параметр, сервер для сетей Майкрософт не устанавливает связь с теми клиентами сети Microsoft, которые не подтвердят подписывание SMB-пакетов. Если параметр отключен (по умолчанию), подписывание SMB-пакетов согласовывается между клиентом и сервером.
