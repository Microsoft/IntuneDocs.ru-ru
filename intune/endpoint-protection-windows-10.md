# <a name="endpoint-protection-settings-for-windows-10-and-later-in-microsoft-intune"></a>Параметры защиты конечных точек для Windows 10 и более поздних версий в Microsoft Intune

[!INCLUDE[azure_portal](./includes/azure_portal.md)]

Профиль защиты конечных точек позволяет управлять функциями безопасности, такими как BitLocker и Защитник Windows, на устройствах с Windows 10.

Ознакомившись с этой статьей, вы научитесь создавать профили защиты конечных точек.

> [!Note]
> Эти параметры не поддерживаются в выпусках Windows 10 Домашняя и Windows 10 Pro.

## <a name="create-an-endpoint-protection-profile"></a>Создание профиля защиты конечных точек

1. Зарегистрируйтесь на портале Azure.
2. Выберите **Больше служб** > **Мониторинг и управление** > **Intune**.
3. В колонке **Intune** выберите пункт **Конфигурация устройства**.
2. В колонке **Конфигурация устройства** выберите **Управление** > **Профили**.
3. В колонке профилей выберите **Создание профиля**.
4. В колонке **Создание профиля** введите **имя** и **описание** для профиля ограничения устройства.
5. В раскрывающемся списке **Платформа** выберите **Windows 10 и более поздних версий**.
6. В раскрывающемся списке **Тип профиля** выберите **Endpoint Protection**.
7. Настройте нужные параметры. Сведения в этой статье помогут вам понять назначение каждого из параметров. После этого нажмите кнопку **ОК**.
8. Вернитесь в колонку **Создание профиля** и выберите команду **Создать**.

Созданный профиль отобразится в колонке со списком профилей.

## <a name="windows-defender-application-guard"></a>Служба Application Guard в Защитнике Windows

Служба Application Guard доступна только для устройств с Windows 10 (64-разрядная версия). При использовании этого профиля устанавливается компонент Win32 для активации Application Guard.

- **Application Guard** — открытие неутвержденных сайтов в виртуализированном контейнере для просмотра Hyper-V.
- **Поведение буфера обмена** — выберите действия копирования и вставки, которые разрешено выполнять между локальным компьютером и виртуальным браузером Application Guard.
- **Внешнее содержимое на сайтах предприятия** — блокирование загрузки содержимого с недоверенных сайтов.
- **Печать из виртуального браузера** — разрешить печать содержимого из виртуального браузера в PDF, XPS, на локальных или сетевых принтерах.
- **Собрать журналы** — сбор журналов событий, происходящих в рамках сеанса просмотра Application Guard.
- **Сохранять пользовательские данные браузера** — разрешить сохранение пользовательских данных (таких как пароли, избранное и файлы cookie), создаваемых во время сеанса работы с виртуальным браузером Application Guard.


## <a name="windows-defender-firewall"></a>Брандмауэр Защитника Windows

### <a name="global-settings"></a>Глобальные параметры

Эти параметры применимы к сетям всех типов.

- **Протокол FTP** — блокирование протокола FTP с отслеживанием состояния.
- **Время простоя для сопоставлений безопасности перед удалением** — сопоставления безопасности удаляются, если сетевой трафик не регистрируется в течение *n* с.
- **Кодирование предварительных ключей** — общие ключи кодируются с использованием стандарта UTF-8.
- **Исключения IPsec** — настройте трафик, к которому не должна применяться защита IPsec, включая **Коды типов ICMP IPv6 для обнаружения соседей**, **ICMP**, **Коды типов ICMP IPv6 для обнаружения маршрутизаторов** и **Сетевой трафик DHCP по протоколам IPv4 и IPv6**.
- **Проверка для списка отзыва сертификатов** — задайте значение, определяющее то, как должен проверяться список отзыва сертификатов, включая **Отключить проверку списка отзыва сертификатов**, **Сбой проверки для списка отзыва сертификатов только при отозванном сертификате** и **Сбой проверки для списка отзыва сертификатов при любой ошибке**.
- **Рационально сопоставлять набор проверки подлинности по модулю ключей** — укажите, должны ли модули ключей игнорировать весь набор проверки подлинности, если они поддерживают не все пакеты проверки подлинности в этом наборе.
- **Очередь пакетов** — укажите, как обеспечивается масштабирование для ПО на принимающей стороне в сценарии приема зашифрованных данных и пересылки в виде обычного текста при использовании шлюза туннеля IPsec. Это позволяет сохранить очередность пакетов.

### <a name="network-settings"></a>Параметры сети

Эти параметры применяются к сетям определенных типов, включая следующие: **Доменная (рабочая) сеть**, **Частная сеть (с обнаружением)** и **Общедоступная сеть (без обнаружения)**.

#### <a name="general-settings"></a>Общие параметры

- **Брандмауэр Защитника Windows** — включите этот параметр, чтобы блокировать сетевой трафик.
- **Скрытый режим** — запретить работу брандмауэра в скрытом режиме. Это позволяет также блокировать **исключение пакетов, защищенных IPsec**.
- **Экранированный** — включение этого параметра и параметра брандмауэра позволяет блокировать весь входящий трафик.
- **Одноадресные отклики на многоадресный широковещательный трафик** — блокировать одноадресные ответы на многоадресную рассылку. Как правило, получать одноадресные ответы на многоадресную рассылку или широковещательные сообщения нежелательно, так как такие ответы могут свидетельствовать об атаке типа "отказ в обслуживании" или о том, что злоумышленник пытается найти работающий компьютер.
- **Входящие уведомления** — не показывать уведомления пользователям, если приложению запрещено ожидание передачи данных через порт.
- **Действие по умолчанию для входящих подключений** — блокирование действия, которое выполняется по умолчанию в брандмауэре для входящих подключений.

#### <a name="rule-merging"></a>Объединение правил

- **Авторизованные правила брандмауэра Защитника Windows для приложения из локального хранилища** — распознавание и применение авторизованных правил брандмауэра в локальном хранилище.
- **Глобальные правила для портов брандмауэра Защитника Windows из локального хранилища** — распознавание и применение правил брандмауэра в отношении глобальных портов в локальном хранилище.
- **Правила брандмауэра Защитника Windows из локального хранилища** — распознавание и применение глобальных правил брандмауэра в локальном хранилище.
- **Правила IPsec из локального хранилища** — применение правил защиты подключений из локального хранилища вне зависимости от схемы или версии правила.

## <a name="windows-defender-smartscreen-settings"></a>Параметры SmartScreen Защитника Windows

- **SmartScreen для приложений и файлов** — включение Windows SmartScreen для выполнения файлов и приложений.
- **Выполнение непроверенных файлов** — запрет запуска пользователями файлов, которые не были проверены Windows SmartScreen.

## <a name="windows-encryption"></a>Шифрование Windows

### <a name="windows-settings"></a>Параметры Windows

Эти параметры применяются ко всем версиям Windows 10.

- **Шифрование устройств** — когда этот параметр включен, пользователям предлагается включить шифрование устройств. Кроме того, им предлагается убедиться, что не включено шифрование от другого поставщика. Если включить шифрование Windows когда активен другой метод шифрования, устройство может работать нестабильно.
- **Шифрование карты памяти** — включите этот параметр для шифрования всех съемных карт памяти, используемых на устройстве.


### <a name="bitlocker-base-settings"></a>Базовые параметры BitLocker

Базовые параметры — это универсальные параметры BitLocker для всех типов дисков с данными. Параметры групповой политики BitLocker определяют то, какие задачи шифрования дисков или параметры конфигурации пользователь может изменять для всех типов дисков с данными.

- **Предупреждение при шифровании других дисков** — отключить вывод предупреждения для другого типа шифрования дисков на компьютерах пользователей.
- **Настройка методов шифрования**. Включите этот параметр, чтобы настроить алгоритмы шифрования для операционной системы, данных и съемных дисков.
    - **Шифрование дисков операционных систем**. Выберите метод шифрования для дисков операционной системы. Рекомендуется использовать алгоритм XTS-AES.
    - **Шифрование несъемных дисков с данными**. Выберите метод шифрования для фиксированных (встроенных) дисков с данными. Рекомендуется использовать алгоритм XTS-AES.
    - **Шифрование съемных дисков с данными**. Выберите метод шифрования для съемных дисков с данными. Если съемный диск используется с устройствами, работающими не под управлением Windows 10, рекомендуется использовать алгоритм AES-CBC.

### <a name="bitlocker-os-drive-settings"></a>Параметры дисков ОС в BitLocker

Эти параметры применяются к дискам с данными операционной системы.

- **Дополнительная проверка подлинности при запуске** — настройте требования к проверке подлинности при запуске компьютера, включая использование доверенного платформенного модуля (TPM).
    - **BitLocker с несовместимой микросхемой доверенного платформенного модуля**
    - **Запуск совместимого доверенного платформенного модуля** — укажите, является ли микросхема доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным компонентом.
    - **ПИН-код запуска совместимого доверенного платформенного модуля** — укажите, является ли использование ПИН-кода запуска для микросхемы доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным.
    - **Ключ запуска совместимого доверенного платформенного модуля** — укажите, является ли использование ключа запуска для микросхемы доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным.
    - **Ключ и ПИН-код запуска совместимого доверенного платформенного модуля** — укажите, является ли использование ПИН-кода и ключа запуска для микросхемы доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным.
- **Минимальная длина ПИН-кода**. Включите этот параметр, чтобы настроить минимальную длину ПИН-кода запуска для доверенного платформенного модуля (TPM).
    - **Минимальное число символов**. Введите обязательное число символов для ПИН-кода запуска из диапазона **4**-**20**.
- **Восстановление дисков ОС** — включите этот параметр, чтобы управлять восстановлением дисков операционной системы, защищенных с помощью BitLocker, при отсутствии требуемых сведений о запуске.
    - **Агент восстановления данных на основе сертификатов**. Включите этот параметр, чтобы агенты восстановления данных можно было использовать вместе с дисками операционной системы, защищенными BitLocker.
    - **Создание пользователем пароля восстановления**. Укажите, является ли создание 48-значного пароля восстановления разрешенной, запрещенной или обязательной операцией для пользователей.
    - **Создание пользователем ключа восстановления**. Укажите, является ли создание 256-разрядного ключа восстановления разрешенной, запрещенной или обязательной операцией для пользователей.
    - **Варианты восстановления в мастере установки BitLocker** — включите этот параметр, чтобы запретить пользователям просматривать или изменять параметры восстановления при включении BitLocker.
    - **Сохранять сведения о восстановлении BitLocker в AD DS**. Включает хранение сведений о восстановлении BitLocker в Active Directory.
    - **Сведения о восстановлении BitLocker, хранящиеся в AD DS** — укажите, какие части сведений о восстановлении BitLocker хранятся в Active Directory. Выберите один из следующих типов.
        - **Архивировать пароли и пакеты ключей восстановления**
        - **Архивировать только пароли восстановления**
    - **Сохранить сведения о восстановлении в AD DS перед включением BitLocker** — включите этот параметр, чтобы разрешить пользователям включать BitLocker только в том случае, если устройство присоединено к домену, а сведения о восстановлении BitLocker успешно сохранены в Active Directory.
- **Предзагрузочное сообщение о восстановлении и URL-адрес** — включите этот параметр, чтобы настроить сообщение и URL-адрес, отображаемые на предзагрузочном экране восстановления ключей.
    - **Предзагрузочное сообщение о восстановлении**. Настройте способ отображения предзагрузочного сообщения о восстановлении, выводимого пользователям. Выберите один из следующих типов.
        - **Использовать сообщение о восстановлении и URL-адрес по умолчанию**
        - **Использовать пустое сообщение о восстановлении и URL-адрес**
        - **Использовать пользовательское сообщение о восстановлении**
        - **Использовать пользовательский URL-адрес восстановления**

    - **Создание пользователем ключа восстановления**. Укажите, является ли создание 256-разрядного ключа восстановления разрешенной, запрещенной или обязательной операцией для пользователей.

### <a name="bitlocker-removable-data-drive-settings"></a>Параметры удаляемых дисков с данными в BitLocker

- **Доступ на запись к съемным дискам с данными, не защищенным BitLocker** — укажите, требуется ли шифрование BitLocker для съемных запоминающих устройств.
  - **Доступ на запись к устройствам, настроенным в другой организации** — укажите, можно ли производить запись на съемные диски с данными, принадлежащие другой организации.

## <a name="windows-defender-exploit-guard"></a>Exploit Guard в Защитнике Windows

С помощью [Exploit Guard в Защитнике Windows](https://docs.microsoft.com/windows/threat-protection/windows-defender-exploit-guard/windows-defender-exploit-guard) можно управлять приложениями, используемыми сотрудниками, и снижать их уязвимость.

### <a name="attack-surface-reduction"></a>Уменьшение уязвимой зоны

Позволяет [блокировать действия и приложения](https://docs.microsoft.com/windows/threat-protection/windows-defender-exploit-guard/attack-surface-reduction-exploit-guard), которые обычно используются вредоносным ПО для поиска уязвимостей и заражения компьютеров.

#### <a name="rules-to-prevent-office-macro-threats"></a>Правила предотвращения угроз, связанных с макросами Office

Вы можете блокировать выполнение следующих действий приложениями Office:

- **внедрение приложений Office в другие процессы (без исключений);**
- **создание исполняемого содержимого приложениями или макросами Office;**
- **запуск дочерних процессов приложениями Office;**
- **импорт Win32 из кода макросов Office.**

#### <a name="rules-to-prevent-script-threats"></a>Правила предотвращения угроз, связанных со скриптами

Для защиты от угроз, связанных со скриптами, можно блокировать следующие компоненты и действия:

- **код JS, VBS, PS или макросов с маскированием;**
- **выполнение полезных данных, скачанных из Интернета, кодом JS или VBS (без исключений).**

#### <a name="rules-to-prevent-email-threats"></a>Правила предотвращения угроз, связанных с электронной почтой

Для защиты от угроз, связанных с электронной почтой, можно блокировать следующее действие:

- **выполнение исполняемого содержимого (EXE, DLL, PS, JS, VBS и т. д.), полученного по электронной почте через веб-клиент или почтовую программу (без исключений).**

#### <a name="attack-surface-reduction-exceptions"></a>Исключения из правил сокращения потенциальных направлений атак

- **Файлы и папки, которые следует исключить из правил уменьшения уязвимой зоны** — импортируйте или добавьте список расположений, исключаемых из настроенных правил.

### <a name="controlled-folder-access"></a>Контролированный доступ к папкам

Позволяет защищать ценные данные от доступа со стороны вредоносных приложений, а также от других угроз, таких как программы-шантажисты.

- **Защита папок** — защита файлов и папок от нежелательного изменения вредоносными приложениями. Вы можете импортировать **список приложений, имеющих доступ к защищенным папкам**, или добавить их в вручную. Вы также можете отправить **список дополнительных папок, требующих защиты**, или добавить их вручную.

### <a name="network-filtering"></a>Фильтрация сети

Блокируйте исходящие подключения из любого приложения к IP-адресам и доменам с низкой репутацией.

### <a name="exploit-protection"></a>Защита от эксплойтов

Блокируйте **изменение интерфейса защиты от эксплойтов пользователем**, отправив XML-файл, позволяющий настроить ограничения памяти, потока управления и политик для защиты приложения от эксплойтов.

Чтобы включить защиту от эксплойтов, создайте XML-файл с необходимыми параметрами устранения рисков в системе и приложениях. Это можно сделать одним из двух способов.

 1. PowerShell: используйте один из командлетов PowerShell Get-ProcessMitigation, Set-ProcessMitigation и ConvertTo-ProcessMitigationPolicy или несколько из них, чтобы настроить параметры устранения рисков и экспортировать их в формате XML.

 2. Пользовательский интерфейс Центра безопасности Защитника Windows: в Центре безопасности Защитника Windows выберите пункт "Управление приложениями и браузером" и прокрутите содержимое экрана вниз до раздела "Защита от эксплойтов". Настройте параметры устранения рисков на вкладках "Системные параметры" и "Параметры программ". Чтобы экспортировать параметры в формате XML, в нижней части экрана щелкните ссылку "Экспорт параметров".

## <a name="windows-defender-application-control"></a>Управление приложениями в Защитнике Windows

С помощью **политик целостности кода в компоненте "Управление приложениями"** можно выбрать дополнительные приложения, аудит которых должен проводиться посредством управления приложениями в Защитнике Windows или которые должны быть доверенными. Компоненты Windows и все приложения из Магазина Windows являются доверенными по умолчанию.

При работе в режиме **Только аудит** приложения блокироваться не будут. В режиме **Только аудит** в журнал записываются все события из журналов событий локальных клиентов.

После включения управление приложениями можно отключить, только изменив режим **Принудительно** на **Только аудит**. Смена режима **Принудительно** на **Не настроено** не приводит к отключению управления приложениями на назначенных устройствах.

## <a name="windows-defender-security-center"></a>Центр безопасности Защитника Windows

Центр безопасности Защитника Windows работает как отдельное приложение или процесс независимо от отдельных компонентов и выводит уведомления посредством Центра уведомлений. Он позволяет просматривать состояние и настраивать ряд параметров для каждого из компонентов в одном месте. Дополнительные сведения см. в документах по [Защитнику Windows](https://docs.microsoft.com/windows/threat-protection/windows-defender-security-center/windows-defender-security-center).

#### <a name="windows-defender-security-center-app-and-notifications"></a>Приложение и уведомления Центра безопасности Защитника Windows

Вы можете запретить пользователям доступ к определенным разделам приложения "Центр безопасности Защитника Windows". При скрытии раздела также блокируются связанные с ним уведомления.

- **Защита от угроз и вирусов**
- **Работоспособность устройства**
- **Брандмауэр и безопасность сети**
- **Управление приложениями и браузером**
- **Параметры для семьи**
- **Уведомления из отображаемых областей приложения** — выберите уведомления, которые должны отображаться для конечных пользователей. Некритические уведомления содержат сводки действий антивирусной программы "Защитник Windows", включая уведомления о завершении сканирования. Остальные уведомления считаются критически важными.

#### <a name="it-contact-information"></a>Контактные данные ИТ-отдела

Укажите контактные данные ИТ-отдела, которые должны отображаться в приложении "Центр безопасности Защитника Windows" и его уведомлениях. Можно выбрать один из следующих параметров: **Показывать в приложении и уведомлениях**, **Показывать только в приложении**, **Показывать только в уведомлениях** или **Не показывать**. Необходимо указать **название ИТ-организации** и по меньшей мере один из следующих способов связи.

- **Номер телефона или логин Skype ИТ-отдела**
- **Адрес электронной почты ИТ-отдела**
- **URL-адрес веб-сайта для службы поддержки ИТ-отдела**
