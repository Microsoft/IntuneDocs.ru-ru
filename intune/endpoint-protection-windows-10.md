# <a name="endpoint-protection-settings-for-windows-10-and-later-in-microsoft-intune"></a>Параметры защиты конечных точек для Windows 10 и более поздних версий в Microsoft Intune

[!INCLUDE[azure_portal](./includes/azure_portal.md)]

Профиль защиты конечных точек позволяет управлять функциями безопасности, такими как BitLocker, на устройствах с Windows 10.

Ознакомившись с этой статьей, вы научитесь создавать профили защиты конечных точек.

## <a name="create-an-endpoint-protection-profile"></a>Создание профиля защиты конечных точек

1. Зарегистрируйтесь на портале Azure.
2. Выберите **Больше служб** > **Мониторинг и управление** > **Intune**.
3. В колонке **Intune** выберите пункт **Конфигурация устройства**.
2. В колонке **Конфигурация устройства** выберите **Управление** > **Профили**.
3. В колонке профилей выберите **Создание профиля**.
4. В колонке **Создание профиля** введите **имя** и **описание** для профиля ограничения устройства.
5. В раскрывающемся списке **Платформа** выберите **Windows 10 и более поздних версий**.
6. В раскрывающемся списке **Тип профиля** выберите **Endpoint Protection**. 
7. В колонке **Шифрование Windows** настройте нужные вам параметры. Сведения в этой статье помогут вам понять назначение каждого из параметров. После этого нажмите кнопку **ОК**.
8. Вернитесь в колонку **Создать профиль** и выберите пункт **Создать**.

Созданный профиль отобразится в колонке со списком профилей.

## <a name="endpoint-protection-profile-settings-reference"></a>Справочник по параметрам профиля защиты конечных точек

### <a name="windows-settings"></a>Параметры Windows

- **Обязательное шифрование устройств (только для настольной версии)**. Когда параметр включен, пользователям предлагается включить шифрование устройств. Кроме того, им предлагается убедиться, что не включено шифрование от другого поставщика. Если включить шифрование Windows когда активен другой метод шифрования, устройство может работать нестабильно.
- **Обязательное шифрование карты памяти (только для мобильных устройств)**. Включите этот параметр для шифрования всех съемных карт памяти, используемых на устройстве.


### <a name="bitlocker-base-settings"></a>Базовые параметры BitLocker

- **Настройка методов шифрования**. Включите этот параметр, чтобы настроить алгоритмы шифрования для операционной системы, данных и съемных дисков.
    - **Шифрование дисков операционных систем**. Выберите метод шифрования для дисков операционной системы. Рекомендуется использовать алгоритм XTS-AES.
    - **Шифрование несъемных дисков с данными**. Выберите метод шифрования для фиксированных (встроенных) дисков с данными. Рекомендуется использовать алгоритм XTS-AES.
    - **Шифрование съемных дисков с данными**. Выберите метод шифрования для съемных дисков с данными. Если съемный диск используется с устройствами, работающими не под управлением Windows 10, рекомендуется использовать алгоритм AES-CBC.


### <a name="bitlocker-os-drive-settings"></a>Параметры дисков ОС в BitLocker

- **Требовать дополнительную проверку подлинности при запуске** - 
    - **Запретить BitLocker на устройствах без совместимой микросхемы доверенного платформенного модуля** - 
    - **Запуск доверенного платформенного модуля**. Укажите, является ли микросхема доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным компонентом. 
    - **ПИН-код запуска доверенного платформенного модуля**. Укажите, является ли использование ПИН-кода запуска для микросхемы доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным. 
    - **Ключ запуска доверенного платформенного модуля**. Укажите, является ли использование ключа запуска для микросхемы доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным. 
    - **Ключ и ПИН-код запуска доверенного платформенного модуля**. Укажите, является ли использование ПИН-кода и ключа запуска для микросхемы доверенного платформенного модуля (TPM) допустимым, запрещенным или обязательным.
- **Минимальная длина ПИН-кода**. Включите этот параметр, чтобы настроить минимальную длину ПИН-кода запуска для доверенного платформенного модуля (TPM).
    - **Минимальное число символов**. Введите обязательное число символов для ПИН-кода запуска из диапазона **4**-**20**.
- **Включить восстановление дисков ОС**. Включите этот параметр, чтобы управлять восстановлением дисков операционной системы, защищенных с помощью BitLocker, при отсутствии требуемых сведений о запуске.
    - **Разрешить агент восстановления данных на основе сертификатов**. Включите этот параметр, чтобы агенты восстановления данных можно было использовать вместе с дисками операционной системы, защищенными BitLocker.
    - **Создание пользователем пароля восстановления**. Укажите, является ли создание 48-значного пароля восстановления разрешенной, запрещенной или обязательной операцией для пользователей.
    - **Создание пользователем ключа восстановления**. Укажите, является ли создание 256-разрядного ключа восстановления разрешенной, запрещенной или обязательной операцией для пользователей.
    - **Скрыть варианты восстановления в мастере установки BitLocker**. Включите этот параметр, чтобы запретить пользователям просматривать или изменять параметры восстановления при включении BitLocker.
    - **Сохранять сведения о восстановлении BitLocker в AD DS**. Включает хранение сведений о восстановлении BitLocker в Active Directory.
    - **Настроить хранение сведений о восстановлении BitLocker в AD DS**. Укажите, какие части сведений о восстановлении BitLocker хранятся в Active Directory. Выберите один из следующих типов.
        - **Архивировать пароли и пакеты ключей восстановления**
        - **Архивировать только пароли восстановления**
    - **Требовать хранить сведения о восстановлении в AD DS перед включением BitLocker**. Включите этот параметр, чтобы разрешить пользователям включать BitLocker только в том случае, если устройство присоединено к домену, а сведения о восстановлении BitLocker успешно сохранены в Active Directory.
- **Разрешить предзагрузочное сообщение о восстановлении и URL-адрес**. Включите этот параметр, чтобы настроить сообщение и URL-адрес, отображаемые на предзагрузочном экране восстановления ключей.
    - **Предзагрузочное сообщение о восстановлении**. Настройте способ отображения предзагрузочного сообщения о восстановлении, выводимого пользователям. Выберите один из следующих типов.
        - **Использовать сообщение о восстановлении и URL-адрес по умолчанию**
        - **Использовать пустое сообщение о восстановлении и URL-адрес**
        - **Использовать пользовательское сообщение о восстановлении**
        - **Использовать пользовательский URL-адрес восстановления**

- **Запретить доступ на запись к жестким дискам с данными, не защищенным BitLocker**. Если параметр включен, для записи на любые фиксированные или встроенные диски с данными нужно включить для них защиту BitLocker.
- **Включить восстановление жесткого диска**. Включите этот параметр, чтобы управлять восстановлением жестких дисков, защищенных с помощью BitLocker, при отсутствии требуемых сведений о запуске.

    - **Создание пользователем ключа восстановления**. Укажите, является ли создание 256-разрядного ключа восстановления разрешенной, запрещенной или обязательной операцией для пользователей.

### <a name="bitlocker-removable-data-drive-settings"></a>Параметры удаляемых дисков с данными в BitLocker

- **Запретить доступ на запись к удаляемым дискам с данными, не защищенным BitLocker**. Укажите, требуется ли шифрование BitLocker для съемных запоминающих устройств.
    - **Запретить доступ на запись к устройствам, настроенным в другой организации**. Укажите, можно ли производить запись на съемные диски с данными, принадлежащие другой организации.
