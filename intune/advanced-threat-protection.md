# <a name="enforce-compliance-for-windows-defender-atp-with-conditional-access-in-intune"></a>Обеспечение соответствия требованиям для ATP в Защитнике Windows с помощью условного доступа в Intune

ATP в Защитнике Windows и Microsoft Intune совместно позволяют предотвратить бреши в системе безопасности и ограничить влияние брешей на организацию.

Данная функция применяется к: устройства Windows 10;

Например, кто-то отправляет вложение Word с внедренным вредоносным кодом пользователю в вашей организации. Пользователь открывает вложение и включает содержимое. Начинается атака с более высоким уровнем привилегий, и злоумышленник с удаленного компьютера получает права администратора к устройству жертвы. Затем злоумышленник удаленно получает доступ к другим устройствам пользователя.

Эта брешь в системе безопасности может повлиять на всю организацию.

ATP в Защитнике Windows может устранять события безопасности, такие как в этом сценарии. Центр безопасности Защитника Windows (консоль ATP) сообщает об устройстве как имеющем высокий уровень риска и включает подробный отчет о подозрительной деятельности. В нашем примере ATP в Защитнике Windows обнаруживает, что устройство выполнило ненормальный код, испытало эскалацию привилегий процесса, ввело вредоносный код и сгенерировало подозрительную удаленную оболочку. Затем ATP в Защитнике Windows предоставляет варианты устранения угрозы.

С помощью Intune можно создать политику соответствия требованиям, которая определяет приемлемый уровень риска. Если устройство превышает этот риск, оно становится несовместимым. При объединении с политикой условного доступа Azure Active Directory доступ пользователя из корпоративных ресурсов блокируется.

В этой статье показано, как:

- Включить Intune в ATP и включить ATP в Intune. Эти задачи создают подключение между службами Intune и ATP в Защитнике Windows. Это подключение позволяет ATP в Защитнике Windows записывать связанные с компьютером риски для устройств Intune.
- Создать политику соответствия в Intune.
- Включить условный доступ Azure Active Directory на устройствах в соответствии с уровнем угрозы.

## <a name="prerequisites"></a>Предварительные условия

Чтобы использовать ATP с Intune, убедитесь, что у вас настроено и готово к использованию следующее:

- лицензированный клиент для Enterprise Mobility + Security E3 и Windows E5 (или Microsoft 365 корпоративный E5);
- среда Microsoft Intune с [управляемыми Intune](windows-enroll.md) устройствами Windows 10, которые также присоединены к Azure AD;
- служба [ATP в Защитнике Windows](https://docs.microsoft.com/windows/security/threat-protection/windows-defender-atp/windows-defender-advanced-threat-protection) и доступ к центру безопасности Защитника Windows (портал ATP);

## <a name="enable-windows-defender-atp-in-intune"></a>включенный брандмауэр Защитника Windows в Intune.

1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите **Все службы**, отфильтруйте список по **Intune** и выберите **Microsoft Intune**.
3. Выберите **Соответствие устройства политике** > **ATP в Защитнике Windows** > **Открыть Центр безопасности Защитника Windows**.

    ![Выберите, чтобы открыть Центр безопасности Защитника Windows](./media/atp-device-compliance-open-windows-defender.png)

4. В **центре безопасности Защитника Windows**:
    1. Выберите **Параметры** > **Дополнительные параметры**.
    2. Для **подключения Microsoft Intune** выберите **Вкл.**:

        ![Включение подключения к Intune](./media/atp-security-center-intune-toggle.png)

    3. Выберите **Сохранить параметры**.

5. Вернитесь в Intune, **Соответствие устройства** > **ATP в Защитнике Windows**. Установите параметр **Подключение устройств Windows версии 10.0.15063 и более поздней к ATP в Защитнике Windows** в значение **Вкл**.
6. Нажмите кнопку **Сохранить**.

Обычно эта задача выполняется один раз. Поэтому, если ATP уже включено в ресурсе Intune, значит это не нужно повторять.

## <a name="onboard-devices-using-a-configuration-profile"></a>Подключение устройств с использованием профиля конфигурации

При регистрации конечного пользователя в Intune можно с помощью профиля конфигурации передать на устройство различные параметры. Это также применимо к ATP в Защитнике Windows.

В Защитник Windows входит подключаемый пакет конфигурации, который взаимодействует со [службами ATP в Защитнике Windows](https://docs.microsoft.com/windows/security/threat-protection/windows-defender-atp/windows-defender-advanced-threat-protection), чтобы проверять файлы, обнаруживать угрозы и сообщать о рисках в ATP в Защитнике Windows.

При подключении Intune получает автоматически созданный пакет конфигурации из ATP в Защитнике Windows. После передачи профиля на устройство или его развертывания этот пакет конфигурации также передается на устройство. Это позволяет ATP в Защитнике Windows отслеживать устройство на предмет угроз.

После подключения устройства с помощью пакета конфигурации вам больше не понадобится делать это снова. Устройства также можно подключить с помощью [групповой политики или System Center Configuration Manager (SCCM)](https://docs.microsoft.com/windows/security/threat-protection/windows-defender-atp/configure-endpoints-windows-defender-advanced-threat-protection).

### <a name="create-the-configuration-profile"></a>Создание профиля конфигурации

1. На [портале Azure](https://portal.azure.com) выберите **Все службы**, отфильтруйте список по **Intune** и выберите **Microsoft Intune**.
2. Выберите **Конфигурация устройства** > **Профили** > **Создать профиль**.
3. Введите **имя** и **описание**.
4. В поле **Платформа** выберите **Windows 10 и более поздние версии**.
5. В поле **Тип профиля** выберите **ATP в Защитнике Windows (Windows 10 Desktop)**.
6. Настройте параметры:

  - **Windows Defender ATP client configuration package type** (Тип пакета конфигурации клиента ATP в Защитнике Windows). Выберите **Onboard** (Подключить), чтобы добавить пакет конфигурации в профиль. Выберите **Отключить**, чтобы удалить пакет конфигурации из профиля.
  
    > [!NOTE] 
    > Если соединение с ATP в Защитнике Windows установлено правильно, Intune будет автоматически **подключать** профиль конфигурации, а параметр **Windows Defender ATP client configuration package type** (Тип пакета конфигурации клиента ATP в Защитнике Windows) будет недоступен.
  
  - **Общий доступ к примерам для всех файлов**. Если выбран параметр **Включить**, разрешено собирать примеры и предоставлять к ним доступ в ATP в Защитнике Windows. Например, если вы видите подозрительный файл, его можно отправить в ATP в Защитнике Windows для подробного анализа. Если выбран вариант **Не настроено**, примеры не будут доступны в ATP в Защитнике Windows.
  - **Увеличить частоту отчетов о телеметрии**. Для устройств с высоким уровнем риска **включите** этот параметр, чтобы он чаще сообщал данные телеметрии службе ATP в Защитнике Windows.

    В статье [Настройка конечных точек с помощью System Center Configuration Manager](https://docs.microsoft.com/windows/security/threat-protection/windows-defender-atp/configure-endpoints-sccm-windows-defender-advanced-threat-protection) содержатся дополнительные сведения о параметрах ATP в Защитнике Windows.

7. Нажмите кнопки **ОК** и **Создать**, чтобы сохранить изменения. После чего будет создан профиль.

## <a name="create-the-compliance-policy"></a>Создание политики соответствия требованиям
Политика соответствия требованиям определяет приемлемый уровень риска на устройстве.

1. На [портале Azure](https://portal.azure.com) выберите **Все службы**, отфильтруйте список по **Intune** и выберите **Microsoft Intune**.
2. Выберите **Соответствие устройства** > **Политики** > **Создать политику**.
3. Введите **имя** и **описание**.
4. В поле **Платформа** выберите **Windows 10 и более поздние версии**.
5. В разделе **ATP в Защитнике Windows** установите для параметра **Оценка риска устройства должна быть не ниже**:

  - **Очистить**. Этот уровень обеспечивает максимальную защиту. Устройство не может осуществлять доступ к ресурсам организации при наличии каких-либо угроз. При обнаружении любых угроз устройство переходит в состояние несоответствия.
  - **Низкий**. Если обнаруженные на устройстве угрозы имеют низкий уровень, считается, что устройство соответствует требованиям. Устройства со средним или высоким уровнем угроз считаются несоответствующими требованиям.
  - **Средний**. Если обнаруженные на устройстве угрозы имеют низкий или средний уровень, считается, что устройство соответствует требованиям. Если на устройстве найдены угрозы высокого уровня, устройство переходит в состояние несоответствия.
  - **Высокий**. Этот уровень является наименее безопасным, так как на нем разрешены все уровни угроз. Таким образом, устройства с высоким, средним или низким уровнями угроз считаются совместимыми.

6. Нажмите кнопки **ОК** и **Создать**, чтобы сохранить изменения и создать политику.

## <a name="assign-the-policy"></a>Назначение политики

1. На [портале Azure](https://portal.azure.com) выберите **Все службы**, отфильтруйте список по **Intune** и выберите **Microsoft Intune**.
2. Выберите **Совместимость устройства** > **Политики** и выберите политику соответствия требованиям ATP в Защитнике Windows.
3. Выберите **Назначения**.
4. Включайте или исключайте группы Azure AD, чтобы назначить им политику.
5. Чтобы развернуть политику в группы, нажмите кнопку **Сохранить**. Устройства пользователей, на которых распространяется политика, проверяются на соответствие требованиям.

## <a name="create-an-azure-ad-conditional-access-policy"></a>Создание политики условного доступа Azure Active Directory
Политика условного доступа блокирует доступ к ресурсам, *если* устройство не соответствует требования. Таким образом, если устройство превышает уровень угроз, вы можете заблокировать доступ к корпоративным ресурсам, таким как SharePoint или Exchange Online.

1. На [портале Azure](https://portal.azure.com) выберите **Azure Active Directory** > **Условный доступ** > **Новая политика**.
2. Введите **имя** политики и выберите **Пользователи и группы**. Используйте варианты включения или исключения, чтобы добавить группы для политики, и нажмите кнопку **Готово**.
3. Выберите **Облачные приложения**, а затем — приложения, которые следует защищать. Например, выберите **Выбрать приложения**, а затем — **Office 365 SharePoint Online** или **Office 365 Exchange Online**.

    Нажмите кнопку **Готово**, чтобы сохранить изменения.

4. Выберите **Условия** > **Клиентские приложения**, чтобы применить политику к приложениям и браузерам. Например, выберите **Да**, а затем включите **Браузер** и **Mobile apps and desktop clients** (Мобильные приложения и настольные клиенты).

    Нажмите кнопку **Готово**, чтобы сохранить изменения.

5. Выберите **Предоставить доступ**, чтобы применить условный доступ на основе соответствия устройств. Например, выберите **Предоставить доступ** > **Require device to be marked as compliant** (Требовать, чтобы устройство было отмечено как соответствующее).

    Нажмите кнопку **ОК**, чтобы сохранить изменения.

6. Выберите **Включить политику**, а затем — **Создать**, чтобы сохранить изменения.

Статья [Что такое условный доступ?](conditional-access.md) — это хороший ресурс.

## <a name="monitor-device-compliance"></a>Мониторинг устройств на соответствие требованиям
Затем отслеживайте состояние устройств, у которых есть политика соответствия требованиям ATP в Защитнике Windows.

1. На [портале Azure](https://portal.azure.com) выберите **Все службы**, отфильтруйте список по **Intune** и выберите **Microsoft Intune**.
2. Выберите **Соответствие устройства** > **Соответствие политике**.
3. Найдите в списке политику ATP в Защитнике Windows и узнайте, какие устройства соответствуют, а какие — нет.
