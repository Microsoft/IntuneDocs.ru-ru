# <a name="android-app-protection-policy-settings-in-microsoft-intune"></a>Параметры политики защиты приложений Android в Microsoft Intune
В этой статье описываются параметры политики защиты приложений на устройствах Android. Описываемые параметры политик можно [настроить](app-protection-policies.md) для политики защиты приложений на портале Azure в колонке **Параметры**.
Существует три категории параметров политики: параметры защиты данных, требования доступа и условный запуск. В этой статье термин *управляемые политикой приложения* означает приложения, которые настроены с помощью политик защиты приложений.

> [!IMPORTANT]
> Для получения политики защиты приложений для устройств Android на устройстве должно быть установлено приложение "Корпоративный портал Intune". Дополнительные сведения см. в разделе [Доступ к приложениям](../fundamentals/end-user-mam-apps-android.md).

### <a name="encryption"></a>Encryption
| Параметр | Использование | Значение по умолчанию |
|------|------|------|
| **Шифрование данных организации** | Выберите **Требуется**, чтобы включить шифрование рабочих или учебных данных в этом приложении. Intune использует OpenSSL, 256-битовое шифрование AES вместе с системой хранилища ключей Android для безопасного шифрования данных приложения. Данные шифруются синхронно во время задач ввода-вывода. Содержимое в хранилище устройства всегда находится в зашифрованном виде. Новые файлы будут шифроваться с помощью ключей длиной 256 бит. Для существующих файлов, зашифрованных 128-битными ключами, будет предпринята попытка перехода на ключи длиной 256 бит, но успешный результат не гарантируется. Файлы, зашифрованные 128-битными ключами, останутся доступными для чтения. <br><br> Метод шифрования соответствует стандарту FIPS 140-2.     |  **Требуется**|  
| <ul><ui>**Шифровать данные организации на зарегистрированных устройствах** | Выберите **Требовать** для принудительного шифрования данных организации с помощью шифрования уровня приложения Intune на всех устройствах. Выберите **Не требуется**, чтобы не применять принудительное шифрование данных организации с помощью шифрования уровня приложения Intune на зарегистрированных устройствах.| **Требуется** |

## <a name="data-transfer-exemptions"></a>Исключения передачи данных

  Среди приложений и служб платформ существует несколько исключений, в которых политика защиты приложений Intune может разрешить передачу данных. Например, во всех управляемых приложениях Intune на мобильных устройствах с Android должна быть возможность передавать данные в службу преобразования текста в речь Google и получать данные из нее. Этот список может измениться. В нем представлены полезные службы и приложения для безопасной работы.

### <a name="full-exemptions"></a>Полные исключения

  В этих приложениях и службах полностью разрешен обмен данными с приложениями под управлением Intune.

  |Имя службы или приложения | Описание |
  | ------ | ---- |
  | com.android.phone | Встроенное приложение телефона
  | com.android.vending | Магазин Google Play |
  | com.android.documentsui | Средство выбора документов Android|
  | com.google.android.webview | [Веб-представление](https://developer.android.com/reference/android/webkit/WebView.html), необходимое для многих приложений, включая Outlook. |
  | com.android.webview |[Веб-представление](https://developer.android.com/reference/android/webkit/WebView.html), необходимое для многих приложений, включая Outlook.|
  | com.google.android.tts | Преобразование текста в речь Google |
  | com.android.providers.settings | Параметры системы Android |
  | com.android.settings | Параметры системы Android |
  | com.azure.authenticator | Приложение Azure Authenticator, которое нужно для проверки подлинности во многих сценариях. |
  | com.microsoft.windowsintune.companyportal | Intune Портал компании|

### <a name="conditional-exemptions"></a>Условные исключения
  В этих приложениях и службах разрешен обмен данными с приложениями под управлением Intune только при определенных условиях.

  |Имя службы или приложения | Описание | Условие исключения|
  | ------ | ---- | --- |
  | com.android.chrome | Браузер Google Chrome | Chrome используется для некоторых компонентов веб-представления в ОС Android 7.0+ и никогда не скрывается в представлении. Поток данных в приложение и из него всегда ограничен.  |
  | com.skype.raider | Skype | В Skype всегда разрешены только определенные действия, которые приводят к телефонному звонку. |
  | com.android.providers.media | Поставщик содержимого мультимедиа Android | Поставщик содержимого мультимедиа разрешен только при выборе мелодии звонка. |
  | com.google.android.gms; com.google.android.gsf | Пакеты сервисов Google Play | Эти пакеты разрешены для действий Google Cloud Messaging, например push-уведомлений. |
  | com.google.android.apps.maps | Google Карты | Для навигации можно использовать адреса. |

Дополнительные сведения см. в разделе [Исключения политики передачи данных для приложений](app-protection-policies-exception.md).

## <a name="access-requirements"></a>Требования для доступа

| Параметр | Использование |  
|------|------| 
| **Требовать ПИН-код для доступа** | Выберите **Да**, чтобы для использования этого приложения требовалось вводить ПИН-код. Пользователю предлагается настроить ПИН-код при первом запуске приложения в рабочем или учебном контексте. <br><br> Значение по умолчанию — **Да**.<br><br> Настройте следующие параметры для стойкости ПИН-кода: <br> <ul><li>**Выбрать тип**: укажите тип ПИН-кода (цифровой или секретный код), который необходимо ввести для доступа к приложению с примененными политиками защиты. Цифровой ПИН-код состоит только из цифр, а секретный код должен включать в себя по крайней мере одну букву **или** специальный символ. <br><br> Значение по умолчанию — **Цифровой**.<br><br> **Примечание.** Допустимые специальные символы включают английские специальные символы и символы на клавиатуре Android.</li></ul>  <ul><li>**Число попыток до сброса ПИН-кода**: укажите количество попыток, за которое пользователь должен успешно ввести свой ПИН-код, прежде чем он будет сброшен. <br><br> Значение по умолчанию — **5**. </li> <br> <li> **Разрешить простой ПИН-код**: выберите **Да**, чтобы разрешить пользователям применять простые ПИН-коды, например *1234*, *1111*, *abcd* или *aaaa*.
