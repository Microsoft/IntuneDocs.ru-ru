# <a name="the-early-edition-for-microsoft-intune---november-2018"></a>Ранний выпуск Microsoft Intune — ноябрь 2018 г.

> [!Note]
> Уведомление о соглашении о неразглашении. Следующие изменения находятся в разработке для Intune. Эти сведения предоставляются в рамках соглашения о неразглашении на ограниченной основе. Не публикуйте эти сведения в социальных сетях или на общедоступных веб-сайтах, таких как Twitter, UserVoice, Reddit и т. д. 

**Ранний выпуск** содержит список функций, предоставляемых в рамках соглашения о неразглашении, которые будут включены в предстоящие выпуски Microsoft Intune. Эта информация предоставляется в ограниченном объеме и может изменяться. Не публикуйте эти сведения в Twitter, UserVoice и не делитесь этими сведениями за пределами вашей организации. Некоторые из указанных здесь компонентов могут быть не готовы к сроку, поэтому их выпуск может быть отложен. Другие функции проходят проверку в пилотном проекте, которая должна доказать их готовность для развертывания у клиентов. При наличии любых вопросов или проблем свяжитесь с представителем службы поддержки вашей группы продуктов Майкрософт.

Эта страница периодически обновляется. Проверьте наличие дополнительных обновлений.

### <a name="uninstalling-apps-on-corporate-owned-supervised-ios-devices----1281677---"></a>Удаление приложения на корпоративных защищенных устройствах iOS <!-- 1281677 -->

### <a name="temporarily-pause-kiosk-mode-on-android-devices-to-make-changes----3041935---"></a>Временная приостановка режима киоска на устройствах Android для внесения изменений <!-- 3041935 -->
При использовании устройств Android в режиме киоска с несколькими приложениями ИТ-администратору может потребоваться внести изменения на устройстве. С новым параметром киоска с несколькими приложениями ИТ-администратор сможет временно приостановить режим киоска с помощью ПИН-кода и получить доступ ко всему устройству.
Текущие параметры киоска см. в разделе [Параметры киоска Android](android-kiosk-settings.md).

### <a name="set-custom-background-in-managed-home-screen-app-----3041945---"></a>Настройка пользовательского фона в приложении управляемого начального экрана <!-- 3041945 -->
Мы добавим параметр, который позволит настраивать внешний вид фона приложения управляемого начального экрана в Android для бизнеса на устройствах в режиме киоска с несколькими приложениями.  Чтобы настроить **Пользовательский URL-адрес фона**, перейдите в Intune на портале Azure > Конфигурация устройства. Выберите текущий профиль конфигурации устройства или создайте новый, чтобы изменить параметры киоска.

### <a name="enable-virtual-home-button-on-android-enterprise-kiosk-devices-----3042021---"></a>Включение виртуальной кнопки домашней страницы на устройствах киоска Android для бизнеса <!-- 3042021 -->
Новый параметр позволит пользователям с помощью программной клавиши на устройстве переключаться между приложением управляемого начального экрана и другими назначенными приложениями в киоске с несколькими приложениями. Этот параметр особенно полезен в сценариях, где приложение киоска пользователя некорректно отвечает на нажатие кнопки "Назад". Вы сможете настроить этот параметр для корпоративных устройств Android единого назначения. Чтобы включить или отключить **виртуальную кнопку домашней страницы**, перейдите в Intune на портале Azure > конфигурация устройства. Выберите текущий профиль конфигурации устройства или создайте новый, чтобы изменить параметры киоска.

Вы получите больше возможностей управления назначениями политики защиты приложений. С помощью сохранения и применения назначений политики защиты приложений можно распространять политику только на некоторых пользователей.

### <a name="new-microsoft-edge-browser-settings-for-windows-10-and-later----3174639---"></a>Новые параметры браузера Microsoft Edge для Windows 10 и более поздней версии <!-- 3174639 -->

### <a name="select-apps-tracked-on-the-enrollment-status-page---2531007---"></a>Выбор отслеживаемых приложений на странице состояния регистрации<!-- 2531007 -->

### <a name="intune-app-protection-policies-ui-update----3251427---"></a>Обновление пользовательского интерфейса политики защиты приложений Intune <!-- 3251427 -->

Политики защиты приложений Intune позволяют настраивать различные параметры защиты данных для приложений, защищаемых Intune, таких как Microsoft Outlook и Word. Мы меняем метки параметра и кнопки, чтобы они были понятнее. Элементы управления будет изменены с **Да**/**Нет** на **Запретить**/**Разрешить** или **Отключить**/**Включить**, и метки также будут изменены для ясности. Будет изменен формат параметров, чтобы параметр и метка были расположены рядом на элементе управления для более удобной навигации. Параметры по умолчанию и число параметров останется прежним, но с этим изменением будет проще и удобнее использовать параметры и применять выбранные политики защиты приложения.

### <a name="use-microsoft-recommended-settings-with-security-baselines----2055484---"></a>Использование параметров, рекомендуемых корпорацией Майкрософт, с базовыми показателями системы безопасности <!-- 2055484 -->
Intune интегрируется с другими службами, включая ATP в Защитнике Windows и ATP в Office 365, предназначенными для обеспечения безопасности. Клиентам требуется общая стратегия и полный набор готовых рабочих процессов по безопасности в службах Microsoft 365. Наша цель — обеспечить соответствие стратегий решениям для разработки, в которых объединяются операции безопасности и общие задачи администрирования. Мы планируем решить эту задачу в Intune путем публикации набора рекомендуемых корпорацией Майкрософт базовых показателей системы безопасности (**Intune** > **Базовые показатели системы безопасности**).  Администраторы смогут создавать политики безопасности непосредственно на основе этих базовых показателей, а затем развертывать их для пользователей. Они также могут настраивать рекомендации в соответствии с потребностями своей организации. Intune обеспечивает соответствие устройств этим базовым показателям и уведомляет администраторов пользователей или устройств о несоответствии.

### <a name="scope-tags-for-apps---1081941---"></a>Теги области для приложений <!--1081941 -->
Вы сможете создавать теги области, чтобы ограничивать доступ к ресурсам Intune. Добавьте тег области к назначению роли, а затем добавьте тег области в профиль конфигурации. Роль будет иметь доступ только к ресурсам с профилями конфигурации, которые имеют соответствующие теги области (или не имеют тега области вообще).
Чтобы создать тег области, выберите **Роли Intune** > **Область (теги)** > **Создать**.
Чтобы добавить тег области к назначению роли, выберите **Роли Intune** > **Все роли** > **Диспетчер политик и профилей** > **Назначения** > **Область (теги)**.
Чтобы добавить тег области к профилю конфигурации, выберите **Конфигурация устройств** > **Профили** > выберите профиль > **Свойства** > **Область (теги)**.

### <a name="tenant-health-dashboard----1124854---"></a>Панель мониторинга работоспособности клиента <!-- 1124854 -->
На странице состояния клиента в Intune сведения о состоянии клиента будут доступны в одном месте. Страница разделена на четыре части:  
- **Сведения о клиенте**: содержит такие сведения, как центр MDM, общее число зарегистрированных устройств в клиенте и число лицензий. Здесь также указан текущий выпуск службы для клиента.
- **Состояние соединителя**: содержит сведения о настроенных соединителях, таких как соединители Apple VPP, соединители Microsoft Store для бизнеса и соединители сертификатов. Исходя из текущего состояния, соединители помечаются как *Работает*, *Предупреждение* или *Не работает*.
- **Работоспособность службы Intune**: содержит перечень активных инцидентов или сбоев для клиента. Сведения в этом разделе извлекаются напрямую из Центра сообщений Office ([https://portal.office.com](https://portal.office.com)).
- **Новости Intune**: содержит активные сообщения для клиента, в число которых входят уведомления, информирующие о получении клиентом новейших функций Intune. Сведения в этом разделе извлекаются напрямую из Центра сообщений Office ([https://portal.office.com](https://portal.office.com)).


### <a name="deployed-wip-policies-without-user-enrollment----1434452---"></a>Развернутые политики WIP без регистрации пользователей <!-- 1434452 -->
Появится возможность развертывания политик Windows Information Protection (WIP) без необходимости регистрации устройств Windows 10 пользователей. Такая конфигурация позволяет компаниям защищать корпоративные документы на основе конфигурации WIP, при этом пользователь может управлять собственными устройствами Windows. Администратор Intune может выполнять выборочную очистку данных в документах, защищенных с помощью политики WIP. После выбора пользователя и устройства и отправки запроса на очистку все данные, которые были защищены с помощью политики WIP, станут непригодными для использования. В Intune на портале Azure выберите **Мобильное приложение** > **Выборочная очистка приложений**.

### <a name="apple-vpp-token-used-by-another-mdm----1488946---"></a>Использование токена Apple VPP другой системой MDM <!-- 1488946 -->
Intune обнаружит и отобразит соответствующую информацию, если токен программы корпоративных покупок Apple (VPP) одновременно используется и Intune, и другой системой MDM.

### <a name="ios-and-macos-version-numbers-and-build-numbers-are-available-in-compliance-policies----1892471---"></a>Номера версии и номера сборки iOS и macOS доступны в политиках соответствия требованиям <!-- 1892471 -->
В разделе **Соответствие устройства политике** > **Соответствие устройства политике** отображаются версии операционных систем iOS и macOS, которые доступны для использования в политиках соответствия требованиям. В будущем обновлении для обеих платформ также можно будет настроить номер сборки.

При выпуске обновлений безопасности Apple, как правило, оставляет номер версии прежним, но меняет номер сборки. Используя номер сборки в политике соответствия, можно легко проверить, установлено ли обновление для защиты от уязвимостей.

### <a name="retired-devices-in-the-device-compliance-dashboard----1981119---"></a>Неиспользуемые устройства на информационной панели о соответствии устройств политике <!-- 1981119 -->
В будущем обновлении неиспользуемые устройства будут удалены с информационной панели о соответствии устройств политике. Это приведет к изменению ваших показателей соответствия.


### <a name="change-in-the-update-process-for-on-premises-connectors----2277554---"></a>Изменение процесса обновления для локальных соединителей<!-- 2277554 -->
С учетом отзывов клиентов мы изменим процедуру обновления локальных соединителей. После первоначальной установки локального соединителя обновления будут выполняться автоматически. Это изменение будет сначала представлено на новом соединителе сертификата PFX для Microsoft Intune, а впоследствии будет развернуто и на другие типы локальных соединителей.

### <a name="check-for-configuration-manager-compliance----2192052---"></a>Проверка соответствия Configuration Manager <!-- 2192052 -->
Будущее обновление включает новый параметр соответствия System Center Configuration Manager (последовательно выберите **Соответствие устройства** > **Политики** > **Создать политику** > **Windows 10**). Configuration Manager отправит в Intune сигналы о соответствии. С помощью параметра Intune можно настроить отправку сигналами Configuration Manager сообщения о соответствии.

Например, можно затребовать установку всех обновлений программного обеспечения на устройствах. В Configuration Manager это требование имеет состояние "Установлено". Если состояние каких-либо программ на устройстве неизвестно, устройство будет указано как несоответствующее в Intune.

Применяется к Windows 10 и более поздним версиям

### <a name="alerts-for-expiring-vpp-token-or-company-portal-license-running-low----2237572---"></a>Оповещения об истечении срока действия токена VPP или недостаточном числе лицензий корпоративного портала <!-- 2237572 -->
При использовании Volume Purchase Program (VPP) для предварительной подготовки корпоративного портала во время регистрации DEP Intune будет уведомлять вас об истечении срока действия токена VPP и недостаточном числе лицензий корпоративного портала.
