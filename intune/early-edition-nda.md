> [!Note]
> Уведомление о неразглашении информации. Следующие изменения находятся в разработке для Intune. Эти сведения предоставляются в рамках соглашения о неразглашении на ограниченной основе. Не публикуйте эти сведения в социальных сетях или на общедоступных веб-сайтах, таких как Twitter, UserVoice, Reddit и т. д. 

**Ранний выпуск** содержит список функций, предоставляемых в рамках соглашения о неразглашении, которые будут включены в предстоящие выпуски Microsoft Intune. Другие функции проходят проверку в пилотном проекте, которая должна доказать их готовность для развертывания у клиентов. При наличии любых вопросов или проблем свяжитесь с представителем службы поддержки вашей группы продуктов Майкрософт.

### <a name="powershell-scripts-can-run-in-a-64-bit-host-on-64-bit-devices----1862675----"></a>Скрипты PowerShell могут выполняться в 64-разрядном узле на 64-разрядных устройствах <!-- 1862675  -->
При добавлении скрипта PowerShell в профиль конфигурации устройства он всегда выполняется в 32-разрядном режиме даже в 64-разрядных операционных системах. Благодаря этому обновлению администраторы могут выполнять скрипт в 64-разрядном узле PowerShell на 64-разрядных устройствах (**Конфигурация устройства** > **Сценарии PowerShell** > **Добавить** > **Настроить** > **Выполнить скрипт в 64-разрядном узле PowerShell**).
Дополнительные сведения об использовании PowerShell см. в статье [Скрипты PowerShell в Intune](intune-management-extension.md).
Применяется к: Windows 10 и более поздней версии

### <a name="rename-an-enrolled-windows-device----1911112----"></a>Переименование зарегистрированного устройства Windows <!-- 1911112  -->
Вы можете переименовать зарегистрированное устройство Windows 10 (RS4 или более поздней версии). Для этого последовательно выберите **Intune** > **Устройства** > **Все устройства**, выберите устройство и нажмите **Переименовать устройство**.

### <a name="assign-scep-certificates-to-a-userless-macos-device-------2340521-----"></a>Назначение сертификатов SCEP устройству macOS без пользователей <!-- 2340521   -->
Вы сможете назначать сертификаты службы регистрации сертификатов для сетевых устройств (SCEP) устройствам macOS без пользователей и связывать эти сертификаты с профилями Wi-Fi или VPN. Таким образом расширяется поддержка [назначения сертификатов устройствам без пользователей, которая уже была реализована для Windows, iOS и Android](certificates-scep-configure.md#create-a-scep-certificate-profile).

### <a name="intune-conditional-access-ui-update------2432313----"></a>Обновление пользовательского интерфейса для настройки условного доступа в Intune <!-- 2432313  -->
Мы улучшаем пользовательский интерфейс для настройки условного доступа в консоли Intune. К ним можно отнести следующие.
- Колонку *Условный доступ* в Intune можно заменить аналогичной колонкой из Azure Active Directory. Таким образом обеспечивается доступ ко всем настройкам и параметрам условного доступа, имеющимся в Azure AD.
- Настройка *соединителя со службой Exchange* переносится в колонку, которая сейчас называется *Локальный доступ*. При этом колонка переименовывается в *Доступ к Exchange*. Благодаря этому все операции по настройке и мониторингу Exchange Online и локальной среды Exchange объединяются в одном месте.

### <a name="intune-will-leverage-google-play-protect-apis-on-android-devices----2577355----"></a>Intune будет использовать интерфейсы API Google Play Защита на устройствах Android <!-- 2577355  -->
Некоторые ИТ-администраторы сталкиваются с проблемой управления собственными мобильными телефонами сотрудников (BYOD), которые подвергаются угрозе предоставления прав суперпользователя или джейлбрейка. Эти операции могут совершаться самими пользователями без злого умысла, однако они могут приводить к обходу политик Intune, настроенных для защиты данных организации на устройствах пользователей. Поэтому Intune предоставляет функции обнаружения предоставления прав суперпользователя и джейлбрейка как для зарегистрированных, так и для незарегистрированных устройств. В этом выпуске Intune начнет использовать интерфейсы API Google Play Защита для более тщательной проверки предоставления прав суперпользователя на незарегистрированных устройствах. Хотя компания Google не полностью раскрывает информацию о проводимых проверках, ожидается, что эти интерфейсы API будут обнаруживать пользователей, которые получили права суперпользователя на своих устройствах в любых целях: от настройки устройства до получения обновлений ОС на старых устройствах. Таким пользователям можно будет запрещать доступ к корпоративным данным либо можно будет удалять их корпоративные учетные записи из приложений с действующими политиками. Кроме этого, ИТ-администраторам станет доступна дополнительная информация в колонке "Защита приложений Intune": в отчете "Помеченные пользователи" будут представлены пользователи, обнаруженные в результате проверки с помощью API SafetyNet из пакета Google Play Защита, а в отчете "Потенциально опасные приложения" — приложения, обнаруженные в результате проверки с помощью API Verify Apps от Google. Эта возможность доступна в Android. 

### <a name="win32-app-information-available-in-troubleshooting-blade----2617342------"></a>Сведения о приложениях Win32 в колонке "Устранение неполадок" <!-- 2617342    -->
В колонке **Устранение неполадок** можно будет собирать файлы журналов ошибок, связанных с установкой приложений Win32. Дополнительные сведения об устранении неполадок, связанных с установкой приложений, см. в статье [Устранение проблем с установкой приложений](troubleshoot-app-install.md).

### <a name="kiosk-browser-and-microsoft-edge-browser-apps-can-run-on-windows-10-devices-in-kiosk-mode----2935135----"></a>Приложения Kiosk Browser и Microsoft Edge могут выполняться на устройствах с Windows 10 в режиме терминала <!-- 2935135  -->
Устройства с Windows 10 теперь можно использовать в режиме терминала для запуска одного или нескольких приложений. Это обновление включает в себя ряд изменений, касающихся использования браузеров в режиме терминала:

- Браузеры Microsoft Edge и Kiosk Browser теперь могут выполняться как приложения в режиме терминала (последовательно выберите **Конфигурация устройства** > **Профили** > **Создать профиль** > **платформа Windows 10 и более поздние версии** > **Киоск** для типа профиля).
- Можно запретить или разрешить выполнение Microsoft Edge в режиме терминала (последовательно выберите **Конфигурация устройства** > **Профили** > **Создать профиль** > **платформа Windows 10 и более поздние версии** > **Ограничения для устройств** для типа профиля > **Microsoft Edge Browser**). Если браузер Microsoft Edge выполняется не в режиме терминале, пользователи могут изменять его параметры.

Текущий список параметров см. в следующих статьях:

- [Настройка параметров устройства Windows 10 (и более поздних версий) для запуска в качестве киоска](kiosk-settings-windows.md)
- [Ограничения на использование браузера Microsoft Edge для устройств](device-restrictions-windows-10.md#microsoft-edge-browser)

Применяется к: Windows 10 и более поздней версии

### <a name="enabling-restrictions-in-the-device-settings-is-renamed-to-screen-time-on-ios-devices----3699164----"></a>Параметр "Включение ограничений в параметрах устройства" переименовывается во "Время работы экрана" на устройствах iOS <!-- 3699164  -->
На защищенных устройствах iOS можно настраивать параметр **Включение ограничений в параметрах устройства** (**Конфигурация устройства** > **Профили** > **Создать профиль** > **iOS** (платформа) > **Ограничения для устройства** (тип профиля) > **Общие**). В этом обновлении этот параметр переименовывается во **Время работы экрана (только в защищенном режиме)**. Принцип его работы не изменился. В частности, внесены следующие изменения. 

- В iOS 11.4.1 и более ранних версиях: При выборе значения **Блокировать** пользователям запрещается настраивать собственные ограничения в параметрах устройства. 
- В iOS 12.0 и более поздних версиях: При выборе значения **Блокировать** пользователям запрещается настраивать параметры **времени работы экрана** в параметрах устройства, включая конфиденциальность и ограничения на содержимое. На устройствах, обновленных до версии iOS 12.0, вкладка ограничений в параметрах устройства больше не отображается. Эти параметры перенесены в раздел **Время работы экрана**. 

Список текущих параметров см. в описании [ограничений для устройств iOS](device-restrictions-ios.md).

Применяется к: 
- iOS

- сбой из-за отключения App Store;

### <a name="android-enterprise-app-we-app-deployment----1171203---"></a>Развертывание приложения APP-WE на платформе "Android для бизнеса"<!-- 1171203 -->
Для устройств Android, задействованных в сценарии развертывания в рамках политики защиты приложений без регистрации (APP-WE), вы сможете с помощью управляемого магазина Google Play развертывать приложения из магазина и бизнес-приложения для пользователей. ### <a name="intune-policies-update-authentication-method-and-company-portal-app-installation-----1927359---"></a>Обновлены политики Intune для метода аутентификации и установки приложения "Корпоративный портал" <!-- 1927359 -->
В Intune пользователи больше не смогут вручную установить приложение "Корпоративный портал" из магазина приложений на устройствах, зарегистрированных с использованием помощника по настройке и с применением одного из методов регистрации корпоративных устройств Apple. Это изменение применяется, только если при регистрации для аутентификации использовался помощник по настройке Apple. Это изменение также касается только устройств iOS, зарегистрированных с использованием таких средств и программы:  
* Apple Configurator;
* Apple Business Manager;
* Apple School Manager
* программа регистрации устройств Apple (DEP).

Если пользователи установят приложение "Корпоративный портал" из App Store, а затем попытаются зарегистрировать устройства с помощью этого приложения, поступит сообщение об ошибке. Предполагается, что эти устройства будут использовать приложение "Корпоративный портал", только если оно автоматически отправлено Intune при регистрации. Профили регистрации в Intune на портале Azure будут обновлены, чтобы вы могли выбрать метод аутентификации устройств и указать, нужно ли получать приложение "Корпоративный портал". Если пользователям устройств в рамках программы DEP нужно предоставить приложение "Корпоративный портал", укажите это требование в профиле регистрации. Кроме того, экран **Идентификация устройства** в приложении "Корпоративный портал" скоро будет отмечен как устаревший.  
Чтобы установить приложение "Корпоративный портал" на устройстве, зарегистрированном в рамках программы DEP, последовательно выберите Intune > "Клиентские приложения" и отправьте это приложение в качестве управляемого с помощью политик конфигурации приложений. Более подробное описание этих действий будет представлено в последующих документах.

### <a name="administrative-templates-are-in-public-preview-and-moved-to-their-own-configuration-profile----3322847---"></a>Административные шаблоны реализованы в предварительной общедоступной версии и перемещены в отдельный профиль конфигурации <!-- 3322847 -->
Сейчас в Intune доступна закрытая предварительная версия административных шаблонов (**Конфигурация устройства** > **Административные шаблоны**). В этом обновлении: административные шаблоны содержат более 300 параметров, которыми можно управлять в Intune. Ранее эти параметры существовали только в редакторе групповых политик.
Административные шаблоны реализованы в общедоступной предварительной версии и перемещены из меню **Конфигурация устройства** > **Административные шаблоны**. Чтобы найти их, последовательно выберите **Конфигурация устройства** > **Профили** >**Создать профиль**, в разделе **Платформы** выберите **Windows 10 и более поздних версий**, в разделе **Тип профиля** выберите **Административные шаблоны**.
Доступна возможность создания отчетов, в следующих ОС: Windows 10 и более поздней версии
