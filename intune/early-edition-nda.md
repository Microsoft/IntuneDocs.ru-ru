**Ранний выпуск** содержит список функций, предоставляемых в рамках соглашения о неразглашении, которые будут включены в предстоящие выпуски Microsoft Intune. ### <a name="android-enterprise-apps----1352553----"></a>Приложения Android для бизнеса <!-- 1352553  -->
Вы сможете удалять приложения управляемого Google Play из Microsoft Intune. Чтобы удалить приложение управляемого Google Play, откройте Microsoft Intune на портале Azure и выберите **Клиентские приложения** > **Приложения**. В списке приложений щелкните многоточие (...) справа от приложения управляемого Google Play и в отобразившемся меню выберите **Удалить**. Приложение управляемого Google Play, удаленное из списка приложений, автоматически становится неутвержденным.

### <a name="managed-google-play-app-type----1352580---"></a>Тип приложения "Управляемый Google Play" <!-- 1352580 -->
Тип приложения **Управляемый Google Play** позволяет непосредственно добавлять [приложения управляемого Google Play](https://play.google.com/work/search?q=microsoft&c=apps) в Intune. Как администратор Intune вы сможете просматривать, искать, утверждать, синхронизировать и назначить утвержденные приложения управляемого Google Play в Intune. Вам больше не нужно специально переходить в консоль приложений управляемого Google Play и повторно проходить проверку подлинности. В Intune выберите **Клиентские приложения** > **Приложения** > **Добавить**. В списке **Тип приложения** выберите тип приложения **Управляемый Google Play**.

### <a name="deployment-of-online-licensed-microsoft-store-for-business-apps----1672660----"></a>Развертывание лицензированных в подключенном режиме приложений Microsoft Store для бизнеса <!-- 1672660  -->
Вы сможете назначать устройствам лицензированные в подключенном режиме приложения Microsoft Store для бизнеса. Такое развертывание лицензированных в подключенном режиме приложений Microsoft Store для бизнеса делает их доступными для всех пользователей устройства. Эта возможность поддерживается только на настольных компьютерах с Windows 10, начиная со сборки RS4. Возможность установки на устройство доступна на странице назначения клиентских приложений для лицензированных в подключенном режиме приложений Microsoft Store для бизнеса.

### <a name="configure-profile-to-skip-some-screens-during-setup-assistant----2276470----"></a>Настройка профиля для пропуска некоторых экранов в работе с помощником по настройке <!-- 2276470  -->
При создании профиля регистрации в macOS вы сможете настроить его так, чтобы пропускать при работе с помощником по настройке следующие экраны:
- Перенос на Android
- Тон дисплея
- Конфиденциальность
- iCloudStorage

### <a name="assign-autopilot-profiles-to-the-all-devices-virtual-group---2715522----"></a>Назначение профилей Autopilot виртуальной группе "Все устройства" <!--2715522  -->
Вы сможете назначать профили Autopilot виртуальной группе "Все устройства". Для этого последовательно выберите **Регистрация устройств** > **Регистрация Windows** > **Профили развертывания** > выберите профиль > **Назначения**, в разделе **Назначить** выберите **Все устройства**. Дополнительные сведения о профилях Autopilot см. в статье [Регистрация устройств Windows с помощью Windows Autopilot](enrollment-autopilot.md).

### <a name="customize-wallpaper-on-supervised-ios-devices-using-a-device-configuration-profile----2809324----"></a>Настройка фонового рисунка на защищенных устройствах iOS с помощью профиля конфигурации устройства <!-- 2809324  -->
При создании профиля конфигурации устройства для устройств iOS вы сможете настраивать некоторые параметры в меню **Конфигурация устройств** > **Профили** > **Создать профиль** > **iOS** (платформа) > **Ограничения для устройств** (тип профиля). Это обновление содержит новые параметры **Фоновый рисунок**, позволяющие администратору использовать изображения в формате PNG, JPG или JPEG в качестве фонового рисунка, предварительно просматривать рисунок и запрещать пользователям изменять его. Эти параметры фонового рисунка применяются только для защищенных устройств. Список текущих параметров см. в описании [параметров ограничения для устройств iOS](device-restrictions-ios.md).

Вы можете настроить игнорирование пользовательских всплывающих уведомлений пользователя для каждого назначения приложения. В Intune выберите **Клиентские приложения** > **Приложения** > выберите приложение > **Назначения** > **Включить группы**. 

### <a name="contact-sharing-via-bluetooth-is-removed-in-device-restrictions--device-owner-for-android-enterprise----3598396---"></a>Удаление параметра "Обмен контактами через Bluetooth" в разделе "Ограничения для устройств" > "Владелец устройства Android для бизнеса" <!-- 3598396 -->
После создания профиля ограничений устройства для устройств Android для бизнеса доступен параметр **Обмен контактами через Bluetooth**. В этом обновлении параметр **Обмен контактами через Bluetooth** будет удален (**Конфигурация устройства** > **Профили** > **Создать профиль** > **Android для бизнеса** (платформа) > **Ограничения для устройств > Владелец устройства** (тип профиля) > **Общие**). 

Параметр **Обмен контактами через Bluetooth** не поддерживается для управления профилем "Владелец устройства Android для бизнеса". Поэтому при удалении этого параметра устройства и клиенты не затрагиваются, даже если этот параметр включен и настроен в вашей среде.

Текущий список параметров см. в описании [параметров устройств Android для бизнеса, которые разрешают или ограничивают определенные функции](device-restrictions-android-for-work.md).

### <a name="android-enterprise-app-we-app-deployment----1171203---"></a>Развертывание приложения APP-WE на платформе "Android для бизнеса"<!-- 1171203 -->
Для устройств Android, задействованных в сценарии развертывания в рамках политики защиты приложений без регистрации (APP-WE), вы сможете с помощью управляемого магазина Google Play развертывать приложения из магазина и бизнес-приложения для пользователей. Кроме того, в этом сценарии развертывания пользователям будут предоставляться улучшенные условия работы.

### <a name="the-intune-app-sdk-will-support-256-bit-encryption-keys----1832174---"></a>Пакет SDK для приложений Intune будет поддерживать 256-битные ключи шифрования <!-- 1832174 -->
В пакете SDK для приложений Intune для Android предусмотрено использование 256-битных ключей шифрования, если шифрование разрешено политиками защиты приложений. Пакет SDK будет по-прежнему поддерживать 128-битные ключи шифрования для совместимости с содержимым и приложениями, для которых используются предыдущие версии SDK.


### <a name="intune-policies-update-authentication-method-and-company-portal-app-installation-----1927359---"></a>Обновлены политики Intune для метода аутентификации и установки приложения "Корпоративный портал" <!-- 1927359 -->
В Intune пользователи больше не смогут вручную установить приложение "Корпоративный портал" из магазина приложений на устройствах, зарегистрированных с использованием помощника по настройке и с применением одного из методов регистрации корпоративных устройств Apple. Это изменение применяется, только если при регистрации для аутентификации использовался помощник по настройке Apple. Это изменение также касается только устройств iOS, зарегистрированных с использованием таких средств и программы:  
* Apple Configurator;
* Apple Business Manager;
* Apple School Manager
* программа регистрации устройств Apple (DEP).

Если пользователи установят приложение "Корпоративный портал" из App Store, а затем попытаются зарегистрировать устройства с помощью этого приложения, поступит сообщение об ошибке. Предполагается, что эти устройства будут использовать приложение "Корпоративный портал", только если оно автоматически отправлено Intune при регистрации. Профили регистрации в Intune на портале Azure будут обновлены, чтобы вы могли выбрать метод аутентификации устройств и указать, нужно ли получать приложение "Корпоративный портал". Если пользователям устройств в рамках программы DEP нужно предоставить приложение "Корпоративный портал", укажите это требование в профиле регистрации. Кроме того, экран **Идентификация устройства** в приложении "Корпоративный портал" скоро будет отмечен как устаревший.  
Чтобы установить приложение "Корпоративный портал" на устройстве, зарегистрированном в рамках программы DEP, последовательно выберите Intune > "Клиентские приложения" и отправьте это приложение в качестве управляемого с помощью политик конфигурации приложений. Более подробное описание этих действий будет представлено в последующих документах.

Список текущих параметров см. в статье [Параметры защиты конечных точек для Windows 10 и более поздних версий в Intune](endpoint-protection-windows-10.md#windows-encryption).

### <a name="use-microsoft-recommended-settings-with-security-baselines----2055484---"></a>Использование параметров, рекомендуемых корпорацией Майкрософт, с базовыми показателями системы безопасности <!-- 2055484 -->
Intune интегрируется с другими службами, включая ATP в Защитнике Windows и ATP в Office 365, предназначенными для обеспечения безопасности. Клиентам требуется общая стратегия и полный набор готовых рабочих процессов по безопасности в службах Microsoft 365. Наша цель — обеспечить соответствие стратегий решениям для разработки, в которых объединяются операции безопасности и общие задачи администрирования. Мы планируем решить эту задачу в Intune путем публикации набора рекомендуемых корпорацией Майкрософт базовых показателей системы безопасности (**Intune** > **Базовые показатели системы безопасности**).  Администраторы смогут создавать политики безопасности непосредственно на основе этих базовых показателей, а затем развертывать их для пользователей. Они также могут настраивать рекомендации в соответствии с потребностями своей организации. Intune обеспечивает соответствие устройств этим базовым показателям и уведомляет администраторов пользователей или устройств о несоответствии.

### <a name="selective-wipe-support-for-wip-without-enrollment-devices----1434452---"></a>Поддержка выборочной очистки для WIP без регистрации устройств <!-- 1434452 -->
Технология Windows Information Protection Without Enrollment (WIP-WE) позволяет клиентам защитить корпоративные данные на устройствах с Windows 10 без необходимости полной регистрации в MDM. Администратор Intune может выполнять выборочную очистку данных в документах, защищенных с помощью политики WIP-WE. После выбора пользователя и устройства и отправки запроса на очистку все данные, которые были защищены с помощью политики WIP-WE, станут непригодными для использования. В Intune на портале Azure выберите **Мобильное приложение** > **Выборочная очистка приложений**.

### <a name="check-for-configuration-manager-compliance----2192052---"></a>Проверка соответствия Configuration Manager <!-- 2192052 -->
Будущее обновление включает новый параметр соответствия System Center Configuration Manager (последовательно выберите **Соответствие устройства** > **Политики** > **Создать политику** > **Windows 10**). Configuration Manager отправит в Intune сигналы о соответствии. С помощью параметра Intune можно настроить отправку сигналами Configuration Manager сообщения о соответствии.

Например, можно затребовать установку всех обновлений программного обеспечения на устройствах. В Configuration Manager это требование имеет состояние "Установлено". Если состояние каких-либо программ на устройстве неизвестно, устройство будет указано как несоответствующее в Intune.
