### <a name="android-enterprise-app-we-app-deployment----1171203---"></a>Развертывание приложения APP-WE на платформе "Android для бизнеса"<!-- 1171203 -->
Для устройств Android, задействованных в сценарии развертывания в рамках политики защиты приложений без регистрации (APP-WE), вы сможете с помощью управляемого магазина Google Play развертывать приложения из магазина и бизнес-приложения для пользователей. В частности, ИТ-компании могут предоставить пользователям каталог приложений и среду установки. Пользователям не потребуется понижать уровень защиты на своих устройствах, разрешая установку из неизвестных источников. Кроме того, в этом сценарии развертывания пользователям будут предоставляться улучшенные условия работы.

### <a name="new-options-to-automatically-connect-and-persist-rules-when-using-dns-settings-on-windows-10-and-later-devices----1333665-2999078---"></a>Новые параметры для автоматического подключения и хранения правил при использовании параметров DNS на устройствах с Windows 10 и более поздних версий <!-- 1333665, 2999078 -->
На устройствах с Windows 10 и более поздних версий вы сможете создать профиль конфигурации VPN со списком DNS-серверов для разрешения доменов, например contoso.com. Сюда входят новые параметры для разрешения имен (**Конфигурация устройства** > **Профили** > **Создать профиль**, для платформы выберите **Windows 10 и более поздних версий**, для типа профиля укажите **VPN**, последовательно выберите **Параметры DNS** >**Добавить**). 

- **Подключаться автоматически**. Если здесь выбран вариант **Включено**, устройство автоматически подключается к виртуальной частной сети при обращении к введенному домену, например contoso.com.
- **Сохранение**. По умолчанию все правила таблицы политики разрешения имен (NRPT) действуют, только пока устройство подключено с использованием этого профиля VPN. Если этот параметр **включен** в правиле NRPT, оно будет действовать на устройстве, даже если VPN-подключение будет разорвано или профиль VPN будет удален. Правило будет действовать, пока вы не удалите его вручную. Это можно сделать с помощью PowerShell.

Список доступных параметров VPN для Windows 10 см. [здесь](vpn-settings-windows-10.md). 

### <a name="use-smime-to-encrypt-and-sign-a-users-multiple-devices-----1333642-eeready---"></a>Использование S/MIME для шифрования и подписывания нескольких устройств пользователя <!-- 1333642 eeready -->
Будет поддерживаться шифрование электронной почты S/MIME с помощью нового импортированного профиля сертификата (**Конфигурация устройства** > **Профили** > **Создать профиль** > выберите платформу > тип профиля **Импортированный сертификат PKCS**). В Intune сертификаты можно импортировать в формате PFX. Intune затем может доставить эти сертификаты на несколько устройств, зарегистрированных одним пользователем. Другие функции:

- Собственный профиль электронной почты iOS поддерживает включение шифрования S/MIME с помощью импортированных сертификатов в формате PFX.
- Собственное почтовое приложение на устройствах Windows Phone 10 автоматически использует сертификат S/MIME.
- Частные сертификаты могут доставляться для нескольких платформ. Тем не менее, не все почтовые системы поддерживают S/MIME.
- На других платформах может потребоваться вручную настроить почтовой приложение для включения S/MIME.  
- Почтовые приложения, поддерживающие шифрование S/MIME, могут обрабатывать получение сертификатов для шифрования электронной почты S/MIME способом, который не поддерживается MDM, например путем чтения из хранилища сертификатов их издателя.

Поддерживается на: Windows, Windows Phone 10, macOS, iOS, Android

### <a name="intune-policies-update-authentication-method-and-company-portal-app-installation-----1927359---"></a>Обновлены политики Intune для метода аутентификации и установки приложения "Корпоративный портал" <!-- 1927359 -->
Intune больше не будет поддерживать установку приложения "Корпоративный портал" из App Store на некоторых устройствах. Это изменение применяется, только если при регистрации для аутентификации использовался помощник по настройке Apple. Это изменение также касается только устройств iOS, зарегистрированных с использованием таких средств и программы:  
* Apple Configurator;
* Apple Business Manager;
* Apple School Manager
* программа регистрации устройств Apple (DEP).

Если пользователи установят приложение "Корпоративный портал" из App Store, а затем попытаются зарегистрировать устройства с помощью этого приложения, поступит сообщение об ошибке. Предполагается, что эти устройства будут использовать приложение "Корпоративный портал", только если оно автоматически отправлено Intune при регистрации. Профили регистрации в Intune на портале Azure будут обновлены, чтобы вы могли выбрать метод аутентификации устройств и указать, нужно ли получать приложение "Корпоративный портал". Если пользователям устройств в рамках программы DEP нужно предоставить приложение "Корпоративный портал", укажите это требование в профиле регистрации. Кроме того, экран **Идентификация устройства** в приложении "Корпоративный портал" скоро будет отмечен как устаревший.  
Чтобы установить приложение "Корпоративный портал" на устройстве, зарегистрированном в рамках программы DEP, последовательно выберите Intune > "Клиентские приложения" и отправьте это приложение в качестве управляемого с помощью политик конфигурации приложений. Более подробное описание этих действий будет представлено в последующих документах.

### <a name="shared-device-configuration-is-renamed-to-lock-screen-message-for-ios-devices-in-the-azure-portal----2809362---"></a>Параметр конфигурации общего устройства для устройств iOS переименован в Lock Screen Message (Сообщение на экране блокировки) на портале Azure <!-- 2809362 -->
При создании профиля конфигурации для устройств iOS, вы сможете добавить параметр **Shared Device Configuration** (Конфигурация общего устройства) для отображения определенного текста на экране блокировки. Сюда входят следующие изменения: 

- Параметр **конфигурации общего устройства** на портале Azure переименован в "Lock Screen Message (supervised only)" (Сообщение на экране блокировки (только в защищенном режиме)). Выберите **Конфигурации устройства** > **Профили** > **Создать профиль**, для платформы укажите **iOS**, для типа профиля укажите **Функции устройства** и выберите **Lock Screen Message** (Сообщение на экране блокировки).
- При добавлении сообщения для экрана блокировки вы можете указать серийный номер, имя устройства или другое значение для устройства в качестве переменной элемента **Сведения о теге актива**. Например, можно ввести `Device name: {{device name}}` или `Serial number is {{serial number}}` в фигурных скобках. Список доступных токенов iOS см. [здесь](app-configuration-policies-use-ios.md#tokens-used-in-the-property-list).

Список текущих параметров для отображения сообщений на экране блокировки см. [здесь](shared-device-settings-ios.md).

Новые функции: 
- возможность отключить системные уведомления, в том числе входящие вызовы, системные предупреждения, системные ошибки и т. д.;
- отображение предложений пропустить руководство по началу работы и подсказок для приложений, открытых впервые;
- возможность отключить дополнительные параметры экрана блокировки, например параметры камеры, уведомлений, отпечатков пальцев и т. д.

Список текущих параметров см. в статье [Параметры ограничений для рабочих устройств в Intune](device-restrictions-android-for-work.md).

### <a name="android-enterprise-device-owner-devices-can-use-always-on-vpn-connections----3202194---"></a>На корпоративных устройствах Android можно использовать VPN-подключения AlwaysOn от имени владельца устройства <!-- 3202194 -->
В этом обновлении вы можете использовать VPN-подключения AlwaysOn на корпоративных устройствах Android от имени владельца устройства. При перезапуске или разблокировке устройства, а также при смене беспроводной сети VPN-подключение не будет разорвано или будет сразу же восстановлено. Также можно включить режим блокировки, при котором весь сетевой трафик будет заблокирован до момента активации VPN-подключения.
Чтобы включить VPN-подключение AlwaysOn, последовательно выберите **Конфигурация устройства** > **Профили** > **Создать профиль** > , в качестве платформы укажите **Android для бизнеса**, для параметра "Только владелец устройства" выберите значение **Ограничения для устройств** и щелкните **Подключение**. Список текущих параметров см. в статье [Параметры ограничений для рабочих устройств в Intune](device-restrictions-android-for-work.md).

### <a name="new-setting-to-end-processes-in-task-manager-on-windows-10-devices----3285177---"></a>Доступен новый параметр для завершения процессов в диспетчере задач на устройствах с Windows 10 <!-- 3285177 -->

Список текущих параметров см. в статье [Параметры ограничений устройств Windows 10 (и более поздних версий) в Intune](device-restrictions-windows-10.md).

### <a name="administrative-templates-are-in-public-preview-and-moved-to-their-own-configuration-profile----3322847---"></a>Административные шаблоны реализованы в предварительной общедоступной версии и перемещены в отдельный профиль конфигурации <!-- 3322847 -->

### <a name="use-microsoft-recommended-settings-with-security-baselines----2055484---"></a>Использование параметров, рекомендуемых корпорацией Майкрософт, с базовыми показателями системы безопасности <!-- 2055484 -->
Intune интегрируется с другими службами, включая ATP в Защитнике Windows и ATP в Office 365, предназначенными для обеспечения безопасности. Клиентам требуется общая стратегия и полный набор готовых рабочих процессов по безопасности в службах Microsoft 365. Наша цель — обеспечить соответствие стратегий решениям для разработки, в которых объединяются операции безопасности и общие задачи администрирования. Мы планируем решить эту задачу в Intune путем публикации набора рекомендуемых корпорацией Майкрософт базовых показателей системы безопасности (**Intune** > **Базовые показатели системы безопасности**).  Администраторы смогут создавать политики безопасности непосредственно на основе этих базовых показателей, а затем развертывать их для пользователей. Они также могут настраивать рекомендации в соответствии с потребностями своей организации. Intune обеспечивает соответствие устройств этим базовым показателям и уведомляет администраторов пользователей или устройств о несоответствии.

### <a name="scope-tags-for-apps---1081941---"></a>Теги области для приложений <!--1081941 -->
Вы сможете создавать теги области, чтобы ограничивать доступ к ресурсам Intune. Добавьте тег области к назначению роли, а затем добавьте тег области в профиль конфигурации. Роль будет иметь доступ только к ресурсам с профилями конфигурации, которые имеют соответствующие теги области (или не имеют тега области вообще).
Чтобы создать тег области, выберите **Роли Intune** > **Область (теги)** > **Создать**.
Чтобы добавить тег области к назначению роли, выберите **Роли Intune** > **Все роли** > **Диспетчер политик и профилей** > **Назначения** > **Область (теги)**.
Чтобы добавить тег области к профилю конфигурации, выберите **Конфигурация устройств** > **Профили** > выберите профиль > **Свойства** > **Область (теги)**.

### <a name="tenant-health-dashboard----1124854---"></a>Панель мониторинга работоспособности клиента <!-- 1124854 -->
На странице состояния клиента в Intune сведения о состоянии клиента будут доступны в одном месте. Страница разделена на четыре части:  
- **Сведения о клиенте**. Содержит такие сведения, как центр MDM, общее число зарегистрированных устройств в клиенте и число лицензий. Здесь также указан текущий выпуск службы для клиента.
- **Состояние соединителя**. Содержит сведения о настроенных соединителях, таких как соединители Apple VPP, соединители Microsoft Store для бизнеса и соединители сертификатов. Исходя из текущего состояния, соединители помечаются как *Работает*, *Предупреждение* или *Не работает*.
- **Работоспособность службы Intune**. Содержит перечень активных инцидентов или сбоев для клиента. Сведения в этом разделе извлекаются напрямую из Центра сообщений Office ([https://portal.office.com](https://portal.office.com)).
- **Новости Intune**. Содержит активные сообщения для клиента, в том числе уведомления о получении клиентом новейших функций Intune. Сведения в этом разделе извлекаются напрямую из Центра сообщений Office ([https://portal.office.com](https://portal.office.com)).


### <a name="deployed-wip-policies-without-user-enrollment----1434452---"></a>Развернутые политики WIP без регистрации пользователей <!-- 1434452 -->
Появится возможность развертывания политик Windows Information Protection (WIP) без необходимости регистрации устройств Windows 10 пользователей. Такая конфигурация позволяет компаниям защищать корпоративные документы на основе конфигурации WIP, при этом пользователь может управлять собственными устройствами Windows. Администратор Intune может выполнять выборочную очистку данных в документах, защищенных с помощью политики WIP. После выбора пользователя и устройства и отправки запроса на очистку все данные, которые были защищены с помощью политики WIP, станут непригодными для использования. В Intune на портале Azure выберите **Мобильное приложение** > **Выборочная очистка приложений**.

### <a name="check-for-configuration-manager-compliance----2192052---"></a>Проверка соответствия Configuration Manager <!-- 2192052 -->
Будущее обновление включает новый параметр соответствия System Center Configuration Manager (последовательно выберите **Соответствие устройства** > **Политики** > **Создать политику** > **Windows 10**). Configuration Manager отправит в Intune сигналы о соответствии. С помощью параметра Intune можно настроить отправку сигналами Configuration Manager сообщения о соответствии.

Например, можно затребовать установку всех обновлений программного обеспечения на устройствах. В Configuration Manager это требование имеет состояние "Установлено". Если состояние каких-либо программ на устройстве неизвестно, устройство будет указано как несоответствующее в Intune.

Применяется к Windows 10 и более поздним версиям
