# <a name="how-to-assign-apps-to-groups-with-microsoft-intune"></a>Назначение приложений группам с помощью Microsoft Intune

[!INCLUDE [azure_portal](./includes/azure_portal.md)]

Добавив приложение в Microsoft Intune, вы можете назначить его пользователям и устройствам.

Вы можете назначать приложение устройствам независимо от того, находятся ли они под управлением Intune. Сведения в таблице ниже помогут вам понять различные варианты назначения приложений пользователям и устройствам.

||||
|-|-|-|-|
|&nbsp;|Устройства, зарегистрированные в Intune|Устройства, не зарегистрированные в Intune|
|Назначение для пользователей|Да|Да|
|Назначение для устройств|Да|Нет|
|Назначение упакованных приложений или приложений, интегрированных с пакетом SDK для Intune (для политик защиты приложений)|Да|Да|
|Назначение приложений в качестве доступных|Да|Да|
|Назначение приложений в качестве необходимых|Да|Нет|
|Удаление приложений|Да|Нет|
|Получение обновления приложений в Intune|Да|Нет|
|Установка доступных приложений с приложения корпоративного портала|Да|Нет|
|Установка доступных приложений с корпоративного веб-портала|Да|Да|

> [!NOTE]
> Сейчас вы можете назначать приложения iOS и Android (бизнес-приложения и приложения, приобретенные в магазине) устройствам, не зарегистрированным в Intune.<br></br><br></br>
> Чтобы получить обновление приложений на устройствах, которые не зарегистрированы в Intune, пользователи устройств должны перейти на корпоративный портал и установить обновления вручную.

## <a name="how-to-assign-an-app"></a>Как назначать приложения

1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Intune**. Intune находится в разделе **Мониторинг и управление**.
3. В колонке **Intune** выберите **Мобильные приложения**.
4. В рабочей нагрузке **Мобильные приложения** в разделе **Управление** выберите **Приложения**.
5. В списке колонки приложений щелкните приложение, которое необходимо назначить.
6. В колонке **Обзор** для конкретного приложения выберите **Назначения** в разделе **Управление**.
7. Выберите **Add Group** (Добавить группу), чтобы отобразилась колонка **Add group** (Добавление группы) для приложения.
8. Для определенного приложения выберите один из следующих **типов назначения**:
   - **Доступно для зарегистрированных устройств.** Пользователи могут устанавливать приложение из приложения или веб-сайта корпоративного портала.
   - **Available with or without enrollment** (Доступно с регистрацией или без нее). Назначает это приложение группам пользователей, чьи устройства не зарегистрированы в Intune. Обратите внимание, что для типа приложения **Android for Work** этот параметр не поддерживается. 
   - **Обязательно.** Приложение устанавливается на устройствах в выбранных группах.
   - **Удалить.** Приложение удаляется из устройств в выбранных группах.

     > [!NOTE]
     > **Только для приложений iOS**. Если вы создали профиль VPN iOS, который содержит настройки VPN на уровне приложения, его можно выбрать в списке **VPN**. VPN-подключение открывается при запуске приложения. Дополнительные сведения см. в разделе [Параметры VPN для устройств iOS](vpn-settings-ios.md).

9. Щелкните **Включенные группы**, чтобы выбрать группы пользователей, которых затронет это назначение приложения.
10. Выбрав одну или несколько групп для включения, щелкните **Выбрать**.
11. В колонке **Назначить** нажмите кнопку **ОК**, чтобы завершить выбор включенных групп.
12. Щелкните **Exclude Groups** (Исключить группы), если нужно исключить какие-либо группы пользователей, которые затрагивает это назначение приложения.
13. Если вы решили исключить какие-либо группы, в колонке **Выбор групп** щелкните **Выбрать**.
14. Нажмите кнопку **ОК** в колонке **Добавление группы**.
15. Чтобы сохранить назначения, нажмите кнопку **Сохранить** в колонке **Назначения**.

Теперь приложение назначено выбранным группам. Дополнительные сведения о включении и исключении назначений приложений см. в статье [Включение и исключение назначений приложений в Microsoft Intune](apps-inc-exl-assignments.md).
