### <a name="change-in-enrollment-workflow-with-intune-company-portal-on-corporate-ios-devices-authenticating-with-setup-assistant----1927359---"></a>Изменения в рабочем процессе регистрации с помощью корпоративного портала Intune на устройствах iOS корпоративной проверки подлинности с помощью помощника по настройке <!-- 1927359 -->
Нет предстоящих изменений в рабочий процесс для регистрации устройств iOS с помощью одного из методов регистрации Apple корпоративных устройств — Apple Configurator, коммерческий директор Apple, Apple School Manager или Apple устройства регистрации программы (DEP), при использовании программы установки Помощник по для проверки подлинности. Это изменение применяется только к устройствам, зарегистрированным с помощью сопоставления пользователей.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Когда это изменение выйдет в ~~марте~~ или апреле, профили регистрации в Intune на портале Azure будут обновлены, чтобы вы могли выбрать метод аутентификации устройств и указать, нужно ли получать приложение "Корпоративный портал". Будет существовать улучшенный рабочий процесс для регистрации устройств iOS с помощью перечисленных выше методов. 

- При регистрации новых устройств и проверка подлинности с помощью помощника по настройке, вы сможете выбрать ли автоматически развертывать приложение корпоративного портала. Конечные пользователи больше не будут видеть на экране «Идентификация устройства» и экрана «Подтвердите устройства» в потоке регистрации.  
- На устройствах, уже зарегистрированных с использованием помощника с помощью одного из методов регистрации корпоративных устройств Apple необходимо выполнить определенные действия, если вы хотите включить условный доступ. Необходимо настроить политику конфигурации приложений с определенной xml для принудительной отправки на корпоративном портале, эти устройства. Инструкции, чтобы сделать это, в записи блога по ссылке Дополнительные сведения. Если вы решили Push-уведомлений на портале компании, таким образом, конечным пользователям больше не увидите на экране «Идентификация устройства» и экрана «Подтвердите устройства» в потоке регистрации. 
- После этого изменения разворачивается, если вы еще не развернули корпоративный портал с помощью профиля конфигурации приложения, упомянутых выше, и если загрузки конечным пользователям сохранить приложение корпоративного портала из приложения, они смогут войти в систему, но они будут возникать сообщения об ошибках. Они не смогут использовать приложение для условного доступа. 

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Если вы планируете использовать изменения рабочего процесса, необходимо обновить ваш руководство конечного пользователя, чтобы указать, что:

- Конечные пользователи больше не будут видеть два экрана, упомянутых выше в потоке регистрации. 
- Им необходимо зарегистрироваться на корпоративный портал, если оно автоматически развертывается и не для загрузки из магазина приложений. 

Вы можете создать политику конфигурации приложений сейчас, при необходимости в процессе подготовки для этого изменения. При развертывает этот новый рабочий процесс, вы увидите, что профили обновленные регистрации в консоли. Мы также будем проинформировать вас о отведен через Центр сообщений. После этого необходимо предпринять действия, чтобы конечные пользователи могут регистрировать через DEP с проверки подлинности с помощью помощника по настройке и использовании корпоративного портала для условного доступа.

### <a name="plan-for-change-user-experience-update-to-intune-company-portal-app-for-ios"></a>Планирование изменений: обновление пользовательского интерфейса в приложении корпоративного портала Intune для iOS
Мы рады сообщить, что Intune скоро выпустит существенное обновление пользовательского интерфейса для приложения корпоративного портала iOS. В обновлении будет переработан интерфейс домашней страницы, расширены фильтры и ускорен доступ к приложениям и электронной документации.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
В этом обновлении пользовательского интерфейса при сохранении текущих функциональных возможностей корпоративного портала для iOS доступно следующее.
- Домашняя страница с адаптированным под iOS внешним видом. 
- Возможности фильтрации перечней содержимого и поиска, в том числе фильтрация по типу содержимого (приложения или электронные книги) и доступности (управление устройствами обязательно или доступно без регистрации).
- Возможность поиска по электронным книгам.
- Журнал поиска для приложений и электронные книги

Если вы участник программы Apple TestFlight, вы получите уведомление о выходе предварительной версии обновленного приложения корпоративного портала Intune для iOS, когда она станет доступна. Если вы не участвуете в программе Apple TestFlight, еще не поздно зарегистрироваться. Регистрация позволит использовать обновленное приложение "Корпоративный портал" до того, как оно станет доступно конечным пользователям. Кроме того, можно оставлять отзывы непосредственно для группы Intune.  

#### <a name="what-can-i-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Не нужно предпринимать никаких действий; эти изменения будут присутствовать в предстоящем выпуске приложения корпоративного портала для iOS.

### <a name="check-your-delay-visibility-of-software-updates-setting-in-intune"></a>Проверьте ваш параметр «Задержка видимостью обновлений программного обеспечения» в Intune 

Мы совместно в MC171466, мы перемещения несколько параметров в консоли. Благодаря обновлению марта Intune нужно полностью удалить параметр «Задержка видимость программного обеспечения обновления» в колонке политики обновления iOS. Это не изменит способ применять обновления программного обеспечения по расписанию, но это может повлиять на время видимость обновление откладывается для конечных пользователей. Может потребоваться выполнить определенные действия до конца марта, если вы используете этот параметр. 

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
После обновления службы Intune февраля вы заметите, что настройка присутствует в профилей ограничения устройств в консоли и в iOS на обновление политики в колонке обновления программного обеспечения. Если вы видите это изменение, отражаются в консоли, вот что вам может потребоваться выполнить.

- Для существующей политики обновления для iOS: Если у вас есть пользовательский настроить этот параметр, чтобы ничего, кроме порта по умолчанию 30 дней и существующие конфигурации для параметра видимости задержки, чтобы применялись в конце марта, необходимо будет создать новую профиль ограниченного использования устройств iOS. Здесь видимость параметр задержки перед началом нужно будет иметь те же значения, как в существующие политики обновления iOS, а также применяться в те же группы. После обновления марта вы больше не смогут для изменения значения этого параметра в существующих политик обновления iOS, так как он больше не будет отображаться в этой колонке. Вы настроите этот параметр в новые профили вместо этого.
  Если значение для количества дней, которые вы можете отложить видимости не соответствует в обоих расположениях для значений, пользовательские настройки, видимость задержки, параметр не будет работать, и конечные пользователи будут видеть обновление на своих устройствах, как только он доступен. Это может иметь минимальное воздействие на большинство клиентов, поскольку другие параметры в колонке политики обновления программного обеспечения всегда захватили приоритет этот параметр в консоли.
- Для новой политики обновления для iOS: при попытке создать новые политики в колонке обновлений программного обеспечения после обновления службы Intune февраля, вы увидите этот параметр серым. Вы увидите Примечание в консоли, если требуется отложить видимость обновлений перенаправление в колонку настройки устройства.

#### <a name="what-can-i-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Необходимо предпринять действия, если этот параметр не используется или не требуется отложить видимостью обновлений программного обеспечения для конечных пользователей.

Если требуется отложить видимость обновлений, приступить к настройке параметра в новые профили в колонке для настройки устройства в разделе ограничения устройств > Общие. При наличии этот настраиваемый параметр настроен в iOS существующие политики обновления, создания профиля ограниченного использования программ эквивалентное устройства с одинаковым значением для «days» отложить видимость обновлений для пользователей, развертывает после февральской обновления и до марта. 

Вы можете обновить ваши рекомендации ИТ-специалистов и сообщите в службу поддержки.

См. в разделе поддержки записи блога на дополнительные сведения о том настроить этот параметр.

### <a name="plan-for-change-upcoming-fix-for-windows-10-email-profiles-in-intune---3904031--"></a>Планирование изменений: исправления для профилей электронной почты Windows 10 в Intune <!--3904031-->
Мы обновляем способ Intune записывает электронной почты, профили для Windows 10 в апреле обновление службы Intune для исправления ошибки также о том, убедитесь, что ваши профили электронной почты по-прежнему работать в будущих версиях Windows 10. Нет действий, которые необходимо выполнить после развертывания этого исправления.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Это изменение влияет на, если используются профили электронной почты Windows 10 с
- Собственный почтовый клиент на компьютерах Windows 10 или
- Клиент электронной почты Outlook на Windows 10 Mobile

Это влияет на обоих клиентов Intune автономной и гибридной службе управления мобильными устройствами (MDM).

После обновления апреля развертывает, необходимо повторно создать эти профили в консоли Intune (в консоли администрирования Configuration Manager, если вы используете гибридное управление мобильными Устройствами).

Если вы не предпримете действий, вот что вы увидите для профилей, созданных до обновления апреля:

- Существующие профили электронной почты будет отображаться в состоянии ошибки, в консоли Intune или консоли администрирования Configuration Manager, но конечные пользователи по-прежнему будет иметь доступ к электронной почте. Однако после развертывает последующие обновления Windows, эти профили не будет работать. Конечным пользователям на устройствах с этих профилей, потеряют доступ к электронной почте.
- Изменения, внесенные в эти профили после апреля не будут отражены в целевые устройства.
- Выборочная очистка не будет работать для удаления этих профилей, даже после исправления разворачивается в апреле.

Если принять меры и повторного создания профилей электронной почты, конечным пользователям придется выполнить действия, похожие на те при развертывании профиля электронной почты в первый раз. Доступ к электронной почте будет заблокирован синхронизации до принятия обновление, которое применяет новый профиль.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Необходимо выполнить действие, только в том случае, после исправления разворачивается с обновлением апреля. Мы обратимся к вам через Центр сообщений при это изменение в эксплуатацию вы можете начать их для повторного создания профилей в Intune.

При использовании профилей электронной почты Windows 10 в Intune, необходимо выполнить следующие действия:

1. Захват существующих параметров профиля Windows 10
2. Отменить назначение и/или удалить существующие профили
3. Создать новые профили, используя записанные параметры и назначить новые профили в те же группы

Может потребоваться уведомить конечных пользователей и сообщить в службу поддержки знать об этом изменении. См. запись блога поддержки дополнительные данные в сведения об ошибке и инструкции для повторного создания этих профилей.
