Эти уведомления предоставляют важные сведения, которые помогут вам подготовиться к будущим изменениям в Intune.

### <a name="updated-feature-new-rbac-role-coming-to-intune--4253397--"></a>Обновленная функция: новая роль RBAC в Intune<!--4253397-->
В январском обновлении службы Intune мы планируем выпустить новую роль безопасности. Эта роль будет указана в Intune как Endpoint Security Manager (Диспетчер безопасности конечной точки), а сама роль является расширением роли Azure AD "Администратор безопасности".
 
#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
На сегодняшний день в Azure AD доступны три приведенные ниже роли для специалистов по безопасности.
- Роль "Читатель сведений о безопасности" в Azure AD, которая предоставляет к Intune доступ только для чтения.
- Роль "Оператор безопасности" в Azure AD, которая предоставляет к Intune доступ только для чтения.
- Администратор безопасности в Azure AD. Когда Intune выпустит январское обновление с разрешениями только для чтения для Intune, новые разрешения, предоставляемые ролью Endpoint Security Manager (Диспетчер безопасности конечной точки), будут следующими:
    - чтение, создание, обновление, удаление и назначение политик соответствия устройств;
    - чтение, удаление и обновление управляемых устройств;
    - чтение, создание, обновление, удаление и назначение базовых показателей безопасности;
    - чтение и обновление задач безопасности.
 
### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Просмотрите свои роли RBAC в Intune уже сегодня. Если в настоящее время у вас есть только роли глобальных администраторов, никаких изменений не требуется. Если вы используете роли и вам нужна степень детализации, которую обеспечивает роль Endpoint Security Manager (Диспетчер безопасности конечной точки), назначьте эту роль, когда она будет доступна. Свежие сведения о выпуске Intune см. на странице [Новые возможности Microsoft Intune](../fundamentals/whats-new.md). 

### <a name="updated-support-statement-for-adobe-acrobat-reader-for-intune-mobile-app--5746776--"></a>Обновленная инструкция по поддержке для мобильного приложения Adobe Acrobat Reader для Intune<!--5746776-->
В конце августа мы сообщили в MC188653, что 1 декабря 2019 года срок службы мобильного приложения Adobe Acrobat Reader для Intune истекает и Adobe планирует поддерживать политики защиты приложений Intune в своем основном приложении Acrobat Reader. С тех пор мы получили отзывы клиентов о том, что необходимо предоставить больше времени, чтобы позволить ИТ-администраторам применить приложение Adobe Acrobat Reader для Intune в политиках, а пользователям — начать использовать его. Учитывая высокий уровень использования Adobe Acrobat Reader для Intune на устройствах пользователей и его важность в корпоративных сценариях, мы хотим обеспечить соответствие требованиям защиты приложений вашей организации. 

Хотя мы по-прежнему рекомендуем применять общее мобильное приложение Acrobat Reader в ваших политиках, так как мобильное приложение Acrobat Reader поддерживает политики защиты приложений и интегрировано с пакетом SDK для Intune, приложение Adobe Acrobat Reader для Intune будет поддерживаться до 31 марта 2020 г. 

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Вы получаете это сообщение, так как в нашем отчете указано, что одна или несколько политик в вашей организации направлены на приложение Adobe Acrobat Reader для Intune и (или) вы, возможно, получили наше предыдущее сообщение об истечении срока службы. 

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Сообщите своим пользователям и службе поддержки об этих изменениях. Для создания канала для вопросов, связанных с Intune, можно использовать [функцию указания сведений о поддержке на Корпоративном портале](../apps/company-portal-app.md#support-information).

### <a name="end-support-for-windows-phone-81--3544909--"></a>Завершение поддержки для Windows Phone 8.1<!--3544909-->
Срок действия основной фазы поддержки Microsoft Windows Phone 8.1 истек в июле 2017 года, а расширенной — в июне 2019 года. Приложение "Корпоративный портал" для Windows Phone 8.1 находится в режиме непрерывной связи с октября 2017. Microsoft Intune завершит поддержку Windows Phone 8.1 20 февраля 2020 года.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
После 20 февраля 2020 года эти устройства не будут получать обновления безопасности, и вы не сможете регистрировать новые устройства. Существующие устройства Windows Phone 8.1 будут оставаться зарегистрированными (политика, приложения, отчеты). Однако поддержка устранения неполадок существующей регистрации после этой даты не будет осуществляться, поскольку срок действия поддержки платформы для многих компонентов, таких как сторонние сертификаты, уже истек. Intune прекращает тестирование совместимости с Intune и Windows Phone 8.1.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Вы можете просмотреть отчеты Intune, чтобы узнать, на какие устройства или пользователей может повлиять это изменение. Перейдите в раздел "Устройства" > "Все устройства" и выполните фильтрацию по ОС. Вы можете добавить дополнительные столбцы, чтобы узнать, кто в вашей организации пользуется устройствами с Windows Phone 8.1. Предложите пользователям обновить устройства до поддерживаемой версии операционной системы.


### <a name="intune-plan-for-change-windows-10-version-1703-company-portal-moving-out-of-support--5026679--"></a>Запланированное изменение в Intune. Прекращение поддержки Корпоративного портала для Windows 10 версии 1703<!--5026679-->
С 8 октября 2019 г. для выпусков Enterprise и EDU прекращена поддержка Windows 10 версии 1703 (также называется Windows 10 RS2). С 26 декабря 2019 г. Intune прекратит поддержку соответствующего приложения Корпоративного портала для RS2 и RS1.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Вы больше не увидите новые возможности в конкретной версии приложения Корпоративного портала, хотя мы будем продолжать поддерживать эту версию приложения до 26 декабря 2019 г., в том числе при необходимости предоставлять обновления безопасности для этого приложения. Тем не менее для Windows 10 начиная с версии 1703 не будут предоставляться обновления безопасности после прекращения поддержки, поэтому мы настоятельно рекомендуем обновить устройства Windows до более новой версии Windows и убедиться, что вы используете последнюю версию приложения Корпоративного портала, чтобы вы могли получать новые возможности и дополнительный функционал.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Предпринимаемые действия зависят от настройки среды. В общем случае следует определить устройства с более ранней версией ОС или Корпоративного портала и обновить эти версии. Чтобы задать круги обновления Windows 10, войдите в Intune и выберите "Обновления программного обеспечения" — "Круги обновления Windows 10". Последняя версия Корпоративного портала — 10.3.5601.0. Направляйте своих пользователей для ее получения в Microsoft Store, чтобы у них были актуальные выпуски. Через Intune также можно установить последнюю версию на устройствах Windows с помощью [Microsoft Store для бизнеса](https://docs.microsoft.com/intune/windows-store-for-business).

### <a name="take-action-use-microsoft-edge-for-your-protected-intune-browser-experience--5728447--"></a>Меры, которые нужно принять. Использование Microsoft Edge для взаимодействия с защищенным браузером в Intune<!--5728447-->
Как мы уже рассказывали в прошлом году, мобильный браузер Microsoft Edge поддерживает тот же набор компонентов управления, что и Managed Browser, обеспечивая значительно улучшенное взаимодействие с пользователями. Для обеспечения надежной работы в Microsoft Edge мы прекращаем использование Intune Managed Browser. С 27 января 2020 г. Intune Managed Browser больше не будет поддерживаться в Intune.  

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу? 
С 1 февраля 2020 г. Intune Managed Browser больше не будет доступен в Google Play Маркет или в App Store для iOS. В данный момент вы по-прежнему сможете нацелить новые политики защиты приложений на Intune Managed Browser, но новые пользователи не смогут скачать приложение Intune Managed Browser. Кроме того, в iOS новые веб-клипы, которые отправляются на зарегистрированное в MDM устройство, открываются в Microsoft Edge вместо Intune Managed Browser.  

31 марта 2020 г. Intune Managed Browser будет удален из консоли Azure. Это означает, что вы больше не сможете создавать политики для Intune Managed Browser. Если у вас есть политики Intune Managed Browser, они не будут затронуты. Intune Managed Browser будет отображаться в консоли как бизнес-приложение без значка, а существующие политики все еще будут отображаться как предназначенные для приложения. Мы также удалим возможность перенаправлять веб-содержимое в Intune Managed Browser в разделе "Защита данных" политик защиты приложений.  

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению? 
Чтобы обеспечить плавный переход с Intune Managed Browser на Microsoft Edge, мы рекомендуем заранее выполнить следующие действия: 

1. Настройте использование Microsoft Edge для iOS и Android с помощью политики защиты приложений (также называется MAM) и параметров конфигурации приложения. Вы можете повторно использовать политики Intune Managed Browser для Microsoft Edge, применяя существующие политики для Microsoft Edge.  
2. Убедитесь, что для всех приложений, защищенных MAM в вашей среде, параметр политики защиты приложений "Ограничение передачи веб-содержимого другими приложениями" имеет значение "Браузеры, управляемые политиками". 
3. Настройте использование всех объектов с защитой MAM с параметром конфигурации управляемого приложения com.microsoft.intune.useEdge со значением true. Начиная со следующего месяца с выпуска 1911 вы сможете выполнить действия 2 и 3, просто настроив для параметра "Ограничение передачи веб-содержимого другими приложениями" значение "Microsoft Edge", выбрав его в разделе "Защита данных" политик защиты приложений. 

Скоро станет доступна поддержка веб-клипов на iOS и Android. После выпуска этой поддержки необходимо будет перенацелить уже существующие веб-клипы, чтобы они открывались в Microsoft Edge вместо Managed Browser.

### <a name="plan-for-change-updated-experience-when-enrolling-android-enterprise-dedicated-devices-in-intune--5198878--"></a>Подготовьтесь к изменениям. Обновленный процесс регистрации выделенных устройств Android Enterprise в Intune.<!--5198878-->
В выпуске Intune за ноябрь (выпуск 1911) мы добавляем поддержку развертывания сертификатов SCEP на выделенных устройствах Android Enterprise, что обеспечит доступ на основе сертификатов к профилям Wi-Fi. Это обновление также влечет незначительные изменения в потоке регистрации выделенных устройств Android Enterprise.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Если вы управляете выделенными устройствами Android Enterprise в своей среде, в ноябре вы заметите некоторые изменения.

- Для новых развертываний выделенных устройств Android Enterprise: Пользователям будет необходимо выполнить другие действия на устройствах во время регистрации. Регистрация будет начинаться так же, как сейчас (по QR-коду, через NFC, полностью автоматически или по идентификатору устройства), но добавится обязательный шаг установки приложения.
- Для существующих устройств Android, которые зарегистрированы как выделенные устройства: Intune начнет автоматически устанавливать приложение Microsoft Intune на устройства с первых чисел ноября. Никаких дополнительных действий от пользователей не потребуется. Приложение будет автоматически скачано и установлено на устройства. 

#### <a name="what-can-i-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Следует запланировать обновление руководства для пользователей и предоставить службе технической поддержки сведения об этом изменении. Дополнительные сведения и снимки экрана вы найдете по ссылке ниже. Мы обновим страницу новых возможностей, когда начнется развертывание этого изменения.

### <a name="decreasing-support-for-android-device-administrator"></a>Ограничение поддержки функции администратора устройства Android 
Функция администратора устройства Android (иногда называют устаревшим режимом управления Android — выпуск Android 2.2) — это способ управлять устройствами Android. Сейчас улучшенная функция управления доступна в [Android для бизнеса](../enrollment/connect-intune-android-enterprise.md) (выпуск Android 5.0). Чтобы способствовать переходу на современные, расширенные и более безопасные средства управления устройствами, компания Google ограничивает поддержку функции администратора устройства в новых выпусках Android.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Эти изменения, реализуемые компанией Google, повлияют на пользователей Intune следующим образом:  
- Intune будет предоставлять полную поддержку для устройств, управляемых администратором, с Android версии 10 и выше, только до второго квартала 2020 г. С этого момента устройства, управляемые администратором, с Android версии 10 и выше больше не будут поддерживаться в полной мере. В частности, затронутые устройства не получат новые требования к паролю.
    - На устройства Samsung Knox это не повлияет, так как расширенная поддержка обеспечивается с помощью интеграции Intune с платформой Knox. Это даст вам больше времени спланировать отказ от управления администратором.    
- Устройств, управляемых администратором, с Android до версии 10 это изменение не коснется, и этот режим управления будет поддерживаться в полной мере.    
- Для всех устройств Android 10 и последующих версий компания Google ограничила возможность получать доступ к сведениям об идентификаторах устройств для таких агентов управления устройствами администратором, как Корпоративный портал. Это ограничение влияет на следующие функции Intune после обновления устройства в Android 10 или более поздней версии:  
    - контроль доступа к сети для VPN больше не будет работать;   
    - устройства, определяемые как корпоративные с использованием IMEI или серийного номера, не будут автоматически отмечаться как корпоративные;  
    - IMEI и серийный номер больше не будут видны ИТ-администраторам в Intune. 
        > [!NOTE]
        > Это повлияет только на устройства Android 10 и последующих версий с функцией администратора устройства, но не на устройства с функцией управления Android для бизнеса. 

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Чтобы избежать проблем с ограниченной поддержкой этой функции начиная с третьего квартала 2020 г., рекомендуется сделать следующее:
- Не включайте на новых устройствах функцию администратора устройства.
- Если ожидается, что устройство будет обновлено до версии Android 10, замените функцию администратора устройства на функцию управления Android для бизнеса и (или) используйте политики защиты приложений.

### <a name="plan-for-change-intune-app-sdk-and-app-protection-policies-for-android-moving-to-support-android-50-and-higher-in-an-upcoming-release---4911065---"></a>Подготовьтесь к изменениям. В пакете SDK для приложений Intune и политиках защиты приложений для Android поддержка Android версии 5.0 и выше будет реализована в предстоящем выпуске <!--4911065 -->
В Intune поддержка Android версии 5.x (Lollipop) и выше будет реализована в предстоящем выпуске. Обновите все устройства и приложения с применением последней версии пакета SDK для приложений Intune.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Если вы не используете или не планируете использовать пакет SDK или политику защиты приложений для Android, это изменение не повлияет на вас. Если вы используете пакет SDK для приложений Intune, обязательно обновите его до последней версии, а также обновите устройства до Android версии 5.x и выше. В противном случае приложения не будут получать обновления, а качество взаимодействия с ними будет постепенно ухудшаться.

Ниже приведен список распространенных устройств, зарегистрированных в Intune, которые работают под управлением Android версии 4.x. Если у вас есть одно из этих устройств, выполните соответствующие действия. Так вы убедитесь, что устройство будет поддерживать Android 5.0 и последующих версий или что оно будет заменено устройством, поддерживающим эти версии Android. Приведенный ниже список устройств, которые нужно проверить, не является исчерпывающим:

- Samsung SM-T365
- Samsung GT-I9195
- Samsung SM-G800F
- Samsung SM-G357FZ
- Motorola XT1080
- Samsung GT-I9305
- Samsung SM-T231

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Примените к приложениям последнюю версию пакета SDK для приложений Intune. Вы также можете настроить для условного запуска требование минимальной версии ОС (только в качестве предупреждения), чтобы сообщать пользователям персональных устройств о необходимости выполнить обновление.

### <a name="intune-plan-for-change-nearing-end-of-support-for-windows-7---3042987---"></a>Запланированные изменения в Intune. Скорое прекращение поддержки Windows 7<!-- 3042987 -->
Как уже сообщалось в публикациях MC148476 и MC176794 в сентябре 2018 г. и марте 2019 г., расширенная поддержка Windows 7 будет прекращена 14 января 2020 г. С этого момента в Intune прекращается поддержка устройств под управлением Windows 7, что позволит нам сосредоточиться на более современных технологиях и предоставить пользователям новые возможности. С этой даты техническая поддержка и автоматические обновления, обеспечивающие защиту компьютеров под управлением Windows 7, будут недоступны в Intune. Корпорация Майкрософт настоятельно рекомендует перейти на Windows 10 до января 2020 г., чтобы избежать ситуаций с недоступностью необходимых служб или поддержки. Дополнительные сведения о жизненном цикле поддержки Windows см. [здесь](https://support.microsoft.com/help/13853/windows-lifecycle-fact-sheet).

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Вы получили это сообщение, так как управляете компьютерами c Windows 7, используя устаревший агент программного обеспечения Intune для ПК. Так как до окончания расширенной поддержки Windows 7 остается менее года, мы настоятельно рекомендуем вашей организации как можно скорее приступить к обновлению до Windows 10.  

Windows 10 имеет встроенные возможности управления ПК, поэтому вам больше не понадобится устанавливать агент клиента, например агент программного обеспечения Intune для Windows 7. Начиная с Windows 8.1 корпорация Майкрософт использует архитектуру управления мобильными устройствами (MDM) для подготовки, настройки, и обновления компьютеров Windows и управления ими. После настройки Intune можно упростить регистрацию Windows, [зарегистрировав ПК с Windows 10 в Intune](..\windows-enroll.md) через канал MDM. Для управления ПК с Windows 10 рекомендуется использовать это решение MDM для управления компьютерами без использования агента.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Мы рекомендуем вашей организации рассмотреть следующий план действий:

- До 14 января 2020 г. запланировать и осуществить обновление парка компьютеров под управлением Windows 7 до Windows 10.
- См. сведения о том, как [обновить ПК с Windows 7 до Windows 10](https://docs.microsoft.com/windows/deployment/).
- Ознакомиться с возможностями [службы Assure для классических приложений](https://www.microsoft.com/fasttrack/microsoft-365/desktop-app-assure?rtc=1) в рамках программы FastTrack, обеспечивающей совместимость приложений Майкрософт.
- Перейти с использования устаревшего агента программного обеспечения Intune для управления на рекомендуемое Майкрософт решение MDM для управления ПК с Windows 10. Зарегистрировать новые ПК с Windows 10 с помощью решения MDM для Intune на портале Azure.

См. [дополнительные сведения](https://aka.ms/Windows7_Intune).
