# <a name="the-early-edition-for-microsoft-intune---september-2017"></a>Ранний выпуск Microsoft Intune, сентябрь 2017 г.

**Ранний выпуск** содержит список функций, которые будут включены в предстоящие выпуски Microsoft Intune. Эта информация предоставляется в ограниченном объеме и может изменяться. Эта информация не предназначена для распространения за пределами вашей организации. Некоторые из указанных здесь компонентов могут быть не готовы к сроку, поэтому их выпуск может быть отложен. Другие функции проходят проверку в пилотном проекте, которая должна доказать их готовность для развертывания у клиентов. При наличии любых вопросов или проблем свяжитесь с представителем службы поддержки вашей группы продуктов Майкрософт.

Эта страница периодически обновляется. Проверьте наличие дополнительных обновлений.

> [!Note]
>Следующие изменения находятся в разработке для Intune. Дополнительные сведения о новых гибридных функциях см. на [странице новых гибридных функций](/sccm/mdm/understand/whats-new-in-hybrid-mobile-device-management).

### <a name="google-play-protect-support-on-android----908720----"></a>Поддержка пакета "Google Play Защита" в Android <!-- 908720  -->  
В новой ОС Android Oreo компания Google представила пакет средств для обеспечения безопасности под названием "Google Play Защита", который позволяет организациям и пользователям запускать надежные приложения и защищать образы Android. Решение Intune будет поддерживать функции пакета "Google Play Защита", в том числе удаленную аттестацию SafetyNet.  Администраторы могут задать требования политики соответствия для настройки и поддержания работоспособности пакета "Google Play Защита". Для настройки **аттестации устройств SafetyNet** необходимо подключить устройство к службе Google, чтобы убедиться, что оно работоспособно и не скомпрометировано. Администраторы также могут задать настройки профиля конфигурации для Android for Work, чтобы службы Google Play проверяли установленные приложения в обязательном порядке.  Функция условного доступа может заблокировать доступ пользователей к корпоративным ресурсам, если устройство не соответствует требованиям пакета "Google Play Защита". 

### <a name="prevent-users-of-android-devices-from-changing-their-device-date-and-time-----1333292---"></a>Запрет на изменение даты и времени на устройстве Android пользователями <!-- 1333292 -->

### <a name="scan-devices-with-windows-defender----1280988--1280990-----"></a>Проверка устройств с помощью Защитника Windows <!-- 1280988  1280990   -->
С помощью антивирусной программы "Защитник Windows" вы сможете выполнить **быструю проверку** и **полную проверку** устройств с Windows 10, а также **обновить сигнатуры** на них. В колонке сведений об устройстве выберите действие, которое необходимо выполнить на устройстве. Вам будет предложено подтвердить действие перед отправкой команды на устройство. 

**Быстрая проверка.** Позволяет проверить расположения, где вредоносная программа регистрируется для запуска, например разделы реестра и известные папки запуска Windows. Быстрая проверка в среднем занимает пять минут. При использовании с параметром **Always-on real-time protection** (Постоянная защита в режиме реального времени), который позволяет проверять открытые, закрытые и выбранные в папке файлы, быстрая проверка обеспечит защиту от вредоносных программ в системе или в ядре. По завершении сканирования пользователи увидят его результаты на своих устройствах. 

**Полная проверка.** Полезна для проверок по требованию, а также для устройств, которые подверглись угрозам вредоносных программ. Позволяет выявить все неактивные компоненты, для которых требуется более тщательная очистка. Полная проверка может занять около часа. По завершении сканирования пользователи увидят его результаты на своих устройствах. 

**Update signatures** (Обновление сигнатур). Эта команда обновляет определения и сигнатуры вредоносных программ для антивирусной программы "Защитник Windows". Это гарантирует эффективность антивирусной программы "Защитник Windows" при обнаружении вредоносных программ. Эта функция применима только к устройствам с Windows 10, и для нее требуется подключение к Интернету. 

### <a name="bitlocker-device-configuration----1397398---"></a>Настройка BitLocker для устройства <!-- 1397398 -->  
В разделе **базовых параметров шифрования Windows** появится новый параметр **Предупреждение при шифровании других дисков**, который позволяет отключить [предупреждение](https://docs.microsoft.com/en-us/windows/client-management/mdm/bitlocker-csp#allowarningforotherdiskencryption) при шифровании других дисков, которые могут использоваться на устройстве пользователя.  Это предупреждение требует согласия пользователя на настройку BitLocker на устройстве, и настройка BitLocker блокируется, пока пользователь не подтвердит согласие.  Новый параметр позволяет отключить это предупреждение для пользователя.

### <a name="company-portal-for-windows-81-and-windows-phone-81-moving-to-sustaining-mode---1428681--"></a>Переход корпоративного портала для Windows 8.1 и Windows Phone 8.1 в режим поддержки<!--1428681-->

Эти приложения по-прежнему будут доступны для скачивания через существующие каналы для выпуска, такие как магазин Майкрософт. 

После перехода в режим поддержки эти приложения будут получать только критически важные обновления для системы безопасности. Дополнительные обновления или функции больше не будут выпускаться для этих приложений. Для поддержки новых функций мы рекомендуем установить на устройствах Windows 10 или Windows 10 Mobile.

### <a name="intune-mam--outlook-for-android-add-ins-----1450688---"></a>Надстройки MAM и Outlook в Intune для Android <!-- 1450688 -->
Через несколько недель разработчики Office объявят о выпуске надстроек для Outlook в Android. Этот набор функций надстройки уже существует в Outlook Web App и Outlook в Windows, iOS и Mac. Так как управление надстройками осуществляется через Exchange, пользователи смогут копировать данные и сообщения и совместное использовать их в Outlook и неуправляемых приложениях, если администратор не отключил доступ к надстройкам в Exchange. 

Чтобы управлять правами пользователя на доступ к надстройкам, узнайте у администратора Exchange, применимы ли ваши политики защиты данных MAM к надстройкам.

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Если политики Exchange уже настроены, чтобы предотвращать загрузку неопубликованных надстроек или их установку, можете пропустить оставшуюся часть этого раздела. Политики MAM будут применены должным образом. Если же в политиках MAM настроен запрет на вырезание, копирование и вставку в Outlook на Android, а политика расширений не настроена в Exchange, следует знать, что по умолчанию пользователи смогут устанавливать надстройки в Outlook. Эти надстройки могут получать доступ к тексту, теме и другим свойствам сообщений. Вы можете отключить установку надстроек для пользователей, попросив администратора Exchange удалить роли My Marketplace Apps (Мои приложения Marketplace) и My Custom Apps (Мои пользовательские приложения). Дополнительные сведения см. по ссылке ниже.

Обратите внимание, что изменения настроек в Exchange применяются к Outlook Web App, Outlook Mobile и Outlook для Windows, iOS и Mac. 

#### <a name="what-do-i-need-to-do"></a>Что нужно сделать?
Просмотрите свои политики Exchange сейчас. Свяжитесь с ИТ-отделом и службой поддержки. Если у вас возникли вопросы или проблемы, обратитесь в нашу службу поддержки.

### <a name="actions-for-non-compliance----730266--"></a>Действия для несоответствия <!--730266-->     
*Действия для несоответствия* — это новая функция политик соответствия, позволяющая принимать меры для устройств, которые не соответствуют требованиям. Можно указать одно или несколько действий и период, когда они могут выполняться. Например, можно уведомить пользователя устройства по электронной почте сразу после нарушения соответствия либо заблокировать несоответствующему устройству доступ к корпоративным ресурсам по истечении 3-дневного льготного периода с помощью условного доступа.

Для каждого устройства можно узнать причины отсутствия автоматического обновления.

### <a name="zimperium---new-mobile-threat-defense-partner------954681---"></a>Zimperium — новый партнер Mobile Threat Defense <!-- 954681 -->

#### <a name="how-integration-with-intune-works"></a>Принцип работы интеграции с Intune
Оценка рисков основана на данных телеметрии, собранных с устройств, на которых выполняется приложение Zimperium. Вы можете настроить политики условного доступа EMS на основе оценки рисков Zimperium, реализуемой с помощью политик соответствия устройств Intune. Эти политики также можно использовать для разрешения или запрета доступа несоответствующих устройств к корпоративным ресурсам на основании обнаруженных угроз.

### <a name="new-azure-ad-conditional-access-policy-ui-link-from-intune-----1016201---"></a>Ссылка на новый пользовательский интерфейс политики условного доступа к Azure из Intune <!-- 1016201 -->
ИТ-администраторы могут задавать условные политики в новом пользовательском интерфейсе политики условного доступа в рабочей нагрузке Azure AD. Политики условного доступа на основе приложений, находящиеся в разделе "Защита приложений Intune" в Azure, останутся там и будут применяться параллельно. Также будет доступна удобная ссылка на новый пользовательский интерфейс политики условного доступа в Azure AD из рабочей нагрузки Intune.

### <a name="on-premises-exchange-connector-high-availability-support-----676614---"></a>Поддержка высокой доступности локального соединителя Exchange <!-- 676614 -->   
Вы можете использовать несколько ролей сервера клиентского доступа (CAS) для локального соединителя Exchange. Например, при сбое основного сервера клиентского доступа соединитель Exchange получает запрос на переключение на другие CAS. Эта функция обеспечивает непрерывную работу службы.

### <a name="system-center-operations-manager-management-pack-for-exchange-connector----885457---"></a>Пакет управления System Center Operations Manager для соединителя Exchange <!-- 885457 -->   
Будет представлен пакет управления System Center Operations Manager для соединителя Exchange, с помощью которого вы сможете анализировать журналы соединителя Exchange. Вы получаете дополнительные способы мониторинга службы Intune для устранения неполадок.

### <a name="end-of-support-for-ios-80----1164477---"></a>Окончание поддержки iOS 8.0 <!---1164477--->
Для доступа к корпоративным ресурсам управляемым приложениям и приложению корпоративного портала для iOS потребуется операционная система iOS 9.0 или более поздней версии. Устройства, которые не будут обновлены до сентября этого года, не смогут осуществлять доступ к указанным приложениям и корпоративному порталу. К декабрю любой доступ к корпоративным ресурсам, включая электронную почту, будет отключен. 

### <a name="end-of-support-for-android-43-and-lower----1171127-1326920----"></a>Окончание поддержки Android 4.3 и более ранних версий <!---1171127, 1326920 --->
Для доступа к корпоративным ресурсам управляемым приложениям и приложению корпоративного портала для Android потребуется Android 4.4 или более поздней версии. Устройства, которые не будут обновлены до начала октября, не смогут осуществлять доступ к указанным приложениям и корпоративному порталу. В декабре все зарегистрированные устройства будут принудительно сняты с учета, что приведет к утрате доступа к корпоративным ресурсам. Если вы используете политики защиты приложений без MDM, приложения не будут получать обновления, а качество взаимодействия с ними будет постепенно ухудшаться.

### <a name="platform-support-reminder-windows-phone-81-mainstream-support-will-end-july-11-2017----1327781---"></a>Напоминание о поддержке платформы: основная фаза поддержки Windows Phone 8.1 закончится 11 июля 2017 г. <!-- 1327781 -->  
11 июля 2017 г. заканчивается основная фаза поддержки платформы Windows Phone 8.1. Поддержка настольных систем на базе Windows 8.1 затронута не будет.

Это не окажет немедленное влияние на все устройства Windows Phone 8.1, управляемые службой Intune. Зарегистрированные устройства, как и все политики, конфигурации и приложения, продолжат работать должным образом. Обратите внимание на отсутствие усовершенствований для платформы Windows Phone 8.1 в службе Intune, а также для приложения корпоративного портала Windows Phone 8.1.

Рекомендуется при первой возможности обновить соответствующие устройства Windows Phone 8.1 до Windows 10 Mobile.
