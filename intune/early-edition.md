# <a name="the-early-edition-for-microsoft-intune---august-2017"></a>Ранний выпуск Microsoft Intune, август 2017 г.

**Ранний выпуск** содержит список функций, которые будут включены в предстоящие выпуски Microsoft Intune. Эта информация предоставляется в ограниченном объеме и может изменяться. Эта информация не предназначена для распространения за пределами вашей организации. Некоторые из указанных здесь компонентов могут быть не готовы к сроку, поэтому их выпуск может быть отложен. Другие функции проходят проверку в пилотном проекте, которая должна доказать их готовность для развертывания у клиентов. При наличии любых вопросов или проблем свяжитесь с представителем службы поддержки вашей группы продуктов Майкрософт.

Эта страница периодически обновляется. Проверьте наличие дополнительных обновлений.

> [!Note]
>Следующие изменения находятся в разработке для Intune. Дополнительные сведения о новых гибридных функциях см. на [странице новых гибридных функций](/sccm/mdm/understand/whats-new-in-hybrid-mobile-device-management).

## <a name="intune-on-the-azure-portal"></a>Intune на портале Azure

### <a name="actions-for-non-compliance----730266--"></a>Действия для несоответствия <!--730266-->     
*Действия для несоответствия* — это новая функция политик соответствия, позволяющая принимать меры для устройств, которые не соответствуют требованиям. Можно указать одно или несколько действий и период, когда они могут выполняться. Например, можно уведомить пользователя устройства по электронной почте сразу после нарушения соответствия либо заблокировать несоответствующему устройству доступ к корпоративным ресурсам по истечении 3-дневного льготного периода с помощью условного доступа.

### <a name="new-report-that-lists-ios-devices-with-older-ios-versions------1352223---"></a>Новый отчет, перечисляющий устройства iOS с более старыми версиями iOS <!-- 1352223 -->
Отчет **Устаревшие устройства iOS** будет доступен из рабочей области **Обновления программного обеспечения**. В отчете можно просмотреть список защищенных устройств iOS, которые были затронуты политикой обновления iOS и для которых существуют доступные обновления. Для каждого устройства можно узнать причины отсутствия автоматического обновления. 

### <a name="new-settings-to-allow-and-block-apps-on-samsung-knox-standard-devices-----822899--1305423--"></a>Новые параметры для разрешения и блокировки приложений на устройствах Samsung KNOX Standard <!-- 822899,  1305423-->   
Мы добавляем новые [параметры ограничения для устройств](device-restrictions-android.md), позволяющие указать следующие списки приложений:
  - приложения, которые пользователям разрешено устанавливать;
  - приложения, которые пользователям запрещено запускать;
  - приложения, которые скрыты от пользователей на устройстве.  

Вы можете указать приложение по URL-адресу, имени пакета или из списка управляемых вами приложений.

### <a name="new-settings-for-windows-10-device-restriction-profile"></a>Новые параметры для профиля ограниченного использования устройств Windows 10
<!--- 978575, 1308849, -->
Мы добавляем новые параметры в профиль ограниченного использования устройств Windows 10 в категории "SmartScreen Защитника Windows".

Дополнительные сведения о профиле ограниченного использования устройств Windows 10 см. в разделе [Параметры ограничений для применения к устройствам Windows 10 и более поздних версий]( device-restrictions-windows-10.md).




### <a name="android-for-work-support-for-lookout----1087312---"></a>Поддержка Android for Work для Lookout <!-- 1087312 -->   
Соединитель Intune с Lookout будет поддерживать устройства Android for Work при работе с приложением Lookout for Work. Вы можете развертывать приложение Lookout как внутри контейнера, так и за его пределами.


### <a name="intune-app-protection-and-citrix-mdx-development-tools----709185---"></a>Защита приложений Intune и средства разработки Citrix MDX <!-- 709185 -->
Устройствами и приложениями можно управлять с помощью комбинации Citrix XenMobile MDX и Microsoft Intune. Это позволяет управлять приложениями с политикой защиты приложений Intune при одновременном использовании технологии mVPN в Citrix.

Вы можете найти репозиторий кода, содержащий инструмент упаковки для приложений Intune и пакет SDK для приложений Intune для iOS и Android, интегрированный с технологией mVPN Citrix MDX.


### <a name="zimperium---new-mobile-threat-defense-partner------954681---"></a>Zimperium — новый партнер Mobile Threat Defense <!-- 954681 -->
Вы можете управлять доступом к корпоративным ресурсам с мобильных устройств посредством условного доступа на основании оценки рисков, проведенной Zimperium — решением для защиты от угроз на мобильных устройствах, интегрированным с Microsoft Intune.

#### <a name="how-integration-with-intune-works"></a>Принцип работы интеграции с Intune
Оценка рисков основана на данных телеметрии, собранных с устройств, на которых выполняется приложение Zimperium. Вы можете настроить политики условного доступа EMS на основе оценки рисков Zimperium, реализуемой с помощью политик соответствия устройств Intune. Эти политики также можно использовать для разрешения или запрета доступа несоответствующих устройств к корпоративным ресурсам на основании обнаруженных угроз.

### <a name="new-azure-ad-conditional-access-policy-ui-link-from-intune-----1016201---"></a>Ссылка на новый пользовательский интерфейс политики условного доступа к Azure из Intune <!-- 1016201 -->
ИТ-администраторы могут задавать условные политики в новом пользовательском интерфейсе политики условного доступа в рабочей нагрузке Azure AD. Политики условного доступа на основе приложений, находящиеся в разделе "Защита приложений Intune" в Azure, останутся там и будут применяться параллельно. Также будет доступна удобная ссылка на новый пользовательский интерфейс политики условного доступа в Azure AD из рабочей нагрузки Intune.


### <a name="on-premises-exchange-connector-high-availability-support-----676614---"></a>Поддержка высокой доступности локального соединителя Exchange <!-- 676614 -->   
Вы можете использовать несколько ролей сервера клиентского доступа (CAS) для локального соединителя Exchange. Например, при сбое основного сервера клиентского доступа соединитель Exchange получает запрос на переключение на другие CAS. Эта функция обеспечивает непрерывную работу службы.

### <a name="system-center-operations-manager-management-pack-for-exchange-connector----885457---"></a>Пакет управления System Center Operations Manager для соединителя Exchange <!-- 885457 -->   
Будет представлен пакет управления System Center Operations Manager для соединителя Exchange, с помощью которого вы сможете анализировать журналы соединителя Exchange. Вы получаете дополнительные способы мониторинга службы Intune для устранения неполадок.

### <a name="end-of-support-for-ios-80----1164477---"></a>Окончание поддержки iOS 8.0 <!---1164477--->
Для доступа к корпоративным ресурсам управляемым приложениям и приложению корпоративного портала для iOS потребуется операционная система iOS 9.0 или более поздней версии. Устройства, которые не будут обновлены до сентября этого года, не смогут осуществлять доступ к указанным приложениям и корпоративному порталу. К декабрю любой доступ к корпоративным ресурсам, включая электронную почту, будет отключен. 

### <a name="end-of-support-for-android-43-and-lower----1171127-1326920----"></a>Окончание поддержки Android 4.3 и более ранних версий <!---1171127, 1326920 --->
Для доступа к корпоративным ресурсам управляемым приложениям и приложению корпоративного портала для Android потребуется Android 4.4 или более поздней версии. Устройства, которые не будут обновлены до начала октября, не смогут осуществлять доступ к указанным приложениям и корпоративному порталу. В декабре все зарегистрированные устройства будут принудительно сняты с учета, что приведет к утрате доступа к корпоративным ресурсам. Если вы используете политики защиты приложений без MDM, приложения не будут получать обновления, а качество взаимодействия с ними будет постепенно ухудшаться.


### <a name="platform-support-reminder-windows-phone-81-mainstream-support-will-end-july-11-2017----1327781---"></a>Напоминание о поддержке платформы: основная фаза поддержки Windows Phone 8.1 закончится 11 июля 2017 г. <!-- 1327781 -->  
11 июля 2017 г. заканчивается основная фаза поддержки платформы Windows Phone 8.1. Поддержка настольных систем на базе Windows 8.1 затронута не будет.

Это не окажет немедленное влияние на все устройства Windows Phone 8.1, управляемые службой Intune. Зарегистрированные устройства, как и все политики, конфигурации и приложения, продолжат работать должным образом. Обратите внимание на отсутствие усовершенствований для платформы Windows Phone 8.1 в службе Intune, а также для приложения корпоративного портала Windows Phone 8.1.

Рекомендуется при первой возможности обновить соответствующие устройства Windows Phone 8.1 до Windows 10 Mobile. 

## <a name="intune-apps"></a>Приложения Intune

### <a name="intune-managed-browser-support-for-ios-and-android----1374196---"></a>Поддержка Intune Managed Browser для iOS и Android <!---1374196--->

Начиная с октября 2017 г. приложение Intune Managed Browser для Android будут поддерживать только устройства под управлением Android 4.4 и более поздних версий. Приложение Intune Managed Browser для iOS будет поддерживать только устройства под управлением iOS 9.0 и более поздних версий. В более ранних версиях iOS и Android можно продолжить использовать приложение Managed Browser, но нельзя установить новые версии приложения и использовать все возможности приложения. Мы рекомендуем обновить эти устройства до поддерживаемой версии операционной системы.

### <a name="allow-end-users-to-access-the-company-portal-app-for-android-without-enrollment----1169910---"></a>Разрешение конечным пользователям доступа к приложению корпоративного портала для Android без регистрации <!---1169910--->  
Скоро конечным пользователям не придется регистрировать свои устройства, чтобы получить доступ к приложению корпоративного портала для Android. Пользователи в организациях, где применяются политики защиты приложений, больше не будут получать запросы на регистрацию при открытии приложения корпоративного портала. Конечные пользователи также смогут устанавливать приложения с корпоративного портала без регистрации устройства. 

### <a name="improved-error-message-for-when-a-user-reaches-the-maximum-number-of-devices-allowed-to-enroll----1270370---"></a>Улучшенное сообщение об ошибке, отображаемое, когда пользователь достигает максимально разрешенного числа устройств для регистрации <!-- 1270370 -->
Вместо обычного сообщения об ошибке конечные пользователи видят понятное сообщение об ошибке: "Вы зарегистрировали максимальное число устройств, разрешенное ИТ-администратором. Удалите зарегистрированное устройство или обратитесь за помощью к ИТ-администратору".


### <a name="inform-end-users-what-device-information-can-be-seen-for-ios---739894--"></a>Информирование конечных пользователей о том, какие сведения можно увидеть об устройстве iOS <!--739894-->
Мы добавляем **тип владения** на экран сведений об устройстве в приложении корпоративного портала для iOS. Это позволит пользователям получать дополнительные сведения о конфиденциальности непосредственно с этой страницы из документации для конечных пользователей Intune. Они также смогут найти эти сведения на экране "О программе".

### <a name="apps-details-pages-display-new-information-for-android-devices---1287476--"></a>На страницах сведений о приложениях отображаются новые данные для устройств Android <!--1287476-->
На странице сведений о приложениях приложения корпоративного портала для Android будут отображаться категории приложений, которые ИТ-администратор определил для этого приложения.

### <a name="intune-mobile-application-management-mam-dialog-boxes-now-have-a-modern-interface----1199015---"></a>Диалоговые окна управления мобильными приложениями (MAM) Intune теперь имеют современный интерфейс <!-- 1199015 -->
Диалоговые окна управления мобильными приложениями (MAM) Intune были обновлены и теперь имеют современное оформление. Новые диалоговые окна функционируют так же, как и прежние.

На современных устройствах Android диалоговые окна ошибок или уведомлений, отображаемые в Intune, теперь будут иметь новое оформление.

### <a name="new-setting-in-the-android-company-portal-app-to-toggle-battery-optimization---1405990--"></a>Новый параметр в приложении корпоративного портала Android для включения или отключения оптимизации батареи <!--1405990-->
На странице **Параметры** в приложении корпоративного портала для Android будет находиться новый параметр, позволяющий отключать оптимизацию батареи для приложения корпоративного портала и Microsoft Authenticator. Имя приложения, отображаемое в параметре, зависит от того, какое приложение управляет рабочей учетной записью. Корпорация Майкрософт рекомендует пользователям отключить оптимизацию батареи для повышения производительности рабочих приложений, которые синхронизируют сообщения электронной почты и данные. 




## <a name="notices"></a>Объявления

### <a name="apple-to-require-updates-for-application-transport-security---748318--"></a>Требуемые обновления Application Transport Security от Apple <!--748318-->   
Компания Apple объявила, что с весны 2017 г. к Application Transport Security (ATS) будут применяться определенные требования. ATS позволяет обеспечить высокий уровень безопасности при передаче данных в приложениях по протоколу HTTPS. Это изменение затрагивает клиентов Intune, которые используют приложения корпоративного портала для iOS. Дополнительные сведения см. в [блоге службы поддержки Intune](https://aka.ms/compportalats).

### <a name="direct-access-to-apple-enrollment-scenarios---951869--"></a>Прямой доступ к сценариям регистрации Apple <!--951869-->   
Для учетных записей Intune, созданных после января 2017 г., включен прямой доступ к сценариям регистрации Apple с помощью рабочей нагрузки регистрации устройств на портале предварительной версии Azure. Ранее доступ к предварительной версии регистрации Apple можно было получить, только перейдя по ссылкам на классическом портале Intune. Учетные записи Intune, созданные до января 2017 г., потребуется перенести (единожды), перед тем как эти возможности станут доступны в Azure. Расписание переноса еще не объявлено, но мы предоставим сведения о нем в кратчайшие сроки. Если вам не удалось получить доступ к предварительной версии, используя имеющуюся учетную запись, мы настоятельно рекомендуем создать пробную учетную запись, чтобы протестировать новые возможности.

### <a name="plan-for-change-intune-is-changing-the-intune-partner-portal-experience----1050016---"></a>Планирование изменений: Intune изменяет интерфейс портала партнеров<!-- 1050016 -->
Мы удалим страницу партнеров Intune с сайта manage.microsoft.com после обновления службы в середине мая 2017 г.  

Если вы являетесь администратором страницы партнеров Intune, вы больше не сможете просматривать страницу и выполнять здесь действия от имени своих клиентов. Вместо этого вам нужно будет войти на один из двух других порталов партнера корпорации Майкрософт.

И в [центре партнеров Майкрософт](https://partnercenter.microsoft.com/), и в [центре администратора партнера Microsoft Office 365](https://portal.office.com/) вы сможете входить в учетные записи клиентов, которыми вы управляете. Если вы являетесь партнером, используйте один из этих сайтов для управления своими клиентами.
