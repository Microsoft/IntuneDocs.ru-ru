## <a name="app-management"></a>Управление приложениями

### <a name="managed-google-play-private-lob-apps----1464182----"></a>Управляемые Google Play закрытые бизнес-приложения <!-- 1464182  -->
Intune позволит ИТ-администраторам публиковать частные бизнес-приложения Android на управляемых Google Play через IFRAME, внедренный в консоль Intune.  В настоящее время ИТ-администраторы должны публиковать бизнес-приложения напрямую в консоли публикации Google Play, которая требует большого количества действий и занимает много времени.  Эта новая функция позволяет легко публиковать бизнес-приложения с минимальным набором шагов без необходимости выхода из консоли Intune.  Любой из сценариев управления предприятием Android, использующих управляемый Google Play, может воспользоваться преимуществами этой функции (рабочего профиля, выделенных, полностью управляемых и не зарегистрированных устройств).  В Intune выберите **Клиентские приложения** > **Приложения** > **Добавить**. Затем выберите **управляемые Google Play** из списка **Тип приложения** . Дополнительные сведения об управляемых Google Play приложениях см. [в статье Добавление управляемых Google Play приложений на устройства Android Enterprise с помощью Intune](apps-add-android-for-work.md).

### <a name="company-portal-app-installation-status-messages----2514416----"></a>Сообщения о состоянии установки корпоративный портал приложения <!-- 2514416  -->
Приложение корпоративный портал отобразит дополнительные сообщения о состоянии установки приложения для конечных пользователей. Следующие условия будут применены к новым функциям зависимости Win32.
- Приложение не удалось установить. Зависимости, определенные администратором, не выполнены.
- Приложение успешно установлено, но требуется перезагрузка.
- Приложение находится в процессе установки, но для продолжения требуется перезагрузка.

### <a name="managed-google-play-iframe-support----2871756----"></a>Поддержка управляемого Google Play IFRAME <!-- 2871756  -->
Intune предоставит поддержку для добавления и управления веб-ссылками непосредственно в консоли InTune с помощью управляемого Google Play IFRAME.  Это позволяет ИТ-администраторам отправлять URL и изображения значков, а затем развертывать эти ссылки на устройствах так же, как и обычные приложения Android. Любой из сценариев управления предприятием Android, использующих управляемый Google Play, может воспользоваться преимуществами этой функции (рабочего профиля, выделенных, полностью управляемых и не зарегистрированных устройств).  В Intune выберите **Клиентские приложения** > **Приложения** > **Добавить**. Затем выберите **управляемые Google Play** из списка **Тип приложения** . Дополнительные сведения об управляемых Google Play приложениях см. [в статье Добавление управляемых Google Play приложений на устройства Android Enterprise с помощью Intune](apps-add-android-for-work.md).

### <a name="macos-support-for-vpp-apps----3173501----"></a>Поддержка macOS для приложений VPP <!-- 3173501  -->
Приложения macOS, приобретенные с помощью Apple Business Manager, отображаются в консоли при синхронизации токенов Apple VPP в Intune. Вы можете назначать, отзывать и переназначать группам лицензии для устройств и пользователей с помощью консоли. Microsoft Intune помогает управлять приложениями VPP, приобретенными для использования в вашей компании, следующим образом:
- Предоставляет сведения о лицензиях из магазина приложений.
- Отслеживает количество уже используемых лицензий.
- Запрещает устанавливать большее количество копий приложения, чем у вас имеется.
См. сведения об Intune и VPP в руководстве по [управлению приложениями и книгами, приобретенными по программе корпоративных закупок, с помощью Microsoft Intune](vpp-apps.md).

### <a name="macos-support-for-web-apps----3174427----"></a>Поддержка macOS для веб-приложений <!-- 3174427  -->
Веб-приложения, которые позволяют добавлять ярлыки для URL-адресов в Интернете, можно закрепить на панели с помощью Корпоративного портала macOS. Пользователи могут получить доступ к действию **установки** на странице сведений о веб-приложении на Корпоративном портале macOS. Дополнительные сведения о типе приложения " **веб-ссылка** " см. [в разделе добавление приложений в Microsoft Intune](apps-add.md).

#### <a name="assign-microsoft-edge-beta-for-macos----4678761----"></a>Назначение бета-версии Microsoft ребра для macOS <!-- 4678761  -->
Вы сможете добавить и назначить последнюю версию бета-версии Microsoft ребра в Intune для устройств macOS. В Intune выберите **клиентские приложения** > **приложения** > **Добавить приложение** > **Microsoft ребро — macOS**. Затем назначьте бета-версию Microsoft ребра для нужных групп. Microsoft Update (MAU) обеспечивает актуальность Microsoft ребра. Дополнительные сведения о Microsoft ребр см. в статье [Управление веб-доступом с помощью Microsoft ребра с Microsoft Intune](manage-microsoft-edge.md).

### <a name="read-and-write-graph-api-operations-for-intune-apps----5031704----"></a>Операции чтения и записи API Graph для приложений Intune <!-- 5031704  -->
Приложения смогут вызывать API Graph Intune с операциями чтения и записи, используя удостоверение приложения без учетных данных пользователя. Дополнительные сведения о доступе к API Microsoft Graph для Intune см. в статье [Работа с Intune в Microsoft Graph](https://docs.microsoft.com/graph/api/resources/intune-graph-overview?view=graph-rest-1.0).

### <a name="configure-app-notification-content-for-organization-accounts----2576686---"></a>Настройка содержимого уведомлений приложения для учетных записей Организации <!-- 2576686 -->
Политики защиты приложений (APP) Intune на устройствах Android и iOS позволяют управлять содержимым уведомлений приложения для учетных записей организации. Эта функция требует поддержки приложений и может быть недоступна для всех приложений с поддержкой приложений. Дополнительные сведения о политиках защиты приложений см. в статье [Что такое политики защиты приложений?](app-protection-policy.md).

### <a name="available-google-play-app-reporting-for-android-work-profiles----3041956----"></a>Доступные отчеты о приложениях Google Play для рабочих профилей Android <!-- 3041956  -->
Для доступных установок приложений на устройствах с рабочими профилями Android вы можете просматривать сведения о состоянии установки приложения, а также установленную версию управляемых приложений Google Play. Дополнительные сведения см. в статьях [Как отслеживать политики защиты приложений](app-protection-policies-monitor.md), [Управление устройствами с рабочим профилем Android в Intune](android-enterprise-overview.md) и [Тип приложения "Управляемый Google Play"](apps-add-android-for-work.md#managed-google-play-app-types).

## <a name="device-configuration"></a>Конфигурация устройства

В Intune вы создаете профили для устройств iOS и macOS (**Профили** > **конфигурации** > устройств**создают профиль** > **iOS** или **macOS** для платформы > **компонентов устройств).** , **Ограничения устройства**или **расширения** для типа профиля). В настоящее время в этих профилях перечислены доступные параметры. 

В будущем обновлении доступные параметры на портале Intune будут классифицироваться по типу регистрации, к которому они относятся:

  - Все типы регистрации
  - Регистрация устройств
  - Пользователь утвердил и автоматически регистрирует устройства
  - Автоматическая регистрация устройств

### <a name="new-voice-control-settings-for-supervised-ios-devices-running-in-kiosk-mode----4892835----"></a>Новые параметры управления голосовыми устройствами для защищенных устройств iOS, работающих в режиме киоска <!-- 4892835  -->

В Intune можно создавать политики для запуска защищенных устройств iOS в качестве киоска или выделенного устройства (**Профили** > **конфигурации** > устройств**создают профиль** > **iOS** для платформы >  **Ограничения на устройства** для типа профиля > **киоска (только**в защищенном режиме)). 

В будущем обновлении будут доступны новые параметры, которыми можно управлять:

- **Голосовое управление**: включает голосовое управление на устройстве в полноэкранном режиме.
- **Изменение управления голосовыми**устройствами: разрешить пользователям изменять параметры управления голосовыми сообщениями на устройстве в режиме киоска.

Чтобы просмотреть текущие параметры, перейдите к [параметрам iOS киоска (только](device-restrictions-ios.md#kiosk)в защищенном режиме).

Применяется к:

- iOS версии 13.0 и выше

### <a name="use-single-sign-on-for-apps-and-websites-on-your-ios-and-macos-devices----4893175----"></a>Использование единого входа для приложений и веб-сайтов на устройствах iOS и macOS <!-- 4893175  -->
В будущем обновлении будут доступны новые параметры единого входа для устройств iOS и macOS (**Профили** > **конфигурации** > устройств**создают профиль** > **iOS** или **macOS** для Платформа > **возможности устройств** для типа профиля).

Используйте эти параметры для настройки единого входа, особенно для приложений и веб-сайтов, использующих проверку подлинности Kerberos. Вы можете выбрать расширение приложения единого входа с универсальными учетными данными и встроенное расширение Kerberos Apple.

Чтобы просмотреть текущие возможности устройств, которые можно настроить, перейдите в раздел [функции устройства iOS](ios-device-features-settings.md) и [функции устройства macOS](macos-device-features-settings.md).

Применяется к:

- iOS 13.0 и более поздние версии.
- macOS 10.15 и более поздние версии;

### <a name="associate-domains-to-apps-on-macos-1015-devices----4898079----"></a>Связывание доменов с приложениями на устройствах macOS 10.15 + <!-- 4898079  -->
На устройствах macOS можно настроить различные функции и отправить эти функции на устройства с помощью политики (**Профили** > **конфигурации** > устройств**создать профиль** > **macOS** для Платформа > **возможности устройств** для типа профиля). В следующем обновлении вы сможете связать домены с вашими приложениями. Эта функция позволяет использовать учетные данные совместно с веб-сайтами, относящимися к вашему приложению, и может использоваться с расширением единого входа Apple, универсальными связями и автозаполнением паролей. 

Чтобы просмотреть текущие компоненты, которые можно настроить, перейдите в раздел [Параметры функций устройства macOS в Intune](macos-device-features-settings.md).

Применяется к:

- macOS 10.15 и более поздние версии;

### <a name="use-itunes-and-apps-in-the-itunes-app-store-url-when-showing-or-hiding-apps-on-ios-supervised-devices----4928474----"></a>Использование "iTunes" и "приложения" в URL-адресе магазина приложений iTunes при отображении или скрытии приложений на защищенных устройствах iOS <!-- 4928474  --> 
В Intune можно создавать политики для отображения или скрытия приложений на защищенных устройствах iOS (**Профили** > **конфигурации** > устройств**Создание профиля** > **iOS** для платформы > **устройство ограничения** для типа профиля > **Показать или скрыть приложения (только**в защищенном режиме)). 

Вы можете ввести URL-адрес магазина приложений iTunes, например `https://itunes.apple.com/us/app/work-folders/id950878067?mt=8`. В следующем обновлении вы сможете использовать `apps` и `itunes` в URL-адресе, например:

- `https://itunes.apple.com/us/app/work-folders/id950878067?mt=8`
- `https://apps.apple.com/us/app/work-folders/id950878067?mt=8`

Дополнительные сведения об этих параметрах см. в разделе [Показать или скрыть приложения](device-restrictions-ios.md#show-or-hide-apps).

Применяется к:

- iOS

### <a name="support-for-ikev2-vpn-profiles-for-ios----1943438---"></a>Поддержка профилей VPN IKEv2 для iOS <!-- 1943438 -->
Вы сможете создавать профили VPN для собственного VPN-клиента iOS с использованием протокола IKEv2. IKEv2 — это новый тип подключения, доступный в разделе **Конфигурация устройства** > **Профили** > **Создать профиль** > **iOS** (платформа) > **VPN** (тип профиля) > **Параметры**.

Эти профили VPN настраивают собственный VPN-клиент. Таким образом, никакие приложения VPN-клиента не устанавливаются на управляемых устройствах и не отправляются на них принудительно. Для использования этой функции нужно зарегистрировать устройства в Intune (регистрация MDM).

Чтобы просмотреть текущие параметры VPN, которые можно настроить, см. раздел [Настройка параметров VPN для устройств iOS в Microsoft Intune](vpn-settings-ios.md).

Применимо к: iOS

## <a name="device-management"></a>Управление устройствами

### <a name="deploy-software-updates-to-macos-devices----3194876---"></a>Развертывание обновлений программного обеспечения на устройствах macOS <!-- 3194876 -->
Вы сможете развертывать обновления программного обеспечения в группах устройств macOS. Эта функция включает в себя критическую версию, встроенное по, файл конфигурации и другие обновления. Вы сможете отправить обновления на следующем устройстве после возврата или выбрать еженедельное расписание для развертывания обновлений в заданных окнах или из них. Это помогает в том случае, если требуется обновить устройства за пределами стандартного рабочего времени или когда служба технической поддержки полностью обработана. Вы также получите подробный отчет обо всех macOS устройствах с развернутыми обновлениями. Можно детализировать отчет для каждого устройства, чтобы просмотреть состояние конкретных обновлений.

### <a name="send-custom-notifications-to-a-device----4928910----"></a>Отправка пользовательских уведомлений на устройство <!-- 4928910  -->
Вы сможете отправлять пользовательские уведомления на определенные устройства, на которых установлено корпоративный портал или приложение Intune. Для этого перейдите в раздел**устройства** >  **Intune** > **все устройства** > выберите устройство > **больше** > **Отправить настраиваемое уведомление**. 

### <a name="updates-to-android-enterprise-fully-managed-features----3464667-5227935-4062195-4631425-4631440---"></a>Обновления полностью управляемых функций Android Enterprise <!-- 3464667, 5227935, 4062195, 4631425, 4631440 -->

Мы добавим следующую поддержку для полностью управляемых устройств Android:

- Сертификаты SCEP для полностью управляемого Android будут доступны для проверки подлинности сертификатов на устройствах, управляемых как владелец устройства. Сертификаты SCEP уже поддерживаются на устройствах рабочих профилей.  С сертификатами SCEP для владельца устройства вы сможете: <!-- 5227935 -->
    - Создание профиля SCEP в разделе "выполнить" в Android Enterprise
    - Связывание сертификатов SCEP с профилем Wi-Fi для проверки подлинности
    - Связывание сертификатов SCEP с профилями VPN для проверки подлинности
    - Связывание сертификатов SCEP с профилями электронной почты для проверки подлинности (через конфигурацию приложения)
- Системные приложения будут поддерживаться на устройствах Android Enterprise. В Intune вы добавите Системное приложение Android Enterprise, выбрав пункт **клиентские** > **приложения** > **Добавить**. В списке **Тип приложения** выберите **Android Enterprise System App**. Дополнительные сведения о добавлении приложений в Intune см. в разделе [Добавление приложений в Microsoft Intune](apps-add.md). <!-- 4062195 -->
- В области **соответствие устройства требованиям** > для**владельца устройства** **Android Enterprise** > вы сможете создать политику соответствия требованиям, которая задает уровень аттестации Google SafetyNet.   <!-- 4631425 -->
- На полностью управляемых устройствах Android Enterprise будут поддерживаться поставщики защиты от угроз для мобильных устройств. В списке **соответствие устройства требованиям** > для**владельца устройства** **Android Enterprise** > можно выбрать приемлемый уровень угроз. <!-- 4631440 --> Текущие параметры — это [параметры Android для бизнеса, позволяющие пометить устройства как соответствующие или не соответствующие политике с помощью Intune](compliance-policy-create-android-for-work.md#device-owner).


Применяется к: 
- Полностью управляемые устройства Android Enterprise

## <a name="security"></a>Безопасность

### <a name="tamper-protection-for-windows-defender-antivirus-----4705448---------"></a>Защита от взлома для антивирусной программы "Защитник Windows"  <!-- 4705448       -->
Мы будем добавлять *защиту от* изменений в параметры, которыми может управлять Intune для антивирусной программы "Защитник Windows". Вы сможете использовать профиль конфигурации устройства для защиты конечных точек Windows 10, чтобы включить или отключить защиту от несанкционированного доступа.  Дополнительные сведения о защите от несанкционированного доступа см. в разделе [предотвращение изменений параметров безопасности с помощью защиты от](https://docs.microsoft.com/windows/security/threat-protection/windows-defender-antivirus/prevent-changes-to-security-settings-with-tamper-protection) изменения в документации по Windows.
