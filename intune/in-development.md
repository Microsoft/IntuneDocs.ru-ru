# <a name="in-development-for-microsoft-intune---june-2019"></a>В разработке для Microsoft Intune, июнь 2019 г.

Для помощи в подготовке и планировании эта страница перечисляет компоненты и обновления пользовательского интерфейса Intune, которые находятся на стадии разработки, но не были выпущены. Дополнительно

- Если мы полагаем, что вам потребуется принять меры до изменения, мы опубликуем дополнительное сообщение в центре сообщений Office.
- Когда функция выпускается в рабочей среде в предварительной или общедоступной версии, описание компонента переместится с этой страницы на страницу [Что нового](whats-new.md).
- Эта страница и страница [Что нового](whats-new.md) периодически обновляются. Проверьте наличие дополнительных обновлений.
- Стратегические цели и шкалу времени см. в документе [Стратегия развития Microsoft 365](https://www.microsoft.com/microsoft-365/roadmap?rtc=2&filters=EMS).

> [!Note]
> Эти элементы отражают текущие ожидания корпорации Майкрософт относительно возможностей Intune, которые ожидаются в будущем выпуске. Даты и отдельные компоненты могут изменяться. Не все элементы в разработке описаны на этой странице.

## <a name="intune-in-the-azure-portal"></a>Intune на портале Azure

### <a name="app-management"></a>Управление приложениями

#### <a name="device-users-can-view-all-managed-apps-theyve-installed-or-tried-to-install----2352913---"></a>Пользователи устройства могут просматривать все управляемые приложения, которые они установили или пытались установить. <!-- 2352913 -->
Корпоративный портал для Windows будет перечислять все управляемые приложения (как обязательные, так и доступные), установленные на устройстве пользователя. Пользователи смогут просматривать попытки установки и ожидающие установки приложения, а также их текущее состояние. Если ваша организация не делает приложения обязательными или доступными, пользователи увидят сообщение о том, что приложения компании не были установлены. Пользователи также смогут сортировать или фильтровать свои приложения по состоянию установки.

#### <a name="configure-which-browser-is-allowed-to-link-to-organization-data----3145939---"></a>Настройка браузера, который можно связывать с данными организации <!-- 3145939 -->
Политики защиты приложений (APP) Intune на устройствах Android и iOS позволят передать веб-ссылки организации в конкретный браузер за пределами Intune Managed Browser или Microsoft Edge.  Дополнительные сведения о политиках защиты приложений см. в статье [Что такое политики защиты приложений?](app-protection-policy.md).

#### <a name="installed-apps-page-on-the-company-portal-website-----4224326---"></a>Страница установленных приложений на веб-сайте корпоративного портала  <!-- 4224326 -->
[Веб-сайт корпоративного портала](https://portal.manage.microsoft.com/) будет содержать новую страницу, показывающую пользователям все приложения, которые были установлены на их устройствах. Этот список включает в себя как доступные приложения, так и приложения, являющиеся обязательными в соответствующей организации. На этой странице пользователи смогут просмотреть состояния установки и требований для приложений на своем устройстве. Дополнительные сведения о веб-сайте корпоративного портала см. в статьях [Использование веб-сайта корпоративного портала Intune](/intune-user-help/using-the-intune-company-portal-website) и [Настройка приложения корпоративного портала Microsoft Intune](company-portal-app.md).

#### <a name="call-graph-api-read-operations-from-an-application-without-user-credentials----4655885---"></a>Вызов операций чтения API Graph из приложения без учетных данных пользователя <!-- 4655885 -->
Приложения смогут вызывать операции чтения API Graph Intune с помощью удостоверения приложения и без учетных данных пользователя. Дополнительные сведения см. в разделе [Получение доступа без пользователя](https://docs.microsoft.com/graph/auth-v2-service).

### <a name="device-configuration"></a>Конфигурация устройства


#### <a name="support-for-ikev2-vpn-profiles-for-ios----1943438---"></a>Поддержка профилей VPN IKEv2 для iOS <!-- 1943438 -->
Вы сможете создавать профили VPN для собственного VPN-клиента iOS с использованием протокола IKEv2. IKEv2 — это новый тип подключения, доступный в разделе **Конфигурация устройства** > **Профили** > **Создать профиль** > **iOS** (платформа) > **VPN** (тип профиля) > **Параметры**.

Эти профили VPN настраивают собственный VPN-клиент. Таким образом, никакие приложения VPN-клиента не устанавливаются на управляемых устройствах и не отправляются на них принудительно. Для использования этой функции нужно зарегистрировать устройства в Intune (регистрация MDM).

Чтобы просмотреть текущие параметры VPN, которые можно настроить, см. раздел [Настройка параметров VPN для устройств iOS в Microsoft Intune](vpn-settings-ios.md).

Применимо к: iOS

#### <a name="configure-settings-for-kernel-extensions-on-macos-devices----20430240---"></a>Настройка параметров модулей ядра на устройствах с macOS <!-- 20430240 -->
На устройствах с macOS можно создать профиль конфигурации устройства (**Конфигурация устройства** > **Профили** > **Создать профиль** > **macOS** для платформы). Будущее обновление будет включать в себя новую группу параметров, позволяющих настраивать и использовать расширения ядра на ваших устройствах.

Применимо к: macOS 10.13.2 и более поздних версий

#### <a name="baseline-support-for-keyword-search-----3082036-----------"></a>Поддержка базовых показателей для поиска по ключевым словам  <!-- 3082036         -->
При создании или редактировании профиля базовых показателей безопасности вы скоро сможете использовать *поиск* для фильтрации отображаемых в консоли настроек.   

#### <a name="use-applicability-rules-when-creating-windows-10-device-configuration-profiles----2549910---"></a>Используйте "правила применимости" при создании профилей конфигурации устройств с Windows 10 <!-- 2549910 -->
Вы создаете профили конфигурации устройств Windows 10 (**Конфигурация устройства** > **Профили** > **Создать профиль**  >  **Windows 10** для платформы). Вы сможете создавать **правила применимости**, чтобы профиль применялся только к определенной версии или выпуску. Например, можно создать профиль, который включает некоторые параметры BitLocker. Когда вы добавите профиль, используйте правило применимости, чтобы профиль применялся только к устройствам под управлением системы Windows 10 Корпоративная.

Применяется к: 
- Windows 10 и более поздней версии

#### <a name="apps-from-the-store-only-setting-for-windows-10-devices-includes-more-configuration-options----2697002----"></a>Параметр "Использование приложений только из Магазина" для устройств Windows 10 включает в себя дополнительные параметры конфигурации <!-- 2697002  -->

При создании профиля ограничений для устройств Windows вы можете использовать параметр **Использование приложений только из Магазина**, чтобы пользователи могли устанавливать только приложения из Магазина приложений Windows (**Конфигурация устройства** > **Профили** > **Создать профиль** > **Windows 10 и более поздних версий** (платформа) > **Ограничения для устройств** (тип профиля)). В будущем обновлении этот параметр будет расширен для поддержки дополнительных параметров. 

Чтобы узнать текущие значения, воспользуйтесь статьей [Параметры устройства с Windows 10 (и более новых версий), которые позволяют разрешить или ограничить некоторые функции через Intune](device-restrictions-windows-10.md#app-store).

Область применения: Windows 10 и более поздние версии

#### <a name="deploy-multiple-zebra-mobility-extensions-device-profiles-to-a-device-same-user-group-or-same-devices-group----4089955---"></a>Развертывание нескольких профилей устройств для расширений Zebra Mobility Extensions на устройстве, в одной группе пользователей или одной группе устройств <!-- 4089955 -->
В Intune можно использовать расширения Zebra Mobility Extensions (MX) в профиле конфигурации устройства для настройки параметров или добавления параметров, не встроенных в Intune. Сейчас можно развернуть один профиль на одном устройстве. В будущем обновлении вы сможете развернуть несколько профилей:

- в одной группе пользователей;
- в одной группе устройств;
- на одном устройстве.

Использование расширений MX в Intune описано в разделе [Использование устройств Zebra с Zebra Mobility Extensions и управление ими в Microsoft Intune](android-zebra-mx-overview.md).

Область применения: Android

#### <a name="some-kiosk-settings-on-ios-devices-are-set-using-block-replacing-allow----4404075----"></a>Некоторые параметры киоска на устройствах iOS задаются с использованием значения "Блокировать", заменяющего "Разрешить" <!-- 4404075  -->
Чтобы создать профиль ограничений для устройств iOS (**Конфигурация устройства** > **Профили** > **Создать профиль** > **iOS** (платформа) > **Ограничения для устройств** (тип профиля) > **Киоск**), задайте параметры **Автоблокировка**, **Переключение звонка**, **Поворот экрана**, **Кнопка спящего режима экрана** и **Кнопки громкости**. 

Сейчас эти параметры настраиваются с помощью значения **Разрешить** (блокирует функцию) или **Не настроено** (разрешает функцию). В будущем обновлении эти значения примут следующий вид: **Блокировать** (блокирует функцию) или **Не настроено** (разрешает функцию).

Чтобы узнать текущие значения, воспользуйтесь статьей [Параметры устройства с iOS, которые позволяют разрешить или ограничить некоторые функции через Intune](device-restrictions-ios.md). 

Применимо к: iOS

#### <a name="use-face-id-for-password-authentication-on-ios-devices----4490704----"></a>Использование Face ID для проверки подлинности пароля на устройствах iOS <!-- 4490704  -->
При создании профиля ограничений для устройств iOS можно использовать отпечаток пальца в качестве пароля. В будущем обновлении параметры использования отпечатка в качестве пароля также будут включать распознавание лиц (**Конфигурация устройства** > **Профили** > **Создать профиль** > **iOS** (платформа) > **Ограничения для устройств** (тип профиля) > **Пароль**). Таким образом, изменяются следующие параметры:

- **Разблокирование с помощью отпечатков пальцев** теперь называется **Разблокировка Touch ID и Face ID**.
- **Модификация отпечатка (только в защищенном режиме)** теперь называется **Модификация Touch ID и Face ID (только в защищенном режиме)** .

Face ID доступен в iOS 11.0 и более поздних версий. Чтобы узнать текущие параметры, см. статью [Параметры устройства с iOS, которые позволяют разрешить или ограничить некоторые функции через Intune](device-restrictions-ios.md#password).

Применимо к: iOS

#### <a name="apps-feature-is-dependent-on-ratings-region-when-restricting-gaming-and-app-store-features-on-ios-devices----4593948----"></a>Функции приложения зависят от региона возрастной категории при ограничении игровых функций и функций магазина приложений на устройствах iOS <!-- 4593948  -->
На устройствах iOS можно разрешить или ограничить функции, связанные с играми, магазином приложений и просмотром документов (**Конфигурация устройства** > **Профили** > **Создать профиль** > **iOS** (платформа) > **Ограничения для устройств** (тип профиля) > **App Store, просмотр документов, игры**). Вы также можете выбрать регион возрастной категории, например США. В будущем обновлении функция **Приложения** станет дочерней по отношению к функции **Регион возрастной категории** и зависит от значения **Регион возрастной категории**.

Чтобы узнать текущие параметры, см. статью [Параметры устройства с iOS, которые позволяют разрешить или ограничить некоторые функции через Intune](device-restrictions-ios.md#app-store-doc-viewing-gaming).

Применимо к: iOS

#### <a name="administrative-templates-for-group-policy---------3510695---"></a>Административные шаблоны для групповой политики     <!--  3510695 -->
Для повышения безопасности устройств в облаке мы выпустим административные шаблоны, которые позволят использовать Intune для настройки отдельных параметров групповой политики для компьютеров под управлением Windows.  Эти шаблоны используют поставщик служб конфигурации (CSP) политики, чтобы предоставить до 2500 дополнительных параметров из Office, Windows и OneDrive.

####  <a name="new-settings-for-the-windows-security-baseline-----3534649-4217151------------"></a>Новые параметры для базовых показателей безопасности Windows  <!-- 3534649, 4217151          -->
Мы добавляем новые параметры в базовые показатели безопасности Windows. Первый параметр предназначен для реализации безопасности на основе виртуализации, для чего требуется безопасная загрузка. Второй параметр позволит управлять активацией голосовых функций для приложений Windows при заблокированном экране.

#### <a name="security-baselines-will-be-generally-available------3785395---------"></a>Базовые показатели безопасности будут общедоступными  <!--  3785395       -->
Функция базовых показателей безопасности скоро перейдет на этап предварительной версии и станет общедоступной. 

#### <a name="the-windows-security-baseline-template-will-be-generally-available-------3794072---------"></a>Шаблон базовых показателей безопасности Windows будет общедоступным   <!--  3794072       -->
Шаблон базовых показателей безопасности Windows скоро перейдет на этап предварительной версии и станет общедоступным. Использование предварительной версии этого шаблона будет прекращено, и будет доступен новый шаблон.

### <a name="device-management"></a>Управление устройствами

#### <a name="assign-scope-tags-to-all-managed-devices-in-a-security-group----3173810---"></a>Назначение тегов области для всех управляемых устройств в группе безопасности <!-- 3173810 -->
Сейчас вы можете назначить устройству тег области, перейдя на страницу **Свойства** отдельного устройства и выбрав теги области. В будущем обновлении вы сможете назначить теги области группе безопасности, при этом все устройства в ней также будут связаны с этими тегами. Для этого выберите **Intune** > **Роли** > **Область (теги)**  > **Создать** > **Область (теги)** и укажите группы, которым хотите назначить тег области. Этот тег области также будет назначен всем устройствам в этих группах. Теги области, заданные с помощью этой функции, будут перезаписывать теги области, заданные с помощью текущего потока тегов области устройства. (В будущем обновлении текущий поток для назначения тегов области устройствам будет сделан доступным только для чтения.)

#### <a name="see-the-security-patch-level-for-android-devices----4461911----"></a>Просмотр уровня обновления для системы безопасности для устройств Android <!-- 4461911  -->
Вы сможете просмотреть уровень обновления для системы безопасности для устройств Android. Для этого выберите **Intune** > **Устройства** > **Все устройства** > укажите устройство > **Монитор** > **Оборудование**.
