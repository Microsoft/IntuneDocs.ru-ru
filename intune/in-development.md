# <a name="in-development-for-microsoft-intune---april-2019"></a>В разработке — Microsoft Intune, апрель 2019 г.

Для помощи в подготовке и планировании эта страница перечисляет компоненты и обновления пользовательского интерфейса Intune, которые находятся на стадии разработки, но не были выпущены. Кроме того:

- Если мы полагаем, что вам потребуется принять меры до изменения, мы опубликуем дополнительное сообщение в центре сообщений Office.
- Когда функция выпускается в рабочей среде в предварительной или общедоступной версии, описание компонента переместится с этой страницы на страницу [Что нового](whats-new.md).
- Эта страница и страница [Что нового](whats-new.md) периодически обновляются. Проверьте наличие дополнительных обновлений.
- Стратегические цели и шкалу времени см. в документе [Стратегия развития Microsoft 365](https://www.microsoft.com/microsoft-365/roadmap?rtc=2&filters=EMS).

> [!Note]
> Эти элементы отражают текущие ожидания корпорации Майкрософт относительно возможностей Intune, которые ожидаются в будущем выпуске. Даты и отдельные компоненты могут изменяться. Не все элементы в разработке описаны на этой странице.

**RSS-канал**. Получите уведомление об обновлении этой страницы, скопировав и вставив следующий URL-адрес в программу чтения каналов: `https://docs.microsoft.com/api/search/rss?search=%22in+development+-+microsoft+intune%22&locale=en-us`

### <a name="set-login-settings-and-control-restart-options-on-macos-devices----1210083---"></a>Задание параметров входа и управление параметрами перезапуска на устройствах macOS <!-- 1210083 -->
На устройствах macOS можно создать профиль конфигурации устройства (**Конфигурация устройства** > **Профили** > **Создать профиль** > **macOS** для платформы > **Функции устройства** для типа профиля). Новые параметры окна входа будут включать такие элементы, как отображение настраиваемого баннера, выбор способа входа для пользователей, отображение или скрытие параметров питания и многое другое.

Чтобы просмотреть текущие параметры, см. описание [параметры функций устройств macOS](macos-device-features-settings.md).

Только для macOS

### <a name="advanced-settings-for-windows-defender-firewall----1311949---"></a>Дополнительные параметры для брандмауэра Защитника Windows <!-- 1311949 -->
Скоро вы сможете использовать Intune для управления особыми правилами брандмауэра на клиентах для Защитника Windows. Правила позволяют указать поведение входящего и исходящего трафика для приложений, сетевых адресов и портов. 

### <a name="require-app-protection-conditional-access----1634317---"></a>Требуется условный доступ для защиты приложения  <!--1634317 -->
Вы сможете использовать *политику требования защиты приложений*, которая подтверждает, что политика применяется к приложению пользователя до завершения входа, чтобы запретить пользователям доступ к данным, которые вы защитили с помощью условного доступа. Хотя применение политики может замедлить первое использование, оно позволяет предотвратить проблемы с сетью, неверные административные настройки или намеренные усилия обойти политики защиты приложений. 

### <a name="retire-noncompliant-devices----1827291---"></a>Снятие несоответствующих устройств с учета <!-- 1827291 -->
Мы собираемся добавить новое действие соответствия, позволяющее снимать с учета несоответствующие устройства. Когда несоответствующие устройство снимается с учета, с него удаляются все корпоративные данные; устройство также удаляется из управления Intune. Это действие выполняется при достижении заданного значения в днях. Минимальное значение — 30 дней. 

### <a name="configure-settings-for-kernel-extensions-on-macos-devices----2043024---"></a>Настройка параметров модулей ядра на устройствах с macOS <!-- 2043024 -->
На устройствах с macOS можно создать профиль конфигурации устройства (**Конфигурация устройства** > **Профили** > **Создать профиль** > **macOS** для платформы). Новая группа параметров позволяет настраивать и использовать расширения ядра на ваших устройствах.

Применимо к: macOS 10.13.2 и более поздних версий

### <a name="configure-profile-to-skip-some-screens-during-setup-assistant----2276470---"></a>Настройка профиля для пропуска некоторых экранов при работе с помощником по настройке <!-- 2276470 -->
При создании профиля регистрации в macOS можно настроить его так, чтобы пропускать в работе с помощником по настройке следующие экраны:
- Перенос на Android
- Тон дисплея
- Конфиденциальность
- iCloudStorage. Если вы изменяете профиль или создаете новый, выбранные пропускаемые экраны должны синхронизироваться с сервером управления мобильными устройствами Apple. Пользователи могут выполнить синхронизацию устройств вручную, чтобы избежать задержек при получении изменений профиля.
Подробные сведения см. в статье [Автоматическая регистрация устройств с macOS с помощью Программы регистрации устройств или Apple School Manager](device-enrollment-program-enroll-macos.md).

### <a name="device-users-can-view-all-managed-apps-theyve-installed-or-tried-to-install----2352913---"></a>Пользователи устройства могут просматривать все управляемые приложения, которые они установили или пытались установить. <!-- 2352913 -->
Корпоративный портал для Windows будет перечислять все управляемые приложения (&ndash;как обязательные, так и доступные&ndash;), установленные на устройстве пользователя. Пользователи смогут просматривать попытки установки и ожидающие установки приложения, а также их текущее состояние. Если ваша организация не делает приложения обязательными или доступными, пользователи увидят сообщение о том, что приложения компании не были установлены. Пользователи также смогут сортировать или фильтровать свои приложения по состоянию установки.

### <a name="scope-tags-for-apple-vpp-tokens---2371886---"></a>Теги области для токенов Apple VPP <!--2371886 -->
Теперь вы сможете добавить теги области в токены Apple VPP. Только пользователи, которым назначен такой же тег области, будут иметь доступ к токену Apple VPP с таким тегом. Приложения VPP и электронные книги, приобретенные с использованием этого токена, наследуют его теги области. Дополнительные сведения о тегах области см. в разделе [Использования RBAC и тегов области](scope-tags.md).

### <a name="use-applicability-rules-when-creating-windows-10-device-configuration-profiles----2549910---"></a>Используйте "правила применимости" при создании профилей конфигурации устройств с Windows 10 <!-- 2549910 -->
Вы создаете профили конфигурации устройств Windows 10 (**Конфигурация устройства** > **Профили** > **Создать профиль**  >  **Windows 10** для платформы). Вы сможете создавать **правила применимости**, чтобы профиль применялся только к определенной версии или выпуску. Например, можно создать профиль, который включает некоторые параметры BitLocker. Когда вы добавите профиль, используйте правило применимости, чтобы профиль применялся только к устройствам под управлением системы Windows 10 Корпоративная.

Область применения: 
- Windows 10 и более поздней версии

### <a name="enable-win32-app-dependencies----2617348---"></a>Включение зависимостей приложения Win32 <!-- 2617348 -->
Общедоступная предварительная версия — как администратор вы сможете запросить установку других приложений в виде зависимостей, прежде чем устанавливать свое приложение Win32. В частности, зависимые приложения необходимо установить на устройстве перед установкой приложения Win32. Эта функциональность будет доступна только после обновления агента управления Intune до версии 1904 (выше 1.18.120.0). Это может занять одну или две дополнительных недели после обновления версии службы до 1904. В Intune выберите **Клиентские приложения** > **Приложения** > **Добавить**, чтобы отобразить колонку **Добавить приложение**. Выберите **Приложение Windows (Win32)** в поле **Тип приложения**. Дополнительные сведения см. в разделе [Изолированная среда Intune — управление приложениями Win32](apps-win32-app-management.md).

### <a name="new-device-restriction-setting-for-android-enterprise-device-owner-let-users-connect-to-wi-fi-networks-on-android-enterprise-dedicated-devices-running-multi-app-kiosk-mode---3041940---"></a>Новый параметр ограниченного использования устройств для Android для бизнеса, владелец устройства: разрешить пользователям подключаться к сетям Wi-Fi на устройствах Android для бизнеса в выделенном режиме киоска с несколькими приложениями <!--3041940 -->
Администраторы смогут включить новый параметр, который позволяет пользователям настраивать Bluetooth на устройствах Android для бизнеса под управлением выделенного режима киоска с несколькими приложениями. Чтобы просмотреть этот параметр в консоли Intune, выберите **Intune** > **Конфигурация устройства** > **Профили** > **Создание профиля** > выберите **Android Enterprise** для платформы > **Только владелец устройства, ограничения устройства** для типа профиля > **Параметры**  > **Выделенные устройства** > выберите **С несколькими приложениями** в раскрывающемся списке **Полноэкранный режим**. Параметр **Конфигурация Wi-Fi** станет доступен для включения. 

Применимо: выделенные устройства Android Enterprise, работающие в режиме киоска с несколькими приложениями.

Чтобы просмотреть этот параметр в консоли Intune, выберите **Intune** > **Конфигурация устройства** > **Профили** > **Создание профиля** > выберите **Android Enterprise** для платформы > **Только владелец устройства, ограничения устройства** для типа профиля > **Параметры**  > **Выделенные устройства** > выберите **С несколькими приложениями** в раскрывающемся списке **Полноэкранный режим**. ### <a name="monitor-security-baseline-status-public-preview----3082047---"></a>Мониторинг состояния базовых показателей безопасности (предварительная версия) <!-- 3082047 --> 
При отслеживании *состояния устройства* для ваших базовых конфигураций системы безопасности представление будет упорядочивать состояние по категориям базовых показателей, например *Выше блокировки*, *BitLocker* и *Браузер*. Будут представлены все доступные категории базовых показателей. Для каждой категории вы увидите, сколько устройств не соответствуют определенной базовой категории, настроены неправильно или не применяются.

###  <a name="intune-security-tasks-for-defender-atp-in-public-preview----3208597---"></a>Задачи безопасности Intune для ATP в Защитнике (в общедоступной предварительной версии) <!-- 3208597 -->
Ожидается в виде общедоступной предварительной версии: Intune скоро добавит задачи по обеспечению безопасности для нового средства Защитника Майкрософт по управлению угрозами и уязвимостями.  Благодаря этой интеграции администраторы операций безопасности в ATP в Защитнике Windows (WDATP) смогут более эффективно применять рекомендуемые исправления для новых угроз, касающихся администраторов Intune. Добавление задач по обеспечению безопасности реализует подход на основе рисков для обнаружения, определения приоритетов и устранения уязвимостей и неправильных конфигураций конечных точек.

Дополнительные сведения о задачах безопасности в Intune см. в записи блога [Использование задач по обеспечению безопасности Intune для расширения управления угрозами и уязвимостями ATP в Защитнике Windows](https://techcommunity.microsoft.com/t5/Enterprise-Mobility-Security/Microsoft-Intune-security-tasks-extend-Microsoft-Defender-ATP-s/ba-p/369857). 

### <a name="create-and-use-oemconfig-device-configuration-profiles-in-intune----3305883---"></a>Создание и использование профилей конфигурации устройств OEMConfig в Intune <!-- 3305883 -->
Intune будет поддерживать настройку устройств Android Enterprise с помощью OEMConfig. В частности, вы можете создать профиль конфигурации устройств и применить параметры к устройствам Android Enterprise с помощью OEMConfig (**Конфигурация устройства** > **Профили** > **Создать профиль** > **Android Enterprise** для платформы).

Поддержка поставщиков вычислительной техники сейчас осуществляется в индивидуальном порядке. Если требуемое приложение OEMConfig отсутствует в списке приложений OEMConfig, обратитесь по адресу `IntuneOEMConfig@microsoft.com`.

Область применения: 
- Android для бизнеса

### <a name="new-device-restriction-settings-for-android-enterprise-device-owner----3574254---"></a>Новые параметры ограничений устройств для Android Enterprise (режим владельца устройства) <!-- 3574254 -->
На устройствах Android Enterprise можно создать профиль ограничения устройств, чтобы разрешить или запретить функции, задать правила паролей и многое другое (**Конфигурация устройства** > **Профили** > **Создать профиль** > выберите **Android Enterprise** для платформы > **Только владелец устройства > Ограничения для устройств** для типа профиля). 

Будут доступны новые параметры, в том числе параметры паролей, которые разрешают полный доступ к приложениям в Google Play Маркете для полностью управляемых устройств, и многое другое. 

Текущий список параметров см. в описании [параметров устройств Android для бизнеса, которые разрешают или ограничивают определенные функции](device-restrictions-android-for-work.md). 

Применимо: полностью управляемые устройства Android для бизнеса

### <a name="device-overview-shows-primary-user---794259---"></a>В обзоре устройства отображается основной пользователь <!--794259 -->
На странице обзора устройства будет отображаться основной пользователь, которого также называют пользователем сопоставления пользователей и устройств (UDA). Чтобы просмотреть основного пользователя для устройства, выберите **Intune** > **Устройства** > **Все устройства** и выберите устройство. Основной пользователь отображается в верхней части страницы **Обзор**.

### <a name="expanded-support-for-android-enterprise-fully-managed-devices----3903241-3903244-3903246---"></a>Расширенная поддержка для полностью управляемых устройств Android для бизнеса <!-- 3903241, 3903244, 3903246 -->
Мы собираемся расширить поддержку полностью управляемых устройств Android Enterprise ([первое объявление в январе 2019 г.](whats-new.md#week-of-january-14-2019)), включив в нее следующее:
- Соответствие требованиям
- Conditional access (Условный доступ)
- Приложение "Добавить конечного пользователя"

Чтобы настроить полностью управляемые устройства Android, последовательно выберите **Регистрация устройства** > **Регистрация Android** > **Corporate-owned, fully managed user devices (Полностью управляемые корпоративные пользовательские устройства)**. Поддержка полностью управляемых устройств Android остается на этапе предварительной версии, и некоторые возможности Intune могут быть частично неработоспособными. 

### <a name="additional-managed-google-play-app-reporting-for-android-enterprise-work-profile-devices----4105925---"></a>Дополнительные отчеты об управляемых приложениях Google Play для устройств с рабочим профилем Android Enterprise <!-- 4105925 -->
Для управляемых приложений Google Play, развернутых на устройствах с рабочим профилем Android Enterprise, можно будет просмотреть конкретный номер версии приложения, установленного на устройстве. Это касается только обязательных приложений. Аналогичная функциональность для доступных приложений будет добавлена в будущем выпуске.

### <a name="ios-third-party-keyboards----4111843---"></a>Клавиатуры сторонних производителей для iOS <!-- 4111843 -->
Поддержка политики защиты приложений Intune для параметра **Клавиатуры сторонних производителей** будет завершена из-за изменения платформы iOS. Вы не сможете настроить этот параметр в консоли администрирования Intune, и он не будет применяться на клиенте в пакете SDK для приложений Intune.

### <a name="block-users-from-scanning-for-windows-updates----3316758---"></a>Запретить пользователям проверять наличие обновлений Windows <!-- 3316758 -->
Мы добавили новый параметр кольца обновления Windows, который можно использовать для блокирования проверки на наличие обновлений Windows у пользователей. Этот параметр не будет доступен на портале, но его можно настроить с помощью API Intune Graph.

### <a name="windows-update-notifications----3316782---"></a>Уведомления Центра обновления Windows <!-- 3316782 -->
Мы добавили поддержку для конфигураций кольца обновления Windows, чтобы вы могли настроить уведомления об обновлениях Windows, которые видят пользователи. Этот параметр не будет доступен на портале, но его можно настроить с помощью API Intune Graph.

### <a name="easier-access-to-diagnostic-settings----3804627---"></a>Упрощенный доступ к параметрам диагностики <!-- 3804627 -->
Мы добавили новый параметр колонки **Журналы аудита** в каждой рабочей нагрузке журнала аудита в консоли Intune, который позволяет напрямую открыть страницу *Параметры диагностики*.
