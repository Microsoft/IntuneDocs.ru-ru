# <a name="in-development-for-microsoft-intune---march-2019"></a>В разработке для Microsoft Intune — март 2019 г.

Для помощи в вашу готовность и планирования, эта страница списка функций и обновляет пользовательский Интерфейс Intune находятся на стадии разработки, но не был освобожден. Кроме того:

- Если мы полагаем, что вам потребуется выполнить действие до изменения, мы опубликуем бесплатных post центре сообщений Office.
- Когда функция запускается в рабочей среде, либо в качестве предварительной версии или общедоступной версии, описание компонента переместит со страницы, а также на [странице](whats-new.md).
- Эта страница и [странице](whats-new.md) периодически обновляются. Проверьте наличие дополнительных обновлений.
- Ссылаться на [стратегия развития Microsoft 365](https://www.microsoft.com/microsoft-365/roadmap?rtc=2&filters=EMS) для стратегического конечные результаты и шкал времени.

> [!Note]
> Эти элементы отражают текущее ожиданиям корпорации Майкрософт о возможностях Intune, ожидается в будущем выпуске. Даты и отдельные компоненты могут изменяться. Не все элементы в разработке на этой странице есть описание компонента.

### <a name="encryption-report-----2351538---"></a>Шифрование отчета  <!-- 2351538 -->
Вы сможете воспользоваться новым отчетом шифрования для получения сведений о состоянии шифрования устройств. Доступные сведения будет включать версию устройства TPM, готовности шифрования и состояние, отчеты об ошибках и многое другое.  

### <a name="access-bitlocker-recovery-keys-from-the-intune-portal-----2351547----"></a>Доступ к ключам восстановления BitLocker на портале Intune  <!-- 2351547  -->
Мы добавили новую точку входа на устройствах, где можно просмотреть сведения из Azure Active Directory (AAD) о ИД ключа BitLocker и ключам восстановления BitLocker.

### <a name="scope-tags-for-app-configuration-policies---2371891---"></a>Теги области для политики конфигурации приложений <!--2371891 -->
Вы сможете добавить тег области политику конфигурации приложений, чтобы только пользователи с ролями, также назначен этот тег области имеют доступ к политику конфигурации приложения. Политики конфигурации приложения можно только назначенные или связанных с приложениями, назначить один и тот же тег области.

### <a name="assign-autopilot-profiles-to-the-all-devices-virtual-group---2715522---"></a>Назначение профилей Autopilot виртуальной группе "Все устройства" <!--2715522 -->
Вы сможете назначать профили Autopilot виртуальной группе "Все устройства". Для этого последовательно выберите **Регистрация устройств** > **Регистрация Windows** > **Профили развертывания** > выберите профиль > **Назначения**, в разделе **Назначить** выберите **Все устройства**. Дополнительные сведения о профилях Autopilot см. в статье [Регистрация устройств Windows с помощью Windows Autopilot](enrollment-autopilot.md).

### <a name="install-available-apps-using-the-company-portal-app-after-windows-bulk-enrollment----2751523----"></a>Установка доступных приложений, с помощью приложения корпоративного портала после Windows Массовая регистрация <!-- 2751523  -->
Устройства Windows, которые зарегистрированы в Intune с помощью [массовой регистрации Windows](windows-bulk-enroll.md) (пакетов подготовки) будут иметь возможность использовать приложение корпоративного портала для установки доступных приложений. Дополнительные сведения о приложении корпоративного портала, см. в разделе [вручную добавить корпоративного портала Windows 10](store-apps-company-portal-app.md) и [как настроить приложение корпоративного портала Microsoft Intune](company-portal-app.md).

### <a name="scope-tags-for-ios-app-provisioning-profiles---2934430---"></a>Теги области для профили подготовки приложений iOS <!--2934430 -->
Вы сможете добавить тег области профиль подготовки приложений iOS, чтобы только пользователи с ролями, также назначен этот тег области имеют доступ к профиль подготовки приложений iOS. 

### <a name="office-deployment-tool-odt-xml-for-office-proplus-deployment----3192477----"></a>Средство развертывания Office (ODT) XML для развертывания Office профессиональный плюс <!-- 3192477  -->
Вы сможете предоставить средства развертывания Office (ODT) XML при создании экземпляра Office профессиональный плюс в консоли администрирования Intune. Таким образом, благодаря больше, если существующие параметры пользовательского интерфейса Intune не соответствуют вашим потребностям. 

###  <a name="block-users-from-scanning-for-windows-updates-------3316758------"></a>Запретить пользователям проверяет наличие обновлений Windows    <!-- 3316758    -->
Мы добавили новый Windows update кольца параметр, который можно использовать, будет блокировать пользователей из проверки на наличие обновлений Windows. Этот параметр не будет доступен из портала, но можно настроить с помощью Intune Graph API.

### <a name="windows-update-notifications-----3316782---"></a>Уведомления об обновлениях Windows  <!-- 3316782 -->
Поэтому вы сможете настроить уведомления об обновлениях Windows, которые пользователи увидят мы добавили поддержку к конфигурациям кольца обновления Windows. Этот параметр не будет доступен из портала, но можно настроить с помощью Intune Graph API.

### <a name="changes-to-company-portal-enrollment-for-ios-12-device-users---3448635---"></a>Изменения в регистрации корпоративного портала для пользователей устройств iOS 12 <!--3448635 -->  
Корпоративного портала для iOS будут обновлять экранов регистрации приложения и действия в соответствии с изменениями регистрации MDM, выпущенная в Apple iOS 12.2. Обновленный рабочий процесс теперь будет предлагать пользователю:

- Разрешить Safari откройте веб-сайт корпоративного портала (с помощью Safari) и загрузить профиль управления перед возвратом в приложение корпоративного портала. 
- Откройте приложение параметры для установки профиля управления на своем устройстве.
- Вернуться в приложение корпоративного портала, чтобы завершить регистрацию.  

Дополнительные сведения о как подготовить эти изменения, см. в разделе [post техническое сообщество Майкрософт](https://techcommunity.microsoft.com/). В то же время, чтобы поддерживать новые регистрации iOS в приложении корпоративного портала, мы обновили действия, описанные в [Регистрация устройства iOS в Intune](https://docs.microsoft.com/en-us/intune/ios-enroll). Эти изменения документы будут динамическими после Apple iOS версии 12.2. 

### <a name="support-for-additional-connectors-on-the-tenant-status-page----3617202-------"></a>Поддержка дополнительных соединителей на странице состояния клиента <!-- 3617202     -->
На странице состояния клиента будут отображаться сведения о состоянии для дополнительных соединителей, включая *Windows Defender Advanced Threat Protection* (ATP) и другие соединители Mobile Threat Defense.

### <a name="granting-intune-read-only-access-to-some-azure-active-directory-roles----3637917---"></a>Предоставление Intune доступ только для некоторых ролей Azure Active Directory для чтения <!-- 3637917 -->
Мы будем предоставлять что Intune доступ только для следующих ролей Azure AD для чтения. Разрешения, предоставленные с ролями Azure AD заменяет разрешения, предоставленные с помощью Intune элемента управления доступом на основе ролей (RBAC).

Доступ только к данным аудита Intune для чтения:

- Администратор соответствия требованиям
- Администратор соответствия требованиям данных

Доступом только для чтения ко всем данным Intune:

- Администратор безопасности
- Оператор безопасности
- Читатель сведений о безопасности
- Глобальный модуль чтения

### <a name="easier-access-to-diagnostic-settings------3804627-----"></a>Упрощенный доступ к параметрам диагностики   <!-- 3804627   -->
Мы добавили новый параметр, позволяющий **журналы аудита** колонке в каждый в каждая рабочая нагрузка журнал аудита в консоли Intune, можно использовать для напрямую открыть *параметров диагностики* страницы.

### <a name="create-and-use-device-configuration-profiles-on-android-zebra-devices-in-intune----3895244----"></a>Создание и использование профилей конфигурации устройств на устройствах Android Zebra в Intune <!-- 3895244  -->
Intune будет поддерживать настройке устройств Android Zebra. В частности вы сможете:

Применяется к:

### <a name="deployment-of-online-licensed-microsoft-store-for-business-apps----1672660----"></a>Развертывание лицензированных в подключенном режиме приложений Microsoft Store для бизнеса <!-- 1672660  -->
Вы сможете назначать устройствам лицензированные в подключенном режиме приложения Microsoft Store для бизнеса. Такое развертывание лицензированных в подключенном режиме приложений Microsoft Store для бизнеса делает их доступными для всех пользователей устройства. Эта возможность поддерживается только на настольных компьютерах с Windows 10, начиная со сборки RS4. Возможность установки на устройство доступна на странице назначения клиентских приложений для лицензированных в подключенном режиме приложений Microsoft Store для бизнеса.
