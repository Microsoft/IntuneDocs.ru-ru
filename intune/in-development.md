### <a name="include-and-exclude-mixture-of-user-groups-and-device-groups-when-assigning-policies-and-profiles----1807547---"></a>Включать и исключать группы пользователей и группы устройств, и, при назначении политик и профилей <!-- 1807547 -->

В настоящее время можно включать и исключать только группы пользователей, *или* включать и исключать только группы устройств. Не удается включить и исключить смешивая эти группы, такие как включать группы пользователей *и* исключить группу устройств.

Вы сможете включать и исключать группы пользователей и групп устройств. Можно включить группу пользователей и исключать группы устройств. Например можно назначить или развертывания профиля конфигурации устройства для группы пользователей, но исключить личных устройств.

[Назначение профилей конфигурации устройств](device-profile-assign.md) включает дополнительные сведения о назначении профилей для групп пользователей и групп устройств.

### <a name="expanded-support-for-android-enterprise-fully-managed-devices----3903241-3903244-3903246---"></a>Расширенная поддержка для Android для бизнеса, полностью управляемые устройства <!-- 3903241, 3903244, 3903246 -->
Мы собираемся расширить поддержку устройств Android для бизнеса управляемые ([представлена в января 2019](whats-new.md#week-of-january-14-2019) путем включения следующих:
- Соответствие требованиям
- Conditional access (Условный доступ)
- Новый пользователь приложения

Чтобы настроить полностью управляемые устройства Android, последовательно выберите **Регистрация устройства** > **Регистрация Android** > **Corporate-owned, fully managed user devices (Полностью управляемые корпоративные пользовательские устройства)**. Поддержка остается полностью управляемых устройств Android в предварительной версии и некоторые возможности Intune не может быть полностью работоспособными. 

### <a name="additional-managed-google-play-app-reporting-for-android-enterprise-work-profile-devices----4105925---"></a>Дополнительные управляемый Google Play в отчетах приложения для устройства с рабочим профилем Android для бизнеса <!-- 4105925 -->
Управляемый Google Play приложения, развернутые на устройства с рабочим профилем Android для бизнеса она позволяет просматривать определенный номер версии приложения, установленные на устройстве. Это касается только необходимые приложения. Такую же функцию для приложения, доступные для будет добавлена в будущем выпуске.

### <a name="ios-third-party-keyboards----4111843---"></a>Клавиатуры сторонних производителей для iOS <!-- 4111843 -->

### <a name="changes-to-company-portal-enrollment-for-ios-12-device-users---3448635---"></a>Изменения в регистрации корпоративного портала для пользователей устройств iOS 12 <!--3448635 --> 
Корпоративного портала для iOS будут обновлять экранов регистрации приложения и действия в соответствии с изменениями регистрации MDM, выпущенная в Apple iOS 12.2. Обновленный рабочий процесс теперь будет предлагать пользователю:

- Разрешить Safari откройте веб-сайт корпоративного портала (с помощью Safari) и загрузить профиль управления перед возвратом в приложение корпоративного портала.
- Откройте приложение параметры для установки профиля управления на своем устройстве.
- Вернуться в приложение корпоративного портала, чтобы завершить регистрацию.

Дополнительные сведения о как подготовить эти изменения, см. в разделе [post техническое сообщество Майкрософт](https://aka.ms/CP_changes_iOS12). В то же время, чтобы поддерживать новые регистрации iOS в приложении корпоративного портала, мы обновили действия, описанные в [Регистрация устройства iOS в Intune](https://docs.microsoft.com/en-us/intune/ios-enroll). Эти изменения документа будут динамическими после Apple iOS версии 12.2.
