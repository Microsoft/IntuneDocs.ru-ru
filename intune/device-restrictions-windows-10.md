# <a name="windows-10-and-newer-device-restriction-settings-in-microsoft-intune"></a>Параметры ограничений для устройств с ОС Windows 10 и более поздних версий в Microsoft Intune

В этой статье перечислены и описаны все параметры настраиваемых ограничений для устройств Windows 10. Эти параметры добавляются в профиль конфигурации устройства, а затем назначаются или развертываются на устройствах Windows 10 с помощью Microsoft Intune.

> [!Note]
> Доступность тех или иных параметров зависит от выпуска Windows.

## <a name="general"></a>Общие

- **Захват экрана (только для мобильных устройств).** Разрешает пользователю сохранять содержимое экрана устройства в виде изображения.
- **Копирование и вставка (только для мобильных устройств).** Разрешает действия копирования и вставки между приложениями на устройстве.
- **Ручная отмена регистрации.** Разрешает пользователю вручную удалить рабочую учетную запись с устройства.
  - Этот параметр политики не применяется, если компьютер присоединен к Azure Active Directory и включена автоматическая регистрация. 
  - Этот параметр политики не применяется к компьютерам с ОС Windows 10 Домашняя.
- **Ручная установка корневого сертификата (только для мобильных устройств).** Запрещает ручную установку корневых сертификатов и промежуточных CAP-сертификатов.

- **Камера.** Разрешает или запрещает использование камеры на устройстве.
- **Синхронизация файлов OneDrive.** Блокирует синхронизацию файлов устройства в OneDrive.
- **Съемные носители**. Указывает, можно ли использовать на устройстве внешние запоминающие устройства, например SD-карты.
- **Геолокация.** Указывает, может ли устройство использовать сведения о службах определения расположения.
- **Общий Интернет.** Разрешает использовать на устройстве общий доступ к подключению к Интернету.
- **Сброс телефона**. Управляет возможностью очистки устройства пользователем.
- **USB-подключение (только для мобильных устройств).** Указывает, разрешен ли доступ устройства к внешним запоминающим устройствам через USB-подключение.
- **Режим AntiTheft (только для мобильных устройств).** Указывает, включен ли режим защиты от кражи Windows.
- **Кортана.** Включает или отключает голосовой помощник — Кортану.
- **Запись речи (только для мобильных устройств).** Разрешает или запрещает использование записи речи на устройстве.
- **Изменение имени устройства.** Запрещает пользователю изменять имя устройства (только для Windows 10 Mobile).
- **Добавление пакетов подготовки**. Блокирует агент конфигурации среды выполнения, который устанавливает пакеты подготовки.
- **Удаление пакетов подготовки.** Блокирует агент конфигурации среды выполнения, который удаляет пакеты подготовки.
- **Обнаружение устройств.** Блокирует обнаружение устройства другими устройствами.
- **Диспетчер задач (только для мобильных устройств).** Блокирует диспетчер задач на устройстве.
- **Диалоговое окно ошибки SIM-карты (только для мобильных устройств).** Блокирует отображение сообщения об ошибке на устройстве, если SIM-карта не обнаружена.
- **Ink Workspace.** Запрет доступа к Ink Workspace. **Не настроено** включает компонент Ink Workspace, и пользователи могут использовать его поверх экрана блокировки.
- **Автоматическое повторное развертывание.** Разрешает пользователям с правами администратора удалять все пользовательские данные и параметры с помощью клавиш **CTRL+WIN+R** на экране блокировки устройства. Устройство будет автоматически перенастроено и повторно зарегистрировано в системе управления.
- **Требовать от пользователей подключаться к сети при настройке устройства (только для предварительной оценки Windows)** — выберите **Требовать**, чтобы устройство нужно было подключить к сети, прежде чем продолжить процедуру после отображения страницы "Сеть" во время установки Windows 10. Пока эта функция находится на этапе предварительной версии, для использования этого параметра необходима версия Windows Insider для предварительной оценки со сборкой 1809 или более поздней.

## <a name="password"></a>Пароль

- **Пароль.** Пользователь должен ввести пароль для доступа к устройству.
  - **Требуемый тип пароля.** Указывает, должен ли пароль содержать только цифры или буквы и цифры.
  - **Минимальная длина пароля.** Минимальная длина пароля (только для Windows 10 Mobile)
  - **Число неудачных попыток входа перед очисткой устройства.** Если для устройства включен BitLocker, после указанного числа неудачных попыток входа оно переключается в режим восстановления BitLocker (для устройств под управлением Windows 10). Если режим BitLocker для устройства не включен, этот параметр не применяется. После указанного числа неудачных попыток входа устройство будет очищено (для устройств под управлением Windows 10 Mobile).
  - **Максимальное время бездействия (в минутах), по истечении которого экран блокируется.** Указывает период бездействия устройства, по истечении которого экран блокируется.
  - **Срок действия пароля (в днях).** Указывает время, по истечении которого требуется сменить пароль для устройства.
  - **Запретить повторное использование предыдущих паролей.** Указывает число предыдущих паролей, сохраняемых на устройстве.
  - **Требовать пароль при выходе устройства из состояния простоя (только для Mobile).** Указывает, что пользователь должен ввести пароль для разблокирования устройства (только для Windows 10 Mobile).
  - **Простые пароли**. Разрешает использовать простые пароли, например 1111 или 1234. Кроме того, этот параметр разрешает или блокирует использование графических паролей Windows.
- **Шифрование**. Включает шифрование на целевых устройствах.

## <a name="personalization"></a>Personalization

- **URL-адрес фонового изображения рабочего стола (только для настольных компьютеров)**. Введите URL-адрес изображения в формате JPEG, используемого в качестве фонового рисунка рабочего стола Windows. Пользователи не могут изменить этот рисунок.

## <a name="privacy"></a>Конфиденциальность

- **Персонализация ввода**. Разрешает отключить использование облачных служб речи для Кортаны, диктофона или приложений Магазина Майкрософт. Если вы разрешите эти службы, корпорация Майкрософт может собирать голосовые данные для улучшения обслуживания.
- **Автоматическое согласие с запросами к пользователю о связывании и конфиденциальности**. Разрешает Windows автоматически принимать сообщения о связывании и согласии на обеспечение конфиденциальности при запуске приложений.
- **Опубликовать действия пользователей**. Выберите значение **Блокировать**, чтобы запретить публикацию и обнаружение недавно использовавшихся ресурсов в диспетчере задач.
- **Только локальные действия**. Выберите значение **Блокировать**, чтобы запретить публикацию и обнаружение недавно использовавшихся ресурсов в диспетчере задач только для локальных действий.

Вы можете настраивать сведения, к которым получат доступ все приложения на устройстве. Можно задать исключения для каждого приложения с помощью **исключений конфиденциальности на уровне приложения**.

### <a name="exceptions"></a>Исключения

- **Сведения об учетной записи**. Определите, может ли приложение получать доступ к имени пользователя, фотографии и другим контактным данным.
- **Фоновые приложения**. Определите, можно ли запускать это приложение в фоновом режиме.
- **Календарь**. Определите, может ли это приложение получать доступ к календарю.
- **Журнал вызовов**. Определите, может ли это приложение получать доступ к журналу вызовов.
- **Камера**. Определите, может ли это приложение получать доступ к камере.
- **Контакты**. Определите, может ли это приложение получать доступ к контактам.
- **Электронная почта**. Определите, может ли это приложение получать доступ к электронной почте.
- **Расположение**. Определите, может ли это приложение получать доступ к информации о расположении.
- **Обмен сообщениями**. Определите, может ли это приложение читать или отправлять текстовые и MMS-сообщения.
- **Микрофон**. Определите, может ли это приложение использовать микрофон.
- **Перемещение**. Определите, может ли это приложение получать доступ к информации о перемещении устройства.
- **Уведомления**. Определите, может ли это приложение получать доступ к уведомлениям.
- **Телефон**. Определите, может ли это приложение получать доступ к телефону.
- **Радио**. Некоторые приложения используют радиосвязь (например, Bluetooth) на устройстве для отправки и получения данных, и им требуется включать или отключать эту радиосвязь. Определите, может ли это приложение управлять радиосвязью.
- **Задачи**. Определите, может ли это приложение получать доступ к задачам.
- **Доверенные устройства**. Выберите, может ли это приложение использовать доверенные устройства, то есть оборудование, которое использовалось ранее или поставлялось вместе с этим устройством. В качестве доверенных устройств могут выступать телевизоры, проекторы и т. д.
- **Отзывы и диагностика**. Выберите, может ли это приложение получать доступ к данным диагностики.
- **Синхронизация с устройствами** — определите, может ли это приложение автоматически предоставлять доступ к информации беспроводным устройствам, которые явно не связаны с этим компьютером, планшетом или телефоном, и синхронизироваться с этим устройствами.

## <a name="locked-screen-experience"></a>Интерфейс заблокированного экрана

- **Уведомления центра уведомлений (только для мобильных устройств)**. Разрешает отображать уведомления из центра уведомлений на экране блокировки устройства (только Windows 10 Mobile).
- **URL-адрес изображения на экране блокировки (только для настольных компьютеров).** Введите URL-адрес изображения в формате JPEG, которое используется в качестве фона на экране блокировки Windows. Пользователи не могут изменить этот параметр.
- **Настраиваемое пользователем время ожидания экрана (только для мобильных устройств)**. Разрешает настроить период времени. 
- **Кортана на заблокированном экране (только для компьютеров)**. Запрещает пользователю взаимодействовать с Кортаной, когда устройство находится в режиме блокировки (только Windows 10 Desktop).
- **Всплывающие уведомления на заблокированном экране**. Запрещает отображение оповещений на экране блокировки устройства.
- **Время ожидания экрана (только для мобильных устройств)**. Указывает период в секундах после блокировки экрана, по истечении которого экран отключается.

## <a name="app-store"></a>Магазин App Store

- **App Store (только для мобильных устройств).** Разрешает или запрещает использование Магазина приложений на устройствах Windows 10 Mobile.
- **Автообновление приложений из Магазина.** Разрешает автоматически обновлять приложения, установленные из Microsoft Store.
- **Установка доверенного приложения.** Разрешает загружать неопубликованные приложения, подписанные доверенным сертификатом.
- **Разблокировка для разработчиков.** Разрешает параметры разработчика Windows, например изменение пользователем неопубликованных приложений.
- **Совместное использование данных приложения пользователями.** Разрешает пользователям одного приложения совместно использовать одни и те же данные на одном устройстве.
- **Использовать только частный магазин**. Разрешает пользователям скачивать приложения только из частного магазина.
- **Запуск приложений из Store.** Отключает все приложения, ранее установленные на устройстве или скачанные из Microsoft Store.
- **Устанавливать данные приложения на системный том.** Блокирует сохранение данных приложений на системном томе устройства.
- **Устанавливать приложения на системный диск.** Блокирует сохранение данных приложений на системном диске устройства.
- **DVR для игр (только для настольных систем).** Указывает, разрешена ли запись и трансляция игр.
- **Только приложения из магазина.** Определяет, могут ли пользователи устанавливать приложения из других мест, кроме магазина приложений.

## <a name="microsoft-edge-browser"></a>Браузер Microsoft Edge

### <a name="start-experience"></a>Начало работы

- **Отображаемые страницы при запуске Microsoft Edge**. Выберите страницы, которые открываются при запуске Microsoft Edge. Доступны следующие параметры:
  - **Начальные страницы**. Microsoft Edge открывается со стандартной начальной страницей, которая определена в ОС.
  - **Страница новой вкладки**. Microsoft Edge загружает все, что определено в параметре **URL-адрес страницы новой вкладки**.
  - **Страница последнего сеанса**. Microsoft Edge загружает страницу с последним открытым сеансом. 
  - **Пользовательская начальная страница**. Microsoft Edge загружает начальную страницу, определенную ИТ-администратором.
- **Пользователь может изменить начальные страницы**. Значение **Разрешить** разрешает пользователям изменять начальные страницы. Администраторы могут с помощью `EdgeHomepageUrls` указать начальные страницы, которые будут открываться по умолчанию для пользователей при запуске Microsoft Edge. Вариант **Не настроено** запрещает пользователям изменять начальные страницы.
- **URL-адрес страницы новой вкладки**. Введите URL-адрес, который будет открываться на странице новой вкладки. Например, введите `https://www.bing.com`.
- **Открывать веб-содержимое на странице новой вкладки**. Значение **Блокировать** запрещает Microsoft Edge открывать веб-сайт на новой вкладке. Если эта функция заблокирована, открывается пустая новая вкладка. Вариант **Не настроено** сохраняет на устройстве поведение операционной системы по умолчанию, например возможность пользователя выбирать отображаемое содержимое.
- **Кнопка домашней страницы**. Выберите действие, выполняемое при нажатии кнопки домашней страницы. Доступны следующие параметры:
  - **Начальные страницы**. Откроется тот вариант, который вы выбрали для параметра **Отображаемые страницы при запуске Microsoft Edge**.
  - **URL-адрес страницы новой вкладки**. Откроется тот вариант, который вы выбрали для параметра **URL-адрес страницы новой вкладки**.
  - **Настраиваемый URL-адрес кнопки домашней страницы**. Откроется тот вариант, который вы выбрали для параметра **URL-адрес кнопки домашней страницы**.
  - **Скрыть кнопку домашней страницы**. Скрывает кнопку домашней страницы.
- **Пользователь может изменить кнопку домашней страницы**. Значение **Разрешить** разрешает пользователям изменять кнопку домашней страницы. Изменения, внесенные пользователем, переопределяют любые настройки администратора для кнопки домашней страницы. Вариант **Не настроено** использует поведение по умолчанию операционной системы устройства, которое может запрещать пользователям изменять настроенные администратором параметры кнопки домашней страницы.
- **Отобразить страницу первого запуска**. Значение **Блокировать** отключает отображение вводной страницы при первом запуске Microsoft Edge. Эта функция блокирует вводную страницу, например в организациях с конфигурацией нулевых выбросов. Вариант **Не настроено** отображает вводную страницу.
  - **URL-адрес первого запуска**. Введите URL-адрес страницы, которая будет отображаться для пользователей при первом запуске Microsoft Edge (только для Windows 10 Mobile).
- **Всплывающие окна**. Выберите **Блокировать**, чтобы запретить отображение всплывающих окон в браузере. Применяется только к Windows 10 Desktop. Если выбрать вариант **Не настроено**, будет разрешено отображение всплывающих окон в веб-браузере.
- **Отправлять трафик интрасети в Internet Explorer.** Значение **Разрешить** разрешает пользователям открывать веб-сайты интрасети в Internet Explorer вместо Microsoft Edge (только для Windows 10 Desktop). Вариант **Не настроено** разрешает пользователям использовать Microsoft Edge.
- **Список сайтов, открываемых в режиме предприятия**. Введите URL-адрес, по которому размещен список веб-сайтов, которые будут открываться в режиме предприятия. Пользователи не могут изменить этот список. Применяется только к Windows 10 Desktop.
- **Сообщение при открытии сайтов в Internet Explorer**. Используйте этот параметр, чтобы Microsoft Edge отображал уведомления перед открытием сайта в Internet Explorer 11. Доступны следующие параметры:
  - **Не настроено**. Используется поведение операционной системы по умолчанию, которое не всегда включает отображение сообщений.
  - **Отображать сообщение без возможности открывать сайты в Microsoft Edge**. При попытке открыть сайт в Internet Explorer отображается сообщение. Этот сайт будет открыт в Internet Explorer. Возможность открывать сайты в Microsoft Edge не предусмотрена.
  - **Отображать сообщение при открытии сайтов в Microsoft Edge**. При попытке открыть сайт в Internet Explorer отображается сообщение. В этом сообщении есть ссылка **Продолжить работу в Microsoft Edge**, которая разрешает пользователям использовать Microsoft Edge вместо IE.

  > [!IMPORTANT]
  > Этот вариант требует включить параметр **Настроить список сайтов в режиме предприятия** и (или) параметр **Отправить все сайты интрасети в Internet Explorer 11**.

- **Список совместимости Майкрософт**. Значение **Блокировать** отключает список совместимости Майкрософт в Microsoft Edge. Этот список предоставляется корпорацией Майкрософт для того, чтобы в Edge правильно отображались сайты с известными проблемами совместимости. Вариант **Не настроено** разрешает использовать список совместимости Майкрософт.
- **Предварительная загрузка начальных страниц и страницы новой вкладки**. Выберите **Блокировать**, чтобы в Microsoft Edge не выполнялась предварительная загрузка начальных страниц и страницы новой вкладки. Предварительная загрузка ускоряет запуск Microsoft Edge и загрузку новой вкладки. Вариант **Не настроено** использует стандартное поведение операционной системы, которое не всегда включает предварительную загрузку этих страниц.
- **Предварительный запуск начальной страницы и страницы новой вкладки**. Выберите **Блокировать**, чтобы в Microsoft Edge не выполнялся предварительный запуск начальных страниц и страницы новой вкладки. Предварительный запуск повышает производительность Microsoft Edge и ускоряет запуск Microsoft Edge. Вариант **Не настроено** использует стандартное поведение операционной системы, которое не всегда включает предварительный запуск этих страниц.

### <a name="favorites-and-search"></a>Избранное и поиск

- **Панель избранного**. Выберите поведение для панели избранного на каждой странице Microsoft Edge. Доступны следующие параметры:
  - Вариант **Не настроено** использует стандартное поведение операционной системы, которое не всегда включает предварительный запуск этих страниц. Пользователь может изменить этот параметр.
  - **Скрыть**. Скрывает панель избранного на всех страницах. Пользователь не может изменить этот параметр.
  - **Отобразить**. Отображает панель избранного на всех страницах. Пользователь не может изменить этот параметр.
- **Список избранного**. Добавьте список URL-адресов в файл избранного. Например, добавьте адрес `http://contoso.com/favorites.html`.
- **Ограничить внесение изменений в избранное**. Значение **Блокировать** запрещает пользователям добавлять, импортировать, сортировать или изменять элементы списка избранного. Вариант **Не настроено** использует стандартное поведение операционной системы, которое может разрешать пользователям изменять этот список.
- **Синхронизация избранного между браузерами Майкрософт (только компьютеры)**. Значение **Требовать** принуждает Windows синхронизировать избранное между Internet Explorer и Microsoft Edge. Все операции добавления, удаления, изменения и упорядочивания в списке избранного будут синхронизироваться между браузерами.  Вариант **Не настроено** использует стандартное поведение операционной системы, разрешая пользователям самостоятельно выбрать режим синхронизации между браузерами.
- **Поисковая система по умолчанию.** Выберите для устройства поисковую систему по умолчанию. Это значение можно изменить в любое время. Доступны следующие параметры:
  - Диапазон
  - Bing
  - Google
  - Yahoo
  - Пользовательское значение
- **Варианты поиска.** Значение **Не настроено** разрешает поисковой системе предлагать сайты по мере ввода фраз в строке поиска. Если выбрать **Блокировать**, использовать эту функцию будет запрещено.

### <a name="privacy-and-security"></a>Конфиденциальность и безопасность

- **Просмотр InPrivate.** Значение **Блокировать** запрещает пользователю открывать сеансы просмотра InPrivate. Если выбрать вариант **Не настроено**, можно будет использовать эту функцию.
- **Сохранить журнал браузера**. Значение **Блокировать** запрещает Microsoft Edge сохранять журнал просмотра. Вариант **Не настроено** разрешает сохранять журнал браузера.
- **Очистить данные просмотра веб-страниц при выходе (только для компьютеров)**. Значение **Требовать** очищает журнал и данные браузера при выходе пользователя из Microsoft Edge. Вариант **Не настроено** использует стандартное поведение операционной системы, которое может включать кэширование данных о просмотренных страницах.
- **Синхронизировать параметры браузера между устройствами пользователя**. Выберите способ синхронизации параметров браузера между устройствами. Доступны следующие параметры:
  - **Разрешить**. Включает синхронизацию параметров браузера Microsoft Edge между устройствами пользователя
  - **Блокировать и разрешить переопределение пользователем**. Блокирует синхронизацию параметров браузера Microsoft Edge между устройствами пользователя. Пользователи могут переопределить этот выбор.
  - **Блокировать**. Запрещает синхронизацию параметров браузера Microsoft Edge между устройствами пользователя Пользователи не могут переопределить этот выбор.
- **Диспетчер паролей**. Значение **Блокировать** отключает функцию диспетчера паролей в Microsoft Edge. Если выбрать вариант **Не настроено**, можно будет использовать эту функцию.
- **Файлы cookie**. Выберите режим применения файлов cookie в веб-браузере. Доступны следующие параметры:
  - **Разрешить**. Файлы cookie сохраняются на устройстве.
  - **Блокировать все файлы cookie**. Файлы cookie не сохраняются на устройстве.
  - **Блокировать только сторонние файлы cookie**. Файлы cookie сторонних производителей и партнеров не сохраняются на устройстве.
- **Автозаполнение**. Значение **Блокировать** отключает на устройстве функцию автозаполнения. Вариант **Не настроено** разрешает пользователю изменять параметры автозаполнения в браузере (только для Windows 10 Desktop).
- **Отправить заголовки do-not-track**. Значение **Не настроено** требует, чтобы устройства отправляли заголовки do-not-track на веб-сайты, запрашивающие сведения для отслеживания (рекомендуется). Выберите **Блокировать**, чтобы запретить устройству отправку таких заголовков, которые разрешают веб-сайтам отслеживать пользователя.
- **IP-адрес LocalHost для протокола WebRTC**. Значение **Блокировать** запрещает отображать IP-адрес локального узла пользователя при телефонных звонках по протоколу WebRTC. Вариант **Не настроено** разрешает отображать IP-адрес локального узла пользователя при телефонных звонках по этому протоколу.
- **Сбор данных динамических плиток**. Значение **Блокировать** отключает в Windows сбор информации из динамических плиток, на которых пользователь закрепляет сайт в меню "Пуск" из Microsoft Edge. Вариант **Не настроено** разрешает сбор такой информации.
- **Пользователь может игнорировать ошибки сертификата**. Значение **Блокировать** запрещает пользователям доступ к веб-сайтам, которые содержат ошибки SSL или TLS. Вариант **Не настроено** разрешает пользователям использовать такие сайты.

### <a name="additional"></a>Дополнительно

- **Браузер Microsoft Edge (только для мобильных устройств)**. Выберите **Блокировать**, чтобы запретить использование Microsoft Edge на устройстве. Если вы заблокируете Microsoft Edge, отдельные параметры будут применяться только к рабочему столу. Вариант **Не настроено** разрешает использовать на устройстве веб-браузер Microsoft Edge.
- **Раскрывающийся список адресной строки (только компьютеры)**. Значение **Блокировать** запрещает браузеру Microsoft Edge отображать при вводе предложения в виде раскрывающегося списка. Этот параметр помогает снизить нагрузку на сеть между Microsoft Edge и службами Майкрософт. Вариант **Не настроено** разрешает Microsoft Edge отображать список предложений.
- **Во весь экран**. Выберите **Блокировать**, чтобы запретить в Microsoft Edge полноэкранный режим отображения (только веб-содержимое без интерфейса Microsoft Edge). Вариант **Не настроено** использует стандартное поведение операционной системы, что разрешает использовать полноэкранный режим в Microsoft Edge.
- **О флагах**. Значение **Блокировать** запрещает пользователям доступ к странице `about:flags` в Microsoft Edge, которая содержит экспериментальные параметры и параметры разработчика. Вариант **Не настроено** использует стандартное поведение операционной системы, которое может разрешать доступ к странице `about:flags`.
- **Средства для разработчиков**. Значение **Блокировать** запрещает пользователям открывать средства для разработчиков Microsoft Edge. Вариант **Не настроено** разрешает пользователям открывать средства для разработчиков.
- **Расширения**. Значение **Не настроено** разрешает пользователям устанавливать на устройстве расширения Microsoft Edge. Вариант **Блокировать** запрещает такую установку.
- **Расширения разработчика для корпоративных пользователей**. Значение **Блокировать** запрещает в Microsoft Edge загрузку неопубликованных приложений, то есть установку и запуск непроверенных расширений с помощью функции **Загрузить расширения** в Microsoft Edge. Вариант **Не настроено** использует стандартное поведение операционной системы, которое может разрешать загрузку неопубликованных приложений.
- **Обязательные расширения**. Выберите, какие расширения пользователи не смогут отключить в Microsoft Edge. Введите имя семейства пакетов и выберите **Добавить**. Списки [Поиск имени семейств пакетов](https://docs.microsoft.com/sccm/protect/deploy-use/find-a-pfn-for-per-app-vpn) содержат некоторые рекомендации.

  Вы также можете **импортировать** CSV-файл с именами семейств пакетов.

- **JavaScript**. Выберите **Блокировать**, чтобы запретить выполнять скрипты JavaScript в браузере на устройстве. **JavaScript**. Разрешает выполнять скрипты, например JavaScript, в браузере Microsoft Edge.

## <a name="cellular-and-connectivity"></a>Сотовая сеть и подключение

- **Канал сотовых данных**. Запретите пользователям использовать данные, например просматривать страницы в Интернете, когда они подключены к сети мобильной связи. 
- **Передача данных в роуминге.** Разрешите роуминг между сетями при доступе к данным.
- **VPN по сотовой сети.** Определяет, может ли устройство получать доступ к VPN-подключениям при подключении к сети мобильной связи.
- **Передача данных VPN в роуминге по сотовой сети.** Определяет, может ли устройство получить доступ к VPN-подключениям при роуминге в сети мобильной связи.
- **Bluetooth.** Управляет возможностью включать и настраивать Bluetooth на устройстве.
- **Возможность обнаружения Bluetooth.** Разрешает обнаружение устройства другими устройствами, поддерживающими Bluetooth.
- **Предварительное сопряжение Bluetooth**. Разрешает настраивать отдельные устройства Bluetooth для автоматического связывания с главным устройством.
- **Реклама через Bluetooth.** Разрешает устройству получать объявления через Bluetooth.
- **Служба подключенных устройств**. Разрешает использовать службу подключенных устройств, с помощью которой можно обнаруживать другие устройства Bluetooth и подключаться к ним.
- **NFC.** Разрешает пользователю включать и настраивать функции радиочастотной связи ближнего действия.
- **Wi-Fi.** Разрешает пользователю включать и настраивать Wi-Fi на устройстве (только Windows 10 Mobile).
- **Автоматически подключаться к хот-спотам Wi-Fi.** Разрешает устройству автоматически подключаться к точкам доступа Wi-Fi и автоматически принимать любые условия, действующие для такого соединения.
- **Ручная настройка Wi-Fi.** Указывает, может ли пользователь настраивать подключения Wi-Fi либо использовать только подключения на базе профиля Wi-Fi (только Windows 10 Mobile).
- **Интервал сканирования Wi-Fi**. Разрешает задавать частоту сканирования сетей Wi-Fi на устройстве. Укажите значение от 1 (как можно чаще) до 500 (как можно реже).
- **Разрешенные службы Bluetooth**. Разрешает указывать список разрешенных профилей и служб Bluetooth в виде шестнадцатеричных строк.

- **Макет меню "Пуск"**. Для настройки меню "Пуск" на настольных компьютерах можно отправить XML-файл, который включает настройки, порядок отображения приложений в списке и многое другое. Пользователи не смогут изменить настроенный макет меню "Пуск".
- **Закрепить веб-сайты на плитках в меню "Пуск"**. Импортируйте изображения из Microsoft Edge, которые будут отображаться как ссылки в меню "Пуск" в Windows для настольных компьютеров.
- **Открепить приложения от панели задач**. Выберите **Блокировать**, чтобы запретить пользователям откреплять приложения из меню "Пуск".
- **Быстрое переключение пользователей**. Выберите **Блокировать**, чтобы запретить переключение между пользователями, вошедшими в систему, без выхода из системы.
- **Часто используемые приложения**. Выберите **Блокировать**, чтобы скрыть часто используемые приложения в меню "Пуск". При этом будет отключен соответствующий переключатель в приложении "Параметры".
- **Недавно добавленные приложения**. Выберите **Блокировать**, чтобы скрыть отображение недавно добавленных приложений в меню "Пуск". При этом будет отключен соответствующий переключатель в приложении "Параметры".
- **Режим запуска экрана**. Выберите способ отображения начального экрана. Выберите отображение **во весь экран** или **оконный режим**.
- **Последние открытые элементы в списках переходов**. Выберите **Блокировать**, чтобы скрыть последние списки переходов на панели задач и в меню "Пуск". При этом будет отключен соответствующий переключатель в приложении "Параметры".
- **Список приложений**. Выберите способ отображения приложения "Параметры". Доступны следующие параметры: 
  - Свернуть
  - Свернуть и отключить приложение "Параметры" 
  - Удалить и отключить приложение "Параметры"
- **Кнопка питания**. Выберите **Блокировать**, чтобы скрыть кнопку питания в меню "Пуск".
- **Плитка пользователя**. Выберите **Блокировать**, чтобы скрыть плитку пользователя в меню "Пуск".
  - **Блокировка**. Выберите **Блокировать**, чтобы скрыть вариант `Lock` на плитке пользователя в меню "Пуск".
  - **Выйти**. Выберите **Блокировать**, чтобы скрыть вариант `Sign out` на плитке пользователя в меню "Пуск".
- **Завершение работы**. Выберите **Блокировать**, чтобы скрыть варианты `Update and shut down` и `Shut down` на кнопке питания в меню "Пуск".
- **Сон**. Выберите **Блокировать**, чтобы скрыть вариант `Sleep` на кнопке питания в меню "Пуск".
- **Гибернация**. Выберите **Блокировать**, чтобы скрыть вариант `Hibernate` на кнопке питания в меню "Пуск".
- **Переключить учетную запись**. Выберите **Блокировать**, чтобы скрыть вариант `Switch account` на плитке пользователя в меню "Пуск".
- **Параметры перезагрузки**. Выберите **Блокировать**, чтобы скрыть варианты `Update and restart` и `Restart` на кнопке питания в меню "Пуск".
- **Папка "Документы" в меню "Пуск".** Скрывает или показывает папку "Документы" в меню "Пуск" Windows.
- **Папка "Загрузки" в меню "Пуск".** Скрывает или показывает папку "Загрузки" в меню "Пуск"Windows.
- **Папка "Проводник" в меню "Пуск".** Скрывает или показывает папку "Проводник" в меню "Пуск" Windows.
- **Папка "Домашняя группа" в меню "Пуск"**. Скрывает или показывает папку "Домашняя группа" в меню "Пуск" Windows.
- **Папка "Музыка" в меню "Пуск".** Скрывает или показывает папку "Музыка" в меню "Пуск" Windows.
- **Папка "Сеть" в меню "Пуск".** Скрывает или показывает папку "Сеть" в меню "Пуск" Windows.
- **Папка "Личные" в меню "Пуск".** Скрывает или показывает папку "Личные" в меню "Пуск" Windows.
- **Папка "Изображения" в меню "Пуск".** Скрывает или показывает папку "Изображения" в меню "Пуск" Windows.
- **Папка "Параметры" в меню "Пуск".** Скрывает или показывает папку "Параметры" в меню "Пуск" Windows.
- **Папка "Видео" в меню "Пуск".** Скрывает или показывает папку "Видео" в меню "Пуск" Windows.

## <a name="kiosk-preview---obsolete"></a>Киоск (предварительная версия) — устарело

Эти параметры доступны только для чтения и не могут быть изменены. Сведения о включении полноэкранного режима см. в разделе [Параметры киоска для Windows 10 и более поздних версий](kiosk-settings.md).

На устройстве киоска обычно выполняется одно приложение или конкретный набор приложений. Пользователи не могут получить доступ к другим компонентам или функциям на устройстве.

- **Режим киоска** определяет тип режима киоска, поддерживаемый политикой. Возможны следующие значения.

  - **Не настроен** (по умолчанию). Политика не поддерживает режим киоска.
  - **Киоск с одним приложением**. Профиль разрешает использовать на устройстве только одно приложение. Определенное приложение запускается при входе пользователя в систему. Этот режим также не позволяет открывать новые приложения или изменять выполняющиеся.
  - **Киоск с несколькими приложениями**. Профиль разрешает использовать на устройстве несколько приложений. Пользователю доступны только добавляемые вами приложения. Преимущество киоска с несколькими приложениями (или устройства фиксированного предназначения) заключается в предоставлении пользователям понятных возможностей взаимодействия за счет доступа только к необходимым приложениям и скрытия тех приложений, которые им не нужны.

#### <a name="single-app-kiosks"></a>Киоски с одним приложением

Введите следующие параметры:

- **Учетная запись пользователя**. Введите локальную (по отношению к устройству) учетную запись пользователя, учетную запись домена AD или учетную запись Azure AD, связанную с приложением киоска.
  - Локальная учетная запись: введите как `devicename\accountname`, `.\accountname` или `accountname`.
  - Учетная запись домена: введите как `domain\accountname`.
  - Учетная запись Azure AD: введите как `AzureAD\emailaddress`. Обязательно введите "AzureAD", так как это фиксированное доменное имя. Затем укажите адрес электронной почты Azure AD. Например, введите `AzureAD\user@contoso.onmicrosoft.com`.

    Для киосков в общедоступных средах с включенным автоматическим входом в систему следует использовать тип пользователя с минимальными привилегиями (например, локальную учетную запись обычного пользователя). При использовании учетной записи Azure AD для режима киоска обязательно введите `AzureAD\user@yourorganization.com`.

- **Идентификатор модели пользователя приложения (AUMID)**. Введите AUMID для приложения киоска. Дополнительные сведения см. в статье [Поиск идентификатора модели пользователя для установленного приложения](https://docs.microsoft.com/windows-hardware/customize/enterprise/find-the-application-user-model-id-of-an-installed-app).

#### <a name="multi-app-kiosks"></a>Киоски с несколькими приложениями

[Киоски с несколькими приложениями](https://docs.microsoft.com/windows/configuration/lock-down-windows-10-to-specific-apps#configure-a-kiosk-in-microsoft-intune) используют конфигурацию киоска, в которой указаны разрешенные приложения и другие параметры. 

Нажмите кнопку **Добавить**, чтобы создать конфигурацию киоска или выбрать уже существующую. Затем введите следующие параметры.

- **Имя конфигурации киоска**. Введите понятное имя, используемое для идентификации заданной конфигурации.

- **Приложения киоска**. Введите приложения, которые доступны в меню "Пуск". Пользователь может открывать только добавляемые вами приложения.

  - **Тип приложения**. Выберите тип приложения киоска.
    - **Приложение Win32** — обычное классическое приложение. Вам нужен полный путь к исполняемому файлу по отношению к устройству.
    - **Приложение UWP** — универсальное приложение для Windows. Вам требуется [AUMID приложения](https://docs.microsoft.com/windows-hardware/customize/enterprise/find-the-application-user-model-id-of-an-installed-app).

  - **Идентификатор**. Введите полный путь к исполняемому файлу (приложения Win32) или [AUMID приложения](https://docs.microsoft.com/windows-hardware/customize/enterprise/find-the-application-user-model-id-of-an-installed-app) (приложения UWP).

- **Панель задач**. Выберите **Включить** (показывать) панель задач или оставить ее в состоянии **Не настроено** (скрыто) в киоске.

- **Макет меню "Пуск"**. Введите XML-файл, описывающий, как приложения отображаются в меню "Пуск". [Customize and export Start layout](https://docs.microsoft.com/windows/configuration/customize-and-export-start-layout) (Настроить и экспортировать макет меню "Пуск") — предоставляет некоторые рекомендации и образец XML-файла.

  [Создать киоск Windows 10, выполняющий приложения](https://docs.microsoft.com/windows/configuration/lock-down-windows-10-to-specific-apps#create-xml-file) — предоставляет дополнительные сведения об использовании и создании XML-файлов.

- **Назначенные пользователи**. Добавьте одну или несколько учетных записей пользователей, которые могут использовать добавляемые вами приложения. Учетной записи, выполнившей вход в систему, доступны только приложения, определенные в конфигурации. Это может быть локальная по отношению к устройству учетная запись или учетная запись Azure AD, связанная с приложением киоска.

    Для киосков в общедоступных средах с включенным автоматическим входом в систему следует использовать тип пользователя с минимальными привилегиями (например, локальную учетную запись обычного пользователя). Чтобы настроить учетную запись Azure Active Directory (AD) для режима киоска, используйте формат `domain\user@tenant.com`.

## <a name="windows-defender-antivirus"></a>Антивирусная программа "Защитник Windows"

- **Мониторинг в реальном времени.** Разрешает в режиме реального времени проверять наличие вредоносных программ, шпионского ПО и другого нежелательного программного обеспечения.
- **Контроль поведения.** Разрешает Защитнику отслеживать наличие определенных типичных подозрительных действий на устройствах.
- **Система проверки сети (NIS)**. Система NIS помогает защитить устройства от сетевых эксплойтов. Она использует сигнатуры известных уязвимостей из Microsoft Endpoint Protection Center, чтобы обнаружить и заблокировать вредоносный трафик.
- **Сканировать все скачиваемые файлы.** Указывает, проверяет ли Защитник все файлы, скачиваемые из Интернета.
- **Сканировать скрипты, загруженные в веб-браузеры Майкрософт.** Разрешает Защитнику проверять скрипты, используемые в Internet Explorer.
- **Доступ конечных пользователей к Защитнику.** Определяет, скрыт ли пользовательский интерфейс Защитника Windows от конечных пользователей. Изменение этого параметра вступает в силу при следующей перезагрузке компьютера.
- **Интервал обновления сигнатур (в часах)**. Укажите интервал, с которым Защитник проверяет наличие новых файлов сигнатур.
- **Мониторинг действий файлов и программ.** Разрешает Защитнику отслеживать действия, выполняемые с файлами и программами на устройствах.
- **Срок (в днях) перед удалением вредоносных программ в карантине.** Разрешает Защитнику продолжить отслеживание устраненной вредоносной программы определенное число дней, чтобы можно было вручную проверить затронутые ранее устройства. Если задать число дней равным **0**, вредоносная программа остается в папке карантина и не удаляется автоматически.
- **Предел загрузки ЦП при сканировании.** Разрешает ограничить объем ресурсов ЦП, который может использовать проверка (от **1** до **100**).
- **Проверять архивные файлы.** Разрешает Защитнику проверять архивные файлы, например ZIP- или CAB-файлы.
- **Сканировать входящие сообщения электронной почты.** Разрешает Защитнику проверять электронные письма при их поступлении на устройство.
- **Сканировать съемные диски при полной проверке.** Разрешает Защитнику проверять съемные носители, например USB-накопители.
- **Сканировать сопоставленные сетевые диски при полной проверке.** Разрешает Защитнику проверять файлы на сопоставленных сетевых дисках.
  Если файлы на диске доступны только для чтения, Защитник не сможет удалить найденные на нем вредоносные программы.
- **Сканировать файлы, открываемые из сетевых папок**. Разрешает Защитнику проверять файлы на общих сетевых дисках (например, доступные по UNC-пути). Если файлы на диске доступны только для чтения, Защитник не сможет удалить найденные на нем вредоносные программы.
- **Защита облака.** Разрешает или запрещает Microsoft Active Protection Service получать с управляемых устройств сведения об активности вредоносных программ. Эти сведения используются для дальнейшего улучшения службы.
- **Выдавать запрос перед отправкой образцов**. Указывает, следует ли отправлять потенциально вредоносные файлы, для обработки которых может потребоваться дополнительный анализ, в корпорацию Майкрософт автоматически.
- **Время для выполнения ежедневной быстрой проверки.** Разрешает запланировать ежедневную быструю проверку в заданное время.
- **Тип выполняемой проверки системы**. Введите уровень сканирования, которое выполняется во время запланированной проверки системы.
- **Обнаружить потенциально нежелательные приложения**. Разрешает выбрать уровень защиты при обнаружении Windows потенциально нежелательных приложений:
  - **Заблокировать**
  - **Аудит**. Дополнительные сведения о потенциально нежелательных приложениях см. в разделе [об обнаружении и блокировании потенциально нежелательных приложений](https://docs.microsoft.com/windows/threat-protection/windows-defender-antivirus/detect-block-potentially-unwanted-apps-windows-defender-antivirus).
- **Действия с обнаруженными угрозами от вредоносных программ**. Укажите в этом параметре действия, которые Защитник должен выполнять для каждого обнаруженного уровня угроз (низкий, средний, высокий и критический). Доступны следующие параметры:
  - **Очистить**
  - **Карантин**
  - **Remove**
  - **Разрешить**
  - **Определяется пользователем**
  - **Заблокировать**

### <a name="windows-defender-antivirus-exclusions"></a>Исключения антивирусной программы "Защитник Windows"

- **Файлы и папки, исключаемые из проверок и защиты в реальном времени.** Добавляет один или несколько файлов либо папок, таких как **C:\Путь** или **%ProgramFiles%\Путь\имя_файла.exe**, в список исключений. Эти файлы и папки не учитываются при проверках в режиме реального времени или по расписанию.
- **Расширения файлов, исключаемые из проверок и защиты в реальном времени.** Добавляет одно или несколько расширений файла, например **JPG** или **TXT**, в список исключений. Все файлы с такими расширениями не учитываются при проверках в реальном времени или по расписанию.
- **Процессы, исключаемые из проверок и защиты в реальном времени.** Добавляет один или несколько процессов типа **EXE**, **COM** или **SCR** в список исключений. Эти процессы не учитываются при проверках в реальном времени или по расписанию.

## <a name="network-proxy"></a>Прокси-сервер сети

- **Автоматически определять параметры прокси-сервера**. Если этот параметр включен, устройство попытается найти путь к скрипту PAC.
- **Использовать скрипт прокси-сервера**. Используйте этот параметр, чтобы указать путь к скрипту PAC для настройки прокси-сервера.
  - **URL-адрес скрипта установки.** Введите URL-адреса скрипта PAC, который необходимо использовать для настройки прокси-сервера.
- **Использовать ручную настройку прокси-сервера**. Используйте этот параметр, чтобы вручную указать сведения о прокси-сервере.
  - **Адрес.** Введите имя или IP-адрес прокси-сервера.
  - **Номер порта.** Введите номер порта прокси-сервера.
  - **Исключения прокси-сервера.** Введите URL-адреса, которые не должны использовать прокси-сервер. Для разделения элементов используйте точку с запятой.
  - **Обход прокси-сервера для локального адреса**. Включите этот параметр, чтобы не использовать прокси-сервер для локальных адресов интрасети.

## <a name="windows-spotlight"></a>Windows: интересное

- **Windows: интересное.** Используйте этот параметр, чтобы блокировать все функциональные возможности "Windows: интересное" на устройствах Windows 10. Если вы отключите этот параметр, перечисленные ниже параметры будут недоступны.
  - **Windows: интересное на экране блокировки**. Запрещает компоненту "Windows: интересное" отображать информацию на экране блокировки устройства.
  - **Сторонние предложения в разделе Windows: интересное**. Запрещает компоненту "Windows: интересное" предлагать содержимое, которое опубликовано не корпорацией Майкрософт.
  - **Функции клиентов.** Разрешает блокировать функции клиентов, например рекомендации по использованию меню "Пуск" и уведомления о членстве.
  - **Советы по использованию Windows.** Разрешает блокировать всплывающие окна с советами в Windows.
  - **Windows: интересное в центре уведомлений**. Запрещает компоненту "Windows: интересное" отображать предложения, например новое содержимое о приложении или безопасности, в центре уведомлений Windows.
  - **Персонализация раздела Windows: интересное**. Запрещает компоненту "Windows: интересное" персонализировать результаты в зависимости от использования устройства.
  - **Экран приветствия Windows**. Отключает экран приветствия Windows, где пользователю отображаются сведения о новых и обновленных возможностях.

## <a name="projection"></a>Прогнозирование

- **Ввод пользователя с беспроводных дисплеев-приемников.** Блокирует ввод пользователя с беспроводных дисплеев-приемников.
- **Проекция на этот компьютер.** Блокирует обнаружение ПК устройствами для выполнения проекции.
- **Требовать ПИН-код для связывания.** Требует запрашивать ПИН-код при подключении к устройству проекции.

## <a name="cloud-printer"></a>Облачный принтер

- **URL-адрес обнаружения принтера**. Введите URL-адрес для обнаружения облачных принтеров.
- **URL-адрес центра доступа к принтеру**. Введите URL-адрес конечной точки аутентификации для получения токенов OAuth. Например, можно ввести `https://login.microsoftonline.com/your Azure AD Tenant ID`.
- **GUID приложения собственного клиента Azure**. Введите GUID клиентского приложения, авторизованного для получения токенов OAuth из OAuthAuthority.
- **URI ресурса службы печати**. Введите URI ресурса OAuth для службы печати, настроенный на портале Azure. Например, можно ввести `http://MicrosoftEnterpriseCloudPrint/CloudPrint`.
- **Максимальное число опрашиваемых принтеров (только для мобильных устройств)**. Введите максимальное число принтеров, которое можно опрашивать. Например, введите `10`.
- **URI ресурса службы обнаружения принтеров**. Введите URI ресурса OAuth для службы обнаружения принтера, настроенный на портале Azure. Например, можно ввести `http://MopriaDiscoveryService/CloudPrint`.

> [!TIP]
> После настройки [гибридной облачной печати Windows Server](https://docs.microsoft.com/windows-server/administration/hybrid-cloud-print/hybrid-cloud-print-overview) можно настроить эти параметры, а затем выполнить развертывание на устройствах Windows.

## <a name="local-printer"></a>Локальный принтер

- **Принтеры** — список локальных принтеров, которые были добавлены.
- **Принтер по умолчанию** — задайте принтер по умолчанию.
- **Доступ пользователей для добавления новых принтеров** — разрешите или запретите использование локальных принтеров.

## <a name="reporting-and-telemetry"></a>Отчетность и телеметрия

- **Публикация данных об использовании**. Выберите уровень предоставляемых диагностических данных. Доступны следующие параметры:
  - Безопасность
  - базовой;
  - Расширенный
  - Полная проверка
- **Отправлять данные браузера Microsoft Edge в Microsoft 365 Analytics**. Чтобы использовать эту функцию, задайте для параметра **Публикация данных об использовании** значение **Расширенный** или **Полный**. Этот параметр контролирует, какие данные браузер Microsoft Edge отправляет в Microsoft 365 Analytics с корпоративных устройств, для которых настроен коммерческий идентификатор. Доступны следующие параметры:
  - Вариант **Не настроено** использует стандартное поведение операционной системы, которое может не включать отправку данных о просмотренных страницах.
  - **Отправлять данные только об интрасети** — разрешает администратору отправлять данные о доступе к интрасети.
  - **Отправлять данные только об Интернете** — разрешает администратору отправлять данные о доступе к Интернету.
  - **Отправлять данные об интрасети и Интернете** — разрешает администратору отправлять данные о доступе к интрасети и Интернету.
- **Прокси-сервер телеметрии**. Укажите полное доменное имя (FQDN) или IP-адрес прокси-сервера для переадресации запросов к данным телеметрии и подключенным пользовательским интерфейсам помощью подключения Secure Sockets Layer (SSL). Для этого параметра используется формат *сервер*:*порт*. Если включена эта политика, то при сбое указанного прокси-сервера или при отсутствии настроенного прокси-сервера, данные телеметрии и подключенного пользовательского интерфейса не передаются и сохраняются на локальном устройстве.

  Примеры форматов:

  ```
  IPv4: 192.246.246.106:100
  IPv6: [2001:4898:4010:4013:95c1:a8b2:953c:c633]:100
  FQDN: www.contoso.com:345
  ```

## <a name="messaging"></a>Messaging

- **Синхронизация сообщений (только для мобильных устройств)** — отключите универсальную службу, а также резервное копирование и восстановление текстовых сообщений.
- **MMS-сообщения (только для мобильных устройств)** — отключите возможность отправки и получения MMS-сообщений на устройстве.
- **RCS (только для мобильных устройств)** — отключите функции приема и отправки через протокол Rich Communication Services (RCS).
