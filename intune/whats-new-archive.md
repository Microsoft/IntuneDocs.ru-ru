# <a name="whats-new-in-the-microsoft-intune---previous-months"></a>Новые возможности в Microsoft Intune — предыдущие месяцы

[!INCLUDE[azure_portal](./includes/azure_portal.md)]

## <a name="september-2017"></a>Сентябрь 2017 г.

### <a name="inform-end-users-what-device-information-can-be-seen-for-ios---739894--"></a>Информирование конечных пользователей о том, какие сведения можно увидеть об устройстве iOS <!--739894-->

Мы добавили **тип владения** на экран сведений об устройстве в приложении корпоративного портала для iOS. Теперь пользователи смогут узнать больше о конфиденциальности из документации для пользователя Intune непосредственно на этой странице. Они также смогут найти эти сведения на экране "О программе".

### <a name="allow-end-users-to-access-the-company-portal-app-for-android-without-enrollment----1169910---"></a>Разрешение конечным пользователям доступа к приложению корпоративного портала для Android без регистрации <!---1169910--->

Скоро конечным пользователям не придется регистрировать свои устройства, чтобы получить доступ к приложению корпоративного портала для Android. Пользователи в организациях, где применяются политики защиты приложений, больше не будут получать запросы на регистрацию при открытии приложения корпоративного портала. Конечные пользователи также смогут устанавливать приложения с корпоративного портала без регистрации устройства. 


### <a name="easier-to-understand-phrasing-for-the-company-portal-app-for-android----1396349---"></a>В приложении корпоративного портала для Android текст стал более понятным <!---1396349--->.  

Упрощен процесс регистрации пользователей в приложении корпоративного портала для Android благодаря обновленному тексту. Если у вас есть документация по пользовательской регистрации, необходимо обновить ее, чтобы отразить новые экраны. Примеры можно найти на странице [Обновление элементов пользовательского интерфейса для приложения конечного пользователя Intune](whats-new-app-ui.md#week-of-september-11-2017).

### <a name="windows-10-company-portal-app-added-to-windows-information-protection-allow-policy----677129---"></a>Добавление приложения корпоративного портала Windows 10 в политику разрешения Windows Information Protection <!-- 677129 -->

Приложение корпоративного портала Windows 10 обновлено для поддержки Windows Information Protection (WIP). Теперь его можно добавить в политику WIP. После этого изменения вам больше не потребуется добавлять приложение в список **исключений**.


## <a name="august-2017"></a>Август 2017 г.

### <a name="improvements-to-device-overview----1404453---"></a>Усовершенствования обзора устройств <!-- 1404453 -->  
Улучшенное представление обзора устройств отображает зарегистрированные устройства, но исключает устройства, управляемые Exchange ActiveSync. Параметры управления устройствами Exchange ActiveSync отличаются от параметров зарегистрированных устройств. ### <a name="conditional-access-support-for-macos-devices"></a>Поддержка условного доступа для устройств macOS 
<!-- 720172 -->

- Подробнее о [поддержке условного доступа для устройств macOS](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-azure-portal).

## <a name="july-2017"></a>Июль 2017 г.

### <a name="restrict-android-and-ios-device-enrollment-restriction-by-os-version------1333256--1245463----"></a>Ограничения регистрации устройств Android и iOS по версии ОС <!--- 1333256,  1245463 --->
Intune поддерживает ограничение регистрации устройств Android и iOS по номеру версии операционной системы. В разделе **Ограничение по типу устройств** ИТ-администратор теперь может задавать конфигурации платформ, позволяющие ограничить доступные для регистрации операционные системы минимальным и максимальным номером версии. Версии ОС Android должны задаваться в виде строки "Основная.Вспомогательная.Сборка.Редакция", в которой атрибуты вспомогательной версии, сборки и редакции являются необязательными. Версии iOS должны задаваться в виде строки "Основная.Вспомогательная.Сборка", где атрибуты вспомогательной версии и сборки являются необязательными. Дополнительные сведения об [ограничениях на регистрацию устройств](enrollment-restrictions-set.md).

>[!NOTE]
>Не ограничивайте регистрацию с помощью программы регистрации Apple или Apple Configurator.

### <a name="restrict-android-ios-and-macos-device-personally-owned-device-enrollment------1333272--1333275-1245709----"></a>Ограничение регистрации личных устройств с Android, iOS и macOS <!--- 1333272,  1333275, 1245709 --->
Intune может ограничивать регистрацию личного устройства, применяя утвержденные номера IMEI корпоративных устройств. Эта функциональность Intune расширена для iOS, Android и macOS, где используются серийные номера устройств. Отправив серийные номера в Intune, можно предварительно объявить устройства как корпоративные. С помощью ограничений регистрации вы можете заблокировать личные устройства (BYOD), разрешив только регистрацию корпоративных устройств. Дополнительные сведения об [ограничениях на регистрацию устройств](enrollment-restrictions-set.md).

Чтобы импортировать серийные номера, перейдите в раздел **Регистрация устройства** > **Идентификаторы корпоративных устройств** и нажмите кнопку **Добавить**, а затем отправьте CSV-файл (без заголовка, два столбца для серийных номеров и таких сведений, как номера IMEI).  Чтобы ограничить личные устройства, перейдите в раздел **Регистрация устройства** > **Ограничения регистрации**. В области **Ограничения по типу устройств** выберите **По умолчанию**, а затем **Конфигурации платформ**. Вы можете **Разрешить** или **Блокировать** личные устройства на базе iOS, Android и macOS. 


### <a name="new-device-action-to-force-devices-to-sync-with-intune----711369---"></a>Новое действие для принудительной синхронизации устройств с Intune <!-- 711369 -->
В этом выпуске мы добавили новое действие, которое заставляет выбранное устройство немедленно выполнить возврат в Intune. При возврате устройство сразу же получает все ожидающие действия или политики, которые были ему назначены.  Это действие позволяет сразу проверять назначенные политики и устранять возникшие с ними неполадки, не дожидаясь следующего запланированного возврата.
Дополнительные сведения см. в статье [Синхронизация устройств](device-sync.md).

### <a name="force-supervised-ios-devices-to-automatically-install-the-latest-available-software-update----777100---"></a>Принудительно защищенные устройства iOS для автоматической установки последнего доступного обновления программного обеспечения <!-- 777100 -->
Новая политика доступна в рабочей области "Обновления программного обеспечения", где вы можете принудительно защитить устройства iOS для автоматической установки последнего доступного обновления программного обеспечения. Дополнительные сведения см. в статье [Настройка политик обновления iOS](/intune/software-updates-ios).

### <a name="check-point-sandblast-mobile---new-mobile-threat-defense-partner-----954651-1172027---"></a>Check Point SandBlast Mobile — новый партнер Mobile Threat Defense <!-- 954651, 1172027 -->
Вы можете управлять доступом к корпоративным ресурсам с мобильных устройств посредством условного доступа на основании оценки рисков, проведенной Checkpoint SandBlast Mobile — решением для защиты от угроз на мобильных устройствах, интегрированным с Microsoft Intune.

#### <a name="how-integration-with-intune-works"></a>Принцип работы интеграции с Intune
Оценка рисков основана на данных телеметрии, собранных с устройств, на которых выполняется приложение Checkpoint SandBlast Mobile. Вы можете настроить политики условного доступа EMS на основе оценки рисков, выполненной Checkpoint SandBlast Mobile с применением политик соответствия устройств Intune. Вы можете разрешить или запретить доступ несоответствующих устройств к корпоративным ресурсам в зависимости от обнаруженных угроз.


### <a name="deploy-an-app-as-available-in-the-microsoft-store-for-business----748101---"></a>Развертывание приложения как доступного в Магазине Майкрософт для бизнеса <!-- 748101 -->

### <a name="ui-updates-to-the-company-portal-website---1313244-part-1--"></a>Обновления пользовательского интерфейса для веб-сайта корпоративного портала <!--1313244 part 1-->
Мы внесли несколько обновлений в пользовательский интерфейс [веб-сайта корпоративного портала](https://portal.manage.microsoft.com) для улучшения работы пользователей.

- __Улучшения плиток приложений.__ Теперь значки приложений отображаются с автоматически создаваемым фоном на основе базового цвета значка (если его можно определить). Там, где применимо, этот фон заменяет серую границу, которая ранее отображалась на плитках приложений.

    На веб-сайте корпоративного портала отображаются крупные значки, когда это возможно, в следующих выпусках. Рекомендуется, чтобы ИТ-администраторы публиковали приложения с использованием значков с высоким разрешением и минимальным размером 120 x 120 пикселей. 

- __Изменения навигации.__ Элементы панели навигации перемещены в меню в верхнем левом углу. Страница "Категории" удалена. Теперь пользователи могут фильтровать содержимое по категориям при просмотре.

- __Обновления для рекомендуемых приложений.__ Мы добавили выделенную страницу для сайта, где пользователи могут просматривать выбранные вами приложения, и определенным образом оптимизировали пользовательский интерфейс раздела "Рекомендуемые" на домашней странице.

Приложения iOS, в которые интегрирован пакет SDK, также могут задавать требование минимальной версии на уровне пакета SDK. Пользователь не сможет получить доступ к целевому приложению, если на трех разных уровнях, описанных выше, не выполнены минимальные требования политики защиты приложения. На этом этапе пользователь может удалить учетную запись (для приложений с несколькими удостоверениями), закрыть приложение или обновить ОС либо приложение до требуемой версии.

Кроме того, вы также можете настраивать дополнительные параметры, чтобы пользователю отображалось уведомление с рекомендацией обновить ОС или приложение. При этом доступ к приложению не будет блокироваться. Пользователь сможет закрыть это уведомление и использовать приложение в обычном режиме.

Дополнительные сведения см. в статьях [Параметры политики защиты приложений в iOS](app-protection-policy-settings-ios.md) и [Параметры политики защиты приложений в Android](app-protection-policy-settings-android.md).

### <a name="configure-windows-update-for-business-settings---776716--"></a>Настройка параметров Центра обновления Windows для бизнеса <!--776716-->

Windows как служба — это новый механизм обеспечения обновления для Windows 10. Начиная с Windows 10, все новые обновления компонентов и исправления будут включать в себя содержимое всех предыдущих обновлений. Это означает, что если установлено последнее обновление, то устройства с Windows 10 полностью обновлены. В отличие от предыдущих версий Windows, теперь необходимо устанавливать не отдельную часть обновления, а полное обновление.

С помощью Центра обновления Windows для бизнеса процесс управления обновлением можно упростить, чтобы не утверждать отдельные обновления для групп устройств. Вы по-прежнему можете управлять рисками в своей среде благодаря настройке стратегии накопительных обновлений, а Центр обновления Windows будет обеспечивать установку обновлений в нужное время. Microsoft Intune позволяет настраивать параметры обновления на устройствах, а также устанавливать обновления в отложенном режиме. Intune хранит не обновления, а только назначение политики обновления. Устройства напрямую обращаются к Центру обновления Windows и отслеживают наличие обновлений. Для настройки **колец обновления Windows 10** и управления ими используйте Intune. Кольцо обновления содержит группу параметров, позволяющих указать время и способ установки обновлений Windows 10. Дополнительные сведения см. в разделе [Настройка параметров Центра обновления Windows для бизнеса](windows-update-for-business-configure.md).
