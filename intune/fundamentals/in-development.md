## <a name="app-management"></a>Управление приложениями

### <a name="additional-app-configuration-variable-available----4969237----"></a>Доступна дополнительная переменная конфигурации приложения <!-- 4969237  -->
При создании политики конфигурации приложений вы сможете включить переменную конфигурации `AAD Device ID` как часть параметров конфигурации. В Intune выберите **Клиентские приложения** > **Политики конфигурации приложений** > **Добавить**. Введите сведения о политике конфигурации и выберите **Параметры конфигурации** , чтобы просмотреть колонку **Параметры конфигурации** .

### <a name="dark-mode-for-ios-company-portal----4911422----"></a>Темный режим для корпоративный портал iOS <!-- 4911422  -->
Для корпоративный портал iOS планируется темный режим. Скачайте корпоративные приложения, управляйте своими устройствами и получайте поддержку в цветовой схеме по своему усмотрению. Сведения о настройке приложения "Корпоративный портал" iOS см. в статье [Настройка приложения корпоративного портала Microsoft Intune](../apps/company-portal-app.md).

### <a name="prevent-installation-of-apps-from-unknown-sources-on-android-enterprise-devices----4760025----"></a>Запретить установку приложений из неизвестных источников на устройствах Android Enterprise <!-- 4760025  -->
Для устройства рабочего профиля Android Enterprise можно запретить пользователям устанавливать приложения из неизвестных источников в рабочий профиль. При необходимости вы сможете также расширить это ограничение для личного профиля. Если включить это ограничение, конечные пользователи на устройствах с рабочим профилем Android Enterprise также будут предотвращены от загрузки неопубликованных приложений из неизвестных источников в личную сторону устройства. 

### <a name="update-to-app-protection-ui-and-ios-app-provisioning-ui----4102027-4102029----"></a>Обновление пользовательского интерфейса защиты приложений и пользовательского интерфейса подготовки приложений iOS <!-- 4102027, 4102029  -->
Пользовательский интерфейс для создания и изменения политик защиты приложений и профилей подготовки приложений iOS в Intune будет обновлен. Ниже приведены изменения пользовательского интерфейса.
- Упрощенный интерфейс с использованием формата мастера, сжатого в пределах одной колонки. 
- Обновление для потока создания, включающее назначения.
- Сводная страница всех элементов, заданных при просмотре свойств до создания новой политики или изменения свойства. Кроме того, при изменении свойств в сводке отображается только список элементов из категории изменяемых свойств.

### <a name="create-groups-of-management-objects-called-policy-sets----3762880----"></a>Создание групп управляющих объектов, называемых наборами политик <!-- 3762880  -->
Наборы политик позволяют создать набор ссылок на уже существующие сущности управления, которые должны быть идентифицированы, нацелены и отслеживаться как единое концептуальное устройство. Наборы политик не заменяют существующие основные понятия или объекты. Администратор может продолжить назначать отдельные объекты, как это делается сегодня. На отдельные объекты ссылается набор политик. Таким образом, любые изменения отдельных объектов будут отражены в наборе политик.  В Intune выберите **наборы политик** > **создать** , чтобы создать новый набор политик. 

### <a name="win32-apps-on-windows-10-s-mode-devices----3747604----"></a>Приложения Win32 на устройствах с Windows 10 в режиме S <!-- 3747604  --> 
Вы сможете устанавливать и запускать приложения Win32 на устройствах, управляемых в режиме Windows 10 S. Вы сможете создать одну или несколько дополнительных политик для режима S с помощью средств PowerShell для управления приложениями в Защитнике Windows (WDAC). Подпишите дополнительные политики с помощью портала подписывания Device Guard, а затем отправьте и распределите политики через Intune. В Intune эту возможность можно найти, выбрав **клиентские приложения** > **Дополнительные политики Windows 10 S**. 

### <a name="set-app-availability-based-on-a-date-and-time----3510685----"></a>Установка доступности приложения на основе даты и времени <!-- 3510685  -->
Как администратор вы сможете настроить время начала и время крайнего срока для требуемого приложения. В момент запуска расширение управления Intune запустит содержимое приложения и загрузит его в кэш. Приложение будет установлено в крайнем крайнем случае. Для доступных приложений время начала будет определяться при отображении приложения в корпоративный портал. В Intune выберите **Клиентские приложения** > **Приложения**. Затем выберите определенное приложение из списка или нажмите кнопку **Добавить** , чтобы добавить новое приложение. В колонке приложения выберите **назначения** > **Добавить группу**. Задайте для параметра **тип назначения** значение **обязательно** , а затем выберите **включаемые группы**. Установите флажок **сделать это приложение обязательным для всех пользователей** , а затем выберите **изменить** **, чтобы изменить** параметры **взаимодействия с конечным пользователем** . В колонке **взаимодействие с пользователем** задайте **время доступности программного обеспечения** по мере необходимости. Дополнительные сведения о добавлении приложений в Intune см. в [этой статье](../apps/apps-add.md).

### <a name="require-win32-apps-to-restart----3136567----"></a>Запрашивать перезапуск приложений Win32 <!-- 3136567  -->
После успешной установки можно будет потребовать, чтобы приложение Win32 было необходимо перезапустить. Кроме того, вы сможете выбрать время (льготный период) перед перезапуском.

### <a name="company-portal-app-on-windows----1808082----"></a>корпоративный порталное приложение в Windows <!-- 1808082  -->
Приложение корпоративный портал на устройствах Windows будет обновлено для вывода всплывающих уведомлений пользователям, даже если приложение закрыто. При обновлении будут отображаться только уведомления о доступных приложениях, когда состояние установки — "завершено" или "сбой". Приложение корпоративный портал не будет показывать уведомления для требуемых приложений.

### <a name="company-portal-app-installation-status-messages----2514416----"></a>Сообщения о состоянии установки корпоративный портал приложения <!-- 2514416  -->
Приложение корпоративный портал отобразит дополнительные сообщения о состоянии установки приложения для конечных пользователей. Следующие условия будут применены к новым функциям зависимости Win32.
- Приложение не удалось установить. Зависимости, определенные администратором, не выполнены.
- Приложение успешно установлено, но требуется перезагрузка.
- Приложение находится в процессе установки, но для продолжения требуется перезагрузка.

#### <a name="assign-microsoft-edge-beta-for-macos----4678761----"></a>Назначение бета-версии Microsoft ребра для macOS <!-- 4678761  -->
Вы сможете добавить и назначить последнюю версию бета-версии Microsoft ребра в Intune для устройств macOS. В Intune выберите **клиентские приложения** > **приложения** > **Добавить приложение** > **Microsoft ребра-macOS**. Затем назначьте бета-версию Microsoft ребра для нужных групп. Microsoft Update (MAU) обеспечивает актуальность Microsoft ребра. Дополнительные сведения о Microsoft ребр см. в статье [Управление веб-доступом с помощью Microsoft ребра с Microsoft Intune](../apps/manage-microsoft-edge.md).

### <a name="configure-app-notification-content-for-organization-accounts----2576686---"></a>Настройка содержимого уведомлений приложения для учетных записей Организации <!-- 2576686 -->
Политики защиты приложений (APP) Intune на устройствах Android и iOS позволяют управлять содержимым уведомлений приложения для учетных записей организации. Эта функция требует поддержки приложений и может быть недоступна для всех приложений с поддержкой приложений. Дополнительные сведения о политиках защиты приложений см. в статье [Что такое политики защиты приложений?](../apps/app-protection-policy.md).

### <a name="available-google-play-app-reporting-for-android-work-profiles----3041956----"></a>Доступные отчеты о приложениях Google Play для рабочих профилей Android <!-- 3041956  -->
Для доступных установок приложений на устройствах с рабочими профилями Android вы можете просматривать сведения о состоянии установки приложения, а также установленную версию управляемых приложений Google Play. Дополнительные сведения см. в статьях [Как отслеживать политики защиты приложений](../apps/app-protection-policies-monitor.md), [Управление устройствами с рабочим профилем Android в Intune](../enrollment/android-enterprise-overview.md) и [Тип приложения "Управляемый Google Play"](../apps/apps-add-android-for-work.md#managed-google-play-app-types).

## <a name="device-configuration"></a>Конфигурация устройства

На устройствах iOS и Ипадос можно создать профиль для ограничения возможностей и параметров на устройствах (**Конфигурация устройства** > **Профили** > **создать профиль** > **iOS/ипадос** для устройства > платформы  **ограничения** для типа профиля). Доступны новые параметры, которыми можно управлять: 
- Доступ к сетевому диску в приложении "файлы"  
- Доступ к USB-диску в приложении "файлы" 
- Wi-Fi всегда включен 

Чтобы узнать текущие параметры, см. статью [Параметры устройства с iOS, которые позволяют разрешить или ограничить некоторые функции через Intune](../configuration/device-restrictions-ios.md).

Область применения:
- iOS 13.0 и более поздние версии.
- iPadOS 13.0 и более поздние версии.

### <a name="connect-automatically-setting-is-removed-in-wi-fi-profiles-on-android-and-android-enterprise----5021055----"></a>Параметр подключения автоматически удаляется в профилях Wi-Fi в Android и Android Enterprise. <!-- 5021055  -->
На устройствах Android и Android Enterprise можно создать профиль Wi-Fi для настройки различных параметров (**Конфигурация устройства** > **Профили** > **создать профиль** > **Android** или **Android Enterprise.** для типа профиля > платформы **Wi-Fi** ). Параметр **подключиться автоматически** будет удален, так как он [не поддерживается Android](https://developer.android.com/reference/android/net/wifi/WifiManager.html#enableNetwork%28int%2c%20boolean%29). 

Если вы используете этот параметр в профиле Wi-Fi, вы можете заметить, что **подключение автоматически** не будет работать. Не нужно предпринимать никаких действий, но имейте в виду, что этот параметр удаляется в пользовательском интерфейсе Intune.

Чтобы просмотреть текущие параметры, перейдите в раздел [Параметры Wi-Fi Android](../configuration/wi-fi-settings-android.md) или [Android Enterprise Wi-Fi Settings](../configuration/wi-fi-settings-android-enterprise.md).

Область применения:
- Android
- Android для бизнеса

### <a name="create-a-global-http-proxy-on-android-enterprise-device-owner-devices----4816339----"></a>Создание глобального прокси-сервера HTTP на устройствах с владельцами устройств Android Enterprise <!-- 4816339  -->
На устройствах Android Enterprise можно настроить глобальный прокси-сервер HTTP в соответствии со стандартами веб-обзора вашей организации (**Конфигурация устройства** > **Профили** > **создать профиль** > **Android Enterprise** для Платформа > **Владелец устройства > ограничения на устройства** для типа профиля > **Подключение**). После настройки весь трафик HTTP будет использовать этот прокси-сервер.

Область применения:
- владельцам устройств Android для бизнеса

### <a name="new-device-firmware-configuration-interface-profile-for-windows-10-and-later-devices----2266073----"></a>Новый профиль интерфейса конфигурации встроенного по устройства для устройств Windows 10 и более поздних версий <!-- 2266073  -->
В Windows 10 и более поздних версиях можно создать профиль конфигурации устройства для управления параметрами и компонентами (**Конфигурация устройства** > **Профили** > **создать профиль** > **Windows 10 и более поздней версии** для платформы). Будет создан новый тип профиля интерфейса конфигурации встроенного по устройства, позволяющий Intune управлять параметрами UEFI (BIOS).

Обзор всех текущих параметров, которые можно настроить, см. в разделе [применение функций и параметров на устройствах с помощью профилей устройств в Microsoft Intune](../configuration/device-profiles.md).

Область применения:
- Windows 10 RS5 (1809) и более поздние версии на выбранных устройствах

### <a name="pkcs-certificates-for-macos-----1333650------------------"></a>Сертификаты PKCS для macOS  <!-- 1333650                -->
Мы будем добавлять полную поддержку сертификатов PKCS на устройствах под управлением macOS. Пользователи смогут развертывать сертификаты пользователей и устройств с полями настройки субъекта и альтернативное имя субъекта. У нас также будет новый параметр Разрешить доступ ко всем приложениям, что позволяет всем связанным приложениям получить доступ к закрытому ключу. Дополнительные сведения об этом параметре см. в следующей документации Apple: https://developer.apple.com/business/documentation/Configuration-Profile-Reference.pdf.

###   <a name="derived-credentials-to-provision-mobile-devices-with-certificates----------1736036-1736037-1772050--------"></a>Производные учетные данные для инициализации мобильных устройств с помощью сертификатов      <!--  1736036, 1736037, 1772050      --> 
Мы будем добавлять поддержку для производных учетных данных, которые поддерживают стандарт *национального института стандартов и технологий (NIST) 800-157* для развертывания сертификатов на устройствах.  Производные учетные данные полагаются на использование платы личной идентификации (PIV) или карты общей карты доступа (CAC), например смарт-карты. Пользователи проходят проверку подлинности с помощью смарт-карты на компьютере, а затем передают запрос на сертификат для управляемого устройства, следуя процессу, который требуется производному поставщику учетных данных.   

Процесс использования производных учетных данных для получения сертификата отличается от использования профилей сертификатов SCEP или PKCS, но конечный результат — мобильные устройства с сертификатами для проверки подлинности, которые можно использовать для проверки подлинности приложений, VPN, Wi-Fi или электронной почты. профили.   

Дополнительные сведения см. в разделе [DERIVED PIV Credentials](https://www.nccoe.nist.gov/projects/building-blocks/piv-credentials) on www.nccoe.NIST.gov.

Первоначальный выпуск производных учетных данных будет поддерживать Entrust Datacard, Интерцеде и DISA Пуребред в iOS. Дополнительные платформы и производные поставщики учетных данных будут поддерживаться в последующих выпусках.

### <a name="ui-update-for-creating-and-editing-windows-10-update-rings-----4099089------------"></a>Обновление пользовательского интерфейса для создания и изменения колец для обновления Windows 10  <!-- 4099089          -->

### <a name="ui-update-for-creating-and-editing-ios-software-updates-----4099090----------"></a>Обновление пользовательского интерфейса для создания и изменения обновлений программного обеспечения iOS  <!-- 4099090        -->   
Мы будем развертывать обновленный интерфейс создания и редактирования пользовательского интерфейса для обновлений программного обеспечения iOS в Intune. Изменения пользовательского интерфейса будут включать:
- Упростите существующий интерфейс с помощью формата мастера, сжатого в пределах одной колонки. Это обновление пользовательского интерфейса будет доойтись с пределами колонок, требующих от ИТ-специалистов детализации углублением колонок.  
- Процесс создания политики обновления программного обеспечения iOS будет обновлен для включения назначений 
- Политика обновления программного обеспечения iOS будет включать сводную страницу всех элементов, установленных при просмотре свойств, до создания новой политики и изменения свойств. При редактировании в сводке показан только список элементов, заданных в пределах одной категории редактируемых свойств.

### <a name="target-macos-user-groups-to-require-jamf-management----4061739---"></a>Целевые группы пользователей macOS, которым требуется управление Jamf <!-- 4061739 -->
Вы сможете назначать определенные группы пользователей, чтобы их macOS устройства управлялись Jamf. Эта настройка позволит применить интеграцию Jamf к подмножеству устройств macOS, в то время как другие устройства продолжают управляться Intune. Это также позволит постепенно перенести устройства пользователей из одного MDM в другое.

### <a name="new-restrictions-for-renaming-windows-devices----3478938---"></a>Новые ограничения для переименования устройств Windows <!-- 3478938 -->
Имена устройств должны соответствовать следующим правилам:
- не более 15 символов (должен быть меньше или равен 63 байт, не включая завершающее значение NULL)
- Не нулевая или пустая строка
- Разрешенные символы ASCII: буквы (a – z, A – Z), цифры (0-9) и дефисы
- Допускается Юникод: символов > = 0x80, должен быть допустимым UTF8, должен быть IDN-сопоставляемые (Ртлиднтонамепрепуникоде выполнен). см. RFC 3492)
- Имена не должны содержать только цифры или начинаться с цифры
- В имени нет пробелов
- Недопустимые символы: {|} ~ [\] ^ ':; < = >? & @! " # $ % ` ( ) + / , . _ *)

### <a name="deploy-software-updates-to-macos-devices----3194876---"></a>Развертывание обновлений программного обеспечения на устройствах macOS <!-- 3194876 -->
Вы сможете развертывать обновления программного обеспечения в группах устройств macOS. Эта функция включает в себя критическую версию, встроенное по, файл конфигурации и другие обновления. Вы сможете отправить обновления на следующем устройстве после возврата или выбрать еженедельное расписание для развертывания обновлений в заданных окнах или из них. Эта функция позволяет обновлять устройства, находящиеся за пределами стандартного рабочего времени, или когда служба технической поддержки полностью укомплектована персоналом. Вы также получите подробный отчет обо всех macOS устройствах с развернутыми обновлениями. Можно детализировать отчет для каждого устройства, чтобы просмотреть состояние конкретных обновлений.
