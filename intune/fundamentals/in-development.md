## <a name="app-management"></a>Управление приложениями

### <a name="apply-dark-mode-in-ios-company-portal----4911422----"></a>Применение темного режима в iOS корпоративный портал <!-- 4911422  -->
Темный режим планируется для iOS корпоративный портал. Вы сможете скачать приложения компании, управлять устройствами и получить поддержку в цветовой схеме по своему усмотрению. Сведения о настройке приложения "Корпоративный портал" iOS см. в статье [Настройка приложения корпоративного портала Microsoft Intune](../apps/company-portal-app.md).

### <a name="run-win32-apps-on-windows-10-s-mode-devices----3747604----"></a>Запуск приложений Win32 на устройствах с Windows 10 в режиме S <!-- 3747604  --> 
Вы сможете устанавливать и запускать приложения Win32 на устройствах, управляемых в режиме Windows 10 S. Создайте одну или несколько дополнительных политик для режима S с помощью средств PowerShell для управления приложениями в Защитнике Windows (WDAC). Используйте портал подписывания Device Guard для подписания дополнительных политик. Затем отправьте и распределите политики с помощью Intune. 

В Intune эту возможность можно найти, выбрав **клиентские приложения**  > **дополнительных политик Windows 10 S**. 

### <a name="set-app-availability-based-on-a-date-and-time----3510685----"></a>Установка доступности приложения на основе даты и времени <!-- 3510685  -->
Как администратор вы сможете настроить время начала и крайний срок для требуемого приложения. В момент запуска расширение управления Intune загрузит содержимое приложения и кэширует его. Приложение будет установлено в крайнем крайнем случае. Для доступных приложений время начала будет определяться при отображении приложения в корпоративный портал. 

Установка доступности приложения на основе даты и времени:

1. В Intune выберите **Клиентские приложения** > **Приложения**. 
1. Выберите приложение из списка или добавьте новое, нажав кнопку **Добавить**. 
1. В колонке приложения выберите **назначения** > **Добавить группу**. 
1. Задайте для параметра **тип назначения** значение **обязательно** , а затем выберите **включаемые группы**. 
1. Задайте для параметра **сделать это приложение обязательным, чтобы все пользователи** могли **Да**. 
1. Выберите **изменить** , чтобы изменить параметры **взаимодействия с конечным пользователем** . 
1. В колонке **взаимодействие с пользователем** задайте **время доступности программного обеспечения** по мере необходимости. 

Дополнительные сведения см. в статье [Добавление приложений в Microsoft Intune](../apps/apps-add.md).

### <a name="require-win32-apps-to-restart----3136567----"></a>Запрашивать перезапуск приложений Win32 <!-- 3136567  -->
После успешной установки вы сможете потребовать перезапуска приложения Win32. Можно выбрать время (льготный период) перед перезагрузкой.

### <a name="display-notifications-for-the-company-portal-app-on-windows----1808082----"></a>Отображение уведомлений для приложения корпоративный портал в Windows <!-- 1808082  -->
Мы будем обновлять приложение корпоративный портал на устройствах Windows, чтобы отображать всплывающие уведомления для пользователей, даже если приложение закрыто. Обновление будет показывать уведомления для доступных приложений только в случае, если состояние установки — "завершено" или "сбой". Приложение корпоративный портал не будет показывать уведомления для требуемых приложений.

### <a name="display-installation-status-messages-for-the-company-portal-app----2514416----"></a>Отображение сообщений о состоянии установки для приложения корпоративный портал <!-- 2514416  -->
Приложение корпоративный портал отобразит дополнительные сообщения о состоянии установки приложения для конечных пользователей. Следующие условия будут применены к новым функциям зависимости Win32.
- Приложение не удалось установить. Зависимости, определенные администратором, не выполнены.
- Приложение успешно установлено, но требуется перезагрузка.
- Приложение находится в процессе установки, но для продолжения требуется перезагрузка.

### <a name="assign-the-microsoft-edge-beta-for-macos----4678761----"></a>Назначение бета-версии Microsoft ребра для macOS <!-- 4678761  -->
Вы сможете добавить и назначить последнюю версию бета-версии Microsoft ребра в Intune для устройств macOS. 

Чтобы назначить бета-версию Microsoft ребра для устройств macOS, сделайте следующее:
1. В Intune выберите **клиентские приложения**  > **приложения**  > **Добавить приложение**  > **Microsoft ребра — macOS**. 
1. Назначьте бета-версию Microsoft ребра предполагаемым группам. Microsoft Update (MAU) поддерживает Microsoft ребро в актуальном состоянии. 
 
Дополнительные сведения о Microsoft ребр см. в статье [Управление веб-доступом с помощью Microsoft ребра с Microsoft Intune](../apps/manage-microsoft-edge.md).

### <a name="configure-app-notification-content-for-organization-accounts----2576686---"></a>Настройка содержимого уведомлений приложения для учетных записей Организации <!-- 2576686 -->
ПРИЛОЖЕНИЕ Intune на устройствах Android и iOS позволит управлять содержимым уведомлений приложения для учетных записей организации. Эта функция требует поддержки приложений и может быть недоступна для всех приложений с поддержкой приложений. Дополнительные сведения о приложении см. в статье [Общие сведения о политиках защиты приложений](../apps/app-protection-policy.md).

## <a name="device-configuration"></a>Конфигурация устройства

### <a name="new-device-firmware-configuration-interface-profile-for-devices-that-run-windows-10-and-later----2266073----"></a>Новый профиль интерфейса конфигурации встроенного по устройства для устройств под управлением Windows 10 и более поздних версий <!-- 2266073  -->
В Windows 10 и более поздних версиях можно создать профиль конфигурации устройства для управления параметрами и функциями: 

1. Выберите **Конфигурация устройства** > **Профили** > **Создать профиль**.
1. В поле платформы выберите **Windows 10 и более поздних версий**. 
 
Новый тип профиля интерфейса конфигурации встроенного по устройства позволяет Intune управлять параметрами UEFI (BIOS).

Сведения о текущих параметрах, которые можно настроить, см. [в разделе Применение компонентов и параметров на устройствах с помощью профилей устройств в Microsoft Intune](../configuration/device-profiles.md).

Эта функция применима к Windows 10 RS5 (1809) и более поздних версий на выбранных устройствах.

## <a name="device-management"></a>Управление устройствами


### <a name="edit-the-group-tag-value-for-autopilot-devices---4816775---"></a>Изменение значения тега группы для устройств с автопилотой<!-- 4816775 -->
Вы сможете изменить значение **тега группы** для устройств с автопилотой:

1. Выберите **Intune**  > **Регистрация устройств**  > **регистрации Windows**  > **устройств**с**Windows автопилота**  > .
1. Выберите устройство.
1. На панели справа измените значение **тега группы** .
1. Нажмите кнопку **Сохранить**.

### <a name="target-macos-user-groups-to-require-jamf-management----4061739---"></a>Целевые группы пользователей macOS, которым требуется управление Jamf <!-- 4061739 -->
Вы сможете выбрать определенные группы пользователей, чтобы они macOS устройства, которые должны управляться Jamf. Эта настройка позволит применить интеграцию Jamf к подмножеству устройств macOS, в то время как другие устройства продолжают управляться Intune. Нацеливание также позволит постепенно перенести устройства пользователей из одной системы управления мобильными устройствами (MDM) в другую.

### <a name="deploy-software-updates-to-macos-devices----3194876---"></a>Развертывание обновлений программного обеспечения на устройствах macOS <!-- 3194876 -->
Вы сможете развертывать обновления программного обеспечения в группах устройств macOS. Эта функция включает в себя критическую версию, встроенное по, файл конфигурации и другие обновления. Обновления можно отправить на следующей записи после изменения устройства. Кроме того, можно выбрать еженедельное расписание развертывания обновлений в заданных периодах или за их пределами. 

Эта функция позволяет обновлять устройства, находящиеся за пределами стандартного рабочего времени или за его пределами, когда служба технической поддержки полностью обработана. Кроме того, вы получите подробный отчет обо всех macOS устройствах с развернутыми обновлениями. Можно детализировать отчет по устройствам, чтобы просмотреть состояние конкретного обновления.

## <a name="monitoring-and-troubleshooting"></a>Мониторинг и устранение неполадок

### <a name="android-report-on-the-devices-overview-page----2984353----"></a>Отчет о Android на странице обзора устройств <!-- 2984353  -->
Мы добавим новый отчет на странице **Обзор устройств** . В отчете отображается количество устройств Android, зарегистрированных в каждом решении по управлению устройствами. На диаграмме показаны счетчики устройств для рабочего профиля, полностью управляемый, выделенный и зарегистрированный администратором устройства. 

Чтобы просмотреть отчет, выберите **Intune** > **устройства** > **Обзор**.

### <a name="updated-support-experience-------5012398------"></a>Обновленный интерфейс поддержки   <!--  5012398    -->
В рамках непрерывных улучшений мы будем обновлять интерфейс поддержки в консоли для Intune.  Мы улучшаем Поиск в консоли и обратную связь для распространенных проблем, и мы будем оптимизировать рабочий процесс для обращения в службу поддержки.
