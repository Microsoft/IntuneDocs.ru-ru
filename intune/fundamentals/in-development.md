## <a name="app-management"></a>Управление приложениями

### <a name="smime-support-for-microsoft-outlook-mobile----2669398----"></a>Поддержка S/MIME для Microsoft Outlook Mobile <!-- 2669398  -->
Intune будет поддерживать доставку сертификатов подписывания и шифрования S/MIME, которые можно использовать с Outlook Mobile на iOS и Android. Дополнительные сведения см. в разделе [Параметры электронной почты для устройств iOS](~/configuration/email-settings-ios.md) и [Параметры электронной почты для устройств Android](~/configuration/email-settings-android.md).

### <a name="custom-settings-support-for-macos-applications----4736278----"></a>Поддержка пользовательских параметров для приложений macOS <!-- 4736278  -->
Intune будет поддерживать пользовательские параметры, позволяя добавлять определенные ключи и значения в существующий файл списка свойств (. plist) для настройки приложений macOS и устройства. Не все приложения поддерживают управляемые настройки, и в некоторых случаях можно управлять только определенными параметрами. Параметры развертываются только через канал устройства. Следует отправлять только файлы списка свойств или XML-файлы, предназначенные для параметров канала устройства.

### <a name="assignment-type-value-in-windows-company-portal----5459950----"></a>Значение типа назначения в корпоративный портал Windows <!-- 5459950  -->
Страница **установленные приложения** приложения корпоративный портал Windows будет обновлена. Столбец " **тип назначения** " на странице " **установленные приложения** " был обновлен с именем "требуется для вашей организации". Возможные значения: **Да** или **нет** , чтобы обозначить требуемые и доступные приложения. Это изменение выполняется в ответ на некоторую путаницу с конечным пользователем. Дополнительные сведения о приложении "Корпоративный портал" для Windows см. в статье [Настройка приложения корпоративного портала Microsoft Intune](~/apps/company-portal-app.md).

### <a name="run-win32-apps-on-windows-10-s-mode-devices----3747604----"></a>Запуск приложений Win32 на устройствах с Windows 10 в режиме S <!-- 3747604  --> 
Вы сможете устанавливать и запускать приложения Win32 на устройствах, управляемых в режиме Windows 10 S. Создайте одну или несколько дополнительных политик для режима S с помощью средств PowerShell для управления приложениями в Защитнике Windows (WDAC). Используйте портал подписывания Device Guard для подписания дополнительных политик. Затем отправьте и распределите политики с помощью Intune. 

В Intune эту возможность можно найти, выбрав **клиентские приложения**  > **дополнительных политик Windows 10 S**. 

### <a name="set-app-availability-based-on-a-date-and-time----3510685----"></a>Установка доступности приложения на основе даты и времени <!-- 3510685  -->
Как администратор вы сможете настроить время начала и крайний срок для требуемого приложения. В момент запуска расширение управления Intune загрузит содержимое приложения и кэширует его. Приложение будет установлено в крайнем крайнем случае. Для доступных приложений время начала будет определяться при отображении приложения в корпоративный портал. 

Установка доступности приложения на основе даты и времени:

1. В Intune выберите **Клиентские приложения** > **Приложения**. 
1. Выберите приложение из списка или добавьте новое, нажав кнопку **Добавить**. 
1. В колонке приложения выберите **назначения** > **Добавить группу**. 
1. Задайте для параметра **тип назначения** значение **обязательно** , а затем выберите **включаемые группы**. 
1. Задайте для параметра **сделать это приложение обязательным, чтобы все пользователи** могли **Да**. 
1. Выберите **изменить** , чтобы изменить параметры **взаимодействия с конечным пользователем** . 
1. В колонке **взаимодействие с пользователем** задайте **время доступности программного обеспечения** по мере необходимости. 

Дополнительные сведения см. в статье [Добавление приложений в Microsoft Intune](../apps/apps-add.md).

### <a name="require-win32-apps-to-restart----3136567----"></a>Запрашивать перезапуск приложений Win32 <!-- 3136567  -->
После успешной установки вы сможете потребовать перезапуска приложения Win32. Можно выбрать время (льготный период) перед перезагрузкой.

### <a name="display-notifications-for-the-company-portal-app-on-windows----1808082----"></a>Отображение уведомлений для приложения корпоративный портал в Windows <!-- 1808082  -->
Мы будем обновлять приложение корпоративный портал на устройствах Windows, чтобы отображать всплывающие уведомления для пользователей, даже если приложение закрыто. Обновление будет показывать уведомления для доступных приложений только в случае, если состояние установки — "завершено" или "сбой". Приложение корпоративный портал не будет показывать уведомления для требуемых приложений.

### <a name="display-installation-status-messages-for-the-company-portal-app----2514416----"></a>Отображение сообщений о состоянии установки для приложения корпоративный портал <!-- 2514416  -->
Приложение корпоративный портал отобразит дополнительные сообщения о состоянии установки приложения для конечных пользователей. Следующие условия будут применены к новым функциям зависимости Win32.
- Приложение не удалось установить. Зависимости, определенные администратором, не выполнены.
- Приложение успешно установлено, но требуется перезагрузка.
- Приложение находится в процессе установки, но для продолжения требуется перезагрузка.

### <a name="configure-app-notification-content-for-organization-accounts----2576686---"></a>Настройка содержимого уведомлений приложения для учетных записей Организации <!-- 2576686 -->
ПРИЛОЖЕНИЕ Intune на устройствах Android и iOS позволит управлять содержимым уведомлений приложения для учетных записей организации. Эта функция требует поддержки приложений и может быть недоступна для всех приложений с поддержкой приложений. Дополнительные сведения о приложении см. в статье [Общие сведения о политиках защиты приложений](../apps/app-protection-policy.md).

## <a name="device-configuration"></a>Конфигурация устройства

### <a name="use-pkcs-certificates-with-wi-fi-profiles-on-windows-10-and-later-devices----3246388----"></a>Использование сертификатов PKCS с профилями Wi-Fi на устройствах Windows 10 и более поздних версий <!-- 3246388  -->
Сейчас можно проверить подлинность профилей Wi-Fi Windows с сертификатами SCEP (**Конфигурация устройства** > **Профили** > **создать профиль** > **Windows 10 и более поздних версий** для платформы > **Wi-Fi** для Тип профиля > **тип EAP** **Enterprise** > ). Вы сможете использовать сертификаты PKCS с профилями Wi-Fi Windows. Эта функция позволяет пользователям проходить проверку подлинности профилей Wi-Fi с помощью новых или существующих профилей сертификатов PKCS в клиенте. 

Дополнительные сведения о профилях Wi-Fi см. [в статье Добавление параметров Wi-Fi для устройств Windows 10 и более поздних версий в Intune](../configuration/wi-fi-settings-windows.md).

Область применения:
- Windows 10 и более поздней версии

### <a name="new-exchangeactivesync-settings-when-creating-an-email-device-configuration-profile-on-ios-devices----4892824----"></a>Новые параметры Ексчанжеактивесинк при создании профиля конфигурации устройства электронной почты на устройствах iOS <!-- 4892824  --> 
На устройствах iOS и Ипадос можно настроить подключение к электронной почте в профиле конфигурации устройства (**Конфигурация устройства** > **Профили** > **создать профиль** > **iOS/ипадос** для платформы > **электронной почты** для типа профиля). 

Будут доступны новые параметры Ексчанжеактивесинк, включая:
- Выберите службы для синхронизации (или блокировать синхронизацию), например "Электронная почта", "Календарь" и "Контакты".
- Разрешает (или запрещает) пользователям изменять параметры синхронизации для этих служб на своих устройствах. 

Чтобы просмотреть текущие параметры, перейдите к разделу [Параметры профиля электронной почты для устройств iOS в Intune](../configuration/email-settings-ios.md).

Область применения:
- iOS 13.0 и более поздние версии.
- iPadOS 13.0 и более поздние версии.

### <a name="prevent-users-from-adding-personal-google-accounts-to-android-enterprise-device-owner-and-dedicated-devices----5353228----"></a>Запретить пользователям добавлять персональные учетные записи Google к владельцу устройства Android Enterprise и выделенным устройствам <!-- 5353228  -->
Вы сможете запретить пользователям создавать персональные учетные записи Google на владельцах устройств Android Enterprise и на выделенных устройствах (**Конфигурация устройств** > **Профили** > **создать профиль** > **Android Enterprise** для **владельца устройства > только > ограничения на устройства** для типа профиля > **параметры пользователей и учетных записей**).

Полный список текущих параметров, которые вы можете настроить, см. в описании [параметров устройств Android для бизнеса, разрешающих или ограничивающих определенные функции Intune](../configuration/device-restrictions-android-for-work.md).

Область применения:
- владельцы устройств Android Enterprise
- Выделенные устройства Android для бизнеса

### <a name="server-side-logging-for-siri-commands-setting-is-removed-in-ios-device-restrictions-profile----5468501----"></a>Параметр ведения журнала на стороне сервера для команд Siri удален в профиле ограничений устройства iOS <!-- 5468501  -->
На устройствах iOS можно создать профили ограничений устройств, которые настраивают ведение журнала на стороне сервера для команд Siri (**Конфигурация устройства** > **Профили** > **создать профиль** > **iOS/ипадос** для платформы). > **Ограничения на устройства** для типа профиля > **встроенные приложения**). Параметр **ведения журнала на стороне сервера для команд Siri** будет удален.

Этот параметр будет удален из консоли администрирования Intune. Этот параметр не оказывает влияния на устройство, хотя существующие политики, для которых настроен этот параметр, будут по прежнему отображаться. Если вы хотите удалить параметр из существующих политик, перейдите к политике, внесите небольшие изменения, сохраните ее, и политика будет обновлена.

Полный список параметров, которые вы можете настроить, см. в описании [параметров устройств iOS и iPadOS, разрешающих или ограничивающих определенные функции Intune](../configuration/device-restrictions-ios.md).

Область применения:
- iOS

## <a name="device-management"></a>Управление устройствами

### <a name="edit-device-name-value-for-autopilot-devices---2640074----"></a>Изменение значения имени устройства для устройств автопилота<!-- 2640074  -->
Вы сможете изменить значение имени устройства для устройств автопилота, присоединенных к Azure AD. Чтобы сделать это, перейдите в **Intune** > **Регистрация устройств** > **регистрации Windows** > **устройства** **Windows автопилот** > > выбрать устройство > изменить значение **имени устройства** в правой области. > **Сохранить**.


### <a name="edit-the-group-tag-value-for-autopilot-devices---4816775---"></a>Изменение значения тега группы для устройств с автопилотой<!-- 4816775 -->
Вы сможете изменить значение **тега группы** для устройств с автопилотой:

1. Выберите **Intune**  > **Регистрация устройств**  > **регистрации Windows**  > **устройств**с**Windows автопилота**  > .
1. Выберите устройство.
1. На панели справа измените значение **тега группы** .
1. Нажмите кнопку **Сохранить**.

### <a name="target-macos-user-groups-to-require-jamf-management----4061739---"></a>Целевые группы пользователей macOS, которым требуется управление Jamf <!-- 4061739 -->
Вы сможете выбрать определенные группы пользователей, чтобы они macOS устройства, которые должны управляться Jamf. Эта настройка позволит применить интеграцию Jamf к подмножеству устройств macOS, в то время как другие устройства продолжают управляться Intune. Нацеливание также позволит постепенно перенести устройства пользователей из одной системы управления мобильными устройствами (MDM) в другую.
