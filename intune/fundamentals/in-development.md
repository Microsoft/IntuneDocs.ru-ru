# <a name="in-development-for-microsoft-intune---january-2020"></a>В разработке для Microsoft Intune, январь 2020 г.

Эта страница предназначена для помощи в подготовке и планировании, и здесь перечислены компоненты и обновления пользовательского интерфейса Intune, которые находятся на стадии разработки, но не были выпущены. В дополнение к сведениям на этой странице: 

- Если мы полагаем, что вам потребуется принять меры до изменения, мы опубликуем дополнительное сообщение в Центре сообщений Office.
- Если функция входит в рабочую среду, является ли она предварительной или общедоступной, описание функции будет перенесено с этой страницы на [новые](whats-new.md).
- Эта страница и страница [новых возможностей](whats-new.md) периодически обновляются. Проверьте наличие дополнительных обновлений.
- Стратегические цели и временную шкалу см. в документе [Стратегия развития Microsoft 365](https://www.microsoft.com/microsoft-365/roadmap?rtc=2&filters=EMS).

> [!NOTE]
> На этой странице отражены текущие ожидания возможностей Intune в будущем выпуске. Даты и отдельные компоненты могут меняться. На этой странице не описаны все функции в разработке.

## <a name="app-management"></a>Управление приложениями

### <a name="display-notifications-for-the-company-portal-app-on-windows---1808082----"></a>Отображение уведомлений для приложения корпоративный портал в Windows<!-- 1808082  -->
Мы будем обновлять приложение корпоративный портал на устройствах Windows, чтобы отображать всплывающие уведомления для пользователей, даже если приложение закрыто. Обновление будет показывать уведомления для доступных приложений только в случае, если состояние установки — "завершено" или "сбой". Приложение корпоративный портал не будет показывать уведомления для требуемых приложений. 

### <a name="display-installation-status-messages-for-the-company-portal-app---2514416----"></a>Отображение сообщений о состоянии установки для приложения корпоративный портал<!-- 2514416  -->
Приложение корпоративный портал отобразит дополнительные сообщения о состоянии установки приложения для конечных пользователей. Следующие условия будут применены к новым функциям зависимости Win32.
- Приложение не удалось установить. Зависимости, определенные администратором, не выполнены.

### <a name="retarget-web-clips-to-microsoft-edge-on-ios-devices---5455276-idready---"></a>Перенацеливание веб-клипов в Microsoft ребро на устройствах iOS<!-- 5455276 idready -->
Веб-клипы, которые работают как закрепленные веб-приложения на устройствах iOS, потребуется обновить. Недавно развернутые веб-клипы открываются в Microsoft, а не Intune Managed Browser, если это необходимо для открытия в защищенном браузере. Необходимо перенацелить уже существующие веб-клипы, чтобы убедиться, что они открыты в Microsoft ребре вместо Managed Browser. 

### <a name="user-experience-change-when-adding-apps-to-intune---4705829-idready---"></a>Изменение взаимодействия с пользователем при добавлении приложений в Intune<!-- 4705829 idready -->
При добавлении приложений через Intune вы увидите новый пользовательский интерфейс. Этот интерфейс предоставляет те же самые параметры и сведения, которые использовались ранее, однако перед добавлением приложения в Intune в новом интерфейсе следует процесс, аналогичный описанному в мастере. Этот новый интерфейс также содержит страницу проверки перед добавлением приложения. В [центре администрирования диспетчера конечных точек (Майкрософт)](https://go.microsoft.com/fwlink/?linkid=2109431) выберите **Приложения** > **Все приложения** > **Добавить**. Дополнительные сведения см. в статье [Добавление приложений в Microsoft Intune](~/apps/apps-add.md).

#### <a name="require-win32-apps-to-restart----3136567--"></a>Запрашивать перезапуск приложений Win32 <!-- 3136567-->
После успешной установки можно потребовать, чтобы приложение Win32 было перезапущено. Кроме того, можно выбрать время (льготный период) до того, как должно произойти перезагрузка.

## <a name="device-configuration"></a>Конфигурация устройства

### <a name="add-automatic-proxy-settings-to-wi-fi-profiles-for-android-enterprise-work-profiles---4490822-idready---"></a>Добавление автоматических параметров прокси-сервера в профили Wi-Fi для рабочих профилей Android Enterprise<!-- 4490822 idready -->
На устройствах с рабочими профилями Android Enterprise можно создавать профили Wi-Fi. При выборе типа предприятия Wi-Fi можно также ввести тип протокола EAP, используемый в сети Wi-Fi.

В будущем обновлении при выборе типа "Корпоративный" вы сможете вводить автоматические параметры прокси-сервера, в том числе URL-адрес прокси, например `proxy.contoso.com`.

Чтобы просмотреть текущие параметры Wi-Fi, которые можно настроить, перейдите к статье [Добавление параметров Wi-Fi для устройств под управлением Android Enterprise и Android в Microsoft Intune](../configuration/wi-fi-settings-android-enterprise.md).

Область применения:
- рабочий профиль "Android для бизнеса".

### <a name="wired-network-device-configuration-profiles-for-macos-devices---3508686----"></a>Профили конфигурации проводных сетевых устройств для устройств macOS<!-- 3508686  -->
Будет доступен новый профиль конфигурации устройства macOS, который настраивает проводные сети (**Конфигурация устройства** > **Профили** > **создать профиль** > **macOS** для платформы > **проводной сети** для типа профиля). Эта функция предназначена для создания профилей 802.1 x для управления проводных сетей и развертывания этих проводных сетей на устройствах macOS.

Область применения:
- MacOS

### <a name="vpn-profiles-with-ikev2-vpn-connections-can-use-always-on-with-ios-devices----1947932-idready---"></a>Профили VPN с VPN-подключениями IKEv2 могут использовать Always On с устройствами iOS <!-- 1947932 idready -->
На устройствах iOS можно создать профиль VPN, использующий подключение IKEv2 (**Конфигурация устройства** > **Профили** > **создать профиль** > **iOS/ипадос** для платформы > **VPN** для типа профиля). В будущем обновлении можно настроить Always On с помощью IKEv2. При настройке профили VPN IKEv2 подключаются автоматически и остаются подключенными (или быстро подключаются) к VPN. Он остается подключенным даже при перемещении между сетями или при перезагрузке устройств.

В iOS постоянное VPN ограничена профилями IKEv2.

Чтобы просмотреть текущие параметры IKEv2, которые можно настроить, см. раздел [Добавление параметров VPN для устройств с iOS в Microsoft Intune](../configuration/vpn-settings-ios.md#ikev2-settings).

Область применения:
- iOS

### <a name="improved-user-interface-experience-when-creating-configuration-profiles-on-ios-and-macos-devices---5569008-5569039-5569057-5569110-5569116-5569131-5569139-5569153-5859984-idready---"></a>Улучшенный интерфейс пользователя при создании профилей конфигурации на устройствах iOS и macOS<!-- 5569008-5569039-5569057-5569110-5569116-5569131-5569139-5569153-5859984 idready -->
При создании профиля для устройств iOS или macOS интерфейс центра администрирования управления конечными устройствами будет обновлен. Это изменение влияет на следующие профили конфигурации устройств (**устройства** > **профили конфигурации** > **создать профиль** > **iOS** или **macOS** для платформы):

- Custom: iOS, macOS
- Функции устройств: iOS, macOS
- Ограничения устройств: iOS, macOS
- Защита конечных точек: macOS
- Расширения: macOS
- Файл настроек: macOS

### <a name="improved-user-interface-experience-when-creating-oemconfig-configuration-profiles-on-android-enterprise-devices---5568645-idready----"></a>Улучшенный интерфейс пользователя при создании профилей конфигурации Оемконфиг на устройствах Android Enterprise<!-- 5568645 idready  -->
При создании или изменении профиля Оемконфиг для устройств Android Enterprise центр администрирования Endpoint Management обновляется. В обновленном интерфейсе будут предоставлены краткие сведения о параметрах, которые вы настроили. Это изменение влияет на профиль конфигурации устройства Оемконфиг (**устройства** > **профили конфигурации** > **создать профиль** > **Android Enterprise** для платформы > **оемконфиг** для типа профиля).

Данная функция применяется к:
- Android для бизнеса

## <a name="role-based-access-control"></a>управление доступом на основе ролей;

### <a name="new-intune-built-in-role-endpoint-security-manager--4253397-idready--"></a>Новый диспетчер безопасности конечной точки встроенной роли Intune<!--4253397 idready-->
Будет доступна новая встроенная роль Intune: Диспетчер безопасности конечной точки. Эта новая роль дает администраторам полный доступ к узлу диспетчера конечных точек в Intune и предоставляет доступ только к другим областям. Роль является расширением роли "администратор безопасности" из Azure AD. Если в настоящее время у вас есть только глобальные администраторы в качестве ролей, то никаких изменений не требуется. Если вы используете роли и хотите точно задать уровень детализации, предоставляемый диспетчером безопасности конечных точек, назначьте эту роль, когда она будет доступна. Дополнительные сведения о встроенных ролях см. в разделе [Управление доступом на основе ролей](role-based-access-control.md).

### <a name="intune-roles-user-interface-changes-coming--5801612-idready--"></a>Ожидается изменение пользовательского интерфейса ролей Intune<!--5801612 idready-->
Пользовательский интерфейс [центра администрирования Microsoft Endpoint Manager](https://go.microsoft.com/fwlink/?linkid=2109431) > **администрирования клиентов** > **роли** будут изменяться на более понятную и интуитивно понятную структуру. Этот интерфейс предоставляет те же параметры и сведения, которые вы используете сейчас, но в новом интерфейсе используется процесс, аналогичный мастеру.

## <a name="security"></a>Безопасность

### <a name="derived-credentials-support-on-android-cobo-devices--4839592--"></a>Поддержка производных учетных данных на устройствах Android Кобо<!--4839592-->
Вы сможете использовать полученные учетные данные на полностью управляемых устройствах Android Enterprise. Поддержка будет включена для получения производных учетных данных для Entrust Datacard, Интерцеде и DISA Пуребред. Вы сможете использовать производные учетные данные для проверки подлинности приложения, Wi-Fi, VPN или S/MIME, а также для шифрования с приложениями, которые его поддерживают.
