## <a name="app-management"></a>Управление приложениями

### <a name="ios-user-licensed-vpp-apps---5619268-idready---"></a>приложения VPP с лицензией на пользователя iOS<!-- 5619268 idready -->
Для устройств iOS, предназначенных для регистрации пользователей, конечные пользователи больше не будут представлены с лицензированными для устройств приложениями VPP, развернутыми как доступные. Однако конечные пользователи будут по-прежнему видеть все приложения VPP, лицензированные пользователем, в корпоративный портал. См. сведения об [управлении приложениями iOS и macOS, приобретенными по программе Apple Volume Purchase Program, с помощью Microsoft Intune](~/apps/vpp-apps-ios.md).

### <a name="retrieve-personal-recovery-key-from-mem-encrypted-macos-devices---4851745-idready---"></a>Получение личного ключа восстановления из MEM зашифрованных устройств macOS<!-- 4851745 idready -->
Конечные пользователи смогут получить свой личный ключ восстановления (ключ FileVault) с помощью приложения iOS корпоративный портал. Устройство с личным ключом восстановления должно быть зарегистрировано в Intune и зашифровано с помощью FileVault через Intune. Используя приложение корпоративный портал iOS, конечный пользователь может открыть веб-представление Safari и получить свой личный ключ восстановления. В Intune выберите **устройства** > *зашифрованное и зарегистрированное устройство macOS* > **получить ключ восстановления**. Дополнительные сведения о FileVault см. в разделе [Шифрование FileVault для macOS](~/protect/encrypt-devices.md#filevault-encryption-for-macos).

### <a name="microsoft-app-icons-update--4677605--"></a>Обновление значков приложений Майкрософт<!--4677605-->
Будут обновлены значки, используемые для приложений Майкрософт на панели назначения приложений для политик защиты приложений и политик конфигурации приложений.

### <a name="smime-support-for-microsoft-outlook-mobile---2669398----"></a>Поддержка S/MIME для Microsoft Outlook Mobile<!-- 2669398  -->
Intune будет поддерживать доставку сертификатов подписывания и шифрования S/MIME, которые можно использовать с Outlook Mobile на iOS и Android. Дополнительные сведения см. в разделе [Параметры электронной почты для устройств iOS](~/configuration/email-settings-ios.md) и [Параметры электронной почты для устройств Android](~/configuration/email-settings-android.md).

### <a name="custom-settings-support-for-macos-applications---4736278----"></a>Поддержка пользовательских параметров для приложений macOS<!-- 4736278  -->
Intune будет поддерживать пользовательские параметры, позволяя добавлять определенные ключи и значения в существующий файл списка свойств (. plist) для настройки приложений macOS и устройства. Не все приложения поддерживают управляемые настройки, и в некоторых случаях можно управлять только определенными параметрами. Параметры развертываются только через канал устройства. Следует отправлять только файлы списка свойств или XML-файлы, предназначенные для параметров канала устройства.

### <a name="display-notifications-for-the-company-portal-app-on-windows---1808082----"></a>Отображение уведомлений для приложения корпоративный портал в Windows<!-- 1808082  -->
Мы будем обновлять приложение корпоративный портал на устройствах Windows, чтобы отображать всплывающие уведомления для пользователей, даже если приложение закрыто. Обновление будет показывать уведомления для доступных приложений только в случае, если состояние установки — "завершено" или "сбой". Приложение корпоративный портал не будет показывать уведомления для требуемых приложений.

### <a name="display-installation-status-messages-for-the-company-portal-app---2514416----"></a>Отображение сообщений о состоянии установки для приложения корпоративный портал<!-- 2514416  -->
Приложение корпоративный портал отобразит дополнительные сообщения о состоянии установки приложения для конечных пользователей. Следующие условия будут применены к новым функциям зависимости Win32.
- Приложение не удалось установить. Зависимости, определенные администратором, не выполнены.

### <a name="configure-app-notification-content-for-organization-accounts---2576686---"></a>Настройка содержимого уведомлений приложения для учетных записей Организации<!-- 2576686 -->
ПРИЛОЖЕНИЕ Intune на устройствах Android и iOS позволит управлять содержимым уведомлений приложения для учетных записей организации. Эта функция требует поддержки приложений и может быть недоступна для всех приложений с поддержкой приложений. Дополнительные сведения о приложении см. в статье [Общие сведения о политиках защиты приложений](../apps/app-protection-policy.md).

## <a name="device-configuration"></a>Конфигурация устройства

### <a name="block-users-from-configuring-certificate-credentials-in-the-managed-keystore-on-android-enterprise-device-owner-devices---3311998-idready---"></a>Запретить пользователям настраивать учетные данные сертификатов в управляемых хранилищах ключей на устройствах с владельцами устройств Android Enterprise<!-- 3311998 idready -->
На устройствах с владельцами устройств Android Enterprise есть новый параметр, запрещающий пользователям настраивать свои учетные данные сертификатов в управляемых хранилищах ключей (**Конфигурация устройства** > **Профили** > **создать профиль** > **Android Enterprise** для платформы > **Владелец устройства только > ограничения на устройства** для типа профиля > **Пользователи и учетные записи**).

Текущий список параметров устройств Android Enterprise, которые разрешают или ограничивают определенные функции при использовании Intune, см. [здесь](../configuration/device-restrictions-android-for-work.md).

Область применения:
- Владелец корпоративного устройства Android, включая выделенные и полностью управляемые устройства

### <a name="wired-network-device-configuration-profiles-for-macos-devices---3508686-idready---"></a>Профили конфигурации проводных сетевых устройств для устройств macOS<!-- 3508686 idready -->
На устройствах macOS в будущем будет включен новый профиль конфигурации устройства, который настраивает проводные сети (**Конфигурация устройства** > **Профили** > **создать профиль** > **macOS** для платформы > **проводной сети** для типа профиля). Эта функция предназначена для создания профилей 802.1 x для управления проводных сетей и развертывания этих проводных сетей на устройствах macOS.

Область применения:
- macOS

### <a name="add-automatic-proxy-settings-to-wi-fi-profiles-for-android-enterprise-work-profiles---4490822-idready---"></a>Добавление автоматических параметров прокси-сервера в профили Wi-Fi для рабочих профилей Android Enterprise<!-- 4490822 idready -->
На устройствах с рабочими профилями Android Enterprise можно создавать профили Wi-Fi. При выборе типа предприятия Wi-Fi можно также ввести тип протокола EAP, используемый в сети Wi-Fi.

В будущем обновлении при выборе типа "Корпоративный" вы сможете вводить автоматические параметры прокси-сервера, в том числе URL-адрес прокси, например `proxy.contoso.com`.

Чтобы просмотреть текущие параметры Wi-Fi, которые можно настроить, перейдите к статье [Добавление параметров Wi-Fi для устройств под управлением Android Enterprise и Android в Microsoft Intune](../configuration/wi-fi-settings-android-enterprise.md).

Область применения:
- рабочий профиль "Android для бизнеса".

### <a name="enable-network-access-control-nac-with-cisco-anyconnect-vpn-on-ios-devices---4860111-idready---"></a>Включение контроля доступа к сети (NAC) с помощью VPN Cisco AnyConnect на устройствах iOS<!-- 4860111 idready -->
На устройствах iOS можно создать профиль VPN и использовать другие типы подключений, включая Cisco AnyConnect ( **Профили** **конфигурации устройств** >  > **создать профиль** > **iOS** для платформы > **VPN** для типа профиля > **Cisco AnyConnect** для типа подключения). 

В будущих обновлениях Вы сможете включить контроль доступа к сети (NAC) с помощью Cisco AnyConnect. Чтобы использовать эту функцию, сделайте следующее:

1. В [Руководстве администратора по ядру служб удостоверений Cisco](https://www.cisco.com/c/en/us/td/docs/security/ise/2-1/admin_guide/b_ise_admin_guide_21/b_ise_admin_guide_20_chapter_01000.html)выполните действия, описанные в разделе **Настройка Microsoft Intune в качестве сервера MDM** для настройки подсистемы служб удостоверений Cisco (ISE) в Azure.
2. В профиле конфигурации устройства Intune выберите параметр **включить контроль доступа к сети (NAC)** .

Чтобы просмотреть все доступные параметры VPN, перейдите на страницу [Настройка параметров VPN на устройствах iOS](../configuration/vpn-settings-ios.md).

Область применения:
- iOS

### <a name="updated-single-sign-on-experience-for-apps-and-websites-on-your-ios-ipados-and-macos-devices---4999578-idready---"></a>Обновлен интерфейс единого входа для приложений и веб-сайтов на устройствах iOS, Ипадос и macOS.<!-- 4999578 idready -->
Intune добавляет дополнительные параметры единого входа для устройств iOS, Ипадос и macOS. В настоящее время вы можете настроить расширения приложения единого входа учетных данных и встроенное расширение Kerberos Apple в Intune. В следующем обновлении вы сможете настроить расширения приложения для единого входа, созданные вашей организацией или поставщиком удостоверений. 

Используйте эти параметры для настройки простого единого входа для приложений и веб-сайтов, использующих современные методы проверки подлинности, такие как OAuth и SAML2. 

Чтобы просмотреть параметры расширения приложения SSO, которые можно настроить, перейдите на страницу [SSO в iOS](../configuration/ios-device-features-settings.md#single-sign-on-app-extension) и [единый вход на macOS](../configuration/macos-device-features-settings.md#single-sign-on-app-extension).

Область применения:
- iOS/iPadOS
- macOS

### <a name="require-use-of-approved-keyboards-on-android--4761794-idready---"></a>Требовать использования утвержденных клавиатур на Android<!--4761794 IDready -->
Вы сможете указать список утвержденных клавиатур для использования в управляемых приложениях Android. В управляемом приложении пользователю будет предложено переключиться на одну из утвержденных клавиатур, уже установленных на устройстве, или при необходимости они будут направлены на Google Play Маркет для скачивания и настройки одной из утвержденных клавиатур. Пользователь сможет изменять текстовые поля только в управляемом приложении, если их активная клавиатура является одной из утвержденных клавиатур.

### <a name="use-pkcs-certificates-with-wi-fi-profiles-on-windows-10-and-later-devices---3246388----"></a>Использование сертификатов PKCS с профилями Wi-Fi на устройствах Windows 10 и более поздних версий<!-- 3246388  -->
Сейчас можно проверить подлинность профилей Wi-Fi Windows с сертификатами SCEP (**Конфигурация устройства** > **Профили** > **создать профиль** > **Windows 10 и более поздних версий** для платформы > **Wi-Fi** для типа профиля > **Enterprise** > **тип EAP**). Вы сможете использовать сертификаты PKCS с профилями Wi-Fi Windows. Эта функция позволяет пользователям проходить проверку подлинности профилей Wi-Fi с помощью новых или существующих профилей сертификатов PKCS в клиенте. 

Дополнительные сведения о профилях Wi-Fi см. [в статье Добавление параметров Wi-Fi для устройств Windows 10 и более поздних версий в Intune](../configuration/wi-fi-settings-windows.md).

Область применения:
- Windows 10 и более поздней версии

### <a name="new-exchangeactivesync-settings-when-creating-an-email-device-configuration-profile-on-ios-devices---4892824----"></a>Новые параметры Exchange ActiveSync при создании профиля конфигурации электронной почты на устройствах iOS<!-- 4892824  --> 
На устройствах iOS/iPadOS можно настроить возможность подключения по электронной почте в профиле конфигурации устройства (**Конфигурация устройств** > **Профили** > **Создать профиль** > **iOS/iPadOS** (платформа) > **Электронная почта** (тип профиля)). 

Будут доступны новые параметры Ексчанжеактивесинк, включая:
- Выберите службы для синхронизации (или блокировать синхронизацию), например "Электронная почта", "Календарь" и "Контакты".
- Разрешите (или запретите) пользователям изменять параметры синхронизации для этих служб на своих устройствах. 

Чтобы просмотреть текущие параметры, перейдите к разделу [Параметры профиля электронной почты для устройств iOS в Intune](../configuration/email-settings-ios.md).

Область применения:
- iOS 13.0 и более поздние версии.
- iPadOS 13.0 и более поздние версии.

### <a name="prevent-users-from-adding-personal-google-accounts-to-android-enterprise-device-owner-and-dedicated-devices---5353228----"></a>Запрет пользователям добавлять личные учетные записи Google для владельцев устройств и выделенных устройств Android для бизнеса<!-- 5353228  -->
Вы сможете запретить пользователям создавать персональные учетные записи Google для владельцев устройств и выделенных устройств Android для бизнеса. Для этого выберите **Конфигурация устройства** > **Профили** > **Создать профиль** > **Android для бизнеса** (с учетом платформы) > **Только для владельцев устройств > Ограничения устройств** > **Параметры пользователей и учетных записей**.

Полный список текущих параметров, которые вы можете настроить, см. в описании [параметров устройств Android для бизнеса, разрешающих или ограничивающих определенные функции Intune](../configuration/device-restrictions-android-for-work.md).

Область применения:
- владельцы устройств Android Enterprise
- Выделенные устройства Android для бизнеса

### <a name="server-side-logging-for-siri-commands-setting-is-removed-in-ios-device-restrictions-profile---5468501----"></a>Параметр ведения журнала на стороне сервера для команд Siri удален в профиле ограничений устройств iOS<!-- 5468501  -->
На устройствах iOS можно создать профили ограничений для устройств, которые настраивают ведение журнала на стороне сервера для команд Siri (**Конфигурация устройства** > **Профили** > **создания профиля** > **iOS/Ипадос** для > **устройств** типа профиля > **встроенных приложений**). Параметр **ведения журнала на стороне сервера для команд Siri** будет удален.

Этот параметр будет удален из консоли администрирования Intune. Этот параметр не оказывает влияния на устройство, хотя существующие политики, для которых настроен этот параметр, будут по прежнему отображаться. Если вы хотите удалить параметр из существующих политик, перейдите к политике, внесите небольшие изменения, сохраните ее, и политика будет обновлена.

Полный список параметров, которые вы можете настроить, см. в описании [параметров устройств iOS и iPadOS, разрешающих или ограничивающих определенные функции Intune](../configuration/device-restrictions-ios.md).

Область применения:
- iOS

## <a name="security"></a>Безопасность

### <a name="use-pkcs-certificate-profiles-to-provision-devices-with-certificates---2317124-2317130-2317139-2340517-2340528-2340529-idready---"></a>Использование профилей сертификатов PKCS для инициализации устройств с помощью сертификатов<!-- 2317124, 2317130, 2317139, 2340517, 2340528, 2340529 IDready -->
Вы сможете использовать профиль сертификата PKCS, чтобы выдавать сертификаты устройствам, расширяя текущую поддержку сертификатов на основе пользователей. Сертификаты на основе устройств будут поддерживаться на платформах Android, iOS и Windows, и их можно будет использовать для профилей Wi-Fi и VPN.
