## <a name="week-of-september-11-2017"></a>Неделя, начиная с 11 сентября 2017 г.

### <a name="device-enrollment"></a>Регистрация устройств

#### <a name="additional-push-notifications-for-end-users-on-the-company-portal-app-for-android-o----1475932---"></a>Дополнительные push-уведомления для конечных пользователей в приложении корпоративного портала для Android O <!---1475932--->

Конечные пользователи будут видеть дополнительные уведомления, информирующие о выполнении приложением корпоративного портала для Android O фоновых задач, таких как извлечение политик из службы Intune. Такой повышенный уровень прозрачности помогает конечным пользователям определять, когда корпоративный портал выполняет административные задачи на их устройствах. Эта возможность является частью общего процесса [оптимизации пользовательского интерфейса корпоративного портала](https://blogs.technet.microsoft.com/intunesupport/2017/08/21/android-8-0-o-behaviour-changes-and-microsoft-intune) для приложения корпоративного портала для Android O. 

#### <a name="inform-end-users-what-device-information-can-be-seen-for-ios---739894--"></a>Информирование конечных пользователей о том, какие сведения можно увидеть об устройстве iOS <!--739894--> 

Мы добавили **тип владения** на экран сведений об устройстве в приложении корпоративного портала для iOS. Это позволит пользователям получать дополнительные сведения о конфиденциальности непосредственно с этой страницы из документации для конечных пользователей Intune. Они также смогут найти эти сведения на экране "О программе". 

#### <a name="allow-end-users-to-access-the-company-portal-app-for-android-without-enrollment----1169910---"></a>Разрешение конечным пользователям доступа к приложению корпоративного портала для Android без регистрации <!---1169910--->

Скоро конечным пользователям не придется регистрировать свои устройства, чтобы получить доступ к приложению корпоративного портала для Android. Пользователи в организациях, где применяются политики защиты приложений, больше не будут получать запросы на регистрацию при открытии приложения корпоративного портала. Конечные пользователи также смогут устанавливать приложения с корпоративного портала без регистрации устройства. 


#### <a name="easier-to-understand-phrasing-for-the-company-portal-app-for-android----1396349---"></a>В приложении корпоративного портала для Android текст стал более понятным <!---1396349--->.  

Упрощен процесс регистрации пользователей в приложении корпоративного портала для Android благодаря обновленному тексту. Если у вас есть документация по пользовательской регистрации, необходимо обновить ее, чтобы отразить новые экраны. Примеры можно найти на странице [Обновление элементов пользовательского интерфейса для приложения конечного пользователя Intune](whats-new-app-ui.md#week-of-september-11-2017).

#### <a name="windows-10-company-portal-app-added-to-windows-information-protection-allow-policy----677129---"></a>Добавление приложения корпоративного портала Windows 10 в политику разрешения Windows Information Protection <!-- 677129 -->

Приложение корпоративного портала Windows 10 обновлено для поддержки Windows Information Protection (WIP). Это приложение можно будет добавить в политику разрешения Windows Information Protection. После этого изменения вам больше не потребуется добавлять приложение в список **исключений**.

### <a name="new-setting-in-the-android-company-portal-app-to-toggle-battery-optimization---1405990--"></a>Новый параметр в приложении корпоративного портала Android для включения или отключения оптимизации батареи <!--1405990-->

### <a name="use-azure-active-directory-for-conditional-access----967947---"></a>Условный доступ с помощью Azure Active Directory <!-- 967947 -->
В разделе Azure Active Directory на портале Azure реализована возможность условного доступа. Благодаря этому пользователи получают более функциональную и гибкую платформу для настройки политик для таких облачных приложений, как Office 365 Exchange Online и SharePoint Online.  Теперь вместо консоли Intune можно использовать колонку **Условный доступ в Azure Active Directory**. Существующие в консоли Intune политики необходимо заново создать на портале Azure. Дополнительные сведения см. в разделе [Создание политик условного доступа в Azure AD](/intune/conditional-access-exchange-create.md#create-azure-ad-conditional-access-policies-in-intune-azure-preview).

### <a name="direct-access-to-apple-enrollment-scenarios---951869--"></a>Прямой доступ к сценариям регистрации Apple <!--951869-->
Для учетных записей Intune, созданных после января 2017 г., включен прямой доступ к сценариям регистрации Apple с помощью рабочей нагрузки регистрации устройств на портале Azure. Ранее доступ к предварительной версии регистрации Apple можно было получить, только перейдя по ссылкам на классическом портале Intune. Учетные записи Intune, созданные до января 2017 г., требуется перенести (единожды), перед тем как эти возможности станут доступны в Azure. Расписание переноса еще не объявлено, но мы предоставим сведения о нем в кратчайшие сроки. Если вам не удалось получить доступ к порталу Azure, используя имеющуюся учетную запись, мы настоятельно рекомендуем создать пробную учетную запись, чтобы протестировать новые возможности.

### <a name="administration-roles-being-replaced-in-azure-portal"></a>Замена ролей администрирования на портале Azure
Существующие роли администрирования для управления мобильными приложениями (MAM) (участник, владелец и только для чтения), используемые на классическом портале Intune (Silverlight), заменяются набором новых элементов управления администрированием на основе ролей (RBAC) на портале Intune Azure. После миграции на портал Azure вам понадобится повторно назначить администраторам эти новые роли администрирования. Дополнительные сведения об элементах управления администрированием на основе ролей и новых ролях см. в статье [Роли Intune (RBAC) для Microsoft Intune](/intune/role-based-access-control).



## <a name="whats-coming"></a>Предстоящие изменения

#### <a name="ios-11-mail-app-will-support-oauth----1196951---"></a>Почтовое приложение iOS 11 будет поддерживать OAuth <!---1196951--->
Условный доступ с помощью Intune поддерживает более надежную проверку подлинности для устройств iOS с помощью OAuth. Для этого в приложении корпоративного портала для iOS теперь создается другой поток, обеспечивающий более надежную проверку подлинности. При попытке конечных пользователей выполнить вход в новую учетную запись Exchange в приложении "Почта" они увидят запрос в веб-интерфейсе. После регистрации в Intune пользователи получат запрос на предоставление встроенному приложению "Почта" доступа к сертификату. Большинство конечных пользователей больше не увидит помещенные в карантин сообщения электронной почты. Существующие почтовые учетные записи продолжат использовать обычный протокол проверки подлинности, поэтому такие пользователи по-прежнему будут получать сообщения электронной почты, помещенные в карантин. Этот интерфейс входа для конечных пользователей напоминает мобильные приложения Office.

### <a name="end-of-support-for-ios-80----1164477---"></a>Окончание поддержки iOS 8.0 <!---1164477--->
Для доступа к корпоративным ресурсам управляемым приложениям и приложению корпоративного портала для iOS потребуется операционная система iOS 9.0 или более поздней версии. Устройства, которые не будут обновлены до сентября этого года, не смогут осуществлять доступ к указанным приложениям и корпоративному порталу.

### <a name="end-of-support-for-android-43-and-lower----1171127-1326920----"></a>Окончание поддержки Android 4.3 и более ранних версий <!---1171127, 1326920 --->
Для доступа к корпоративным ресурсам управляемым приложениям и приложению корпоративного портала для Android потребуется Android 4.4 или более поздней версии. Устройства, которые не будут обновлены до начала октября, не смогут осуществлять доступ к указанным приложениям и корпоративному порталу. В декабре все зарегистрированные устройства будут принудительно сняты с учета, что приведет к утрате доступа к корпоративным ресурсам. Если вы используете политики защиты приложений без MDM, приложения не будут получать обновления, а качество взаимодействия с ними будет постепенно ухудшаться.


### <a name="platform-support-reminder-windows-phone-81-mainstream-support-ended-july-11-2017"></a>Напоминание о поддержке платформ: основная фаза поддержки Windows Phone 8.1 завершилась 11 июля 2017 г.
<!-- 1327781 -->
11 июля 2017 г. закончилась основная фаза поддержки платформы Windows Phone 8.1. Поддержка настольных систем на базе Windows 8.1 затронута не будет.

Это не окажет немедленное влияние на все устройства Windows Phone 8.1, управляемые службой Intune. Зарегистрированные устройства, как и все политики, конфигурации и приложения, продолжат работать должным образом. Обратите внимание на отсутствие усовершенствований для платформы Windows Phone 8.1 в службе Intune, а также для приложения корпоративного портала Windows Phone 8.1.

Рекомендуется при первой возможности обновить соответствующие устройства Windows Phone 8.1 до Windows 10 Mobile. 

### <a name="changes-in-support-for-the-intune-ios-company-portal-app-----1164474----"></a>Изменения в поддержке для приложения корпоративного портала Intune для iOS <!-- 1164474  -->
Скоро будет выпущена новая версия приложения корпоративного портала Microsoft Intune для iOS. Она будет поддерживать только устройства под управлением iOS 9.0 или более поздней версии. Версия корпоративного портала, поддерживающая iOS 8, останется доступной в течение очень короткого периода. Тем не менее, обратите внимание, что при использовании приложений iOS с поддержкой MAM мы поддерживаем iOS 9.0 и более поздней версии, поэтому вам нужно проследить, чтобы ваши конечные пользователи перешли на самую новую версию ОС. 

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Мы сообщая об этом заранее, но конкретные даты еще не установлены, поэтому у вас есть время на планирование. Проследите, что ваши пользователи обновились до iOS 9+, а после выхода приложения корпоративного портала попросите пользователей обновить приложение корпоративного портала.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Рекомендуйте пользователям обновиться до iOS 9.0 или более поздней версии, чтобы они могли воспользоваться всеми преимуществами новых функций Intune.  Рекомендуйте пользователям установить новую версию корпоративного портала и использовать предоставляемые ей преимущества.

Перейдите в Intune на портале Azure, откройте "Устройства" > "Все устройства" и отфильтруйте по версии iOS, чтобы просмотреть все текущие устройства с операционной системой, предшествующей iOS 9.


### <a name="apple-to-require-updates-for-application-transport-security---748318--"></a>Требуемые обновления Application Transport Security от Apple <!--748318-->
Компания Apple объявила, что к Application Transport Security (ATS) будут применяться определенные требования. ATS позволяет обеспечить высокий уровень безопасности при передаче данных в приложениях по протоколу HTTPS. Это изменение затрагивает клиентов Intune, которые используют приложения корпоративного портала для iOS.

Мы выпустили доступную версию приложения Корпоративного портала для iOS, созданную с помощью программы Apple TestFlight, в которой реализованы новые требования к ATS. Если вы хотите оценить ее и проверить соответствие требованиям ATS, отправьте письмо по адресу <a href="mailto:CompanyPortalBeta@microsoft.com?subject=Register to TestFlight ATS Company Portal app">CompanyPortalBeta@microsoft.com</a> со своим именем, фамилией, адресом электронной почты и названием организации. Дополнительные сведения см. в [блоге службы поддержки Intune](https://aka.ms/compportalats).
