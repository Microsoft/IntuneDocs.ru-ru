### <a name="device-enrollment"></a>Регистрация устройств  

#### <a name="restrict-android-and-ios-device-enrollment-restriction-by-os-version------1333256--1245463----"></a>Ограничения регистрации устройств Android и iOS по версии ОС <!--- 1333256,  1245463 --->
Intune поддерживает ограничение регистрации устройств Android и iOS по номеру версии операционной системы. В разделе **Ограничение по типу устройств** ИТ-администратор теперь может задавать конфигурации платформ, позволяющие ограничить доступные для регистрации операционные системы минимальным и максимальным номером версии. Версии ОС Android должны задаваться в виде строки "Основная.Вспомогательная.Сборка.Редакция", в которой атрибуты вспомогательной версии, сборки и редакции являются необязательными. Версии iOS должны задаваться в виде строки "Основная.Вспомогательная.Сборка", где атрибуты вспомогательной версии и сборки являются необязательными. Дополнительные сведения об [ограничениях на регистрацию устройств](enrollment-restrictions-set.md).

>[!NOTE]
>Не ограничивайте регистрацию с помощью программы регистрации Apple или Apple Configurator.

#### <a name="restrict-android-ios-and-macos-device-personally-owned-device-enrollment------1333272--1333275-1245709----"></a>Ограничение регистрации личных устройств с Android, iOS и macOS <!--- 1333272,  1333275, 1245709 --->
Intune может ограничивать регистрацию личного устройства, применяя утвержденные номера IMEI корпоративных устройств. Эта функциональность Intune расширена для iOS, Android и macOS, где используются серийные номера устройств. Отправив серийные номера в Intune, можно предварительно объявить устройства как корпоративные. С помощью ограничений регистрации вы можете заблокировать личные устройства (BYOD), разрешив только регистрацию корпоративных устройств. Дополнительные сведения об [ограничениях на регистрацию устройств](enrollment-restrictions-set.md).

Чтобы импортировать серийные номера, перейдите в раздел **Регистрация устройства** > **Идентификаторы корпоративных устройств** и нажмите кнопку **Добавить**, а затем отправьте CSV-файл (без заголовка, два столбца для серийных номеров и таких сведений, как номера IMEI).  Чтобы ограничить личные устройства, перейдите в раздел **Регистрация устройства** > **Ограничения регистрации**. В области **Ограничения по типу устройств** выберите **По умолчанию**, а затем **Конфигурации платформ**. Вы можете **Разрешить** или **Блокировать** личные устройства на базе iOS, Android и macOS. 


### <a name="device-management"></a>Управление устройствами   

#### <a name="new-device-action-to-force-devices-to-sync-with-intune----711369---"></a>Новое действие для принудительной синхронизации устройств с Intune <!-- 711369 -->
В этом выпуске мы добавили новое действие, которое заставляет выбранное устройство немедленно выполнить возврат в Intune. При возврате устройство сразу же получает все ожидающие действия или политики, которые были ему назначены.  Это действие позволяет сразу проверять назначенные политики и устранять возникшие с ними неполадки, не дожидаясь следующего запланированного возврата.
Дополнительные сведения см. в статье [Синхронизация устройств](device-sync.md).

#### <a name="force-supervised-ios-devices-to-automatically-install-the-latest-available-software-update----777100---"></a>Принудительно защищенные устройства iOS для автоматической установки последнего доступного обновления программного обеспечения <!-- 777100 -->
Новая политика доступна в рабочей области "Обновления программного обеспечения", где вы можете принудительно защитить устройства iOS для автоматической установки последнего доступного обновления программного обеспечения. Дополнительные сведения см. в статье [Настройка политик обновления iOS](/intune/software-updates-ios).

#### <a name="check-point-sandblast-mobile---new-mobile-threat-defense-partner-----954651-1172027---"></a>Check Point SandBlast Mobile — новый партнер Mobile Threat Defense <!-- 954651, 1172027 -->
Вы можете управлять доступом к корпоративным ресурсам с мобильных устройств посредством условного доступа на основании оценки рисков, проведенной Checkpoint SandBlast Mobile — решением для защиты от угроз на мобильных устройствах, интегрированным с Microsoft Intune.

##### <a name="how-integration-with-intune-works"></a>Принцип работы интеграции с Intune
Оценка рисков основана на данных телеметрии, собранных с устройств, на которых выполняется приложение Checkpoint SandBlast Mobile. Вы можете настроить политики условного доступа EMS на основе оценки рисков, выполненной Checkpoint SandBlast Mobile с применением политик соответствия устройств Intune. Вы можете разрешить или запретить доступ несоответствующих устройств к корпоративным ресурсам в зависимости от обнаруженных угроз.

### <a name="use-azure-active-directory-for-conditional-access----967947---"></a>Условный доступ с помощью Azure Active Directory <!-- 967947 -->
В разделе Azure Active Directory консоли Azure реализована возможность условного доступа, что позволяет сформировать более функциональную и гибкую платформу для настройки политик для таких облачных приложений, как Office 365 Exchange Online и SharePoint Online.  Теперь вместо классической консоли Intune можно использовать колонку **Условный доступ в Azure Active Directory**. Существующие в классической консоли Intune политики необходимо заново создать в консоли Azure. Дополнительные сведения см. в разделе [Создание политик условного доступа к Azure AD](/intune/conditional-access-exchange-create.md#create-azure-ad-conditional-access-policies-in-intune-azure-preview)

### <a name="direct-access-to-apple-enrollment-scenarios---951869--"></a>Прямой доступ к сценариям регистрации Apple <!--951869-->
Для учетных записей Intune, созданных после января 2017 г., включен прямой доступ к сценариям регистрации Apple с помощью рабочей нагрузки регистрации устройств на портале Azure. Ранее доступ к предварительной версии регистрации Apple можно было получить, только перейдя по ссылкам на классическом портале Intune. Учетные записи Intune, созданные до января 2017 г., требуется перенести (единожды), перед тем как эти возможности станут доступны в Azure. Расписание переноса еще не объявлено, но мы предоставим сведения о нем в кратчайшие сроки. Если вам не удалось получить доступ к порталу Azure, используя имеющуюся учетную запись, мы настоятельно рекомендуем создать пробную учетную запись, чтобы протестировать новые возможности.

### <a name="administration-roles-being-replaced-in-azure-portal"></a>Замена ролей администрирования на портале Azure
Существующие роли администрирования для управления мобильными приложениями (MAM) (участник, владелец и только для чтения), используемые на классическом портале Intune (Silverlight), заменяются набором новых элементов управления администрированием на основе ролей (RBAC) на портале Intune Azure. После миграции на портал Azure вам понадобится повторно назначить администраторам эти новые роли администрирования. Дополнительные сведения об элементах управления администрированием на основе ролей и новых ролях см. в статье [Роли Intune (RBAC) для Microsoft Intune](/intune/role-based-access-control).

### <a name="platform-support-reminder-windows-phone-81-mainstream-support-ended-july-11-2017"></a>Напоминание о поддержке платформ: основная фаза поддержки Windows Phone 8.1 завершилась 11 июля 2017 г.

Это не окажет немедленное влияние на все устройства Windows Phone 8.1, управляемые службой Intune. Скоро будет выпущена новая версия приложения корпоративного портала Microsoft Intune для iOS. Она будет поддерживать только устройства под управлением iOS 9.0 или более поздней версии. Версия корпоративного портала, поддерживающая iOS 8, останется доступной в течение очень короткого периода. Тем не менее, обратите внимание, что при использовании приложений iOS с поддержкой MAM мы поддерживаем iOS 9.0 и более поздней версии, поэтому вам нужно проследить, чтобы ваши конечные пользователи перешли на самую новую версию ОС. 

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Мы сообщая об этом заранее, но конкретные даты еще не установлены, поэтому у вас есть время на планирование. Проследите, что ваши пользователи обновились до iOS 9+, а после выхода приложения корпоративного портала попросите пользователей обновить приложение корпоративного портала.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Рекомендуйте пользователям обновиться до iOS 9.0 или более поздней версии, чтобы они могли воспользоваться всеми преимуществами новых функций Intune.  Рекомендуйте пользователям установить новую версию корпоративного портала и использовать предоставляемые ей преимущества.

Перейдите в Intune на портале Azure, откройте "Устройства" > "Все устройства" и отфильтруйте по версии iOS, чтобы просмотреть все текущие устройства с операционной системой, предшествующей iOS 9.


### <a name="apple-to-require-updates-for-application-transport-security---748318--"></a>Требуемые обновления Application Transport Security от Apple <!--748318-->
Компания Apple объявила, что к Application Transport Security (ATS) будут применяться определенные требования. ATS позволяет обеспечить высокий уровень безопасности при передаче данных в приложениях по протоколу HTTPS. Это изменение затрагивает клиентов Intune, которые используют приложения корпоративного портала для iOS.

Мы выпустили доступную версию приложения Корпоративного портала для iOS, созданную с помощью программы Apple TestFlight, в которой реализованы новые требования к ATS. Если вы хотите оценить ее и проверить соответствие требованиям ATS, отправьте письмо по адресу <a href="mailto:CompanyPortalBeta@microsoft.com?subject=Register to TestFlight ATS Company Portal app">CompanyPortalBeta@microsoft.com</a> со своим именем, фамилией, адресом электронной почты и названием организации. Дополнительные сведения см. в [блоге службы поддержки Intune](https://aka.ms/compportalats).
