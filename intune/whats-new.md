### <a name="device-enrollment"></a>Регистрация устройств
#### <a name="windows-autopilot-deployment-program-support-in-microsoft-intune-----747617----"></a>Поддержка программы Windows AutoPilot Deployment в Microsoft Intune <!-- 747617  -->
Теперь вы можете использовать программу Windows AutoPilot Deployment в Microsoft Intune, чтобы дать пользователям возможность подготавливать свои корпоративные устройства, не прибегая к помощи ИТ-отдела. Вы можете настроить интерфейс запуска при первом включении (OOBE) и предоставить пользователям указания по присоединению устройств к Azure AD и регистрации в Intune. Совместное использование Microsoft Intune и Windows AutoPilot устраняет необходимость в развертывании образов операционных систем, их обслуживании и управлении ими. Подробные сведения см. в статье [Регистрация устройств Windows с помощью программы Windows AutoPilot Deployment](https://docs.microsoft.com/intune/enrollment-autopilot).

#### <a name="quick-start-for-device-enrollment-----1425655---"></a>Краткое руководство по регистрации устройств <!-- 1425655 --> 
В области **Регистрация устройства** теперь доступно краткое руководство по началу работы и приводится таблица со ссылками на информацию об управлении платформами и настройке процесса регистрации. Краткое описание каждого элемента и ссылки на документацию с пошаговыми инструкциями могут помочь упростить начало работы.

#### <a name="device-categorization----1427491---"></a>Классификация устройств <!-- 1427491 -->
На диаграмме платформ зарегистрированных устройств в колонке **Устройства > Обзор** устройства упорядочены по платформам, включая Android, iOS, macOS, Windows и Windows Mobile.  Устройства с другими операционными системами отнесены к категории "Другие".  Сюда относятся устройства, произведенные компаниями Blackberry, NOKIA и другими.  

Чтобы узнать, какие устройства имеются в вашем клиенте, выберите **Управление > Все устройства**, а затем с помощью **фильтра** ограничьте значения в поле **ОС**.



### <a name="device-management"></a>Управление устройствами
#### <a name="zimperium---new-mobile-threat-defense-partner------954681---"></a>Zimperium — новый партнер Mobile Threat Defense <!-- 954681 -->  
Вы можете управлять доступом к корпоративным ресурсам с мобильных устройств посредством условного доступа на основании оценки рисков, проведенной Zimperium — решением для защиты от угроз на мобильных устройствах, интегрированным с Microsoft Intune.

##### <a name="how-integration-with-intune-works"></a>Принцип работы интеграции с Intune
Оценка рисков основана на данных телеметрии, собранных с устройств, на которых выполняется приложение Zimperium. Вы можете настроить политики условного доступа EMS на основе оценки рисков Zimperium, реализуемой с помощью политик соответствия устройств Intune. Эти политики также можно использовать для разрешения или запрета доступа несоответствующих устройств к корпоративным ресурсам на основании обнаруженных угроз.

#### <a name="new-settings-for-windows-10-device-restriction-profile------978575-1308849---"></a>Новые параметры для профиля ограниченного использования устройств Windows 10 <!--- 978575, 1308849, -->  
Мы добавляем новые параметры в профиль ограниченного использования устройств Windows 10 в категории "SmartScreen Защитника Windows".

Дополнительные сведения о профиле ограниченного использования устройств Windows 10 см. в разделе [Параметры ограничений для применения к устройствам Windows 10 и более поздних версий]( device-restrictions-windows-10.md).

#### <a name="remote-support-for-windows-and-windows-mobile-devices------1070473---"></a>Удаленная поддержка устройств с Windows и Windows Mobile <!-- 1070473 -->  
Благодаря Intune пользователи устройств с ОС Windows и Windows Mobile теперь могут получать от вас удаленную поддержку с помощью программы [TeamViewer](https://www.teamviewer.com) (приобретается отдельно).

#### <a name="scan-devices-with-windows-defender----1280988--1280990-----"></a>Проверка устройств с помощью Защитника Windows <!-- 1280988  1280990   -->
С помощью антивирусной программы "Защитник Windows" вы теперь можете выполнять **быструю проверку** и **полную проверку** устройств с Windows 10, а также **обновлять сигнатуры** на них. В колонке сведений об устройстве выберите действие, которое необходимо выполнить на устройстве. Вам будет предложено подтвердить действие перед отправкой команды на устройство. 

**Быстрая проверка.** Позволяет проверить расположения, где вредоносная программа регистрируется для запуска, например разделы реестра и известные папки запуска Windows. Быстрая проверка в среднем занимает пять минут. При использовании с параметром **Always-on real-time protection** (Постоянная защита в режиме реального времени), который позволяет проверять открытые, закрытые и выбранные в папке файлы, быстрая проверка обеспечит защиту от вредоносных программ в системе или в ядре. По завершении сканирования пользователи увидят его результаты на своих устройствах. 

**Полная проверка.** Полезна для проверок по требованию, а также для устройств, которые подверглись угрозам вредоносных программ. Позволяет выявить все неактивные компоненты, для которых требуется более тщательная очистка. Полная проверка может занять около часа. По завершении сканирования пользователи увидят его результаты на своих устройствах. 

**Update signatures** (Обновление сигнатур). Эта команда обновляет определения и сигнатуры вредоносных программ для антивирусной программы "Защитник Windows". Это гарантирует эффективность антивирусной программы "Защитник Windows" при обнаружении вредоносных программ. Эта функция применима только к устройствам с Windows 10, и для нее требуется подключение к Интернету. 

#### <a name="the-enabledisable-button-is-removed-from-the-intune-certificate-authority-page-of-the-intune-azure-portal-----1400455---"></a>Кнопка "Включить/Отключить" удалена со страницы "Центр сертификации Intune" на портале Intune Azure <!-- 1400455 -->
 Мы удалили лишнее действие при настройке Certificate Connector в Intune. В настоящее время необходимо скачать компонент Certificate Connector, а затем включить его в консоли Intune. Но если вы отключите этот компонент в консоли Intune, он продолжит выдавать сертификаты.

##### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Начиная с октября кнопка "Включить/Отключить" больше не будет отображаться на странице "Центр сертификации" на портале Azure. На работу соединителя это не повлияет. Сертификаты будут по-прежнему развертываться на устройствах, зарегистрированных в Intune. Вы можете скачать и установить Certificate Connector. Чтобы прекратить выдачу сертификатов, теперь необходимо удалить компонент Certificate Connector, а не отключить его.

##### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Если компонент Certificate Connector у вас в настоящее время отключен, его необходимо удалить.

#### <a name="prevent-users-of-android-devices-from-changing-their-device-date-and-time-----1333292---"></a>Запрет на изменение даты и времени на устройстве Android пользователями <!-- 1333292 -->
С помощью [настраиваемой политики для устройства Android](custom-settings-android.md) можно запретить пользователям изменять дату и время на устройстве Android.

Для этого задайте для настраиваемой политики Android параметр URI ./Vendor/MSFT/PolicyManager/My/System/AllowDateTimeChange. Установите для него значение **TRUE** и назначьте этот URI необходимым группам.

#### <a name="bitlocker-device-configuration----1397398---"></a>Настройка BitLocker для устройства <!-- 1397398 -->
В разделе **базовых параметров шифрования Windows** есть новый параметр **Предупреждение при шифровании других дисков**, который позволяет отключить [предупреждение](https://docs.microsoft.com/windows/client-management/mdm/bitlocker-csp#allowarningforotherdiskencryption) при шифровании других дисков, которые могут использоваться на устройстве пользователя.  Это предупреждение требует согласия пользователя на настройку BitLocker на устройстве, и настройка BitLocker блокируется, пока пользователь не подтвердит согласие.  Новый параметр позволяет отключить это предупреждение для пользователя.

Такой повышенный уровень прозрачности помогает конечным пользователям определять, когда корпоративный портал выполняет административные задачи на их устройствах.

#### <a name="new-behaviors-for-the-company-portal-app-for-android-with-work-profiles----1485783---"></a>Новые функции приложения корпоративного портала для Android с рабочими профилями <!---1485783--->

При регистрации устройства Android for Work с использованием рабочего профиля именно приложение корпоративного портала в рабочем профиле выполняет задачи управления на устройстве. 

Не считая приложений с поддержкой MAM в личном профиле, приложение корпоративного портала для Android больше не обслуживает другие задачи использования. Чтобы повысить удобство использования рабочего профиля, Intune автоматически скрывает приложение корпоративного портала в личном профиле после успешной регистрации рабочего профиля.

Приложение корпоративного портала для Android можно включить в любое время в личном профиле. Для этого выполните поиск [корпоративного портала в Магазине Google Play](https://play.google.com/store/apps/details?id=com.microsoft.windowsintune.companyportal) и нажмите кнопку **Enable** (Включить).

#### <a name="company-portal-for-windows-81-and-windows-phone-81-moving-to-sustaining-mode---1428681--"></a>Переход корпоративного портала для Windows 8.1 и Windows Phone 8.1 в режим поддержки<!--1428681-->

Начиная с октября 2017 г. приложения корпоративного портала для Windows 8.1 и Windows Phone 8.1 перейдут в режим поддержки. Это означает, что приложения и существующие сценарии, такие как регистрация и обеспечение соответствия, будут по-прежнему поддерживаться на этих платформах. Эти приложения по-прежнему будут доступны для скачивания через существующие каналы для выпуска, такие как магазин Майкрософт. 

После перехода в режим поддержки эти приложения будут получать только критически важные обновления для системы безопасности. Дополнительные обновления или функции больше не будут выпускаться для этих приложений. Для поддержки новых функций мы рекомендуем установить на устройствах Windows 10 или Windows 10 Mobile. 


### <a name="device-enrollment"></a>Регистрация устройств

#### <a name="block-unsupported-samsung-knox-device-enrollment------1490695----"></a>Блокирование регистрации неподдерживаемых устройств Samsung Knox<!--- 1490695 --->

Приложение "Корпоративный портал" пытается регистрировать только поддерживаемые устройства Samsung Knox. Во избежание ошибок активации KNOX, делающих регистрацию в MDM невозможной, попытка регистрации предпринимается только для устройств, включенных в [список, опубликованный компанией Samsung](https://www.samsungknox.com/knox-supported-devices/knox-workspace). Некоторые модели устройств Samsung поддерживают KNOX, а другие нет. Перед приобретением и развертыванием следует уточнить совместимость устройств с решением Knox у торгового посредника. Полный список проверенных устройств можно найти в статье [Параметры политики Android и Samsung KNOX Standard](/intune/supported-devices-browsers.md#intune-supported-devices).

#### <a name="end-of-support-for-android-43-and-lower----1171126-1326920----"></a>Окончание поддержки Android 4.3 и более ранних версий <!---1171126, 1326920 --->
Для доступа к корпоративным ресурсам управляемым приложениям и приложению корпоративного портала для Android потребуется Android 4.4 или более поздней версии. В декабре все зарегистрированные устройства будут принудительно сняты с учета, что приведет к утрате доступа к корпоративным ресурсам. Если вы используете политики защиты приложений без MDM, приложения не будут получать обновления, а качество взаимодействия с ними будет постепенно ухудшаться.

#### <a name="inform-end-users-what-device-information-can-be-seen-on-enrolled-devices---1165314--"></a>Информирование конечных пользователей о том, какие сведения об устройстве можно увидеть на зарегистрированном устройстве <!--1165314-->
Мы добавляем **тип владения** на экран сведений об устройстве во всех версиях приложения "Корпоративный портал". Это позволит пользователям получать дополнительные сведения о конфиденциальности непосредственно в статье [Какие сведения становятся доступными для моей организации?](/intune-user-help/what-info-can-your-company-see-when-you-enroll-your-device-in-intune). Эта возможность будет развернута во всех версиях приложения "Корпоративный портал" в ближайшем будущем. Для iOS она была анонсирована в [сентябре](https://docs.microsoft.com/intune/whats-new#week-of-september-11-2017).

### <a name="new-path-for-managed-devices-in-graph-api----1586728---"></a>Новый путь к управляемым устройствам в API Graph <!-- 1586728 -->
Мы изменяем путь, использовавшийся для доступа к управляемым устройствам в бета-версии API Graph. 

| | |
|--|--|
| Текущий путь |  https://graph.microsoft.com/beta/managedDevices |
| Новый путь | https://graph.microsoft.com/beta/deviceManagement/managedDevices |

В течение октября будут работать оба пути. После октябрьского выпуска службы будет работать только новый путь.  Если вы используете API Graph для доступа к управляемым устройствам, выполните обновление и проверьте доступность скриптов и приложений с помощью нового пути. Дополнительные изменения см. в ежемесячном [журнале изменений API Graph](https://developer.microsoft.com/graph/docs/concepts/changelog).


### <a name="direct-access-to-apple-enrollment-scenarios---951869--"></a>Прямой доступ к сценариям регистрации Apple <!--951869-->
Для учетных записей Intune, созданных после января 2017 г., включен прямой доступ к сценариям регистрации Apple с помощью рабочей нагрузки регистрации устройств на портале Azure. Ранее доступ к предварительной версии регистрации Apple можно было получить, только перейдя по ссылкам на классическом портале Intune. Учетные записи Intune, созданные до января 2017 г., требуется перенести (единожды), перед тем как эти возможности станут доступны в Azure. Расписание переноса еще не объявлено, но мы предоставим сведения о нем в кратчайшие сроки. Если вам не удалось получить доступ к порталу Azure, используя имеющуюся учетную запись, мы настоятельно рекомендуем создать пробную учетную запись, чтобы протестировать новые возможности.

### <a name="administration-roles-being-replaced-in-azure-portal"></a>Замена ролей администрирования на портале Azure
Существующие роли администрирования для управления мобильными приложениями (MAM) (участник, владелец и только для чтения), используемые на классическом портале Intune (Silverlight), заменяются набором новых элементов управления администрированием на основе ролей (RBAC) на портале Intune Azure. После миграции на портал Azure вам понадобится повторно назначить администраторам эти новые роли администрирования. Дополнительные сведения об элементах управления администрированием на основе ролей и новых ролях см. в статье [Роли Intune (RBAC) для Microsoft Intune](/intune/role-based-access-control).



## <a name="whats-coming"></a>Предстоящие изменения

### <a name="changes-in-support-for-the-intune-ios-company-portal-app-----1164474----"></a>Изменения в поддержке для приложения корпоративного портала Intune для iOS <!-- 1164474  -->
Скоро будет выпущена новая версия приложения корпоративного портала Microsoft Intune для iOS. Она будет поддерживать только устройства под управлением iOS 9.0 или более поздней версии. Версия корпоративного портала, поддерживающая iOS 8, останется доступной в течение очень короткого периода. Тем не менее, обратите внимание, что при использовании приложений iOS с поддержкой MAM мы поддерживаем iOS 9.0 и более поздней версии, поэтому вам нужно проследить, чтобы ваши конечные пользователи перешли на самую новую версию ОС. 

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Мы сообщая об этом заранее, но конкретные даты еще не установлены, поэтому у вас есть время на планирование. Проследите, что ваши пользователи обновились до iOS 9+, а после выхода приложения корпоративного портала попросите пользователей обновить приложение корпоративного портала.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?
Рекомендуйте пользователям обновиться до iOS 9.0 или более поздней версии, чтобы они могли воспользоваться всеми преимуществами новых функций Intune.  Рекомендуйте пользователям установить новую версию корпоративного портала и использовать предоставляемые ей преимущества.

Перейдите в Intune на портале Azure, откройте "Устройства" > "Все устройства" и отфильтруйте по версии iOS, чтобы просмотреть все текущие устройства с операционной системой, предшествующей iOS 9.


### <a name="apple-to-require-updates-for-application-transport-security---748318--"></a>Требуемые обновления Application Transport Security от Apple <!--748318-->
Компания Apple объявила, что к Application Transport Security (ATS) будут применяться определенные требования. ATS позволяет обеспечить высокий уровень безопасности при передаче данных в приложениях по протоколу HTTPS. Это изменение затрагивает клиентов Intune, которые используют приложения корпоративного портала для iOS.

Мы выпустили доступную версию приложения Корпоративного портала для iOS, созданную с помощью программы Apple TestFlight, в которой реализованы новые требования к ATS. Если вы хотите оценить ее и проверить соответствие требованиям ATS, отправьте письмо по адресу <a href="mailto:CompanyPortalBeta@microsoft.com?subject=Register to TestFlight ATS Company Portal app">CompanyPortalBeta@microsoft.com</a> со своим именем, фамилией, адресом электронной почты и названием организации. Дополнительные сведения см. в [блоге службы поддержки Intune](https://aka.ms/compportalats).
