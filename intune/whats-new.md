## <a name="week-of-june-26th-2017"></a>Неделя, начиная с 26 июня 2017 г.

### <a name="role-based-access-control"></a>контроль доступа на основе ролей;
#### <a name="new-role-based-administration-access-for-intune-admins------1099990---"></a>Новая модель доступа с администрированием на основе ролей для администраторов Intune <!-- 1099990 -->  
Добавлена новая роль администратора условного доступа, позволяющая просматривать, создавать, изменять и удалять политики условного доступа Azure AD. Ранее такие разрешения получали только глобальные администраторы и администраторы безопасности. Теперь администраторы Intune могут получать разрешения этой роли и, соответственно, возможность работы с политиками условного доступа.


### <a name="device-enrollment"></a>Регистрация устройств
#### <a name="tag-corporate-owned-devices-with-serial-number----1215070---"></a>Маркировка корпоративных устройств с использованием серийного номера <!-- 1215070 -->  
Intune поддерживает загрузку серийных номеров iOS, macOS и Android в качестве идентификаторов корпоративных устройств. Сейчас серийные номера нельзя использовать для блокирования регистрации личных устройств, так как эти номера не проверяются во время регистрации. Возможность блокирования личных устройств по серийному номеру будет реализована в ближайшем будущем.


### <a name="device-management"></a>Управление устройствами
#### <a name="new-remote-actions-for-ios-devices----854689---"></a>Новые удаленные действия для устройств iOS <!-- 854689 -->
В этом выпуске мы добавили два новых удаленных действия для устройств iOS.

-   [Выход из системы для текущего пользователя](device-logout-user.md). Выполняет выход из системы для текущего пользователя выбранного устройства iOS.
-   [Удалить пользователя](device-remove-user.md). Удаляет выбранного пользователя из локального кэша на устройстве iOS.


С помощью этих удаленных действий администраторы смогут управлять учетными записями пользователей, кэшированными на общем устройстве iPad, а также выполнять выход из системы для пользователя, вошедшего на устройство.

Во время регистрации администратор определяет максимальное число учетных записей пользователей, которые могут кэшироваться на устройстве. Действие "Удалить пользователя" позволяет администраторам удалять конкретных кэшированных пользователей.

Действие "Выход из системы для текущего пользователя" выполняет выход из системы для пользователя, вошедшего на устройство. Это действие можно найти в верхней части колонки "Обзор", где обычно находятся действия устройства.

Действие "Удалить пользователя" удаляет указанного пользователя из локального кэша устройства. Это действие можно найти, выбрав "Мониторинг" -> "Пользователи" и щелкнув правой кнопкой мыши конкретного пользователя в списке. Все данные, связанные с несинхронизированной учетной записью пользователя, будут потеряны. Кроме того, на то, чтобы удаление пользователя было отражено в списке, может потребоваться до 24 часов.

#### <a name="support-for-shared-ipads-with-the-ios-classroom-app----1044681---"></a>Поддержка общих устройств iPad с приложением iOS Класс <!-- 1044681 -->
В этом выпуске мы расширили поддержку для управления приложением iOS Класс, чтобы включить учащихся, которые входят на общие устройства iPad с помощью управляемых идентификаторов Apple.


### <a name="app-management"></a>Управление приложениями  
#### <a name="support-for-offline-apps-from-the-windows-store-for-business-----777044----"></a>Поддержка автономных приложений, приобретенных в Магазине Windows для бизнеса <!--- 777044 --->
Автономные приложения, приобретенные в Магазине Windows для бизнеса, будут синхронизироваться с порталом Intune. После этого вы сможете развертывать такие приложения в группах устройств или пользователей. Автономные приложения устанавливаются с помощью Intune, а не магазина.

#### <a name="microsoft-teams-is-now-part-of-the-app-based-ca-list-of-approved-apps------1257019---"></a>Microsoft Teams теперь входит в список утвержденных приложений для ЦС на основе приложений <!-- 1257019 -->.

Приложение Microsoft Teams для iOS и Android входит в группу утвержденных для политик условного доступа на основе приложений для Exchange и SharePoint Online. Это приложение можно настроить в колонке "Защита приложений Intune" на портале Azure для всех клиентов, использующих условный доступ на основе приложений.

#### <a name="managed-browser-and-app-proxy-integration----1287310---"></a>Интеграция Managed Browser и прокси приложения <!-- 1287310 -->
 Intune Managed Browser теперь можно интегрировать со службой прокси приложения Azure AD, чтобы предоставить пользователям доступ к внутренним веб-сайтам даже при удаленной работе. Пользователи браузера вводят URL-адрес сайта обычным образом, а Managed Browser направляет запрос через веб-шлюз прокси приложения. Дополнительные сведения см. в статье [Manage Internet access using Managed browser policies (Управление доступом в Интернет с помощью политик Managed Browser)](app-configuration-managed-browser.md).


#### <a name="new-app-configuration-settings-for-the-intune-managed-browser----682951---"></a>Новые параметры конфигурации приложений для Intune Managed Browser <!-- 682951 -->
В этом выпуске мы добавили дополнительные конфигурации для приложения Intune Managed Browser для iOS и Android. Теперь вы можете использовать политику конфигурации приложений для настройки домашней страницы по умолчанию и закладок для браузера.
Дополнительные сведения см. в статье [Manage Internet access using Managed browser policies (Управление доступом в Интернет с помощью политик Managed Browser)](app-configuration-managed-browser.md).

Приложение корпоративного портала для Windows 10 теперь будет автоматически инициировать синхронизацию запросов на установку приложений для устройств с обновлением Windows 10 Creators Update (версия 1703). Это сократит проблему замедления установки приложений в состоянии "Ожидание синхронизации". Кроме того, пользователи смогут вручную запускать синхронизацию в приложении. Эти изменения отражены на странице [Новые возможности в пользовательском интерфейсе приложения](whats-new-app-ui.md).

### <a name="new-guided-experience-for-windows-10-company-portal----1058938---"></a>Новые интерактивные возможности приложения "Корпоративный портал" в Windows 10 <!---1058938--->

Корпоративный портал для Windows 10 будет включать интерактивное пошаговое руководство Intune по работе с устройствами, которые не идентифицированы или не зарегистрированы. Это будут пошаговые инструкции для пользователей, с помощью которых можно зарегистрировать устройство в Azure Active Directory (требуется для работы функции условного доступа) и выполнить регистрацию MDM (требуется для функций управления устройствами). Эти пошаговые инструкции будут доступны на главной странице корпоративного портала. Пользователи, не выполнившие регистрацию, по-прежнему смогут использовать это приложение, однако набор доступных им функций будет ограничен.

Это обновление видно только на устройствах под управлением юбилейного обновления Windows 10 (сборка 1607) или более поздней версии. Эти изменения отражены на странице [Новые возможности в пользовательском интерфейсе приложения](whats-new-app-ui.md).

### <a name="microsoft-intune-and-conditional-access-admin-consoles-are-generally-available"></a>Открыт общий доступ к консолям администрирования Microsoft Intune и условного доступа

Мы объявляем о начале общего доступа к новым консолям администрирования Intune на портале Azure и администрирования условного доступа. Теперь с помощью Intune на портале Azure вы можете управлять всеми возможностями MAM и MDM Intune в едином интерфейсе администратора, используя функции группирования и нацеливания Azure AD. Единая консоль управления условным доступом на портале Azure позволяет расширить возможности за счет объединения функций Azure AD и Intune. Intune теперь отображается в консоли Azure на портале portal.azure.com без ярлыка **предварительной версии**.

На данный момент со стороны существующих клиентов никаких действий не требуется, кроме случаев, когда в ваш центр сообщений поступил ряд запросов на выполнение определенных действий, который позволят нам провести миграцию ваших групп. Также в ваш центр сообщений могло поступить уведомление об увеличении продолжительности миграции из-за определенных проблем с нашей стороны. Мы активно работаем над тем, чтобы завершить миграцию для тех клиентов, кого это коснулось.

### <a name="app-management"></a>Управление приложениями

#### <a name="new-app-protection-policies-conditions-for-mam----679864---"></a>Новые условия политик защиты приложений для MAM <!-- 679864 -->

В консоли администрирования вы теперь сможете устанавливать требования для пользователей MAM без регистрации, что позволяет принудительно применять следующие политики:

- минимальная версия приложения;
- Минимальная версия операционной системы.
- Минимальная версия пакета SDK для приложений Intune в целевом приложении (только для iOS).

Эта функция доступна для устройств iOS и Android. Intune поддерживает требование минимальной версии для платформ операционной системы, версий приложений и пакета SDK для приложений Intune. Приложения iOS, в которые интегрирован пакет SDK, также могут задавать требование минимальной версии на уровне пакета SDK. Пользователь не сможет получить доступ к целевому приложению, если на трех разных уровнях, описанных выше, не выполнены минимальные требования политики защиты приложения. На этом этапе пользователь может удалить учетную запись (для приложений с несколькими удостоверениями), закрыть приложение или обновить ОС либо приложение до требуемой версии.

Кроме того, вы также можете настраивать дополнительные параметры, чтобы пользователю отображалось уведомление с рекомендацией обновить ОС или приложение. При этом доступ к приложению не будет блокироваться. Пользователь сможет закрыть это уведомление и использовать приложение в обычном режиме.

Дополнительные сведения см. в статьях [Параметры политики защиты приложений в iOS](app-protection-policy-settings-ios.md) и [Параметры политики защиты приложений в Android](app-protection-policy-settings-android.md).

#### <a name="configure-app-configurations-for-android-for-work----621621---"></a>Настройка конфигураций приложений для Android for Work <!-- 621621 -->
Некоторые приложения Android из магазина поддерживают варианты управляемых конфигураций, с помощью которых ИТ-администратор может контролировать выполнение приложения в рабочем профиле. С помощью Intune вы теперь можете просматривать поддерживаемые приложением конфигурации и настраивать их на портале Intune, используя конструктор конфигураций или редактор JSON. Дополнительные сведения см. в статье [Использование конфигураций приложений для Android for Work](app-configuration-policies-use-android.md).

#### <a name="new-app-configuration-capability-for-mam-without-enrollment----677969---"></a>Новая возможность управления конфигурациями приложений для MAM без регистрации <!-- 677969 -->

Теперь вы можете создавать политики конфигураций приложений в MAM, не используя канал регистрации. Эта возможность аналогична политикам конфигураций приложений, которые доступны в конфигурации управления мобильными устройствами (MDM). Пример конфигурации приложения в MAM без регистрации см. в статье [Управление доступом в Интернет с помощью политик управляемого браузера в Microsoft Intune](app-configuration-managed-browser.md).

#### <a name="configure-allowed-and-blocked-url-lists-for-the-managed-browser----682960---"></a>Настройка списков разрешенных и запрещенных URL-адресов для Managed Browser <!-- 682960 -->

Теперь вы можете настраивать списки разрешенных и запрещенных URL-адресов для Intune Managed Browser с использованием параметров конфигурации приложений на портале Azure. Эти параметры можно настроить как на управляемом, так и на неуправляемом устройстве. Дополнительные сведения см. в статье [Управление доступом в Интернет с помощью политик управляемого браузера в Microsoft Intune](app-configuration-managed-browser.md).

#### <a name="app-protection-policy-helpdesk-view----1069473---"></a>Представление службы поддержки для политики защиты приложений <!-- 1069473 -->

Пользователи службы ИТ-поддержки теперь могут проверить состояние пользовательской лицензии и приложений политики защиты приложений, назначенных пользователям в колонке "Устранение неполадок". Дополнительные сведения см. в разделе [Устранение неполадок](./help-desk-operators.md).

### <a name="device-configuration"></a>Конфигурация устройства

#### <a name="control-website-visits-on-ios-devices----723832---"></a>Контроль посещений веб-сайтов на устройствах iOS <!-- 723832 -->

Теперь вы можете контролировать веб-сайты, которые посещают пользователи устройств iOS, одним из следующих двух способов:

- добавить разрешенные и заблокированные URL-адреса, используя встроенный фильтр веб-содержимого Apple;

- разрешить доступ с помощью браузера Safari только к указанным веб-сайтам. В браузере Safari созданы закладки для каждого указанного сайта.

Дополнительные сведения см. в разделе [Параметры фильтров веб-содержимого для устройств iOS](web-content-filter-settings-ios.md).

#### <a name="preconfigure-device-permissions-for-android-for-work-apps----621614---"></a>Предварительная настройка разрешений устройств для приложений Android for Work <!-- 621614 -->

Для приложений, развернутых в рабочих профилях устройства Android for Work, теперь можно индивидуально настраивать состояния разрешений.  По умолчанию приложения Android, которым требуются разрешения устройства, например доступ к данным о расположении или камере устройства, предлагают пользователям разрешить или запретить доступ.  Например, если приложение использует микрофон устройства, пользователю предлагается предоставить приложению разрешение на использование микрофона. Эта функция позволяет определять разрешения от имени конечного пользователя.  Вы можете настроить разрешения так, чтобы: а) автоматически отказывать, не уведомляя пользователей, б) автоматически утверждать, не уведомляя пользователей, либо в) запрашивать утверждение или отказ со стороны пользователя. Дополнительные сведения см. в разделе [Параметры ограничений для устройств Android for Work в Microsoft Intune](device-restrictions-android-for-work.md).

#### <a name="define-app-specific-pin-for-android-for-work-devices----728976-1102534---"></a>Определение PIN-кода конкретного приложения для устройств Android for Work <!-- 728976, 1102534 -->

Пользователи устройств с Android 7.0 и более поздних версий с управлением рабочим профилем Android for Work могут определять политику секретного кода, которая будет применяться только в рабочем профиле.  Возможны следующие значения.

- Определение политики секретного кода для всего устройства. Это секретный код, который конечный пользователь вводит для разблокировки всего устройства.
 Определение политики секретного кода только для рабочего профиля. Конечному пользователю будет предложено ввести секретный код при открытии любого приложения в рабочем профиле.
- Определение политики и для устройства, и для рабочего профиля. ИТ-администратор может определить политику секретного кода и для устройства, и для рабочего профиля, выбрав разные уровни сложности для кодов (например, 4-значный ПИН-код для разблокировки устройства и 6-значный ПИН-код для открытия любого рабочего приложения).

Дополнительные сведения см. в разделе [Параметры ограничений для устройств Android for Work в Microsoft Intune](device-restrictions-android-for-work.md).

> [!NOTE]
> Эта функция будет доступна только на платформе Android 7.0 и более поздних версий.  По умолчанию конечный пользователь может либо использовать два отдельно определяемых ПИН-кода, либо объединить два ПИН-кода в один (при этом будет использоваться более сложный ПИН-код).

#### <a name="new-settings-for-windows-10-devices----978585---"></a>Новые параметры для устройств Windows 10 <!-- 978585 -->

Мы добавили новые [параметры ограниченного использования устройств Windows](device-restrictions-windows-10.md), чтобы управлять такими функциями, как беспроводной дисплей, обнаружение устройств, переключение задач и сообщения об ошибках SIM-карты.

#### <a name="updates-to-certificate-configuration----918991-and-823198---"></a>Обновления конфигурации сертификата <!-- 918991 and 823198 -->

При создании профиля сертификата SCEP на устройствах с iOS, Android и Windows для параметра **Формат имени субъекта** доступно значение **Другой**. До этого обновления поле **Другой** присутствовало только на устройствах iOS. Дополнительные сведения см. в разделе [Создание профиля сертификата SCEP] (certificates-scep-configure.md#how-to-create-a-scep-certificate-profile).

При создании профиля сертификата PKCS для параметра **Альтернативное имя субъекта** доступен параметр **Настраиваемый атрибут Azure AD**. Теперь доступен параметр **Отдел** при выборе параметра **Настраиваемый атрибут Azure AD**. Дополнительные сведения см. в разделе [Создание профиля сертификата PKCS] (certficates-pfx-configure.md#how-to-create-a-pkcs-certificate-profile).

#### <a name="configure-multiple-apps-that-can-run-when-an-android-device-is-in-kiosk-mode----662059---"></a>Настройка нескольких приложений, выполняемых в режиме терминала устройства Android <!-- 662059 -->

Ранее, если устройство Android находилось в режиме терминала, вы могли настроить только одно выполняющееся приложение. Теперь вы можете использовать идентификатор приложения, URL-адрес хранилища или выбрать существующее управляемое приложение Android, чтобы настроить несколько приложений, выполняющихся в режиме терминала. Дополнительные сведения см. в разделе [Параметры режима терминала](device-restrictions-android.md#kiosk).

После миграции на портал Azure вам понадобится повторно назначить администраторам эти новые роли администрирования. Дополнительные сведения об элементах управления администрированием на основе ролей и новых ролях см. в статье [Роли Intune (RBAC) для Microsoft Intune](/intune/role-based-access-control).

## <a name="whats-coming"></a>Предстоящие изменения

### <a name="platform-support-reminder-windows-phone-81-mainstream-support-will-end-july-11-2017"></a>Напоминание о поддержке платформы: основная фаза поддержки Windows Phone 8.1 закончится 11 июля 2017 г.
<!-- 1327781 -->

11 июля 2017 г. заканчивается основная фаза поддержки платформы Windows Phone 8.1. Поддержка настольных систем на базе Windows 8.1 затронута не будет.

Это не окажет немедленное влияние на все устройства Windows Phone 8.1, управляемые службой Intune. Зарегистрированные устройства, как и все политики, конфигурации и приложения, продолжат работать должным образом. Обратите внимание на отсутствие усовершенствований для платформы Windows Phone 8.1 в службе Intune, а также для приложения корпоративного портала Windows Phone 8.1. 

Рекомендуется при первой возможности обновить соответствующие устройства Windows Phone 8.1 до Windows 10 Mobile. 

### <a name="changes-in-support-for-the-intune-ios-company-portal-app-----1164474----"></a>Изменения в поддержке для приложения корпоративного портала Intune для iOS <!-- 1164474  -->


Скоро будет выпущена новая версия приложения корпоративного портала Microsoft Intune для iOS. Она будет поддерживать только устройства под управлением iOS 9.0 или более поздней версии. Версия корпоративного портала, поддерживающая iOS 8, останется доступной в течение очень короткого периода. Тем не менее, обратите внимание, что при использовании приложений iOS с поддержкой MAM мы поддерживаем iOS 9.0 и более поздней версии, поэтому вам нужно проследить, чтобы ваши конечные пользователи перешли на самую новую версию ОС. 

#### <a name="how-does-this-affect-me"></a>Как это повлияет на работу?
Мы сообщая об этом заранее, но конкретные даты еще не установлены, поэтому у вас есть время на планирование. Проследите, что ваши пользователи обновились до iOS 9+, а после выхода приложения корпоративного портала попросите пользователей обновить приложение корпоративного портала.

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Как можно подготовиться к этому изменению?

Рекомендуйте пользователям обновиться до iOS 9.0 или более поздней версии, чтобы они могли воспользоваться всеми преимуществами новых функций Intune.  Рекомендуйте пользователям установить новую версию корпоративного портала и использовать предоставляемые ей преимущества.

Перейдите в Intune на портале Azure, откройте "Устройства" > "Все устройства" и отфильтруйте по версии iOS, чтобы просмотреть все текущие устройства с операционной системой, предшествующей iOS 9.

### <a name="improved-sign-in-experience-across-company-portal-apps-for-all-platforms---user-story-1132123--"></a>Улучшен интерфейс входа в приложениях корпоративного портала для всех платформ <!--User Story 1132123-->

В ближайшие месяцы мы планируем выпустить обновление, которое позволит улучшить возможности входа приложений корпоративного портала Intune для Android, iOS и Windows. Новый пользовательский интерфейс приложения корпоративного портала автоматически появится на всех платформах после того, как Azure AD внесет это изменение. Кроме того, пользователи теперь смогут войти на корпоративный портал с другого устройства с помощью созданного разового кода. Это особенно полезно, если пользователям нужно войти без использования учетных данных.
