# <a name="android-app-protection-policy-settings-in-microsoft-intune"></a>Параметры политики защиты приложений Android в Microsoft Intune
В этой статье описываются параметры политики защиты приложений на устройствах Android. Описываемые параметры политик можно [настроить](app-protection-policies.md) для политики защиты приложений на портале Azure в колонке **Параметры**.
Существуют две категории параметров политики: для защиты данных и доступа к данным. В этой статье термин *управляемые политикой приложения* означает приложения, которые настроены с помощью политик защиты приложений.

На устройствах Android разрешены другие приложения управляемого браузера, которые поддерживают цель http/https. | Любое приложение |
| **Шифрование данных приложения** | Выберите **Да**, чтобы включить шифрование рабочих или учебных данных в этом приложении.   ## <a name="data-transfer-exemptions"></a>Исключения передачи данных

  Среди приложений и служб платформ существует несколько исключений, в которых политика защиты приложений Intune может разрешить передачу данных. Например, во всех управляемых приложениях Intune на мобильных устройствах с Android должна быть возможность передавать данные в службу преобразования текста в речь Google и получать данные из нее. Этот список может измениться. В нем представлены полезные службы и приложения для безопасной работы.

  ### <a name="full-exemptions"></a>Полные исключения

  В этих приложениях и службах полностью разрешен обмен данными с приложениями под управлением Intune.

  |Имя службы или приложения | Описание |
  | ------ | ---- |
  | com.android.phone | Встроенное приложение телефона
  | com.android.vending | Магазин Google Play |
  | com.android.documentsui | Средство выбора документов Android|
  | com.google.android.webview | [Веб-представление](https://developer.android.com/reference/android/webkit/WebView.html), необходимое для многих приложений, включая Outlook. |
  | com.android.webview |[Веб-представление](https://developer.android.com/reference/android/webkit/WebView.html), необходимое для многих приложений, включая Outlook.|
  | com.google.android.tts | Преобразование текста в речь Google |
  | com.android.providers.settings | Параметры системы Android |
  | com.android.settings | Параметры системы Android |
  | com.azure.authenticator | Приложение Azure Authenticator, которое нужно для проверки подлинности во многих сценариях. |
  | com.microsoft.windowsintune.companyportal | Intune Портал компании|

  ### <a name="conditional-exemptions"></a>Условные исключения
  В этих приложениях и службах разрешен обмен данными с приложениями под управлением Intune только при определенных условиях.

  |Имя службы или приложения | Описание | Условие исключения|
  | ------ | ---- | --- |
  | com.android.chrome | Браузер Google Chrome | Chrome используется для некоторых компонентов веб-представления в ОС Android 7.0+ и никогда не скрывается в представлении. Поток данных в приложение и из него всегда ограничен.  |
  | com.skype.raider | Skype | В Skype всегда разрешены только определенные действия, которые приводят к телефонному звонку. |
  | com.android.providers.media | Поставщик содержимого мультимедиа Android | Поставщик содержимого мультимедиа разрешен только при выборе мелодии звонка. |
  | com.google.android.gms; com.google.android.gsf | Пакеты сервисов Google Play | Эти пакеты разрешены для действий Google Cloud Messaging, например push-уведомлений. |

Дополнительные сведения см. в разделе [Исключения политики передачи данных для приложений](app-protection-policies-exception.md).

Дополнительные сведения об отпечатках пальцев для рабочих профилей Android см. в разделе [Блокирование рабочего профиля](https://support.google.com/work/android/answer/7029958).| 
| **Требовать корпоративные учетные данные для доступа** | Выберите **Да**, чтобы для доступа к приложению пользователь выполнил вход с помощью своей рабочей или учебной учетной записи вместо ввода ПИН-кода. В Android один и тот же ПИН-код используется во всех управляемых приложениях Intune. Таймер ПИН-кода сбрасывается, когда приложение становится неактивным на устройстве. Пользователю не нужно будет вводить ПИН-код в другом приложении, управляемом Intune и использующем этот ПИН-код, в течение времени ожидания, указанного в этом параметре.* <br><br></li> |
| **Блокировать снимки экрана и Android Assistant** | Выберите **Да**, чтобы блокировать создание снимков экранов и возможности **Android Assistant** данного устройства при использовании этого приложения. В случае выбора значения **Да** изображение предварительного просмотра переключателя приложений при использовании этого приложения с рабочей или учебной учетной записью будет размыто. <br><br>Значение по умолчанию — **Нет**. |

## <a name="conditional-launch"></a>Условный запуск
Настройте параметры условного запуска, чтобы задать требования безопасности входа для вашей политики защиты доступа. 

По умолчанию предоставляется несколько параметров с предварительно настроенными значениями и действиями. Можно удалить некоторые из них, например *Мин. версия ОС*. Вы также можете выбрать дополнительные параметры из раскрывающегося списка **Выбрать один элемент**.

Этот формат параметра политики поддерживает положительное целое число. *Действия*: <br><ul><li>**Сбросить ПИН-код** — пользователь должен сбросить ПИН-код.</li></ul> <ul><li>**Очистить данные** — учетная запись пользователя, связанная с приложением, будет удалена с устройства.  </li></ul> </li></ul> Значение по умолчанию — **5**. |

Укажите время (в минутах), по истечении которого выполняется повторная проверка требований доступа для приложения. *Действия*: <br><ul><li>**Блокировать доступ (мин)**: период, в течение которого приложения MAM могут выполняться автономно, в минутах.
