# <a name="what-is-microsoft-intune-app-management"></a>Что такое управление приложениями с помощью Microsoft Intune


[!INCLUDE[azure_portal](./includes/azure_portal.md)]


Microsoft Intune позволяет ИТ-администраторам управлять мобильными приложениями, которые используют сотрудники компании. Эта функция дополняет возможности управления устройствами и защиты данных. Используя эту функцию, вы должны убедиться в том, что конечные пользователи имеют доступ к приложениям, необходимым для работы. Это может быть сложной задачей, так как:
- существует множество платформ устройств и типов приложений;
- может потребоваться управлять приложениями и на устройствах организации, и на личных устройствах пользователей;
- необходимо обеспечить безопасность сети и данных.

Кроме того, может потребоваться назначать приложения и управлять ими на устройствах, которые не зарегистрированы в Intune.

Intune предлагает широкий набор возможностей, позволяющих получить нужные приложения на требуемых устройствах. В таблице ниже перечислены возможности управления приложениями. Под таблицей вы найдете основные сведения для понимания Microsoft Intune на портале Azure.

## <a name="how-to-get-started"></a>Начало работы

Большинство параметров, связанных с приложениями, находятся в рабочей нагрузке **Мобильные приложения**, доступ к которой можно получить следующим образом:

1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Intune**. Intune находится в разделе **Мониторинг и управление**.
3. В колонке **Intune** выберите **Мобильные приложения**.

    ![Рабочая нагрузка "Мобильные приложения"](./media/apps-workload.png)

Ниже описаны параметры, доступные в колонке **Мобильные приложения**.

### <a name="manage"></a>управление
- **Приложения** — выберите этот параметр, чтобы добавлять, просматривать, назначать и отслеживать приложения, используемые вашими сотрудниками. Дополнительные сведения см. в следующих статьях:
    - [Добавление приложений](apps-add.md)
    - [Назначение приложений](apps-deploy.md)
    - [Мониторинг приложений](apps-monitor.md)
- **Политики конфигурации приложений** — политики конфигурации приложений позволяют предоставлять параметры, требуемые при работе с приложением. Дополнительные сведения см. в следующих статьях:
    - [Политики конфигурации приложений для Intune](app-configuration-policies-overview.md)
        - [Политики конфигурации приложений iOS](app-configuration-policies-use-ios.md)
        - [Политики конфигурации приложений Android](app-configuration-policies-use-android.md)
- **Политики защиты приложений** — позволяют связывать параметры с приложением для защиты используемых корпоративных данных. Например, можно ограничить возможности приложения взаимодействовать с другими приложениями или требовать ввода ПИН-кода для доступа к корпоративному приложению. Дополнительные сведения см. в следующих статьях:
    - [Политики защиты приложений](app-protection-policies.md)
- **Выборочная очистка приложений** — вы можете настроить удаление только корпоративных данных из выбранного пользовательского устройства. Дополнительные сведения см. в следующих статьях:
    - [Выборочная очистка приложений](apps-selective-wipe.md)
- **Профили подготовки приложений iOS** — приложения IOS содержат профиль подготовки и код, подписанный сертификатом. Если срок действия сертификата истек, вы не сможете запустить приложение. Intune предоставляет средства для упреждающего назначения новой политики профиля подготовки на устройствах с приложениями, срок действия которых вскоре истекает. Дополнительные сведения см. в следующих статьях:
    - [Предотвращение истечения срока действия сертификата мобильного приложения iOS с помощью профиля подготовки](app-provisioning-profile-ios.md)

Дополнительные сведения см. в статье об[ управлении приложениями](app-management.md).

### <a name="monitor"></a>Монитор
- **Лицензии приложений.** В этом разделе можно просматривать, назначать и отслеживать приложения, приобретенные по программе Volume Purchase Program в магазине приложений. Дополнительные сведения см. в следующих статьях:
    - [Приложения iOS, приобретенные по программе корпоративных закупок](vpp-apps-ios.md)
    - [Приложения, приобретенные в Магазине Майкрософт для бизнеса по программе корпоративного лицензирования](windows-store-for-business.md)
- **Обнаруженные приложения.** Здесь отображаются все приложения, назначенные Intune и установленные на устройстве.
- **Состояние установки приложения.** Здесь отображается состояние созданного назначения приложения.
- **Состояние защиты приложения.** Здесь отображается состояние политики защиты приложения для выбранного пользователя.
- **Журналы аудита.** Здесь отображаются действия, связанные с приложением Intune, которые выполняют все ИТ-администраторы.

Дополнительные сведения см. в статье о [мониторинге приложений](apps-monitor.md).

### <a name="setup"></a>Настройка
- **Токены VPP iOS.** Применение и просмотр лицензий Volume Purchase Program (VPP) iOS.
    - [Приложения iOS, приобретенные по программе корпоративного лицензирования](vpp-apps-ios.md)
- **Сертификат Windows Корпоративная.** Примените сертификат подписи кода, используемый для распространения бизнес-приложений на управляемых устройствах Windows, или просмотрите сведения о его состоянии.
- **Сертификат Windows Symantec.** Примените сертификат подписи кода Symantec, который необходим для распространения APPX-файлов XAP и WP8.x на устройствах Windows 10 Mobile, или просмотрите сведения о его состоянии.
- **Магазин Майкрософт для бизнеса**. В этом разделе можно настроить интеграцию с Магазином Майкрософт для бизнеса. После этого вы сможете синхронизировать приобретенные приложения с Intune, назначать их и отслеживать использование лицензий. Дополнительные сведения см. в следующих статьях:
    - [Приложения, приобретенные в Магазине Майкрософт для бизнеса по программе корпоративного лицензирования](windows-store-for-business.md)
- **Ключи загрузки неопубликованных приложений Windows.** Вы можете добавить ключ загрузки неопубликованных приложений Windows, чтобы установить приложение непосредственно на устройствах вместо его публикации и загрузки из магазина Windows. Дополнительные сведения см. в следующих статьях:
    - [Загрузка неопубликованных приложений Windows](app-sideload-windows.md).
- **Фирменная символика корпоративного портала.** Здесь можно настроить фирменную символику для Корпоративного портала. Дополнительные сведения см. в следующих статьях:
    - [Настройка корпоративного портала](company-portal-app.md)
- **Категории приложений.** Добавляйте, закрепляйте и удаляйте имена категорий приложений.
- **Android for Work.** Утверждайте и синхронизируйте приложения, использование которых вы разрешили в своей организации. Дополнительные сведения см. в следующих статьях:
    - [Приложений Android for Work](apps-add-android-for-work.md)
