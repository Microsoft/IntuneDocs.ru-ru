# <a name="configure-and-manage-pkcs-certificates-with-intune"></a>Настройка инфраструктуры сертификатов Microsoft Intune для PKCS

[!INCLUDE[azure_portal](./includes/azure_portal.md)]

## <a name="requirements"></a>Requirements (Требования)

Чтобы использовать сертификаты PKCS с Intune, необходима следующая инфраструктура:

* Имеющийся настроенный домен доменных служб Active Directory.
 
  Дополнительные сведения об установке и настройке AD DS см. в статье [AD DS Design and Planning](https://docs.microsoft.com/windows-server/identity/ad-ds/plan/ad-ds-design-and-planning) (Планирование и проектирование доменных служб Active Directory).

* Имеющийся настроенный центр сертификации предприятия (ЦС).

  Дополнительные сведения о том, как установить и настроить службы сертификатов Active Directory (AD CS), см. в статье [Пошаговое руководство по службам сертификации Active Directory Windows Server](https://technet.microsoft.com/library/cc772393).

  > [!WARNING]
  > Intune требуется запустить службу сертификации Active Directory с помощью центра сертификации предприятия (ЦС), а не автономного ЦС.

* Клиент, подключенный к ЦС предприятия.
* Экспортированная копия корневого сертификата из ЦС предприятия.
* Microsoft Intune Certificate Connector (NDESConnectorSetup.exe), скачанный с портала Intune.
* Служба Windows Server, доступная для размещения Microsoft Intune Certificate Connector (NDESConnectorSetup.exe).

## <a name="export-the-root-certificate-from-the-enterprise-ca"></a>Экспортирование корневого сертификата из ЦС предприятия

Для выполнения проверки подлинности с помощью VPN, Wi-Fi и других ресурсов необходим корневой или промежуточный сертификат ЦС на каждом устройстве. Далее описывается, как получить необходимый сертификат из ЦС предприятия.

1. Войдите в ЦС предприятия с использованием учетной записи с привилегиями администратора.
2. Откройте командную строку как администратор.
3. Экспортируйте сертификат корневого ЦС в расположение, где вы сможете открыть его позже.

   Пример.

4.  После завершения работы мастера, но перед тем, как закрыть его, щелкните **Запустить пользовательский интерфейс соединителя сертификатов**.

   `certutil -ca.cert certnew.cer`

   Дополнительные сведения см. в разделе [Задачи Certutil для управления сертификатами](https://technet.microsoft.com/library/cc772898.aspx#BKMK_ret_sign).

## <a name="configure-certificate-templates-on-the-certification-authority"></a>Настройка шаблонов сертификатов в центре сертификации

1. Войдите в ЦС предприятия с использованием учетной записи с привилегиями администратора.
2. Откройте консоль **центра сертификации**.
3. Щелкните правой кнопкой мыши элемент **Шаблоны сертификатов** и затем выберите **Управление**.
4. Найдите шаблон сертификата **пользователя**, щелкните его правой кнопкой мыши и выберите **Duplicate Template** (Скопировать шаблон). Откроется окно **Properties of New Template** (Свойства нового шаблона).
5. На вкладке **Compatibility** (Совместимость):
   * Задайте для **центра сертификации** значение **Windows Server 2008 R2**.
   * Задайте для **получателя сертификации** значение **Windows 7 / Server 2008 R2**.
6. На вкладке **Общие**
   * Задайте осмысленное **отображаемое имя шаблона**.

   > [!WARNING]
   > **Имя шаблона** по умолчанию совпадает с **отображаемым именем шаблона** *без пробелов*. Запомните имя шаблона для последующего использования.

7. На вкладке **Request Handling** (Обработка запроса) установите флажок **Allow private key to be exported** (Разрешить экспортировать закрытый ключ).
8. На вкладке **Cryptography** (Шифрование) убедитесь, что **минимальный размер ключа** равен 2048.
9. На вкладке **Subject Name** (Имя субъекта) щелкните переключатель **Supply in the request** (Предоставляется в запросе).
10. На вкладке **Расширения** подтвердите, что шифрованная файловая система, защита электронной почты и проверка подлинности клиента отображаются в разделе **Application Policies** (Политики приложений).
    
      > [!IMPORTANT]
      > Для шаблонов сертификатов iOS и macOS измените на вкладке **Расширения** параметр **Использование ключей** и снимите флажок **Подпись подтверждает подлинность**.

11. На вкладке **Security** (Безопасность) добавьте учетную запись компьютера для сервера, на котором устанавливается Microsoft Intune Certificate Connector.
    * Назначьте для этой учетной записи разрешения на **чтение** и **регистрацию**.
12. Щелкните **Apply** (Применить), а затем **ОК**, чтобы сохранить шаблон сертификата.
13. Закройте **Консоль шаблонов сертификатов**.
14. Из консоли **центра сертификации** щелкните правой кнопкой мыши **Шаблоны сертификатов**, **New** (Создать), **Certificate Template to Issue** (Выдаваемый шаблон сертификата).
    * Выберите шаблон, созданный на предыдущем шаге, и щелкните **ОК**.
15. Чтобы сервер мог управлять сертификатами от имени зарегистрированных устройств и пользователей Intune, сделайте следующее:

    a. Щелкните центр сертификации правой кнопкой мыши и выберите **Свойства**.

    b. На вкладке Security (Безопасность) добавьте учетную запись компьютера для сервера, на котором запущен Microsoft Intune Certificate Connector.
      * Предоставьте для параметров **Issue and Manage Certificates** (Выдача сертификатов и управление ими) и **Request Certificates** (Запросить сертификаты) разрешения на доступ к учетной записи компьютера.
16. Выйдите из ЦС предприятия.

## <a name="download-install-and-configure-the-microsoft-intune-certificate-connector"></a>Скачивание, установка и настройка соединителя Microsoft Intune Certificate Connector

![ConnectorDownload][ConnectorDownload]

1. На портале Azure выберите пункты **Другие службы** > **Мониторинг и управление** > **Intune**.
2. В колонке **Intune** выберите пункт **Конфигурация устройства**. 
3. В колонке **Конфигурация устройства** выберите пункт **Центр сертификации**. 
4. Щелкните **Добавить** и выберите пункт **Скачать файл соединителя**. Сохраните файл в расположении, в котором вы сможете открыть его с сервера, на котором выполняется установка. 
5.  Войдите на сервер, где выполняется установка Microsoft Intune Certificate Connector.
6.  Запустите программу установки и примите расположение по умолчанию. Соединитель установится в папку C:\Program Files\Microsoft Intune\NDESConnectorUI\NDESConnectorUI.exe.
    1. На странице параметров установщика выберите **PFX Distribution** (Распределение PFX) и щелкните **Далее**.
    2. Щелкните **Установить** и дождитесь завершения установки.
    3. На странице завершения установите флажок **Запустить соединитель Intune** и нажмите кнопку **Готово**.
7.  Должно открыться окно "Соединитель NDES" на вкладке **Регистрация**. Чтобы включить подключение к Intune, щелкните **Войти** и укажите данные учетной записи с разрешениями администратора.
8.  На вкладке **Advanced** (Дополнительно) можно оставить выбранным переключатель **Use this computer's SYSTEM account (default)** (Использовать СИСТЕМНУЮ учетную запись этого компьютера (по умолчанию)).
9.  Щелкните **Применить**, а затем **ОК**.
10. Вернитесь на портал Azure. В области **Состояние подключения** в разделе **Intune** > **Конфигурация устройства** > **Центр сертификации** через несколько минут появится зеленая галочка и слово **Активно**. Это подтверждение позволяет узнать, что ваш сервер соединителя может взаимодействовать с Intune.


## <a name="create-a-device-configuration-profile"></a>Создание профиля конфигурации устройства

1. Войдите на [портал Azure](https://portal.azure.com).
2. Перейдите к **Intune**, **Конфигурация устройства**, **Profiles** (Профили) и щелкните **Создать профиль**.

   ![NavigateIntune][NavigateIntune]

3. Укажите следующие сведения:
   * **Имя** для профиля.
   * При необходимости задайте описание.
   * **Платформа**, на которой нужно развернуть профиль.
   * Задайте **доверенный сертификат** в качестве **типа профиля**.
4. Перейдите к **параметрам** и укажите CER-файл ранее экспортированного сертификата корневого ЦС.

   > [!NOTE]
   > В зависимости от платформы, выбранной на **шаге 3**, вы можете (или не можете) выбрать **конечное хранилище** для сертификата.

   ![ProfileSettings][ProfileSettings]

5. Щелкните **ОК**, а затем **Создать**, чтобы сохранить профиль.
6. Чтобы назначить новый профиль для одного или нескольких устройств, см. статью [Как назначать профили устройств Microsoft Intune](device-profile-assign.md).

## <a name="create-a-pkcs-certificate-profile"></a>Создание профиля сертификата PKCS

1. Войдите на [портал Azure](https://portal.azure.com).
2. Перейдите к **Intune**, **Конфигурация устройства**, **Profiles** (Профили) и щелкните **Создать профиль**.
3. Укажите следующие сведения:
   * **Имя** для профиля.
   * При необходимости задайте описание.
   * **Платформа**, на которой нужно развернуть профиль.
   * Задайте **сертификат PKCS** в качестве **типа профиля**.
4. Перейдите к **параметрам** и укажите следующую информацию:
   * **Renewal threshold (%)** (Порог обновления (%)) — рекомендовано 20 %.
   * **Срок действия сертификата** — если вы не изменяли шаблон сертификата, для этого параметра следует задать один год.
   * **Центр сертификации** — этот параметр является внутренним полным доменным именем (FQDN) ЦС предприятия.
   * **Имя центра сертификации** — этот параметр является именем центра сертификации предприятия и может отличаться от предыдущего элемента.
   * **Certificate template name** (Имя шаблона сертификата) — это может быть имя шаблона, созданного ранее. Помните, что **имя шаблона** по умолчанию совпадает с **отображаемым именем шаблона** *без пробелов*.
   * **Формат имени субъекта** — задайте для этого параметра значение **Common name** (Обычное имя) (только если не требуется иное).
   * **Subject alternative name** (Альтернативное имя субъекта) — в качестве значения этого параметра нужно задать **Имя участника-пользователя (UPN)** (только если не требуется иное).
   * **Расширенное использование ключа** — при условии, что используются значения по умолчанию, указанные на шаге 10 в предыдущем разделе, **настройте шаблоны сертификатов в центре сертификации** и добавьте следующие **предварительно определенные значения** из поля выбора:
      * **Любая цель**;
      * **Client Authentication** (Проверка подлинности клиента);
      * **Защищенная электронная почта**.
   * **Корневой сертификат** — (для профилей Android) это может быть CER-файл, экспортированный на шаге 3 в предыдущем разделе, [Экспорт корневого сертификата из ЦС предприятия](#export-the-root-certificate-from-the-enterprise-ca).

5. Щелкните **ОК**, а затем **Создать**, чтобы сохранить профиль.
6. Чтобы назначить новый профиль для одного или нескольких устройств, см. статью [Как назначать профили устройств Microsoft Intune](device-profile-assign.md).
