# <a name="role-based-administration-control-rbac-with-microsoft-intune"></a>Управление доступом на основе ролей (RBAC) с помощью Microsoft Intune

Управление доступом на основе ролей позволяет контролировать, кто может выполнять различные задачи Intune в организации, а также кому могут назначаться такие задачи. Вы можете использовать встроенные роли для некоторых распространенных сценариев Intune или же создавать собственные роли. Параметры, которые определяют роль:

- **Определение роли**. Название роли, управляемые в ее рамках ресурсы, а также разрешения, предоставляемые каждому ресурсу.
- **Участники**. Группы пользователей, которым предоставляются разрешения.
- **Область**. Группы пользователей или устройств, которыми могут управлять участники.
- **Назначение**: Когда определение, участники и область настроены, роль назначается.

![Пример управления доступом на основе ролей в Intune](./media/intune-rbac-1.PNG)

В новой версии портала Azure **Azure Active Directory (Azure AD)** предоставляет две роли каталога, которые могут использоваться в службе Intune. Эти роли получают полные разрешения на выполнение любых действий в Intune:

- **Глобальный администратор**. Пользователи с такой ролью имеют доступ ко всем функциям администрирования в Azure AD, а также к службам, которые включаются в федерацию с Azure AD, таким как Exchange Online, SharePoint Online и Skype для бизнеса Online. Пользователь, зарегистрировавший клиента Azure AD, становится глобальным администратором. Только глобальные администраторы могут назначать другие административные роли Azure AD. В компании может быть несколько глобальных администраторов. Глобальные администраторы могут сбрасывать пароли для любых пользователей и остальных администраторов.

- **Администратор службы Intune**. Пользователи с этой ролью имеют глобальные разрешения в Intune (при наличии этой службы). Кроме того, с учетом применимых ограничений Azure эта роль дает возможность управлять пользователями и устройствами, а также создавать группы Intune и управлять ими.

- **Администратор условного доступа**. Пользователи с этой ролью могут лишь просматривать, создавать, изменять и удалять политики условного доступа.

    > [!IMPORTANT]
    > Роль администратора службы Intune не позволяет управлять параметрами условного доступа Azure AD.

    > [!TIP]
    > Кроме того, в Intune реализовано три расширения Azure AD: **Пользователи**, **Группы** и **Условный доступ**, которые контролируются в рамках системы управления доступом на основе ролей Azure AD. Кроме того, **администратор учетных записей пользователей** выполняет только действия, связанные с пользователями или группами AAD, и не имеет полных разрешений на выполнение любых операций в Intune. Дополнительные сведения см. в разделе [Управление доступом на основе ролей в Azure AD](https://docs.microsoft.com/azure/active-directory/active-directory-assign-admin-roles).

## <a name="roles-created-in-the-intune-classic-portal"></a>Роли, созданные на классическом портале Intune

Из классического портала Intune в Intune на портале Azure переносятся только пользователи с ролью **Администратор службы Intune** и полными разрешениями. Пользователям с ролями "Только для чтения" и "Служба технической поддержки" необходимо заново назначить роль **Администратор службы Intune** в Intune на портале Azure, после чего их следует удалить с классического портала.

> [!IMPORTANT]
> Если вашим администраторам потребуется в дальнейшем управлять ПК с помощью Intune, вы можете сохранить доступ администраторов службы Intune к классическому порталу.

## <a name="built-in-roles"></a>Встроенные роли

Следующие роли встроены в Intune. Их можно назначать группам без дальнейшей настройки:

- **Оператор службы технической поддержки**. Выполняет удаленные задачи для пользователей и устройств, а также может назначать пользователям и устройствам приложения или политики.
- **Диспетчер политик и профилей**. Управляет политикой соответствия требованиям, профилями конфигурации, регистрацией Apple и идентификаторами корпоративных устройств.
- **Оператор только для чтения**. Просматривает сведения о пользователях, устройствах, регистрации, конфигурации и приложениях. Не может вносить изменения в Intune.
- **Диспетчер приложений**. Управляет мобильными и управляемыми приложениями. Может читать информацию об устройстве и просматривать его профили конфигурации.
- **Администратор ролей Intune**. Управляет пользовательскими ролями Intune и добавляет назначения для встроенных ролей Intune. Это единственная роль, позволяющая назначать разрешения для администраторов.
- **Администратор учебного заведения**: управляет устройствами с Windows 10 в [Intune для образовательных учреждений](introduction-intune-education.md) и может выполнять следующие действия. 

|Разрешение|Операция|
|---|---|
|Данные аудита|Чтение|
|DeviceConfigurations|Назначение, создание, удаление, чтение, обновление|
|Диспетчеры регистрации устройств|Чтение, обновление|
|Управляемые устройства|Чтение, обновление<!--, Delete [To be added in 1803]-->|
|Мобильные приложения|Назначение, создание, удаление, чтение, обновление|
|Reports|Чтение|
|Удаленные действия|Очистка ПК, перезагрузка, удаленная блокировка, снятие с учета, синхронизация устройств, очистка|
|Организация|Чтение|

### <a name="to-assign-a-built-in-role"></a>Назначение встроенной роли

1. Войдите на [портал Azure](https://portal.azure.com).
2. Выберите **Все службы** > **Intune**. Intune находится в разделе **Мониторинг и управление**.
3. На панели **Intune** выберите **Роли** > **Все роли**.
1. На панели **Роли Intune — Все роли** выберите встроенную роль, которую требуется назначить.

2. На панели <*имя роли*> — **Обзор** выберите **Управление** и затем **Назначения**.

    > [!NOTE]
    > Удалять или изменять встроенные роли нельзя

3. На панели настраиваемой роли выберите **Назначить**.

4. На панели **Назначения ролей** введите **имя** и необязательное **описание** для назначения, а затем выберите следующее:
    - **Members** (Участники). Выберите группу, содержащую пользователя, которому необходимо предоставить разрешения.
    - **Scope** (Область). Выберите группу, содержащую пользователей, которыми сможет управлять вышеуказанный участник.
<br></br>
5. По завершении нажмите кнопку **ОК**. Новое назначение отобразится в списке назначений.

## <a name="custom-roles"></a>Пользовательские роли

Вы можете создавать настраиваемые роли, которым назначаются любые разрешения, необходимые для выполнения определенных должностных обязанностей. Например, если ИТ-отдел организации управляет приложениями, политиками и профилями конфигурации, все эти разрешения можно добавить в одну настраиваемую роль.

> [!IMPORTANT]
> Для создания, изменения и назначения ролей учетная запись должна иметь одно из следующих разрешений в Azure AD:
> - **Глобальный администратор**
> - **Администратор службы Intune**

### <a name="to-create-a-custom-role"></a>Создание настраиваемой роли

1. Войдите на [портал Azure](https://portal.azure.com), используя учетные данные Intune.

2. Выберите **Все службы** в меню слева и введите **Intune** в текстовое поле фильтра.

3. Выберите **Intune** > **Роли** > **Все роли** > **Добавить пользовательскую**.

4. На панели **Добавить пользовательскую роль** введите имя и описание для новой роли и нажмите кнопку **Разрешения**.

5. На панели **Разрешения** выберите разрешения, которые нужно использовать с этой ролью. Для выбора разрешений, которые требуется назначить, руководствуйтесь [таблицей управления доступом на основе ролей для Intune](https://gallery.technet.microsoft.com/Intune-RBAC-table-2e3c9a1a).

6. По завершении нажмите кнопку **ОК**.

7. На панели **Добавить пользовательскую роль** щелкните **Создать**. Новая роль будет отображаться в списке на панели **Роли Intune — Все роли**.

### <a name="to-assign-a-custom-role"></a>Назначение настраиваемой роли

1. На панели **Роли Intune — Все роли** выберите пользовательскую роль, которую требуется назначить.

2. На панели <*имя роли*> — **Обзор** выберите **Управление** и затем **Назначения**. На этой панели можно также изменить или удалить существующие роли.

3. На панели настраиваемой роли выберите **Назначить**.

4. На панели **Назначения ролей** введите **имя** и необязательное **описание** для назначения, а затем выберите следующее:
    - **Members** (Участники). Выберите группу, содержащую пользователя, которому необходимо предоставить разрешения.
    - **Scope** (Область). Выберите группу, содержащую пользователей, которыми сможет управлять вышеуказанный участник.
<br></br>
5. По завершении нажмите кнопку **ОК**. Новое назначение отобразится в списке назначений.
